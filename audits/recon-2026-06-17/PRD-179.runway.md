## 2. Scope (standard PRD format — DRAFT)

**GOAL** — Extend the PRD-178 preview loop to also render against synthetic fixtures that exercise each
conditional dashboard section state, to non-ui paths only, alongside (never instead of) fresh-data render.

**SCOPE**
- Build/assemble fixture payload sets that deterministically trigger each section state in the table above.
- Extend the preview harness (local script and/or CI) to render each fixture to a non-ui output path.
- Add tests asserting each fixture render hits its intended section-state branch.

**OUT OF SCOPE**
- No change to render logic, the PRD-118/119 gates, or the fresh-data path.
- No fixture render to `ui/` (PRD-118 R5).
- No new prediction/decision logic (VISION non-goal — fixtures are presentation coverage only).

**FILES** (DRAFT — ratify at opening; grep-sweep tests first):
- A `tests/fixtures/` payload sets (or reuse/extend existing builders) — one per section state
- M `scripts/preview_dashboard.sh` (or A `scripts/preview_fixtures.sh`) — iterate fixtures → non-ui renders
- possibly M `.github/workflows/dashboard_preview.yml` (add a fixture-render job)
- A `tests/test_preview_fixtures.py` (or extend `tests/test_dashboard_renderer.py`)
- A `docs/prd_history/PRD-179.md` (+ `.review.claude.md`); M registry/state/index

**REQUIREMENTS**
- R1 — Coverage. Each enumerated section state has a fixture that triggers it.
  FAIL: any listed section state lacks a fixture that exercises it.
- R2 — Non-ui only. Every fixture render targets a non-ui path; none can publish to `ui/`.
  FAIL: a fixture render writes under `ui/` or passes the publish gate.
- R3 — Additive. No render-logic, gate, or fresh-data-path change.
  FAIL: `dashboard_renderer.py` render/gate logic is modified, or the fresh-data preview changes behavior.
- R4 — Assertion. Tests assert each fixture render contains its section-state marker.
  FAIL: a fixture renders without its expected marker and the suite still passes.

`dash_helpers.py` / `test_fixture_mode.py` builders to emit fixture payloads*, adding only what no builder

`scripts/preview_dashboard.sh` (or a sibling `preview_fixtures.sh`) for local iteration, plus an optional
non-blocking CI job in `dashboard_preview.yml`* — the local loop is the primary value; CI is a bonus.

premarket/Sunday, red-folder empty/error/expiring, trend-structure closed/unhealthy/inactive/no-data,
candidate-board disabled/unavailable, macro-tape no-data). Confirm if any state is intentionally deferred.

mirrors PRD-178's local target), gitignored.
