## 1. Recon (manual trace — gitnexus unavailable, no npm egress)

**The bug.** The macro-tape evidence rows compute a vote direction (risk-ON / risk-OFF)
that correctly applies a contra-cyclical flip, but the rationale subtitle is a single
static string per driver that ignores direction. On a rising 10Y the vote is risk-OFF
(correct) while the subtitle reads "easing yields favor risk" (risk-ON) — a visible
contradiction.

**Where it lives:**
- `cuttingboard/delivery/macro_tape_layout.py:87-94` — `MACRO_BIAS_INTERPRETATION`,
  a `MappingProxyType` keyed only by `payload_key` → one static string per driver:
  `volatility`="falling vol favors risk", `dollar`="soft dollar favors risk",
  `rates`="easing yields favor risk", `bitcoin`="crypto bid favors risk".
- `cuttingboard/delivery/dashboard_renderer.py:2077-2104` — evidence-row render.
  - `:2081` `_arrow = _tape_arrow_map.get(_slot.label, _DASH)` (UP/DOWN/FLAT)
  - `:2088-2090` `_risk_on = _arrow == _UP`; flipped if `_slot.payload_key in MACRO_BIAS_CONTRA_CYCLICAL`
  - `:2091/2093` `_vote` = "risk-ON vote" / "risk-OFF vote" / "no vote" (FLAT)
  - `:2094` `_interp = MACRO_BIAS_INTERPRETATION.get(_slot.payload_key, "")` — **the bug:
    static lookup, ignores `_arrow`, the contra-cyclical flip, and the FLAT state.**
- Contra-cyclical set: `MACRO_BIAS_CONTRA_CYCLICAL` (macro_tape_layout.py:78) = volatility,
  dollar, rates. Pro-cyclical: bitcoin.

**Blast radius — tight, presentation-only:**
- The vote tally that feeds `macro_bias_direction` to the integrator
  (dashboard_renderer.py:1281, 1793) derives from `long_votes`/`short_votes`, **not** from
  the rationale string. The subtitle is cosmetic HTML.
- **No payload/contract change:** `macro_drivers` (`change_pct`, `level`, `symbol`) is
  untouched; `output.py` and `ui/contract.json` are not involved.
- Output flows to `ui/dashboard.html` + `ui/index.html` (generated; not hand-edited).
- CSS class `.macro-evidence-interp` (dashboard_renderer.py:733) unchanged.

**Test gap (why CI was green on a bug):**
- `tests/test_dashboard_renderer.py:3541-3555` (`test_prd177_r3_macro_evidence_rows_present`)
  asserts only that all four `macro-evidence-interp` strings are **non-empty** and count==4.
  Nothing asserts vote↔rationale direction agreement.
- `:3558-3565` tests vote wording; `:3567-3577` tests the headline tally — neither cross-checks.
- `tests/test_macro_tape_layout.py:40-69` only validates `MACRO_BIAS_INTERPRETATION` is in
  the allowed pure-semantic exports list (`:53`); no content/shape assertion.

## 2. Scope (standard PRD format)

**GOAL** — Make each macro-evidence rationale subtitle direction-keyed so its direction
always agrees with the row's computed vote; lock the agreement with tests.

**SCOPE**
- Reshape `MACRO_BIAS_INTERPRETATION` to direction-keyed (rising / falling / flat per driver).
- At the single lookup (renderer:2094) select the string using the SAME direction signal
  already computed for the vote (`_arrow` + contra-cyclical flip), with an explicit FLAT case.
- Add tests asserting vote↔rationale agreement across all four drivers × {rising, falling, flat}.

**OUT OF SCOPE**
- No change to vote computation, the contra-cyclical set, the tally, or `macro_bias_direction`.
- No payload/contract change (`macro_drivers`, `output.py`, `ui/contract.json` untouched).
- No new evidence rows; no CSS change.

**FILES**
- M `cuttingboard/delivery/macro_tape_layout.py` (constant reshape; ~87-94)
- M `cuttingboard/delivery/dashboard_renderer.py` (lookup at ~2094; reuse existing direction)
- M `tests/test_macro_tape_layout.py` (export-allowlist test if shape assertion shifts)
- M `tests/test_dashboard_renderer.py` (add agreement assertions ~3541+)
- A `docs/prd_history/PRD-191.review.claude.md`, A `docs/prd_history/PRD-191.review.codex.md`
- M `docs/PRD_REGISTRY.md`, M `docs/PROJECT_STATE.md`, M `docs/prd_index.json` (bookkeeping)

**CHANGE SURFACE** (CONSUMER, dashboard truth semantics — mandatory):
`macro_tape_layout.py`: constant-shape change only, no new exported symbols beyond the reshape.
`dashboard_renderer.py`: append-only at the existing lookup — reuse `_risk_on`/`_arrow`, no
signature changes, no new data reads. Rendered-HTML change → run the `dashboard-publish-refresh`
discipline (render dashboard.html → copy to index.html → byte-identical check) before commit.

**REQUIREMENTS**
- R1 — Direction-keyed structure. `MACRO_BIAS_INTERPRETATION` provides a distinct string for
  rising, falling, and flat for each of the four drivers.
  FAIL: any driver lacks a distinct rising/falling/flat string, or the export-allowlist test fails.
- R2 — Agreement at render. The subtitle for each row is selected by the same direction signal
  (arrow + contra-cyclical flip) used to compute `_vote`; FLAT renders the neutral string.
  **Language classes (review-required, make R2 binary-testable):** risk-ON ⊃ {"favors","bid"};
  risk-OFF ⊃ {"pressures","weakness","risk-off"}; neutral ⊃ {"stable"}. The test asserts each row's
  subtitle belongs to the language class matching its vote.
  FAIL: for any driver, a rendered row exists whose vote is risk-OFF while its subtitle is in the risk-ON
  language class (or vice-versa), or a FLAT row renders a non-neutral subtitle.
- R3 — Test lock. Tests assert vote↔rationale agreement for all four drivers × three directions.
  FAIL: the inverted-string regression (rising 10Y → "easing yields favor risk") passes the suite.

**DATA FLOW**
`macro_drivers.change_pct` → arrow + contra-cyclical flip (existing) → direction key →
`MACRO_BIAS_INTERPRETATION[driver][direction]` → `.macro-evidence-interp` span.

## 5. DECISION SEAMS (pre-surfaced)

**SEAM 191-A — the direction-keyed copy (CONTENT, Dustin owns).** Each driver needs three
strings. The falling/rising of the *current* string is usually already correct; the opposite
and flat are new. Recommended defaults below — approve or edit the wording:

| Driver | rising (→ vote) | falling (→ vote) | flat |
|--------|-----------------|------------------|------|
| volatility (contra) | "elevated vol pressures risk" (risk-OFF) | "falling vol favors risk" (risk-ON) ✓existing | "vol stable" |
| dollar (contra) | "strong dollar pressures risk" (risk-OFF) | "soft dollar favors risk" (risk-ON) ✓existing | "dollar stable" |
| rates (contra) | "tightening yields pressure risk" (risk-OFF) | "easing yields favor risk" (risk-ON) ✓existing | "yields stable" |
| bitcoin (pro) | "crypto bid favors risk" (risk-ON) ✓existing | "crypto weakness signals risk-off" (risk-OFF) | "crypto stable" |

*Recommended default: adopt the table as-is.* Note "rising/falling" rows map to vote via the
contra-cyclical flip — the table already encodes the correct vote per cell.

**SEAM 191-B — flat-string presence.** Today FLAT yields `_vote`="no vote". Decision: render a
neutral subtitle ("… stable") vs an empty string. *Recommended default: neutral string* (keeps
R1's "distinct string per direction" clean and the row visually complete).

**SEAM 191-C — data structure shape.** Nested dict vs a small frozen dataclass/tuple.
*Recommended default: nested `MappingProxyType` of `MappingProxyType`* — minimal churn, keeps the
existing pure-semantic-export test model.
