GOAL
The doctrine governing what `logs/audit.jsonl` records — and what it does not — has lived only in code (the `_run_pipeline` write site at `cuttingboard/runtime.py:1004`, the deliberate absence of writes from `_execute_notify_run` at `runtime.py:436`) and in informal recon notes (`docs/DECISIONS.md` 2026-05-23). Three consumers depend on the file (`moomoo_join.py`, `evaluation.py`, `runtime._load_run_history`), and none of their docstrings flag the structural sparsity that follows from the doctrine. Write `docs/audit_doctrine.md` declaring the two binding rules (pipeline writes only from `_run_pipeline`; one record per invocation with notification events as a separate stream), and reference it from each consumer's docstring. No code behavior changes; no schema lock claimed; no position taken on whether intraday coverage should ever be added.
