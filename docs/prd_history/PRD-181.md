PRD-181 - Short-gate fail-closed during the open window

Status: PROPOSED

LANE
HIGH-RISK (touches the protected runtime/__init__.py decision path; changes
which SHORT candidates are blocked)

CLASS
EXECUTION

WHY NOW (ratified decision D-Q7, 2026-06-12 gate recon)
The 2026-06-12 gate recon (docs/audit/gate_recon_2026-06-12.md, flag O5, open
question 7) found an inert-gate window. cuttingboard/intraday_state_engine.py
emits no intraday state before 09:45 ET (_NOISE_END = time(9, 45), the noise
exclusion window; compute_intraday_state returns None before it). In
cuttingboard/runtime/__init__.py the SHORT gate (_apply_intraday_short_permission,
~line 1108) fails OPEN when state is unavailable (~line 1135: state-unavailable
symbols are left in the candidate set and skipped). Net effect: PRD-151's
OPEN-phase SHORT blocking is inert from the 09:30 open through 09:45 - precisely
the window it was written to cover. Dustin ratified the resolution: fail CLOSED
for SHORT during that specific open window. This PRD encodes that decision; it
is drafted, not implemented.

MAX EXPECTED DELTA
~30 production LOC, confined to cuttingboard/runtime/__init__.py (a bounded
fail-closed branch plus threading the run clock into the gate). No new
production files. If implementation exceeds this, stop and amend.

GOAL
When intraday state is unavailable AND the clock is within the open window
(market open through 09:45 ET), SHORT candidates fail CLOSED (blocked) instead
of fail-open; every other case is unchanged.

SCOPE (filter-side only)
- In _apply_intraday_short_permission, when intraday_state_available is False
  AND the run clock is within the open window [market open 09:30 ET, _NOISE_END
  09:45 ET), block SHORT candidates (remove them from the qualified set) rather
  than preserving them.
- Thread the run's ET clock into the gate so it can decide window membership.
  The open-window bound reuses the existing _NOISE_END boundary
  (intraday_state_engine.py); it is read, not redefined.

OUT OF SCOPE
- Do NOT change the 09:45 emission time or any logic in
  cuttingboard/intraday_state_engine.py. This is a filter-side fix only
  (scope-lock); intraday_state_engine.py is not in FILES.
- No change to LONG-side handling in any window.
- No change to the general fail-open default for missing state OUTSIDE the open
  window (e.g. a mid-session compute failure stays fail-open).
- No change to _downside_permission_from_state or the post-09:45 gating logic
  it feeds.

CHANGE SURFACE
- cuttingboard/runtime/__init__.py: add a bounded open-window fail-closed branch
  inside _apply_intraday_short_permission and thread the ET run clock into it.
  No change to the post-09:45 path (_downside_permission_from_state consumer) or
  to the outside-window fail-open default. No unrelated refactor.
- docs/prd_history/PRD-151.md: append a note documenting the 09:45 emission
  boundary and this open-window fail-closed interaction where PRD-151's gating

REQUIREMENTS

R1 - Open window + no state + SHORT -> blocked
When the run clock is within [09:30 ET, 09:45 ET) and intraday state is
unavailable, a SHORT candidate is removed from the qualified set (fail-closed).

FAIL: A SHORT candidate with unavailable state inside the open window survives
into the qualified/published set.

R2 - Open window + no state + LONG -> unaffected
A LONG candidate under the same condition (open window, state unavailable) is
not blocked by this gate.

FAIL: A LONG candidate is removed by the open-window fail-closed branch.

R3 - Post-09:45 with state present -> existing PRD-151 gating applies as-is
At or after 09:45 ET, when intraday state is available, the SHORT gate behaves
exactly as it does today (existing _downside_permission_from_state logic).

FAIL: A post-09:45 run with state present produces a different SHORT outcome
than the current code for the same inputs.

R4 - Outside the open window + missing state -> fail-open preserved
When state is unavailable and the clock is NOT within the open window, the
existing fail-open default holds (SHORT preserved).

FAIL: A missing-state symbol outside the open window is blocked by this PRD.

R5 - Interaction documented
The 09:45 emission boundary and the open-window fail-closed interaction are
documented where PRD-151's gating is described.

FAIL: No doc records the open-window fail-closed behavior or the 09:45 boundary
interaction after this PRD lands.

DATA FLOW
run clock (ET) + intraday_state_available -> _apply_intraday_short_permission:
  - within [09:30, 09:45) AND state unavailable AND side == SHORT -> block
  - within [09:30, 09:45) AND side == LONG -> pass
  - state available (post-09:45) -> existing downside-permission gating
  - state unavailable AND outside open window -> fail-open (preserve)

FAIL CONDITIONS
- SHORT survives inside the open window with no state (R1).
- LONG blocked by the open-window branch (R2).
- Post-09:45 SHORT outcome diverges from current behavior for equal inputs (R3).
- Missing-state SHORT blocked outside the open window (R4).
- Open-window fail-closed behavior undocumented (R5).

TESTS (fixture-based; home: tests/test_gap_down_permission_integration.py)
- open-window + no-state + SHORT -> blocked (removed from qualified set).
- open-window + no-state + LONG -> passes.
- post-09:45 + state present -> existing PRD-151 gating applies (unchanged
  outcome vs current).
- outside-window (e.g. mid-session) + no-state -> fail-open preserved (SHORT
  survives).
- Boundary: exactly 09:45 ET behaves as post-window (state-driven path), since
  the open window is [09:30, 09:45).
