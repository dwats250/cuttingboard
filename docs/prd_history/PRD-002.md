PRD-002 — Options Chain Validation + Runtime Orchestrator

STATUS
COMPLETE
Commits: 6bc67e6, 3d4a214

GOAL
Add live options chain validation as the final pipeline gate, and introduce a
canonical runtime orchestrator that routes all execution modes through a single
entry point.

SCOPE
- Build chain_validation.py: live OI, spread %, bid/ask sanity checks
- Build runtime.py: sole production orchestrator for all execution modes
- Expose single public CLI via python -m cuttingboard
- Add fixture mode for deterministic replay from normalized snapshots
- Add verify mode for artifact validation without re-running the pipeline
- Harden qualification gate logic and regime vote model

OUT OF SCOPE
- Telegram transport (added PRD-006)
- Notification suppression/dedup (added PRD-018)
- Contract schema freeze (added PRD-011)

FILES
A cuttingboard/chain_validation.py
A cuttingboard/runtime.py
A cuttingboard/__main__.py
M cuttingboard/qualification.py
M cuttingboard/regime.py
M cuttingboard/output.py
M cuttingboard/config.py

REQUIREMENTS

R1 — Chain Validation Layer
chain_validation.py MUST gate each option setup on:
- Open interest minimum
- Bid/ask spread percentage maximum
- Bid/ask sanity (bid > 0, ask > bid)
Results: VALIDATED | REJECTED | UNAVAILABLE per symbol.

R2 — Runtime Orchestrator
runtime.py MUST be the sole production orchestrator.
All modes (live, fixture, sunday, verify) MUST route through cli_main().
No other module may contain top-level pipeline execution logic.

R3 — Execution Modes
- LIVE: fetch live data, run full pipeline, write report + audit
- FIXTURE: load normalized snapshot from tests/fixtures/, start at validation
- SUNDAY: run through regime only, no candidates
- VERIFY: read logs/latest_run.json, return PASS or FAIL without re-running

R4 — Single CLI Entry Point
python -m cuttingboard MUST dispatch all modes.
No separate run_premarket.py or run_intraday.py invocations in production.

R5 — Artifact Guarantee
Every live run MUST write:
- reports/YYYY-MM-DD.md
- logs/latest_run.json

R6 — Qualification Hardening
Qualification gate count expanded. Hard/soft gate split enforced.
STAY_FLAT short-circuits all per-symbol work — no gates run.

VALIDATION
Run: pytest -q
Manual: python -m cuttingboard (live), python -m cuttingboard fixture
Confirm: HALT_SYMBOL failure produces HALT, chain rejection excludes setup from output
