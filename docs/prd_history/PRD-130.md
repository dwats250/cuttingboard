CLASS

WHY NOW
Trend structure (PRD-107 / PRD-110 / PRD-112 / PRD-123) collapses
five distinct upstream conditions — insufficient history, missing
upstream price/VWAP, an inactive session at the renderer layer,
fields that are intentionally not computed, and the exact-equality
case `price == ref` — into a single generic `"UNKNOWN"` token in
both the sidecar artifact and the dashboard panel. Operators cannot
distinguish "we don't have enough bars yet" from "the live quote
feed dropped" from "this state was never asked for" from "price is
exactly at the reference level", which erodes trust in an otherwise
deterministic surface. The equality case is particularly damaging:
it is a *successful* comparison resulting in a neutral/aligned
condition, not an unavailable comparison, yet the current code
emits `"UNKNOWN"` for it (see `tests/test_trend_structure.py`
`test_no_at_enum_value` which pins this behavior). This PRD
normalizes those states into named, deterministic categories
without altering trade decisions, qualification, scoring, or any
artifact path/field-name.

MAX EXPECTED DELTA
<= 40 production-code lines across the two files in FILES, plus the
matching targeted tests. No new modules. No new fields. No removals
of existing fields. Exceeding this ceiling MUST stop implementation,
amend the PRD with a revised ceiling and reason, and re-trigger
review per the CLASS matrix in `docs/PRD_PROCESS.md`.

GOAL
Replace the ambiguous `"UNKNOWN"` value in trend-structure unknown-
state fields with deterministic, condition-specific tokens so that
the sidecar and dashboard distinguish between insufficient history,
unavailable upstream data, an inactive session, intentionally
uncomputed fields, and the successful-comparison equality case —
while preserving every existing contract path, field name, runtime
decision, qualification gate, and trade outcome.

CHANGE SURFACE
- `cuttingboard/trend_structure.py`:
  - `_cmp()` (lines ~87–94): replace the existing UNKNOWN-returning
    branches with three deterministic outcomes:
      * `price == ref` (successful comparison, neutral outcome)
        → `AT_LEVEL`
      * `price > ref` → `ABOVE` (unchanged)
      * `price < ref` → `BELOW` (unchanged)
      * `price is None` or `ref is None` → `_cmp()` does NOT decide
        the token; instead it returns a `None` sentinel (or
        equivalent) so the caller maps `ref is None` to the
        precise upstream cause. `_cmp()` itself never emits
        `DATA_UNAVAILABLE` or `INSUFFICIENT_HISTORY`.
    `AT_LEVEL` is a *successful* comparison resulting in a neutral
    state, NOT an unavailable comparison; collapsing it into
    `DATA_UNAVAILABLE` would lose signal semantics. No signature
    change visible to external callers; internal sentinel is a
    module-private implementation detail.
  - VWAP comparison path (`price_vs_vwap` in `_build_record()`):
    if `_cmp()` returns the `None` sentinel due to missing VWAP or
    price, the caller MUST inspect the upstream `_vwap()` cause
    (`cuttingboard/trend_structure.py:49–71`) and emit:
      * `NOT_COMPUTED` — `_vwap()` returned None because
        `_is_intraday(df)` is False (daily / non-intraday input);
        this is an intentional computation boundary, not a data
        outage. Operators viewing a daily-bar symbol MUST see
        `NOT_COMPUTED`, not `DATA_UNAVAILABLE`.
      * `DATA_UNAVAILABLE` — every other `_vwap()` None path:
        df missing/empty, required OHLCV columns absent, empty
        intraday session slice, or non-positive cumulative volume.
        Also `DATA_UNAVAILABLE` when `current_price` (the `price`
        side of `_cmp()`) is None.
    The VWAP comparison path therefore overrides `_cmp()`'s
    sentinel default with the precise upstream cause.
  - SMA comparison paths (`price_vs_sma_50`, `price_vs_sma_200` in
    `_build_record()`): if `_cmp()` returns the `None` sentinel due
    to `ref is None`, the caller MUST inspect the upstream
    `_close_series()` / `_sma()` cause and emit:
      * `INSUFFICIENT_HISTORY` — `_close_series()` returned a
        valid Series but `len(closes) < window` so `_sma()`
        returned None.
      * `DATA_UNAVAILABLE` — `_close_series()` itself returned
        None (missing df, empty df, no `Close` column, or all-NaN
        per `cuttingboard/trend_structure.py:22–33`).
    The SMA comparison path therefore overrides `_cmp()`'s
    `ref is None` default with the precise upstream cause.
  - `_trend_alignment()` (lines ~97–104): the UNKNOWN-propagation
    branch is partitioned by the upstream cause of SMA None:
      * `INSUFFICIENT_HISTORY` — close series exists and is valid
        but has too few usable observations for the requested
        window (closes shorter than 50 or 200 bars).
      * `DATA_UNAVAILABLE` — close series is missing, empty, lacks
        a `Close` column, or is all-NaN (the other paths through
        `_close_series()` that return None at
        `cuttingboard/trend_structure.py:22–33`).
    The function MUST distinguish these two upstream causes
    directly when emitting the token. No signature change.
  - `_entry_context()` (lines ~107–116): the two UNKNOWN-propagation
    branches map to the upstream token (`DATA_UNAVAILABLE` if
    `data_status == "MISSING"`, otherwise propagate whichever
    specific token caused the upstream UNKNOWN). No signature
    change.
  - `_reason()` (lines ~119–135): update the two reason-text
    branches that today say "trend alignment unknown — sma_50 or
    sma_200 missing" and "VWAP unavailable" to reference the new
    token names so operators reading `reason` see the same
    vocabulary as the structured field. Format string shape is
    preserved.
  - `_build_record()` (lines ~159–195): no field set changes; only
    the value strings change.
- `cuttingboard/delivery/dashboard_renderer.py`:
  - The trend-structure block (lines ~1694–1719): introduce a small
    pure mapping from `{AT_LEVEL, INSUFFICIENT_HISTORY,
    DATA_UNAVAILABLE, NOT_COMPUTED}` (and the renderer-only
    `SESSION_UNAVAILABLE` branch covering the existing
    INACTIVE_SESSION_LABEL path and the snapshot-absent branch
    where applicable) to compact display text. `AT_LEVEL` MUST
    render as an affirmative neutral-comparison string (e.g.
    "AT LEVEL"), not an unknown-glyph fallback. Badge text MAY
    change; badge CSS class names MUST NOT.
- No changes to imports, module exports, or public function
  signatures in either file.

REQUIREMENTS

R1 — Unknown-state value domain is partitioned
For every emission site in `cuttingboard/trend_structure.py` that
previously returned `"UNKNOWN"` for a structured state field
(`price_vs_vwap`, `price_vs_sma_50`, `price_vs_sma_200`,
`trend_alignment`, `entry_context`), the function MUST instead
return exactly one of: `"AT_LEVEL"`, `"INSUFFICIENT_HISTORY"`,
`"DATA_UNAVAILABLE"`, or `"NOT_COMPUTED"`. The choice MUST be
deterministic given the function's inputs:
  * `AT_LEVEL` — comparison succeeded and `price == ref` exactly.
  * `INSUFFICIENT_HISTORY` — close series is present and valid but
    too short for the requested SMA window.
  * `DATA_UNAVAILABLE` — required input (price, VWAP, or close
    series) is None / empty / missing `Close` column / all-NaN.
  * `NOT_COMPUTED` — indicator is intentionally not computed under
    current input conditions (e.g. VWAP on non-intraday / daily
    bars). Distinguishes "intentional computation boundary" from
    "data outage". Today's sole emission site is the VWAP
    comparison path when `_is_intraday(df)` is False.

fields.
