# PRD-120 - Dashboard Source-Health Diagnostics and Permission Display Correction

STATUS: COMPLETE @ d20d906

## GOAL

Two renderer-only corrections to the Signal Forge dashboard:

(a) Surface deterministic per-block source-health diagnostics so an operator
    can tell at a glance whether each major dashboard block is rendering
    from fresh, stale, missing, invalid, fallback, mixed, or
    inactive-session inputs.

(b) Replace the ambiguous em-dash Permission cell rendered today as
    `&#8212;` when `outcome` is set, `permission` is None, and the system
    is not halted, with a deterministic, source-derived label.

Complements PRD-119, Dashboard Publish Freshness Gate. PRD-120 changes only
what the renderer emits in HTML. It does not gate publication.

## SCOPE

1. Add a compact one-line `SOURCE: <STATUS>` diagnostic to each of:
   - System State
   - Macro Tape (`MACRO SOURCE: <STATUS>`)
   - Trend Structure
   - Market Map / Developing Setups (`MARKET MAP SOURCE: <STATUS> - setups <N>`)
2. Classify each block's source state from a fixed enum, using an explicit
   precedence order.
3. Stop rendering Permission as `&#8212;` when enough state exists to
   classify the system; map to MONITOR_ONLY / UNKNOWN deterministically.
4. Add a `TREND SYMBOLS: X/Y` completeness summary inside Trend Structure
   under OK / STALE / FALLBACK states only.
5. Diagnostics are additive. They must not replace, rename, or relocate the
   existing PRD-115 lineage banner, PRD-116 MIXED_ARTIFACTS block,
   PRD-117 INACTIVE_SESSION_LABEL, or the existing
   `SOURCE_MISSING` / `PARSE_ERROR` / `STALE MARKET MAP` strings.
6. All changes are renderer-only and read-only against existing helpers.

## OUT OF SCOPE

1. No trade, regime, qualification, market_map, notification, or runtime
   logic changes.
2. No artifact schema, payload contract, or generation_id format changes.
3. No publish-gating changes. The following symbols are read-only:
   `validate_coherent_publish`, `_parse_payload_timestamp`,
   `_allowed_freshness_window`, `_coherent_generation_ids`,
   `_output_under_ui`, `CoherentPublishError`, `StalePublishError`,
   `INACTIVE_SESSION_TYPES`, `INACTIVE_SESSION_LABEL`.
4. No GitHub Actions changes.
5. No new indicators, sidecars, or fixtures.
6. No changes to `ui/index.html` mirroring. That remains a workflow concern.
7. No weakening of stale, coherent-publish, or mixed-lineage checks.
8. No silent coercion of missing data to OK.
9. The `#sunday-macro-context` block (renders only under coherent Sunday
   lineage) is unchanged and gets no SOURCE line.

## TERMINOLOGY

- source_health_enum:
  `OK | STALE | MISSING | INVALID | FALLBACK | MIXED | INACTIVE_SESSION | UNKNOWN`

- block_source_health(block):
  the source_health value the renderer derives and emits for one of:
  System State, Macro Tape, Trend Structure, Market Map.

- permission_display:
  the visible Permission cell value.

- trend_symbol_completeness:
  `(usable, expected)` tuple where `expected == len(config.TREND_STRUCTURE_SYMBOLS)`
  and `usable` is computed by a new renderer-internal helper
  `_trend_symbols_usable(snapshot)` that returns the count of symbols
  whose record contains every field in `_TREND_STRUCTURE_REQUIRED_FIELDS`,
  evaluated per-symbol (NOT all-or-nothing). The existing
  `_trend_structure_records` helper is unchanged and continues to gate
  row rendering with its existing all-or-nothing semantics.

## SOURCE-HEALTH MAPPING

Each block evaluates the rows below in order. The first matching row wins.

### System State

| Order | Condition                                                                                 | Result   |
|-------|-------------------------------------------------------------------------------------------|----------|
| 1     | `artifact_lineage_state == "MIXED"`                                                       | MIXED    |
| 2     | `artifact_lineage_state in ("STALE", "MISSING")`                                          | match    |
| 3     | `_compute_timestamp_freshness(payload.meta.timestamp) == "PARSE_ERROR"`                   | INVALID  |
| 4     | `_compute_timestamp_freshness(payload.meta.timestamp) == "STALE"`                         | STALE    |
| 5     | `_compute_timestamp_freshness(payload.meta.timestamp) == "FRESH"` and lineage `COHERENT`  | OK       |
| 6     | otherwise                                                                                 | UNKNOWN  |

### Macro Tape

| Order | Condition                                                                            | Result   |
|-------|--------------------------------------------------------------------------------------|----------|
| 1     | `macro_drivers` empty or every value is `"MARKET MAP UNAVAILABLE"`                   | MISSING  |
| 2     | Any slot in `tape_value_slots` equals `"--"` or `"N/A"`                              | FALLBACK |
| 3     | otherwise                                                                            | OK       |

(MIXED is intentionally omitted from Macro Tape: no per-slot freshness
signal exists in `_build_tape_value_slots`.)

### Trend Structure

| Order | Condition                                                                                            | Result            |
|-------|------------------------------------------------------------------------------------------------------|-------------------|
| 1     | `artifact_lineage_state == "MIXED"`                                                                  | MIXED             |
| 2     | `artifact_lineage_state in ("STALE", "MISSING")`                                                     | match             |
| 3     | `inactive_session == True`                                                                           | INACTIVE_SESSION  |
| 4     | `trend_structure_snapshot is None` (or not a dict)                                                   | MISSING           |
| 5     | `_ts_generated_at_raw` is non-empty string and `_compute_timestamp_freshness == "PARSE_ERROR"`       | INVALID           |
| 6     | `_compute_timestamp_freshness == "STALE"`                                                            | STALE             |
| 7     | `_trend_symbols_usable(snapshot) == 0`                                                               | FALLBACK          |
| 8     | otherwise                                                                                            | OK                |

### Market Map

| Order | Condition                                          | Result            |
|-------|----------------------------------------------------|-------------------|
| 1     | `artifact_lineage_state == "MIXED"`                | MIXED             |
| 2     | `artifact_lineage_state == "STALE"`                | STALE             |
| 3     | `artifact_lineage_state == "MISSING"`              | MISSING           |
| 4     | `inactive_session == True`                         | INACTIVE_SESSION  |
| 5     | `_mm_status == "SOURCE_MISSING"`                   | MISSING           |
| 6     | `_mm_status == "PARSE_ERROR"`                      | INVALID           |
| 7     | `_mm_status == "STALE"`                            | STALE             |
| 8     | otherwise                                          | OK                |

`OK` is forbidden whenever any higher-precedence row in the same block
matches.

## REQUIREMENTS

R1. `block_source_health` is computed for each of the four blocks using the
    ordered precedence in SOURCE-HEALTH MAPPING.
    FAIL: Any block omits a SOURCE diagnostic line, or a block emits OK
    while a higher-precedence row matches.

R2. System State emits `SOURCE: <STATUS>` directly above the existing
    `RUN SNAPSHOT - <FRESHNESS>` line. The existing RUN SNAPSHOT line is
    preserved unchanged.
    FAIL: RUN SNAPSHOT label is removed, relocated, or rewritten.

R3. Permission display correction. The renderer replaces the
    `<div class="value">&#8212;</div>` fallback near the current Permission
    rendering with deterministic mapping using values already passed to
    `render_dashboard_html`. Evaluated in order:

    1. `bool(system_halted)` -> HALTED (existing rendering preserved).
    2. `stay_flat_reason is not None` -> existing stay_flat_reason text.
    3. `permission is True` -> existing text.
    4. `permission is False` -> existing text.
    5. `permission is None` and `artifact_lineage_state in ("MIXED","STALE","MISSING")` -> UNKNOWN.
    6. `permission is None` and `outcome in (None, "STAY_FLAT", "NO_TRADE")` and not system_halted -> MONITOR_ONLY.
    7. everything else -> UNKNOWN.

    The existing Reason line below Permission is preserved.

    FAIL: The rendered HTML for an active NO_TRADE run under coherent
    lineage contains `>&#8212;<` inside the `system-state` block's
    Permission field.
    FAIL: Any path that previously rendered HALTED no longer renders
    HALTED.

R4. Macro Tape emits a `MACRO SOURCE: <STATUS>` line. This is separate
    from the existing `MACRO BIAS: <...>` line. Both lines render when
    applicable.
    FAIL: Macro Tape lacks a MACRO SOURCE line, or MACRO BIAS is removed.

R5. Trend Structure emits a `SOURCE: <STATUS>` line above the table (or
    above the PRD-116 disabled message under unhealthy lineage, or above
    the PRD-117 INACTIVE_SESSION_LABEL under coherent inactive session).
    FAIL: Trend Structure renders any body content without a preceding
    SOURCE line, OR PRD-117 INACTIVE_SESSION_LABEL is hidden or replaced,
    OR PRD-116 disabled message is removed.

R6. Trend Structure emits `TREND SYMBOLS: X/Y` immediately under the
    SOURCE line whenever `block_source_health in (OK, STALE, FALLBACK)`.
    Suppressed for `MISSING | INVALID | MIXED | INACTIVE_SESSION`.
    X is `_trend_symbols_usable(snapshot)`; Y is
    `len(config.TREND_STRUCTURE_SYMBOLS)`.
    FAIL: TREND SYMBOLS appears under any suppressed state, or X/Y does
    not match the deterministic per-symbol usable count.

R7. Market Map emits `MARKET MAP SOURCE: <STATUS> - setups <N>` line.
    N is the count of candidate cards actually emitted in the HTML, which
    is 0 under any non-OK block state. Existing `STALE MARKET MAP`,
    `SOURCE_MISSING`, `PARSE_ERROR`, and PRD-116 disabled text are
    preserved verbatim.
    FAIL: Market Map lacks the MARKET MAP SOURCE line, or any existing
    diagnostic text is removed or renamed.

R8. No false freshness. No code path may emit `OK` for any block whose
    underlying signals route to a higher-precedence non-OK row.
    FAIL: A test asserts a non-OK precedent signal and the block emits OK.

R9. Coherent-publish and freshness gates preserved. PRD-118
    `validate_coherent_publish` and PRD-119 freshness logic remain the
    only enforcement points for publish gating. PRD-120 reads their
    intermediate signals only.
    FAIL: `validate_coherent_publish`, `CoherentPublishError`, or
    `StalePublishError` is edited, called from new code paths, or
    bypassed.

R10. Renderer path behavior preserved. `write_dashboard` output-path
     semantics, including `_output_under_ui`, are unchanged.
     FAIL: `write_dashboard` argument list, return type, or output-path
     branching changes.

R11. Determinism. Given fixed
     `(payload, run, market_map, trend_structure_snapshot, frozen now)`,
     `render_dashboard_html` returns byte-identical strings across
     repeated calls. Tests freeze BOTH
     `cuttingboard.delivery.dashboard_renderer.datetime` and
     `cuttingboard.delivery.dashboard_renderer._utcnow` to fixed values.
     FAIL: Identical inputs with frozen `now` produce different bytes
     across two calls in the same test.

R12. Visual compactness. Each SOURCE line is one line, ASCII only, no
     debug dumps.

     Acceptable formats:
     - `SOURCE: STALE`
     - `SOURCE: INVALID`
     - `MACRO SOURCE: FALLBACK`
     - `MARKET MAP SOURCE: OK - setups 1`
     - `TREND SYMBOLS: 0/6`

     FAIL: Any new diagnostic exceeds one line or contains non-ASCII.

R13. ASCII only across `dashboard_renderer.py`, `test_dashboard_renderer.py`,
     and PRD-120 docs. No em dashes in source text. Existing `&#8212;`
     HTML entities elsewhere in templates are preserved.
     FAIL: Non-ASCII byte appears in any FILES entry.

R14. Tests. `tests/test_dashboard_renderer.py` adds deterministic tests:

     1. Permission renders MONITOR_ONLY for active NO_TRADE under coherent
        lineage (no `>&#8212;<` inside Permission field).
     2. Permission renders HALTED for halted state (precedence over
        MONITOR_ONLY).
     3. Permission renders UNKNOWN when permission is None and
        outcome == "TRADE" and not halted (catch-all UNKNOWN branch).
     4. Trend Structure renders SOURCE: MISSING when
        `trend_structure_snapshot is None` under coherent lineage.
     5. Trend Structure renders SOURCE: STALE when timestamp is stale.
     6. Trend Structure renders SOURCE: INVALID when generated_at
        unparsable.
     7. Trend Structure renders `TREND SYMBOLS: 0/6` and SOURCE: FALLBACK
        when snapshot exists with all 6 symbol records missing required
        fields (uses `_trend_symbols_usable` per-symbol count).
     8. Macro Tape renders `MACRO SOURCE: FALLBACK` when any tape slot is
        `--` or `N/A`.
     9. Market Map renders `MARKET MAP SOURCE: OK - setups N` with N
        matching the rendered card count under coherent lineage.
    10. No stale precedent signal path renders the block as OK.
    11. Existing coherent-publish validation tests still pass.
    12. Determinism: identical render with frozen `datetime` and
        `_utcnow` produces byte-identical HTML across two calls.
    13. Mapping table coverage: at least one test per reachable enum
        value for each block.
    14. INACTIVE_SESSION_LABEL precedence over Trend Structure
        missing-symbol diagnostic under coherent lineage.
    15. `artifact_lineage_state == "MIXED"` causes Trend Structure and
        Market Map blocks to render MIXED (lineage precedes `_mm_status`
        and `_ts_records`).
    16. `>&#8212;<` does not appear inside the Permission field for any
        coherent-lineage active NO_TRADE run.
    17. Halted + stay_flat_reason precedence: HALTED wins over
        MONITOR_ONLY and UNKNOWN.
    18. Determinism with frozen `datetime` and `_utcnow`: two renders
        produce identical bytes.
    19. Non-ui render still produces source-health diagnostics even
        though PRD-119 publish gating is bypassed.
    20. Malformed `payload.meta.timestamp` under coherent lineage renders
        System State source health INVALID, not STALE.
    21. `permission is None` under MIXED lineage renders Permission as
        UNKNOWN (not MONITOR_ONLY).
    22. Market Map under MIXED lineage with non-empty `market_map.symbols`
        renders `MARKET MAP SOURCE: MIXED - setups 0`.

    FAIL: Any listed branch lacks a deterministic test.
