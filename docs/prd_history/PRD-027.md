PRD-027 — Context Report Layer — Deterministic Premarket and Postmarket Reports

STATUS
COMPLETE
Commit: b8dc599

GOAL
Add deterministic premarket and postmarket report generators that produce structured dicts from the contract without modifying the contract or any existing artifacts.

SCOPE
- Add cuttingboard/reports/premarket.py: build_premarket_report(contract) → dict
- Add cuttingboard/reports/postmarket.py: build_postmarket_report(contract, run_history) → dict
- Update runtime.py: _load_run_history reads audit.jsonl (run records, max 5); generate reports after contract finalized, wire into PipelineResult
- Add 64 tests

OUT OF SCOPE
- No modification of contract fields
- No modification of existing artifacts (reports/, logs/)
- No new Telegram messages

FILES
A cuttingboard/reports/premarket.py
A cuttingboard/reports/postmarket.py
M cuttingboard/runtime.py

REQUIREMENTS

R1 — Premarket Report Structure
build_premarket_report MUST return a dict with: regime-templated scenarios (2–3 per regime), focus_list from trade_candidates, volatility_state derived from market_regime, correlation fields, invalidation when tradable==False.

FAIL: build_premarket_report returns a dict missing any required top-level key.

R2 — Postmarket Report Structure
build_postmarket_report MUST return a dict with: expectation_vs_reality (MATCH/PARTIAL/MISS/NO_EXPECTATION), rejection_breakdown by stage, regime_validation across run_history, deterministic_observations.

FAIL: expectation_vs_reality contains a value outside MATCH/PARTIAL/MISS/NO_EXPECTATION.

R3 — No Contract Mutation
Neither report generator MUST modify any contract field.

FAIL: contract fields differ before and after calling either report generator.

R4 — No Artifact Mutation
Reports MUST NOT write to reports/ or logs/.

FAIL: Either generator writes to disk.

R5 — Run History Limit
_load_run_history MUST read at most 5 run records from audit.jsonl.

FAIL: _load_run_history reads more than 5 records.

DATA FLOW
contract → build_premarket_report() → premarket dict → PipelineResult
contract + audit.jsonl (max 5 runs) → build_postmarket_report() → postmarket dict → PipelineResult

FAIL CONDITIONS
- Report dict missing required keys
- expectation_vs_reality outside defined values
- Contract fields mutated
- Generator writes to disk
- run_history exceeds 5 records

VALIDATION
Run: pytest -q (64 new tests; 954 total passing)
Manual: live run — confirm PipelineResult contains premarket and postmarket dicts, no new files written.
