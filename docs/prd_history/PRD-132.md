WHY NOW
PRD-131 added an SMA composite display cell but explicitly deferred
VWAP composition to a future PRD ("A separate intraday-context
display layer is reserved for a future PRD if and when a clean
design is proposed"), because mixing VWAP into the SMA composite
created an asymmetric boundary (Codex Round-1 finding O2). The
trend-structure panel already exposes raw `price_vs_vwap` and
`relative_volume` cells, but an operator scanning the panel must
mentally combine intraday price location (VWAP) with intraday
participation (RVOL) on every row to recover the composite
intraday-context state. This PRD adds one deterministic, read-only
composite cell that flattens `(price_vs_vwap, relative_volume)`
into a single short positional phrase using strictly structural
vocabulary — symmetric to the PRD-131 SMA composite but operating
on the orthogonal intraday axis, with zero feedback into
qualification, sizing, or any decision logic.

<= 60 production-code lines in
`cuttingboard/delivery/dashboard_renderer.py` (one literal mapping
table + one RVOL band classifier + one composite helper + one
render-site insertion), plus matching targeted tests. No new

GOAL
Add a deterministic, read-only **"Intraday Context"** cell to the
dashboard trend-structure panel that translates the
`(price_vs_vwap, relative_volume)` pair into a single short
structural phrase using strictly positional vocabulary
(no narrative, no forecast, no quality ranking) — leaving every
artifact, contract, runtime, qualification, sizing, payload,
notifier, and decision surface unchanged.

REQUIREMENTS

R1 — Deterministic (VWAP, RVOL band) composite display mapping
The helper `_trend_structure_intraday_display(record)` MUST be a
pure, total function over the enumerated `(price_vs_vwap,
rvol_band)` domain. The mapping table is closed, binding, and
symmetric (3 VWAP comparison tokens × 3 RVOL bands = 9 rows):

| price_vs_vwap | rvol_band    | Intraday Context display     |
|---------------|--------------|------------------------------|
| ABOVE         | AT_OR_ABOVE  | Above VWAP, RVOL >= 1.5x     |
| ABOVE         | BELOW        | Above VWAP, RVOL < 1.5x      |
| ABOVE         | UNAVAILABLE  | Above VWAP, RVOL unavailable |
| BELOW         | AT_OR_ABOVE  | Below VWAP, RVOL >= 1.5x     |
| BELOW         | BELOW        | Below VWAP, RVOL < 1.5x      |
| BELOW         | UNAVAILABLE  | Below VWAP, RVOL unavailable |
| AT_LEVEL      | AT_OR_ABOVE  | At VWAP, RVOL >= 1.5x        |
| AT_LEVEL      | BELOW        | At VWAP, RVOL < 1.5x         |
| AT_LEVEL      | UNAVAILABLE  | At VWAP, RVOL unavailable    |

The table is fully symmetric — `AT_LEVEL` preserves RVOL banding
because a key-level price state with observable participation is
operationally meaningful as a structural fact, and suppressing the
participation cell would discard information without justification.
There is no AT_LEVEL collapse and no asymmetry to defend.

Vocabulary discipline: the display strings are strictly mechanical
threshold-position phrases. `"RVOL >= 1.5x"` is literally the
numerical predicate `relative_volume >= 1.5`; `"RVOL < 1.5x"` is
literally its negation; `"RVOL unavailable"` is the data-state
sentinel. There are no magnitude adjectives ("elevated", "normal",
"high", "low", "heavy", "light", "strong", "weak", "firm", "soft")
and no quality language. The full PRD-131 forbidden-vocabulary
list applies unchanged, and the additional explicit deny-set
`{"elevated", "normal", "high", "low", "heavy", "light"}` is
asserted at test time.

If a future PRD tunes `_INTRADAY_RVOL_THRESHOLD` away from `1.5`,
the literal `"1.5x"` substring in every R1 display string MUST be
updated in lock-step. The display strings and the threshold
constant are an atomic unit.

FAIL: any input combination `(price_vs_vwap, rvol_band)` from the
enumerated comparison-token domain
`(ABOVE | BELOW | AT_LEVEL) × (AT_OR_ABOVE | BELOW | UNAVAILABLE)`
(3 × 3 = 9 rows) maps to a string not in the table above, OR any
composite display string contains a substring matching the PRD-131
forbidden vocabulary list (case-insensitive), OR any display
string contains a magnitude adjective from the explicit deny-set
above. Detected by a targeted test that enumerates all 9 R1 cells
and runs both literal-equality and forbidden-vocabulary regex
assertions.

R2 — VWAP unknown-state precedence
If `price_vs_vwap` equals an unknown-state token from PRD-130
(`DATA_UNAVAILABLE`, `NOT_COMPUTED`), the helper MUST short-circuit
and emit a deterministic phrase from the closed unknown-state set,
regardless of RVOL value:

1. `price_vs_vwap == "DATA_UNAVAILABLE"` →
   `"Intraday context unavailable"`.
2. `price_vs_vwap == "NOT_COMPUTED"` →
   `"VWAP not applicable"` (e.g., non-intraday session, daily bar
   resolution — the existing PRD-130 boundary).

VWAP unknown-state precedence wins over RVOL state because
without VWAP direction the intraday-context composite has no
anchor. No other unknown-state string may be emitted. The closed
unknown-state vocabulary is exactly:
`{"Intraday context unavailable", "VWAP not applicable"}`.

FAIL: any record where `price_vs_vwap ∈ {DATA_UNAVAILABLE,
NOT_COMPUTED}` produces a string outside the closed set above, OR
where the RVOL state overrides VWAP unknown-state precedence.
Detected by per-branch targeted tests.

R3 — Inactive-session and snapshot-absent short-circuit
The trend-structure panel's existing INACTIVE_SESSION_LABEL branch
and snapshot-absent branch (PRD-117 / PRD-120 / PRD-130 / PRD-131
inherited) MUST short-circuit *before* per-row intraday context
rendering. No intraday context cell is emitted on those branches.

FAIL: the panel renders an intraday context cell on an inactive-
session or missing-snapshot run. Detected by a targeted test
asserting the rendered fragment for those branches contains no

R4 — Pure / read-only / no decision feedback / vocabulary leak gate
`_intraday_rvol_band` and `_trend_structure_intraday_display` MUST
be pure functions: no global mutation, no I/O, no time-of-day
reads, no environment reads. Their outputs MUST NOT be read by
any module under `cuttingboard/` outside `cuttingboard/delivery/`.
No qualification gate, scoring module, sizing module, regime
module, output payload, notifier, market_map, or contract schema
gains a dependency on intraday context output.

R5 — RVOL band classifier semantics
`_intraday_rvol_band(rvol)` MUST be a pure function returning one
of three string tokens:

- `"UNAVAILABLE"` — `rvol is None`, `math.isnan(rvol)`, or
  `not math.isfinite(rvol)`.
- `"AT_OR_ABOVE"` — `rvol >= _INTRADAY_RVOL_THRESHOLD`
  (= 1.5 at first commit).
- `"BELOW"` — otherwise (includes 0.0 and any positive value
  below the threshold).

The classifier MUST NOT mutate the input, MUST NOT call any
external service, and MUST be deterministic. The threshold
`1.5` is fixed in this PRD; any future tuning is a separate PRD.

FAIL: any input outside the trichotomy above resolves to a string
not in `{"UNAVAILABLE", "AT_OR_ABOVE", "BELOW"}`, OR the threshold
boundary value (`rvol == 1.5`) classifies as `"BELOW"` instead of
`"AT_OR_ABOVE"`. Detected by a parametrized test over None, NaN,
+inf, -inf, 0.0, 1.0, 1.49, 1.5 (boundary), 1.51, 5.0.
