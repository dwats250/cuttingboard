# PRD-133 — Telegram Macro Pulse Alert Clarity

STATUS: COMPLETE (commit 391f84c)
LANE: STANDARD
CLASS: CONSUMER (downstream of contract; no decision-logic mutation)

## GOAL

Make the hourly Telegram alert a useful compact macro pulse rather than
a system heartbeat. Surface the market context already computed during
ingestion — macro drivers (VIX, DXY, 10Y, BTC) and the session's
tradables (SPY, QQQ, plus any commodity ETFs in the active universe) —
so the alert is informative at a glance without expanding scope into
data fetching, regime logic, decision-affecting modules, or delivery
behavior.

Target body shape for a STAY-FLAT hourly notification with the current
universe populated:

```
STAY FLAT — 12:04 PM PT

Regime: RISK OFF
Confidence: 0.50
Reason: stay flat posture

Macro Tape:
VIX 18.1 ↓1.5% | DXY 98.5 ↓0.2%
10Y 4.42 ↓0.7% | BTC 81.3K ↑1.1%

Tradables:
SPY 724.59 | QQQ 682.62
GLD 418.94 | SLV 66.28
XLE 59.71 | GDX 86.22

Focus: no active setup
```

The dashboard footer is appended downstream by `send_notification` and
is preserved verbatim.

## SCOPE

- Rewrite the hourly body produced by `format_hourly_notification` in
  `cuttingboard/notifications/__init__.py`.
- Route the already-computed `normalized_quotes` dict from runtime into
  the hourly formatter by adding one keyword argument at one call site
  in `cuttingboard/runtime.py` (line 541) and one parameter on the
  formatter signature. No new computation, fetch, persistence, or
  contract change.
- Drop redundant `Action:` line (action is encoded in the title).
- Drop redundant `Blockers:` line (low-signal in compact macro pulse).
- Drop the trailing ASCII `Generated: HH:MM ET` line — the title now
  carries a PT clock.
- Rename body's `State:` label to `Regime:` and, for the hourly path
  only, render `Regime: {regime}` (no ` / POSTURE` suffix; posture is
  in the title).
- Replace single-line `Macro: VIX …` with a two-row, two-column
  `Macro Tape:` block covering VIX / DXY / 10Y / BTC.
- Add a new `Tradables:` block listing up to six symbols
  (SPY, QQQ, GLD, SLV, XLE, GDX) in two columns, three rows, rendered
  only for symbols that are present in `normalized_quotes`.
- Replace `Focus: none` with `Focus: no active setup`. Populated form
  unchanged.
- Update four test files in lockstep.

## REQUIREMENTS

- **R1 — Routing.** `format_hourly_notification` MUST accept
  `normalized_quotes: dict[str, NormalizedQuote]` as a keyword
  argument. `cuttingboard/runtime.py:541` MUST pass the local
  `normalized_quotes` value.
  FAIL: signature missing parameter, OR call site does not pass it.

- **R2 — STAY-FLAT title shape.** Title MUST match
  `^STAY FLAT — \d{1,2}:\d{2} (AM|PM) PT$`.
  FAIL: regex does not match.

- **R3 — Other hourly titles preserved.** TRADE, MONITOR SETUP,
  MONITOR, and HALT hourly titles MUST be byte-identical to the
  pre-change implementation for identical input.
  FAIL: any non-STAY-FLAT hourly title diverges.

- **R4 — Removed lines.** The hourly body MUST NOT contain any line
  beginning with `Action:`, `Blockers:`, `State:`, `Generated:`, or
  the legacy single-line `Macro:` (the `Macro Tape:` block replaces
  it).
  FAIL: any such line appears in body.

- **R5 — Regime line.** Body MUST contain exactly one line of the
  form `Regime: {label}` where `{label}` is the existing
  `_compact_label(regime.regime)` output.
  FAIL: zero or more than one `Regime:` line, OR label format
  differs.

- **R6 — Macro Tape block.** Body MUST contain a `Macro Tape:`
  header followed by exactly two rows. Row 1 = VIX cell + DXY cell;
  Row 2 = 10Y cell + BTC cell. Cells are separated by ` | ` (single
  space, pipe, single space). All four cells carry pct change with
  arrow glued directly to the absolute pct (no space, no explicit
  sign — sign is encoded in the arrow). Cell formats:
  - VIX: `VIX {level:.1f} {arrow}{abs_pct:.1f}%`
  - DXY: `DXY {level:.1f} {arrow}{abs_pct:.1f}%`
  - 10Y: `10Y {level:.2f} {arrow}{abs_pct:.1f}%`
  - BTC: `BTC {btc_short} {arrow}{abs_pct:.1f}%` where `btc_short` =
    `f"{price/1000:.1f}K"` for price ≥ 1000, else `f"{price:.0f}"`.
  Arrow rule: `↓` when `pct_change_decimal < 0`, else `↑` (zero and
  positive both render `↑`). `abs_pct` is `abs(pct_change_decimal) *
  100`. No column padding — cells are rendered with single ` | `
  separator only; no trailing alignment whitespace.
  FAIL: header missing, row count ≠ 2, cell order wrong, arrow
  direction inverted for non-zero pct, pct sign rendered explicitly,
  space between arrow and pct, padding/alignment whitespace inside
  cells, BTC format wrong.

- **R7 — Macro Tape unavailable handling.** When any of the four
  macro symbols is missing from `normalized_quotes`, the
  corresponding cell MUST render as `{LABEL} n/a` (e.g. `DXY n/a`).
  Block structure (header + two rows) is preserved.
  FAIL: missing-symbol crashes formatter, OR omits row, OR renders
  partial cell.

- **R8 — Tradables block.** Body MUST contain a `Tradables:` header
  followed by up to three rows, each with up to two cells separated
  by ` | `. Symbols are rendered in this fixed order, skipping any
  not present in `normalized_quotes`:
  `SPY, QQQ, GLD, SLV, XLE, GDX`. Cell format:
  `{SYM} {price:.2f}`. Rows pack two cells at a time in order.
  When zero symbols are present, the entire `Tradables:` block
  (header + rows + surrounding blank line) MUST be omitted.
  FAIL: order differs, missing symbols are not skipped, row packing
  differs, zero-symbol case renders an empty header.

- **R9 — Focus line.** Empty focus renders `Focus: no active setup`.
  Populated focus form (`Focus: SYM DIR, …`) is unchanged.
  FAIL: body contains `Focus: none`, OR populated form changes.

- **R10 — Body line order.** For STAY-FLAT with no pending block and
  no lifecycle appendix, body MUST be exactly:
  ```
  Regime: …
  Confidence: …
  Reason: …

  Macro Tape:
  <row 1>
  <row 2>

  Tradables:                  ← omitted entirely when zero symbols
  <row(s)>

  Focus: …
  ```
  Blank lines separate the three blocks (Regime/Confidence/Reason,
  Macro Tape, Tradables, Focus); no leading or trailing whitespace
  on the body.
  FAIL: order or spacing differs.

- **R11 — Optional appendix preserved.** When a `Pending
  confirmation:` block exists or `_append_lifecycle_alerts` would
  append lifecycle lines, those blocks MUST be appended after the
  Focus line in their current relative position, separated by a single
  blank line, with shape unchanged.
  FAIL: appendix dropped, reordered, or reformatted.

- **R12 — Preserved behaviors (no change).** This PRD MUST NOT modify:
  - delivery transport (`send_notification`)
  - suppression logic (`should_suppress`)
  - alert trigger behavior (scheduling, notify-mode selection)
  - audit emission (records in `logs/audit.jsonl`)
  - dashboard footer (URL, label `View Dashboard:`, `---` separator)
  - lifecycle / setup-ready appendix shape
  - any non-hourly formatter
  - market data fetching, normalization, or validation
  - dashboard rendering or artifact generation
  FAIL: diff touches any of the above OR observable behavior changes
  for identical input.

  modified.
  FAIL: diff includes a path not in FILES.
