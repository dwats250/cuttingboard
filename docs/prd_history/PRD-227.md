# PRD-227 — Correct PRD-221 registry provenance (phantom 6b3fef1 → merged 8faa675)

Lane: MICRO
Status: IN PROGRESS

## Problem

PRD-221's recorded commit is `6b3fef1` in both `docs/PRD_REGISTRY.md` (the
PRD-221 row) and `docs/prd_index.json` (the number-221 entry). That SHA is a
phantom: `git cat-file -t 6b3fef1` → `Not a valid object name` — it never landed
on `main`. `main` squash-merges, so PRD-221 (PR #85) landed as a NEW squash
commit, `8faa675` (`PRD-221: level diagram — NOW marker, % distance, entry-gap
band (#85)`), which is an ancestor of `main`/HEAD and resolves. The recorded
`6b3fef1` is the pre-squash branch SHA. `tools/validate_prd_registry.py` flags
`PRD-221 hash 6b3fef1 does not resolve` under commit-resolvability.

Surfaced by the chatgpt-codex-connector audit of PR #85 (2026-07-03).

## Fix

Replace `6b3fef1` with `8faa675` at the two recorded sites. No source, no
behavior, no test change.

- `git cat-file -t 8faa675` → `commit`; `8faa675` is an ancestor of HEAD.
- `tools/validate_prd_registry.py` no longer lists `PRD-221` among unresolvable

- No change to any file under `cuttingboard/` or `tests/`.

  PROJECT_STATE). This PRD fixes ONLY PRD-221 (the audited row); a systematic
  resolvability sweep is a possible follow-up, deliberately out of scope here.
