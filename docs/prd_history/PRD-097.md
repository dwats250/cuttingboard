# PRD-097 — Dashboard Sidecar Freshness and Permission Clarity

**Status:** COMPLETE
**Commit:** —

---

## GOAL

Fix dashboard ambiguity caused by stale or mismatched sidecar artifacts after PRD-096,
and improve dashboard wording when the system is active but no candidate has qualified
for trade permission.

This PRD makes the dashboard clearly explain:
1. Whether the Candidate Board is using fresh or stale market_map data, with timestamps.
2. Why Permission may show no tradable setup during a non-halted regime.
3. Whether Macro Pressure is expandable and what data it contains.
4. Which dashboard sidecars are required for a complete render.

---

## REQUIREMENTS

### R1 — Sidecar dependency audit

Dashboard sidecar dependencies documented in a comment block at the top of
`dashboard_renderer.py`. Confirmed the hourly workflow renders HTML at CI time with
fresh `market_map.json` present. No sidecar publish change required.

Sidecars consumed:
- `logs/latest_payload.json` — primary payload (overridden by `--payload`)
- `logs/latest_run.json` — run metadata (overridden by `--run`)
- `logs/latest_hourly_contract.json` — contract entry prices
- `logs/market_map.json` — market context, loaded from `logs_dir/market_map.json`
- `logs/macro_drivers_snapshot.json` — macro driver fallback
- `logs/run_*.json` — history runs, globbed from `logs_dir`

### R2 — Stale market map diagnostics

When `market_map_stale_for_run` is True, the Candidate Board shows:

```
STALE MARKET MAP
Candidate Board paused because market_map timestamp is older than selected run.
Run: <run_timestamp_value or "unavailable">
Market map: <market_map_timestamp_value or "unavailable">
```

Uses existing `_timestamp_label()` which returns "unavailable" when parsed datetime is None.

### R3 — Sidecar publish (no-op)

Confirmed: hourly workflow renders HTML with fresh `market_map.json` in CI.
Published artifact is rendered HTML. No workflow change required.

### R4 — Permission null display

When `run.get("permission")` is None in a non-halted, non-stay-flat context:

```
PERMISSION
NO QUALIFIED SETUP
Reason: No candidate passed qualification.
```

If `first_error` is present, it is used as the reason instead of the default.
Source `run` dict is never mutated.

### R5 — Macro Pressure visibility

When pressure data exists (valid dict):
- Summary: `MACRO PRESSURE ▶`

When pressure data is missing or empty:
- Summary: `MACRO PRESSURE`
- Content: `MACRO PRESSURE UNAVAILABLE`
- Replaces: `NO PRESSURE DATA` and `FIELD_MISSING`
