PRD-032 — Catastrophic Output and Validation Contract Repair

STATUS
DEPRECATED

DEPRECATION NOTE
R1 (artifact enforcement on notify runs) was explicitly superseded: _execute_notify_run was
deliberately kept as a lightweight path with tests asserting it does NOT write artifacts.
R2–R6 were implemented across PRDs 033–048. R7 test coverage exists for all implemented
requirements. Deprecating rather than completing because R1 is now an intentional non-requirement.

ROOT CAUSE
Missing FAIL conditions in prior PRDs allowed:
- non-canonical runtime paths (notify modes)
- notification/output inconsistency
- stale data propagation
- invalid timestamp acceptance
- fail-open gating behavior

GOAL
Enforce a single canonical outcome and eliminate all silent data or control-flow violations that can permit invalid trades or misleading output.

SCOPE
- Enforce canonical artifact generation for ALL runtime execution paths.
- Make notification titles derive strictly from validated trade readiness.
- Eliminate stale OHLCV fallback in live ingestion.
- Reject future-dated quotes during validation.
- Make late-session gate fail closed with explicit logging.
- Add regression tests covering each repaired invariant.

OUT OF SCOPE
- Strategy logic, signals, scoring, or universe changes.
- Regime, qualification, or chain-validation thresholds.
- Pipeline architecture redesign.
- Notification transport, retry, or suppression logic (except title correctness).
- UI, payload structure, or report formatting changes.
- Legacy scripts (`run_intraday.py`, `watch.py`, etc).

REQUIREMENTS

R1 — Canonical Artifact Enforcement (All Modes)
All execution paths entering `execute_run()` MUST produce:
- logs/latest_run.json
- a canonical contract
- a markdown report

This includes ALL notify modes.

`_execute_notify_run()` MUST NOT return a partial dict or bypass artifact generation.

Implementation constraint:
- Either route notify modes through the same pipeline result builder
- OR `_execute_notify_run()` MUST explicitly construct and persist:
  - canonical contract
  - canonical outcome
  - run summary

FAIL:
- Any notify-mode run completes without writing logs/latest_run.json AND a contract.

---

R2 — Explicit Canonical Outcome Field
The canonical contract MUST include:

    contract["outcome"] ∈ {"TRADE", "NO_TRADE", "HALT"}

Outcome derivation MUST follow:

- HALT:
  validation_summary.system_halted == True

- TRADE:
  ≥1 candidate where setup_quality == "TOP_TRADE_VALIDATED"

- NO_TRADE:
  all other cases

Constraints:
- Outcome MUST NOT be inferred from regime.tradable
- Outcome MUST NOT be hard-coded in any branch

FAIL:
- contract["outcome"] missing
- contract["outcome"] not in allowed set
- contract["outcome"] contradicts validated candidates

---

R3 — Notification Title Must Bind to Outcome
`build_notification_message()` MUST derive title ONLY from:

- contract["outcome"]
- validated candidate presence

Mapping:

- HALT → "HALT - SYSTEM ERROR"
- TRADE → "TRADE READY"
- NO_TRADE:
    if candidates exist → "WATCHLIST - SETUPS FORMING"
    else → "NO TRADE - SYSTEM ACTIVE"

Explicit prohibition:
- MUST NOT use system_state.tradable as a deciding factor

FAIL:
- contract["outcome"] == "NO_TRADE" AND title == "TRADE READY"
- contract["outcome"] == "TRADE" AND title != "TRADE READY"

---

R4 — Stale OHLCV Rejection in Live Ingestion
Definition:

A cache is STALE if:
    (now_utc - last_bar_timestamp) >= config.FRESHNESS_SECONDS

Behavior:

If fresh OHLCV fetch fails:
- STALE cache MUST NOT be returned
- Function MUST return None

Allowed:
- Fresh cache (within freshness threshold)
- Explicit fixture/cache-only modes (outside live ingestion)

Derived layer requirement:
- None input MUST result in "metrics unavailable" (not computed)

FAIL:
- stale cache returned after fresh fetch failure
- derived metrics computed from stale OHLCV

---

R5 — Future Timestamp Rejection
Validation MUST enforce:

    age_seconds = now_utc - fetched_at_utc

Constraints:
- age_seconds < 0 beyond allowed skew → INVALID
- allowed skew MUST be explicitly defined:

    MAX_CLOCK_SKEW_SECONDS (new config constant, e.g. 5s)

Behavior:
- Any quote violating skew → rejected
- If symbol is HALT → system_halted = True

FAIL:
- future-dated quote passes validation
- HALT symbol with future timestamp does not halt system

---

R6 — Late Session Gate Fail-Closed
`_is_late_session()` MUST:

- Wrap all time calls in try/except
- On exception:
    - logger.exception(...)
    - return True  # block entries

Constraint:
- Entry MUST NEVER be permitted if cutoff cannot be evaluated

FAIL:
- exception path returns False
- exception path allows entry

---

R7 — Regression Test Coverage (Mandatory)
Each requirement MUST have a dedicated failing test prior to fix:

1. Notify-mode artifact enforcement
2. contract["outcome"] correctness
3. Notification title alignment with outcome
4. Stale OHLCV rejection (fetch failure path)
5. Future timestamp rejection (+ HALT case)
6. Late-session exception fail-closed

Tests MUST:
- deterministically fail pre-fix
- pass post-fix

FAIL:
- any requirement lacks a direct regression test

---

DATA FLOW
execute_run → pipeline → contract["outcome"] → artifacts (summary, contract, report)
contract → notification builder → title/body
ingestion → OHLCV freshness gate → derived input or None
validation → timestamp gate → valid_quotes/system_halted
time_utils → cutoff evaluation → qualification gate

---

FAIL CONDITIONS
- Missing or multiple outcomes
- Notify-mode run without artifacts
- Notification contradicts contract outcome
- Stale OHLCV used in live ingestion
- Future-dated quote accepted
- HALT symbol fails to halt on invalid timestamp
- Late-session exception allows entry
- Missing regression test for any requirement
