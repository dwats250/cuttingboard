GOAL
Close the false-green package-swap trap at its config root: pytest with
default prepend import mode plus the 0-byte tests/__init__.py inserts
the repo root at sys.path[0], AHEAD of PYTHONPATH, so a package-swap
check (mutation, pre/post comparison) silently tests the repo's own
package. Bitten twice (PRD-262.review.claude.md:91-94; PRD-263
validation); the behavioral rule is docs/DECISIONS.md 2026-07-19, which
names this PRD as its closure. CI already installs the package
(`pip install -e ".[dev]"`) and invokes the bare `pytest` binary, so
removing the rootdir insertion cannot break imports: resolution order
becomes PYTHONPATH (when swapped) -> site-packages editable install
(normal runs) — exactly the order a swap needs.

SCOPE
- Remove the pytest rootdir insertion: delete the 0-byte
  tests/__init__.py and/or add an explicit
  `[tool.pytest.ini_options]` import mode to pyproject.toml (exact
  mechanism chosen at implementation; whichever combination makes R2
  pass with the full suite green).
- Make the "print-and-check the resolved package" half of the
  DECISIONS rule automatic: a session-start line in tests/conftest.py
  printing the resolved `cuttingboard.__file__` on every pytest run.
- Documented limitation (recorded here, not fixable by pytest config):
  `python3 -m pytest` prepends the CWD at interpreter startup, ahead of
  everything — the printed probe line from R1 is the guard on that
  invocation form.
- Track this PRD in registry/state per existing convention.
