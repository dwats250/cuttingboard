PRD-029 — Level Awareness Layer (Derived Levels Only)

STATUS
COMPLETE
Commit: 57c23f9

GOAL
Introduce deterministic, derived price levels sourced from contract and run_history
to anchor premarket scenarios, invalidation logic, and postmarket evaluation without
modifying the execution pipeline or contract schema.

SCOPE
- Add cuttingboard/reports/levels.py: derive_key_levels(contract, run_history) → dict
- Update cuttingboard/reports/premarket.py: inject levels dict into scenario and invalidation logic
- Update cuttingboard/reports/postmarket.py: use level interaction to classify
  MATCH / PARTIAL / MISS / NO_EXPECTATION
- Add tests/test_levels.py: unit and determinism tests for levels layer

OUT OF SCOPE
- No changes to execution pipeline (layers 1–10)
- No new data providers or external API calls
- No intraday bar processing
- No VWAP or ORB logic
- No contract schema modification

FILES
A cuttingboard/reports/levels.py
M cuttingboard/reports/premarket.py
M cuttingboard/reports/postmarket.py
A tests/test_levels.py

REQUIREMENTS

R1 — Level Derivation Function
derive_key_levels(contract, run_history) MUST return a dict with exactly these keys:
prior_high, prior_low, prior_close, current_price, gap_direction, range_mid.
No additional keys. No missing keys.

FAIL: derive_key_levels returns a dict with any missing or extra top-level key.

R2 — Output Shape
The return dict MUST conform to:
{
  "prior_high":     float | null,
  "prior_low":      float | null,
  "prior_close":    float | null,
  "current_price":  float | null,
  "gap_direction":  "UP" | "DOWN" | "FLAT" | null,
  "range_mid":      float | null
}

FAIL: Any value is outside its declared type or enumeration.

R3 — Prior Values Source
prior_high, prior_low, prior_close MUST be sourced from the most recent run_history
entry where status != "ERROR".

"most recent" MUST be defined as:
- the last element in run_history after chronological sorting by timestamp.

If no valid entry exists, all three MUST be null.

FAIL:
- A prior_* value is non-null when no valid run_history entry exists
- OR a prior_* value is sourced from an entry where status == "ERROR"
- OR a prior_* value is sourced from any entry other than the last valid chronological entry

R4 — Current Price Source
current_price MUST be sourced from contract artifacts if present, else null.
No external fetch is permitted.

FAIL:
- current_price is non-null when no contract artifact provides it
- OR any external data source is accessed

R5 — Gap Direction Classification
gap_direction MUST be determined as:
- "UP"   if current_price > prior_close * 1.001
- "DOWN" if current_price < prior_close * 0.999
- "FLAT" otherwise
- null   if current_price or prior_close is null

FAIL:
- gap_direction does not match the threshold rule
- OR gap_direction is non-null when either input is null

R6 — Range Midpoint
range_mid MUST equal (prior_high + prior_low) / 2 when both are non-null.
range_mid MUST be null when either prior_high or prior_low is null.

FAIL:
- range_mid is non-null when either bound is null
- OR range_mid deviates from (prior_high + prior_low) / 2

R7 — Premarket Scenario Level References
Each premarket scenario.condition MUST contain at least one exact token from:
- "prior_high"
- "prior_low"
- "range_mid"
- "gap_direction"

FAIL:
Any scenario.condition string that does not contain at least one exact token from the allowed set.

R8 — Premarket Invalidation Level Reference
The premarket invalidation block MUST contain at least one condition expressed in terms of a derived level:
prior_high, prior_low, range_mid, or gap_direction.

FAIL:
The invalidation block contains zero level-based conditions.

R9 — Postmarket Classification
build_postmarket_report MUST classify expectation_vs_reality as exactly one of:
MATCH, PARTIAL, MISS, NO_EXPECTATION.

Classification MUST follow:

MATCH:
- preferred_direction aligns with realized direction
- AND final_price is on the expected side of the referenced level

PARTIAL:
- preferred_direction aligns with realized direction
- BUT final_price is not clearly on the expected side of the level

MISS:
- preferred_direction does not align with realized direction

NO_EXPECTATION:
- no prior scenario to evaluate against

FAIL:
- expectation_vs_reality contains any value outside {MATCH, PARTIAL, MISS, NO_EXPECTATION}
- OR classification does not follow the defined rules

R10 — Level Interaction Simplification
Level interaction MUST be determined using final_price vs level comparison only.
No intraday bar data is permitted.

FAIL:
Implementation requires intraday data to compute level interaction.

R11 — Determinism
Given identical (contract, run_history) inputs, derive_key_levels MUST return identical output.

FAIL:
Two calls with identical inputs return different output.

R12 — No Mutation
derive_key_levels MUST NOT modify any contract field or write to any file.

FAIL:
- Any contract field differs before and after execution
- OR any file is written

DATA FLOW
run_history → derive_key_levels → levels dict
contract artifacts → derive_key_levels → levels dict
levels dict → build_premarket_report
levels dict → build_postmarket_report

FAIL CONDITIONS
- Missing or extra keys in levels dict
- Invalid types or enum values
- Incorrect run_history selection
- External data access
- Incorrect gap_direction classification
- Invalid range_mid calculation
- Missing level tokens in scenarios
- Missing level-based invalidation
- Invalid classification output
- Intraday data dependency introduced
- Non-deterministic output
- Contract mutation or file write

VALIDATION
Run: pytest tests/test_levels.py -q

Tests MUST cover:
- gap_direction boundary conditions at ±0.1%
- gap_direction null handling
- prior_* null when no valid run_history exists
- deterministic selection of most recent valid entry
- range_mid correctness and null handling
- scenario strings contain exact level tokens
- invalidation contains level references
- classification correctness for MATCH / PARTIAL / MISS / NO_EXPECTATION
- determinism across repeated calls
- no external imports in levels.py
- no contract mutation before/after execution
