CLASS
CONSUMER

WHY NOW
The A+ candidate card is long — IF NOW / ENTRY / INVALIDATION / RISK / REASON /
PLAY / WATCH / WATCH — so on mobile the falsifiable trade fields (ENTRY,
INVALIDATION) scroll past inside a wall of uniform-weight monospace. The palette
uses red/green for direction and amber for caution but has no cue that marks the
one thing to act on.

MAX EXPECTED DELTA
45 production LOC (dashboard_renderer.py _render_candidate_card + _CSS).

GOAL
Tuck REASON/PLAY/WATCH behind a default-collapsed disclosure and give ENTRY and
INVALIDATION a distinct cyan "actionable" accent, so the eye lands on the
executable, falsifiable trade fields instead of the scaffolding.

SCOPE
- Move REASON, PLAY, and WATCH lines of the high-grade card into a
  default-collapsed `<details>` ("DETAIL ▶"); keep SYMBOL/GRADE/BIAS/STRUCTURE/
  STATE/IF NOW/ENTRY/INVALIDATION/RISK visible.
- Add one _CSS class `.value-actionable` (cyan `#29b6f6`, the existing
  VWAP/level colour → reads as "your executable level") and apply it to the
  ENTRY and INVALIDATION value elements (replacing plain `.value-key`).

OUT OF SCOPE
- Applying the accent to grade rails, macro-bias, or any directional element
  (would collide with the red/green/amber semantic).
- Changing field values, grade logic, or the level diagram (PRD-074-owned).

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dashboard_renderer.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — `_render_candidate_card`
(~L1555–1651) and the _CSS constant (~L639–744): add one colour class, wrap three
fields in `<details>`, and swap ENTRY/INVALIDATION to the accent class. No change
to which fields exist or their values.

REQUIREMENTS

R1 — REASON/PLAY/WATCH collapsed
In a high-grade card, REASON/PLAY/WATCH render inside a default-collapsed
`<details>`; ENTRY/INVALIDATION/RISK/IF NOW remain outside it.
FAIL: REASON, PLAY, or WATCH renders outside a `<details>` in a high-grade card.

R2 — Actionable accent on trade fields
The ENTRY and INVALIDATION value elements carry a `.value-actionable` class
defined once in _CSS with colour `#29b6f6`.
FAIL: the ENTRY or INVALIDATION value lacks `.value-actionable`, or the class is
undefined in _CSS.

R3 — No semantic clash
`.value-actionable`'s hex differs from the directional (`#4caf50`/`#f44336`) and
caution (`#ff9800`) hexes and is not applied to any grade-rail or macro-bias
element.
FAIL: the accent hex equals a directional/caution hex, or appears on a
`grade-*` / `macro-bias` element.

DATA FLOW
candidate entry fields → _render_candidate_card → ENTRY/INVALIDATION get accent
class; REASON/PLAY/WATCH wrapped in `<details>` → HTML
_CSS constant → `.value-actionable{color:#29b6f6}` definition

FAIL CONDITIONS
- REASON/PLAY/WATCH rendered outside `<details>` in a high-grade card.
- ENTRY/INVALIDATION value missing `.value-actionable`, or class undefined.
- Accent hex collides with a directional/caution colour or lands on a grade/bias
  element.

VALIDATION
Manual:
- Render a high-grade card; ENTRY and INVALIDATION are cyan and visually dominant;
  REASON/PLAY/WATCH are hidden until the disclosure is expanded.
Automated:
- Update test_high_grade_candidate_entry_invalidation_bold (accent class) and the
  PLAY/WATCH/REASON presence tests (inside disclosure); new test asserts the
  accent hex is disjoint from directional/caution hexes.
