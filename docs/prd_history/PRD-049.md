PRD-049 — Development Process Hardening

STATUS
COMPLETE

GOAL
Eliminate five process gaps: stale active_prd.txt state, no CI test run, no linting gate,
dead next_action snapshot field, and no test verification before git commit.

SCOPE
- Set active_prd.txt to "none" (no active PRD)
- Add pytest and ruff steps to CI before the pipeline runs
- Add ruff to dev dependencies in pyproject.toml
- Add advisory ruff check to test_gate.sh (non-blocking, printed warning only)
- Add blocking ruff + pytest run to git_gate.sh after approval, before commit proceeds
- Remove dead next_action field from stop_snapshot.sh

OUT OF SCOPE
- Changes to qualification, regime, output, or any pipeline logic
- Changes to notification or report formatting
- Any new hook beyond modifying existing ones
- Changes to test content or test structure
- CI schedule, modes, or artifact commit behavior

REQUIREMENTS

R1 — Stale active_prd.txt Cleared
.claude/state/active_prd.txt MUST contain exactly the string "none" with no trailing
whitespace or newline variation that would cause string comparison failures.

FAIL: cat .claude/state/active_prd.txt produces anything other than "none".

---

R2 — ruff Added to Dev Dependencies
pyproject.toml [project.optional-dependencies] dev section MUST include ruff>=0.4.0.
No [tool.ruff] config section is required — default ruff rules apply.

FAIL: pip install -e ".[dev]" does not install ruff.

---

R3 — CI Runs Tests and Linting Before Pipeline
cuttingboard.yml MUST contain, as the first substantive job steps (before engine doctor):

Step 1 — Install dev deps:
    pip install -e ".[dev]"

Step 2 — Lint:
    ruff check cuttingboard/ tests/

Step 3 — Test:
    pytest tests/ -q

Both ruff and pytest steps MUST use `set -eo pipefail` and MUST be blocking (non-zero exit
fails the job). They MUST appear before the "Engine health check" step.

FAIL: CI job completes successfully with a file containing a ruff error.
FAIL: CI job completes successfully with a failing test.
FAIL: ruff or pytest steps appear after the engine doctor step.

---

R4 — Advisory Ruff in test_gate.sh
test_gate.sh MUST run `ruff check "$FILE_PATH"` after pytest completes (pass or fail
is irrelevant — ruff always exits 0 in this hook).

Behavior:
- If ruff finds issues: print "[test_gate] Ruff: <output>" to stderr and continue
- If ruff finds no issues: print nothing for ruff
- ruff check MUST NOT cause test_gate.sh to exit non-zero on its own

FAIL: test_gate.sh exits non-zero due to ruff output alone.
FAIL: ruff is not called in test_gate.sh.

---

R5 — Blocking Ruff + Pytest in git_gate.sh Before Commit
After approval is confirmed (APPROVED == 1), git_gate.sh MUST run:

1. ruff check cuttingboard/ tests/
   - On failure: print "[git_gate] BLOCKED: ruff errors must be fixed before commit." to stderr and exit 1
2. pytest tests/ -q
   - On failure: print "[git_gate] BLOCKED: tests must pass before commit." to stderr and exit 1

These checks MUST only run after approval is confirmed — not on unapproved commit attempts.

FAIL: git commit proceeds with ruff errors present.
FAIL: git commit proceeds with failing tests.
FAIL: ruff or pytest runs on an unapproved commit attempt (wastes time, wrong order).

---

R6 — Dead next_action Field Removed from Snapshot
stop_snapshot.sh MUST NOT include next_action in the snapshot dict or printed output
if .claude/state/next_action.txt does not exist or is empty.

If the file exists and is non-empty, include it. If missing or empty, omit the key entirely
from the JSON and from the printed summary lines.

FAIL: snapshot.json contains "next_action": "unknown".
FAIL: snapshot.json contains next_action key when next_action.txt is missing or empty.

---

DATA FLOW
active_prd.txt → protect_files.sh reads it → must be "none" with no active PRD
pyproject.toml dev deps → pip install -e ".[dev]" → ruff available in CI and locally
CI: install dev deps → ruff check → pytest → engine doctor → pipeline
test_gate.sh: pytest → ruff check (advisory, non-blocking) → exit
git_gate.sh: dirty check → approval check → ruff check (blocking) → pytest (blocking) → exit 0
stop_snapshot.sh: read active_prd, branch, commit, timestamp → write snapshot.json (no next_action if file absent)

FAIL CONDITIONS
- active_prd.txt contains anything other than "none"
- ruff not installable via pip install -e ".[dev]"
- CI succeeds with ruff errors in cuttingboard/ or tests/
- CI succeeds with failing tests
- CI ruff/pytest steps appear after engine doctor
- test_gate.sh exits non-zero due to ruff alone
- git commit proceeds with ruff errors
- git commit proceeds with failing tests
- ruff/pytest runs in git_gate.sh on unapproved commit attempt
- snapshot.json contains next_action key when next_action.txt is absent or empty

VALIDATION
Manual:

1. cat .claude/state/active_prd.txt
   → must print exactly: none

2. pip install -e ".[dev]" && ruff --version
   → exits 0, prints ruff version

3. In CI workflow file, confirm ruff and pytest steps appear before "Engine health check" step
   → grep -n "ruff\|pytest\|Engine health" .github/workflows/cuttingboard.yml shows ruff/pytest at lower line numbers

4. Edit a Python file with a known ruff error (e.g., unused import); save
   → test_gate.sh prints ruff warning to stderr but does NOT exit non-zero

5. With ruff errors present, type APPROVE COMMIT and attempt git commit
   → git_gate.sh prints BLOCKED: ruff errors and exits 1, commit does not proceed

6. With ruff clean but a failing test, type APPROVE COMMIT and attempt git commit
   → git_gate.sh prints BLOCKED: tests must pass and exits 1, commit does not proceed

7. Without APPROVE COMMIT, attempt git commit with failing tests
   → git_gate.sh blocks at approval check, does NOT run ruff or pytest

8. cat .claude/state/snapshot.json after session stop
   → next_action key absent from JSON if next_action.txt does not exist
