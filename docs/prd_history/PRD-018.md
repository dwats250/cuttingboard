PRD-018 — Notification Signal Hierarchy and Suppression

STATUS
COMPLETE
Commit: 0f7c341

GOAL
Add state-key-based notification deduplication and a priority classification system so repeated identical signals are suppressed and high-priority signals always break through.

SCOPE
- Add notifications/state.py: notification_state_key(), classify_notification_priority(), load/save_last_state(), should_send() — pure functions, no side effects
- Update audit.py: add priority and state_key fields to write_notification_audit
- Update output.py: thread notification_priority and notification_state_key as keyword-only args
- Update runtime.py: suppression gate before send; save_last_state only on confirmed successful send
- Add tests/test_prd018_notification_suppression.py (48 tests)

OUT OF SCOPE
- No changes to retry or rate limit logic
- No changes to Telegram transport

REQUIREMENTS

R1 — State Key
notification_state_key() MUST produce a deterministic string from contract fields.
Same contract state MUST always produce the same key.

FAIL: Two calls with identical contract produce different state keys.

R2 — Priority Classification
classify_notification_priority() MUST return one of: CRITICAL / HIGH / MEDIUM / LOW.
No other values MUST be returned.

FAIL: classify_notification_priority() returns a value outside the four defined levels.

R3 — Suppression Gate
runtime.py MUST call should_send() before send_notification.
If should_send() returns False, send_notification MUST NOT be called.

FAIL: send_notification is called when should_send() returns False.

R4 — State Save Timing
save_last_state() MUST be called only after a confirmed successful send.
A failed send MUST NOT update last_state.

FAIL: save_last_state() is called after a failed send attempt.

R5 — Audit Fields
write_notification_audit MUST include priority and state_key on every call.

FAIL: Audit record missing priority or state_key.

DATA FLOW
contract → notification_state_key() → should_send() → suppression decision
confirmed send → save_last_state()
all paths → write_notification_audit(priority, state_key)

FAIL CONDITIONS
- Non-deterministic state key
- send_notification called when suppressed
- State saved after failed send
- Audit record missing priority or state_key

VALIDATION
Run: pytest tests/test_prd018_notification_suppression.py -q (48 tests)
Manual: send identical signal twice — confirm second send is suppressed in audit log.
