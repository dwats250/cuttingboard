# PRD-100 - Standardize Artifact Push Rebase Contract

**Status:** COMPLETE
**Commit:** e983bd0

## Goal

Eliminate push race conditions in CI by centralizing artifact push logic into a shared helper with a deterministic fetch/rebase/push sequence, bounded retry, and no force push.

## Scope

- Introduce `tools/ci_push_artifacts.sh` as the single shared artifact push helper.
- Replace inline push logic in both workflow files with a call to the helper.
- Enforce fetch → rebase → push sequence with one bounded fetch/rebase retry on failure.
- Preserve no-change exits when pre/post SHA are identical or missing.

## Out Of Scope

- Python runtime code, dashboard renderer, qualification, regime, or notification logic.
- Changes to artifact content or generation ordering (covered by PRD-099).
- Force push or `git pull` usage.

## Files

- A `tools/ci_push_artifacts.sh`
- M `.github/workflows/cuttingboard.yml`
- M `.github/workflows/hourly_alert.yml`
- M `docs/PRD_REGISTRY.md`
- M `docs/prd_history/PRD-100.md`

## Requirements

- R1: A single `tools/ci_push_artifacts.sh` helper implements all artifact push logic.
- R2: Helper exits 0 with no push when `PRE_SHA` or `POST_SHA` is missing.
- R3: Helper exits 0 with no push when `PRE_SHA == POST_SHA` (no changes committed).
- R4: Push sequence is: `git fetch origin main` → `git rebase origin/main` → first push attempt.
- R5: On first push failure, one bounded retry: `git fetch origin main` → `git rebase origin/main` → retry push.
- R6: Helper exits 1 if retry push also fails.
- R7: No `git pull` and no `--force` push anywhere in the helper.
- R8: Both workflow files delegate to the helper; no inline push logic remains.

## Data Flow

CI workflow captures `PRE_SHA` before artifact commits and `POST_SHA` after. Helper receives both via env vars, short-circuits on no-change, then executes the rebase/push contract. Workflows no longer contain push logic directly.

## Fail Conditions

- Helper contains `git pull` or `git push --force`.
- Inline push logic remains in workflow files after migration.
- No-change condition (equal SHAs) triggers a push attempt.
- More than one fetch/rebase retry cycle on failure.
