# PRD-076 — Dashboard Live Publishing and Layout Finalization

**Status:** COMPLETE
**Commit:** 3c6bb76
**Depends on:** PRD-073

---

## GOAL

Deploy the Signal Forge dashboard to GitHub Pages automatically on every hourly run, and finalize the visual layout — promoting System State to the top, folding run-health in, removing the redundant posture field, and correcting the Sunday run schedule.

All changes are delivery/CI/cosmetic. No decision logic, qualification gates, or contract fields are affected.

---

## SCOPE

- Mirror `ui/dashboard.html` → `ui/index.html` after each hourly write (CI)
- Pages workflow triggers automatically on hourly alert workflow completion
- System State section promoted to top of dashboard
- Run-health fields folded into System State block (no separate section)
- `posture` field removed from dashboard render (redundant with regime/outcome)
- No-trade banner repositioned below System State
- Dashboard section order finalized
- Sunday hourly cron corrected to 23:30 UTC (30 min after futures open)
- Runtime log artifacts excluded from git tracking via `.gitignore`

---

## OUT OF SCOPE

- No changes to decision logic, qualification gates, or regime engine
- No changes to contract fields, payload schema, or any artifact structure
- No notification changes
- No new data sources
- No scoring changes

---

## REQUIREMENTS

### R1 — Live mirror on every hourly write

After `dashboard_renderer.py` writes `ui/dashboard.html`, the CI copies it to `ui/index.html` and commits both.

FAIL: `ui/index.html` is not updated after a successful hourly run.

---

### R2 — Pages auto-deploy on hourly completion

`pages.yml` triggers on `workflow_run` completion of "Cuttingboard Hourly Alert", not just on push to main.

FAIL: Pages deploy does not fire after a successful hourly run.

---

### R3 — System State at top, run-health folded in

Dashboard renders System State (regime, confidence, outcome, permission, action line) as the first block. No separate run-health section.

FAIL: System State is not the first block rendered.
FAIL: A separate run-health section appears.

---

### R4 — Posture field removed

`posture` is not rendered anywhere in the dashboard output.

FAIL: Any occurrence of posture label in rendered HTML.

---

### R5 — Sunday schedule corrected

`cuttingboard.yml` Sunday cron is `30 23 * * 0` (23:30 UTC).

FAIL: Sunday cron is set to any other schedule.

---

### R6 — No decision output changes

All contract fields, outcome values, policy decisions, and payload schema are identical before and after this PRD.

FAIL: Any decision-related field value changes.
