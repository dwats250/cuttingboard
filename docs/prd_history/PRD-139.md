# PRD-139 — Upstream Macro Collector Sidecar

- **LANE:** STANDARD
- **CLASS:** SIDECAR
- **STATUS:** COMPLETE
- **MAX EXPECTED DELTA:** ≤ 90 production LOC

---

## GOAL
Build an isolated upstream macro collector utility (`tools/macro_collector.py`) that synchronously pulls a fixed allowlist of RSS feeds, submits the concatenated text to the Anthropic API in structured-JSON mode, and atomically writes a typed summary snapshot to `logs/macro_regime_snapshot.json`. The utility is sidecar-only: no cuttingboard module imports it, and no cuttingboard module reads its output.

---

## SCOPE
- Frozen JSON schema contract for `logs/macro_regime_snapshot.json`.
- Standalone production-isolated utility at `tools/macro_collector.py`.
- Localized network resilience: per-feed 10s timeout, 3-attempt exponential backoff on HTTP fetches, single attempt for the LLM call.
- Offline-mocked test suite at `tests/test_macro_collector.py`.

---

## OUT OF SCOPE
- Modifying any of the 11 synchronous execution pipeline layers in `cuttingboard/`.
- Wiring the snapshot file into `cuttingboard/regime.py` or any decision input.
- Adding network-dependent client packages to the runtime import tree of `cuttingboard/runtime.py`.
- Loading config through `cuttingboard/config.py` (would break isolation).

## REQUIREMENTS

**R1 — Import Graph Isolation.** `tools/macro_collector.py` MUST NOT import any module from `cuttingboard/`, and no module under `cuttingboard/` may import `tools.macro_collector`.
- FAIL: `grep -rEn "^(from|import)\s+cuttingboard" tools/macro_collector.py` returns any match.
- FAIL: `grep -rEn "tools\.macro_collector|from\s+tools\s+import\s+macro_collector" cuttingboard/` returns any match.

**R2 — Atomic Standalone Execution.** Running `python3 tools/macro_collector.py` MUST: (a) load `ANTHROPIC_API_KEY` via standalone `os.environ` (with optional `python-dotenv` if available; no `cuttingboard.config` import); (b) synchronously fetch the FEEDS allowlist with a per-request 10s timeout and 3-attempt exponential backoff (1s, 2s, 4s); (c) submit concatenated feed text to the Anthropic SDK using a structured-output schema (tool-use or response_format) targeting model `claude-sonnet-4-6`; (d) write the result to `logs/macro_regime_snapshot.json.tmp` and `os.replace(...)` to the final path. The FEEDS allowlist is declared as a module-level constant containing at least: Federal Reserve press release feed, US Treasury press release feed, BLS news release feed. Adding/removing feeds requires a follow-up PRD.
- FAIL: a successful run does not produce or refresh `logs/macro_regime_snapshot.json` with all keys from R3 populated.
- FAIL: the temp file `logs/macro_regime_snapshot.json.tmp` remains on disk after a successful run.

**R3 — Schema Invariance and Contract Integrity.** The emitted JSON object MUST be a flat, non-nested dict with exactly these keys:
- `macro_bias`: one of `"RISK_ON" | "RISK_OFF" | "NEUTRAL" | "CHAOTIC"` (sidecar vocabulary; intentionally independent of `cuttingboard/regime.py`).
- `confidence_score`: float in `[0.0, 1.0]` inclusive.
- `fundamental_driver`: non-empty string, ≤ 280 chars.
- `timestamp`: UTC ISO-8601 string with explicit `+00:00` or `Z` suffix (no naive datetimes).
- FAIL: any key is missing, extra keys present, `macro_bias` outside enum, `confidence_score` outside `[0.0, 1.0]`, `timestamp` lacks tz suffix.

**R4 — Offline Test Suite.** `tests/test_macro_collector.py` MUST mock all RSS HTTP responses and the Anthropic SDK call. No test may make a live outbound network call. Tests MUST cover: (a) happy path produces a schema-valid snapshot, (b) HTTP failure after 3 retries exits 1 and leaves a pre-existing snapshot byte-identical, (c) LLM returns malformed JSON → exits 1, prior snapshot preserved, (d) `confidence_score` out of range → exits 1.
- FAIL: `pytest tests/test_macro_collector.py -q` performs any real network I/O.
- FAIL: any of (a)–(d) is not asserted.

---

## DATA FLOW

1. RSS allowlist (FEEDS constant) → synchronous `urllib`/`requests` GET, per-call 10s timeout, 3-attempt exponential backoff.
2. `tools/macro_collector.py` → concatenate feed text → build structured-output prompt for Anthropic SDK.
3. Anthropic API (`claude-sonnet-4-6`) → returns structured JSON conforming to R3 schema.
4. `tools/macro_collector.py` → write `logs/macro_regime_snapshot.json.tmp` → `os.replace` → final path.

No cuttingboard module reads or writes any node in this flow.

---

## FAIL CONDITIONS

- **Upstream Deprivation:** All RSS endpoints fail or return non-200 after 3 exponential backoff attempts. Action: emit a single JSON-formatted error line to stderr, exit code 1, leave any prior `logs/macro_regime_snapshot.json` byte-identical (no truncation, no temp leftover).
- **LLM Schema Mismatch:** Anthropic response is not parseable JSON, fails schema validation (R3), or violates the `confidence_score` range. Action: raise `ValueError` with a one-line diagnostic, exit code 1, leave any prior snapshot byte-identical.
- **Missing Credentials:** `ANTHROPIC_API_KEY` env var unset. Action: emit one stderr error line, exit code 1, do not initiate any network call.
