PRD-267 — Coverage-reason surfacing (daily): fix 80-char truncation dropping the vote-coverage clause

Status: IN PROGRESS

LANE
STANDARD

CLASS
CONSUMER

WHY NOW
output.py::_alert_reason's naive [:80] truncation cuts the vote-coverage clause off the end of an already-long stay_flat_reason string on the daily compact alert (sol C2, PRD-263 fast-follow).

SPLIT NOTE
This PRD originally scoped both the daily and hourly halves of the sol C2
gap (as a parked Stage-0 scaffold on branch prd-267-coverage-reason-
surfacing). Split at authoring time: the daily half needs only output.py
(this PRD, STANDARD); the hourly half requires a notification formatter
to couple to qualification.py's trading-logic internals, or a change to
when qualify_all runs -- HIGH-RISK on merit, carried to PRD-268 (queued,
scaffold only).

MAX EXPECTED DELTA
~40 production lines in output.py (rewritten _alert_reason plus one
regex constant), one new test file (~50-70 lines). No other file's
logic changes.

GOAL
The daily compact alert's "Reason: ..." line always includes the
vote-coverage clause (e.g. "5/8 votes cast") when stay_flat_reason
carries one, instead of silently losing it to the existing 80-character
truncation.

SCOPE
- Rewrite output.py::_alert_reason so that when the resolved reason
  string contains a vote-coverage clause matching the pattern
  "<N>/<M> votes cast" (0 < N < M, per qualification.py's
  _check_regime_gates), that exact substring is preserved in the
  rendered output even when the full reason exceeds 80 characters —
  by truncating the preceding narrative text to fit the remaining
  budget, not the clause itself.
- Reasons without a vote-coverage clause keep today's exact behavior:
  str(reason).replace("\n", " ")[:80].
- Add a mutation-verified red test proving the clause survives
  truncation, plus a regression check that the no-clause path is
  unchanged.
- No change to how stay_flat_reason itself is built (qualification.py,
  contract.py, runtime/__init__.py untouched) — the daily path already
  runs qualify_all regardless of posture (runtime/__init__.py:391-424),
  so the rich reason already reaches contract["system_state"]
  ["stay_flat_reason"]; this PRD is a rendering-layer fix only.

OUT OF SCOPE
- The hourly path (notifications/__init__.py, qualification.py) — split
  to PRD-268 (HIGH-RISK, scaffolded, not authored): qualify_all is
  skipped on STAY_FLAT for hourly (runtime/__init__.py:426), so
  _hourly_reason has no rich reason to format at all; that is a
  different defect (missing data, not truncated data) needing a
  protected-file change.
- Any change to the 80-character budget itself, the message format
  beyond the Reason line, or qualification.py's gate logic.
- audit_summary.get("regime_failure_reason") and contract.get
  ("error_detail") fallback branches in _alert_reason — observed to be
  dead in the current runtime (nothing populates
  contract["audit_summary"]["regime_failure_reason"]) but out of scope
  to remove or fix here; noted, not touched.

REQUIREMENTS

R1 — Vote-coverage clause survives 80-char truncation
When contract["system_state"]["stay_flat_reason"] contains a clause
matching r"(\d+)/(\d+) votes cast" with 0 < N < M,
output.py::_alert_reason(contract, has_candidates=False) returns a
string containing that exact "<N>/<M> votes cast" substring, and the
returned string's length is <= 80.
FAIL: given a stay_flat_reason containing "N/M votes cast" (0 < N < M)
whose full text exceeds 80 characters, _alert_reason's return value
does not contain "N/M votes cast", or exceeds 80 characters.

R2 — No regression for reasons without a vote-coverage clause
Any reason without a "<N>/<M> votes cast" substring (including the
existing "no setups" fallback and "STAY_FLAT posture" with full vote
coverage, i.e. total_votes == len(_VOTE_KEYS)) is truncated exactly as
before the change: str(reason).replace("\n", " ")[:80].
FAIL: tests/test_prd017_notification_stabilization.py's existing
"Reason: no setups" assertion (test_stay_flat_prd_shape) fails, or any
non-coverage reason's truncated output differs from
str(reason).replace("\n", " ")[:80].

DATA FLOW
qualification.py::_check_regime_gates (unchanged) -> contract.py::
_build_system_state's stay_flat_reason (unchanged, daily always has
qualification_summary per runtime/__init__.py:391-424) ->
output.py::_alert_reason (changed: coverage-aware truncation) ->
rendered "Reason: ..." line in the daily compact alert body.

FAIL CONDITIONS
- R1 FAIL: given a stay_flat_reason containing "N/M votes cast"
  (0 < N < M) whose full text exceeds 80 characters, _alert_reason's
  return value does not contain "N/M votes cast", or exceeds 80
  characters.
- R2 FAIL: any non-coverage reason's rendering changes from today's
  behavior, including test_prd017_notification_stabilization.py's
  existing "no setups" assertion.
