# PRD-211 — Macro-tape metals display correctness
Lane: MICRO
Status: COMPLETE
Depends on: PRD-198 (applies #1, fail-loud)

## Problem
The Macro Tape metals row misrepresents itself three ways:
1. FREEZE: gold/silver are OPTIONAL drivers (contract.py:59); a failed
   GC=F/SI=F fetch hits silent `continue` (contract.py:510-511,
   521-522 in _build_macro_drivers 502-534). Key left unwritten →
   prior committed value persists by omission → stale value under a
   "<1 min old" banner. (PRD-198 #1 violated at the producer.)
2. LABEL: the row labels GC=F/SI=F as XAU/XAG spot
   (macro_tape_layout.py:25-26). They are front-month FUTURES, not
   spot — the display claims a semantic the data doesn't have.
3. MAPS: docs/SCHEMA_MAP.md macro_drivers row is empty and
   docs/CALL_SITE_MAP.md has no gold/silver entry — stale recon cache
   (CLAUDE.md: fix the map as part of the change). This staleness is
   what made this very investigation a full hand-rebuild.

## Verdict (re-verified, carried from recon)
DISPLAY-ONLY. gold/silver fenced from decision path in two independent
places — MACRO_BIAS_DRIVERS (bias vote) and _COMPONENT_FIELDS
(macro_pressure). No HOT/ACTION/permission read. Renderer innocent:
fresh skip renders N/A, not the old number. Freeze is a PERSISTENCE
artifact. → MICRO justified.

## DO NOT (explicit non-goals for this PRD)
- Do NOT add an ETF-implied-spot correspondence assertion as the freeze
  guard. The data is futures; GC=F legitimately diverges from spot by
  basis/contango. Such a check would false-positive on healthy data.
  The freeze guard is "write explicit absence, not silent skip" —
  nothing about value correspondence.
- Do NOT switch the data source from futures to spot. That's a separate
  product decision, out of scope. This PRD makes the LABEL honest about
  what the source IS; it does not change the source.

## Open question — RESOLVED Phase 1: B = FRESH
The hourly payload is written FRESH (full overwrite): sole writer
`deliver_json` (runtime/__init__.py:1853) → `transport.py:_write_file`
→ `p.write_text` (transport.py:83); no read-merge of the prior file.
The only persistence layer, `macro_drivers_snapshot.json`
(`_write_macro_snapshot`, runtime/__init__.py:1962), is also FRESH and
its renderer fallback is all-or-nothing (fires only when the WHOLE
macro_drivers dict is empty, dashboard_renderer.py:1730), so a
metals-only skip never triggers it. An absent gold/silver key therefore
renders `N/A` per-key (`_build_tape_value_slots`), not a stale number.
→ The contract.py silent `continue` (510/521) is **NOT the freeze
carrier**; fixing it would not change the rendered cell. FREEZE is
dropped from this PRD and spun out (see Future-you note). LABEL + MAPS
are the display-true deliverable and land regardless.

## LABEL approach — Option A (decoupled display label)
The pre-impl grep sweep falsified the "one-line in macro_tape_layout.py"
premise: `slot.label` "XAU"/"XAG" is also the `data-symbol` contract key
(PRD-136/137/138 invariants + the publish-refresh skill marker) and the
value-format dispatch key, in both the dashboard and the notification
builder. Chosen fix (Dustin, Option A): add a display-only `TapeSlot.display`;
render the honest CME futures ticker `GC`/`SI` in the **visible** dashboard
span AND the **notification body** (both user-facing surfaces), while keeping
`data-symbol`/`label`/format-dispatch on the historic `XAU`/`XAG` id for slot
shape stability. The `macro-spot-metals-row` CSS class and `data-symbol` stay
unchanged by design.
