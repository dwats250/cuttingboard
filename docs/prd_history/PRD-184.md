PRD-184 — Auto-merge-via-PR landing flow (Claude push enablement)

Status: IN PROGRESS

LANE
HIGH-RISK

CLASS
GOVERNANCE

WHY NOW
PRD-182/183 each had to stop at "PR-ready" and wait for a manual human
push, because `.claude/settings.json` denied `git push` outright. Dustin
chose the "auto-merge via PR after CI" model. The harness safety
classifier reserves permission changes for the human, so the settings
allowlist and the GitHub repo-admin (auto-merge + branch protection) were
applied by Dustin directly; this PRD records that change and the policy
it establishes. (Applied-then-documented, like PRD-151.)

MAX EXPECTED DELTA
<= 40 changed lines across settings.json + CLAUDE.md + DECISIONS. No
engine/test code.

GOAL
Let Claude land PRD work autonomously via push + PR + CI + GitHub
auto-merge, replacing the manual human-push gate for PRD implementation,
while keeping force-push denied and the CI `test` check as the merge gate.

SCOPE
- `.claude/settings.json`: allow `git push` + `gh pr create/merge/view/
  checks` + `gh run watch`; deny force-push variants.
- `CLAUDE.md`: replace the "No push without approval" rule with the
  auto-merge-via-PR policy; drop "pushes" from the explicit-approval list
  (force-push stays approval-gated/denied).
- `docs/DECISIONS.md`: record the policy change and its rationale.

OUT OF SCOPE
- No change to `ci.yml` or the `test` job itself (the gate is reused).
- No direct-to-main push for PRD *implementation* work — it lands via PR.
  Bookkeeping/closeout commits (docs-only) may push to `main` directly.
- The GitHub repo settings (Allow auto-merge; `main` branch protection
  requiring `test`) are account-owner admin, applied by Dustin via the
  setup one-shot; they are backing state, not repo files.

CHANGE SURFACE
- `.claude/settings.json`: permissions allow/deny arrays only; hooks
  untouched.
- `CLAUDE.md`: the "Review and commit discipline" push bullets only.

REQUIREMENTS

R1 — Push allowlist with force-push denied
`.claude/settings.json` `allow` includes `Bash(git push:*)`,
`Bash(gh pr create:*)`, `Bash(gh pr merge:*)`; `deny` no longer contains
the blanket `Bash(git push:*)` and DOES contain the force-push variants.

FAIL: `deny` still contains `Bash(git push:*)`; OR `allow` lacks
`Bash(git push:*)` or `Bash(gh pr merge:*)`; OR no force-push deny.

R2 — CLAUDE.md policy reflects the new flow
The "Review and commit discipline" section states that PRD work lands via
PR + CI + auto-merge (no manual-push gate), force-push excepted.

FAIL: CLAUDE.md still presents "pushing is never done without explicit
approval, and is denied by this repo's settings" as the active rule.

R3 — Repo backing requires CI before merge
`main` branch protection requires the `test` status check, and the repo
allows auto-merge, so `gh pr merge --auto` queues a merge that fires only
on green CI.

FAIL: `gh api repos/dwats250/cuttingboard/branches/main/protection` does
not list `test` as a required status check.

R4 — Flow proven by dogfooding
PRD-184 itself lands through the new flow: pushed branch -> PR ->
`gh pr merge --auto` -> CI `test` green -> auto-merge to `main`.

FAIL: PRD-184 is merged by any path other than CI-gated auto-merge
(e.g. a manual human merge, or a merge with no CI run).

DATA FLOW
Claude commits on a feature branch -> git push -> gh pr create -> gh pr merge --auto
-> CI `test` runs on the PR -> green -> GitHub auto-merges to main -> Claude closes out

FAIL CONDITIONS
- settings.json deny still blocks push, or allow lacks push/gh pr, or
  force-push not denied.
- CLAUDE.md still states the old no-push rule as active.
- main branch protection missing the `test` required check.
- PRD-184 merged by a non-CI-gated path.

VALIDATION

- `python3 -c "import json;d=json.load(open('.claude/settings.json'));
  print('Bash(git push:*)' in d['permissions']['allow'],
  'Bash(git push:*)' not in d['permissions']['deny'])"` -> `True True`.
- `gh api repos/dwats250/cuttingboard/branches/main/protection
  -q .required_status_checks.contexts` -> includes `test`.
- PRD-184 PR shows the `test` check and merges via auto-merge (dogfood).
