# PRD-203: prd_close.sh rebuilds the PROJECT_STATE baseline line (canonical, no stale provenance)

Status: COMPLETE

LANE
STANDARD

CLASS
INFRA

WHY NOW
Alignment cadence #4 caught the PROJECT_STATE "Test baseline" line going stale:
the count had advanced to 2819 while the provenance still read "after the PRD-195
merge -- run 27732171939 for `470aa2b`". Root cause is in scripts/prd_close.sh: it
updates the count by anchoring on the bold label (always works) but refreshes the
commit ref only when the bullet uses the ``at `<hex>` `` form. The live bullet uses
the ``for `<hash>` `` / "CI truth on `main` -- `test` job of run <id> for `<hash>`"
form, so the commit-ref regex self-skips and leaves stale provenance every closeout.

MAX EXPECTED DELTA
<= 30 changed LOC in scripts/prd_close.sh (the baseline-edit block + one optional
flag) plus test updates in tests/test_prd_close.py. No other closeout edit changes.

GOAL
Make prd_close.sh rebuild the entire Test-baseline bullet into one canonical,
deterministic line each closeout, so the count and provenance can never diverge.

SCOPE
- Replace prd_close.sh's two-step baseline edit (patch count; best-effort patch
  ``at `<hex>` ``) with a whole-bullet rebuild.
- Add an optional --ci-run flag for the CI run id.

OUT OF SCOPE
- The other prd_close.sh edits (Last updated, Active PRD reset, Recent ships row,
  registry/index) -- unchanged.
- The prd-closeout-verified skill -- works unchanged; --ci-run is optional and may
  be adopted later.
- Back-fixing historical baseline lines -- the next real closeout (this PRD's own)
  normalizes the live line.

FILES
M scripts/prd_close.sh

CHANGE SURFACE
- scripts/prd_close.sh: only the "(b) Test baseline bullet" block changes -- the
  count-only substitution and the ``at `<hex>` ``-only commit substitution are
  replaced by a single whole-bullet rebuild; the --ci-summary parse, the PRD-196
  fail-loud error gate, and all other edits are untouched. One new optional CLI
  flag (--ci-run).

REQUIREMENTS

R1 -- Canonical whole-bullet rebuild
prd_close.sh replaces the entire "- **Test baseline:**" bullet item (the label
line plus any soft-wrapped or multi-sentence continuation, up to the next bullet,
heading, or blank line) with one line:
"- **Test baseline:** {N} passing[, {M} xfailed] (CI truth on `main`; `test` job
for `{hash}`[, run {ci-run}])." N is the passing count (from --ci-summary, else
--tests); M xfailed comes from the --ci-summary parse, else is preserved from the
prior bullet, else omitted; {hash} is --hash.

FAIL: after closeout the Test-baseline bullet's commit ref differs from --hash, OR
any prior-bullet token (old count, old commit hash, old run id, trailing prose)
survives in the rebuilt bullet.

R2 -- Optional --ci-run flag
prd_close.sh accepts an optional --ci-run <id>; supplied, the line includes
", run <id>"; omitted, no run clause appears and the line is otherwise complete.

FAIL: --ci-run <id> does not place "run <id>" in the baseline line, OR omitting it
emits a stray "run" clause or errors.

R3 -- Fail-loud when the bullet is absent (preserve PRD-196 atomicity)
The rebuild anchors on the "- **Test baseline:**" label; if absent, the closeout
records an error and writes nothing (no silent skip, no partial write).

FAIL: a PROJECT_STATE.md lacking the Test-baseline bullet lets prd_close.sh exit 0
or write partial edits.

R4 -- Regression test: drifted multi-sentence bullet fully rebuilt (PRD-198 inv 4)
tests/test_prd_close.py feeds a drifted bullet in the live form (e.g. "(CI truth
on `main` -- `test` job of run 27732171939 for `470aa2b`. In this sandbox ... the
CI count.).") and asserts the rebuilt bullet (a) carries the new count and the new
--hash commit, and (b) contains none of: the old count, `470aa2b`, the old run id,
or the trailing prose.

FAIL: no test feeds a drifted multi-sentence bullet and asserts the old
commit/run/prose is gone after rebuild.

R5 -- Existing baseline assertions updated; suite green
The existing baseline tests in tests/test_prd_close.py assert the new canonical
format, and the full suite passes.

FAIL: any test_prd_close.py assertion still expects the retired
"(`python -m pytest tests -q` at `<hash>`)" format, or the suite is red.

DATA FLOW
--ci-summary/--tests (N), --ci-summary (M xfailed), --hash (commit), --ci-run
(optional run id) -> prd_close.sh baseline rebuild -> canonical
"- **Test baseline:**" line in docs/PROJECT_STATE.md

FAIL CONDITIONS
- Rebuilt baseline bullet's commit ref != --hash.
- Any stale prior-bullet token (old count/commit/run/prose) survives.
- --ci-run supplied but no "run <id>" in the line (or omitted yet a run clause appears).
- Missing Test-baseline bullet does not fail loud.
- A test still asserts the retired format, or the suite is red.

VALIDATION
Manual:
- In a tmp fixture whose baseline bullet is the drifted multi-sentence form, run
  prd_close.sh; confirm the rebuilt bullet is canonical with the new count + --hash
  and no stale tokens.
- With --ci-run, confirm ", run <id>"; without it, confirm no run clause.
- pytest tests/test_prd_close.py green; full suite green.
- Dogfood: this PRD's own closeout rebuilds the live PROJECT_STATE baseline line
  into canonical form.
