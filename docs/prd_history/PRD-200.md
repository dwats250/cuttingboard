PRD-200 — NYSE weekday-holiday session-closed detection

Status: PROPOSED

LANE
HIGH-RISK

CLASS
CONTRACT

WHY NOW
The pipeline has no notion of a full-day NYSE holiday: the hourly cron runs
holiday-blind, the dashboard renders a normal intraday session, and Trend
Structure shows intraday context (VWAP/RVOL) computed against non-session data.
Sunday closure is already modeled (SUNDAY_PREMARKET → inactive_session); weekday
holidays are the one unmodeled closure gap.

MAX EXPECTED DELTA
150 production LOC across 4 modified + 2 new code/data files (excludes tests and
the JSON table). Exceeding stops implementation and re-triggers CONTRACT review.

GOAL
Detect a full-day NYSE holiday at run time from a static table, carry it as an
additive payload meta boolean, and reuse the existing inactive_session rail to
collapse the Trend Structure section and badge a "NY SESSION CLOSED" state.

SCOPE
- New static-table + pure-loader triad mirroring PRD-176 red-folder: fail-loud
  load_schedule, is_holiday(et_date) membership, and a coverage-horizon guard that
  fails loud when the run date exceeds the table's covered range.
- New predicate cuttingboard/time_utils.is_session_closed(now_et, holiday_result):
  True only for a table holiday — HOLIDAYS ONLY.
- runtime/__init__.py resolves is_session_closed once against get_now_et().date()
  and sets an additive system_state flag adjacent to the SUNDAY_PREMARKET
  injection; delivery/payload.py copies it to meta (additive, mirrors the
  session_type passthrough). Absent ⇒ open.
- dashboard_renderer.py: OR the meta flag into the inactive_session derivation;
  key _allowed_freshness_window on the meta flag (72h); render a "NY SESSION
  CLOSED" badge in system-state post-title with the holiday name; relabel the
  Trend Structure inactive label on the holiday path only.

OUT OF SCOPE
- Live market-data API or pandas_market_calendars dependency; a new session_type
  enum value; half-day / early-close modeling; any weekend-handling change (Sunday
  stays SUNDAY_PREMARKET, Saturday unchanged); display-timezone config; any change
  to is_market_open wall-clock semantics; any per-field intraday suppression
  (whole-section collapse is accepted); any opportunistic runtime/ refactor.

CHANGE SURFACE
- cuttingboard/nyse_holidays.py: new pure helper; reads only its bundled table,
  takes a date/now arg, returns plain data; no runtime/contract/payload imports;
  no writes.
- cuttingboard/time_utils.py: append-only predicate; no change to get_now_* /
  is_market_open signatures or semantics.
- cuttingboard/runtime/__init__.py: one additive system_state key set adjacent to
  the existing SUNDAY_PREMARKET block; no decision-logic, write-ordering, or
  contract-shape change beyond that key.
- cuttingboard/delivery/payload.py: one additive meta passthrough; no existing
  meta key changed.
- cuttingboard/delivery/dashboard_renderer.py: inactive_session boolean extended
  (OR meta flag); freshness window keyed on the additive flag; additive badge +
  holiday-path relabel. No new session_type value, no per-field gate, no change to
  MARKET_CLOSED / usable_count logic.

REQUIREMENTS

R1 — Static holiday loader, fail-loud
load_schedule reads data/nyse_holidays_2026.json and returns a structured result;
a missing/malformed file or invalid entry yields ok=False (never raises, never a
silent-empty success). is_holiday(et_date) returns table membership.
FAIL: deleting or corrupting the table causes a raise, or an empty result
indistinguishable from "no holiday today".

R2 — Coverage-horizon guard
The loader exposes a coverage signal that fails loud when the run date exceeds the
table's covered range (analog of red_folder.is_expiring).
FAIL: a run date past the table's last covered date returns a result the caller
cannot distinguish from a normal in-coverage day.

R3 — Holidays-only predicate
time_utils.is_session_closed(now_et, holiday_result) returns True iff
now_et.date() is a table holiday; it never returns True for Saturday or Sunday.
FAIL: is_session_closed returns True for any weekend date, or False for a date
present in the holiday table.

R4 — Additive meta carry, backward-compatible
runtime resolves is_session_closed once against get_now_et().date() and sets an
additive system_state flag; payload copies it to meta. An artifact missing the
flag renders as a normal (open) day.
FAIL: a payload without the flag renders a closed-day badge or suppressed section,
OR the flag mutates any existing meta key.

R5 — inactive_session rail + freshness, keyed on the flag
The inactive_session derivation is True when lineage is COHERENT and (session_type
∈ INACTIVE_SESSION_TYPES OR the meta flag is set); _allowed_freshness_window
returns the INACTIVE_SESSION_MAX_AGE_HOURS (72h) window when the meta flag is set.
FAIL: a coherent flagged payload does not collapse Trend Structure, OR the
publish-gate applies the 180m live window to a flagged payload.

R6 — Closed-day badge + holiday-path label; Sunday unchanged
System-state renders a "NY SESSION CLOSED" badge (with the holiday name) post-title
when the flag is set; the Trend Structure inactive label uses the closed-day reason
only on the holiday path; the Sunday SUNDAY_PREMARKET INACTIVE_SESSION_LABEL is
unchanged.
FAIL: the Sunday path renders the closed-day label, OR a flagged holiday renders
the unchanged "SESSION INACTIVE" label, OR the badge omits the holiday name.

DATA FLOW
1. data/nyse_holidays_2026.json (read) + get_now_et().date() →
   nyse_holidays.load_schedule / is_session_closed (runtime) → system_state flag.
2. system_state flag → payload.meta flag → (a) dashboard_renderer.inactive_session
   + badge/label; (b) _allowed_freshness_window 72h window.

FAIL CONDITIONS
- Loader raises or returns indistinguishable-empty on a missing/malformed table.
- Coverage guard is silent past the table's last covered date.
- is_session_closed True on a weekend, or wrong on a tabled holiday.
- Missing flag yields a false-closed render, or mutates an existing meta key.
- Coherent flagged payload fails to collapse the section, or gets the 180m window.
- Sunday label changes, or the holiday path keeps "SESSION INACTIVE", or the badge
  omits the holiday name.

VALIDATION

- pytest tests/test_nyse_holidays.py tests/test_time_utils.py
  tests/test_sunday_premarket.py tests/test_dashboard_renderer.py -q passes; full
  suite green; ruff clean.
- Coherent weekday payload, meta flag set for a tabled holiday: confirm the

- Same payload without the flag (non-holiday weekday): confirm a normal table

- Delete/corrupt the table: confirm ok=False (no raise, no silent empty); set the

KNOWN DEPENDENCY
Suppression and badge fire only on a fresh COHERENT run that carries the meta flag
(inactive_session requires COHERENT lineage). The hourly cron is holiday-blind, so
it runs and emits one such artifact; a skipped or stale run degrades to a
normal-day render — safe failure: no false-closed, no STALE trap. Named here as a
dependency, not a silent assumption.
