PRD-200 — NYSE weekday-holiday session-closed detection

Status: PROPOSED

LANE
HIGH-RISK

CLASS
CONTRACT

WHY NOW
The pipeline has no notion of a full-day NYSE holiday: the hourly cron runs
holiday-blind, the dashboard renders a normal intraday session, and Trend
Structure shows intraday context (VWAP/RVOL) computed against non-session data.
Sunday closure is already modeled (SUNDAY_PREMARKET → inactive_session); weekday
holidays are the one unmodeled closure gap.

MAX EXPECTED DELTA
150 production LOC across 4 modified + 2 new code/data files (excludes tests and
the JSON table). Re-confirmed after the name-lookup + expired-badge amendment
(~10 LOC headroom remains). Exceeding stops implementation and re-triggers
CONTRACT review.

GOAL
Detect a full-day NYSE holiday at run time from a static table, carry the holiday
name as an additive nullable payload meta field, and reuse the existing
inactive_session rail to collapse the Trend Structure section and badge a "NY
SESSION CLOSED" state — with a loud "NYSE CALENDAR COVERAGE EXPIRED" badge when
the table no longer covers the run date.

SCOPE
- New static-table + pure-loader triad mirroring PRD-176 red-folder: fail-loud
  load_schedule, is_holiday(et_date) membership, a name-by-date lookup
  (holiday_for(et_date) → {date, name} entry | None), and a coverage-horizon guard
  that fails loud when the run date exceeds the table's covered range.
- New predicate cuttingboard/time_utils.is_session_closed(now_et, holiday_result):
  a clean bool, True only for a table holiday — HOLIDAYS ONLY. The badge name comes
  from the loader lookup, not from this predicate.
- runtime/__init__.py resolves is_session_closed once against get_now_et().date();
  on a closed day it resolves the holiday name via the loader lookup and sets an
  additive system_state field adjacent to the SUNDAY_PREMARKET injection;
  delivery/payload.py copies it to meta.nyse_session_closed_reason (additive
  nullable string, mirrors the session_type passthrough). Absent OR None ⇒ open;
  presence of a non-None reason is the closed-signal for ALL consumers.
- dashboard_renderer.py: OR "reason present" into the inactive_session derivation;
  key _allowed_freshness_window on "reason present" (72h); render a "NY SESSION
  CLOSED" badge in system-state post-title, reading the name from
  meta.nyse_session_closed_reason; relabel the Trend Structure inactive label on the
  holiday path only; render a "NYSE CALENDAR COVERAGE EXPIRED" badge when the loader
  reports the run date past coverage.

OUT OF SCOPE
- Live market-data API or pandas_market_calendars dependency; a new session_type
  enum value; half-day / early-close modeling; any weekend-handling change (Sunday
  stays SUNDAY_PREMARKET, Saturday unchanged); display-timezone config; any change
  to is_market_open wall-clock semantics; any per-field intraday suppression
  (whole-section collapse is accepted); any opportunistic runtime/ refactor.

REQUIREMENTS

R1 — Static holiday loader, fail-loud, with name lookup
load_schedule reads data/nyse_holidays_2026.json and returns a structured result;
a missing/malformed file or invalid entry yields ok=False (never raises, never a
silent-empty success). is_holiday(et_date) returns table membership;
holiday_for(et_date) returns the matching {date, name} entry or None.
FAIL: deleting or corrupting the table causes a raise, or an empty result
indistinguishable from "no holiday today".

R2 — Coverage-horizon guard
The loader exposes a coverage signal that fails loud when the run date exceeds the
table's covered range (analog of red_folder.is_expiring).
FAIL: a run date past the table's last covered date returns a result the caller
cannot distinguish from a normal in-coverage day.

R3 — Holidays-only predicate
time_utils.is_session_closed(now_et, holiday_result) returns True iff
now_et.date() is a table holiday; it never returns True for Saturday or Sunday.
FAIL: is_session_closed returns True for any weekend date, or False for a date
present in the holiday table.

R4 — Additive nullable meta carry, backward-compatible
runtime resolves is_session_closed once against get_now_et().date(); on a closed
day it resolves the holiday name (loader lookup) and sets an additive system_state
field, which payload copies to meta.nyse_session_closed_reason (a string when
closed, absent/None when open). All consumers use defensive access
(meta.get("nyse_session_closed_reason") → None ⇒ open). Presence of a non-None
reason is the closed-signal for every consumer.
FAIL: a payload with the key absent OR set to None renders a closed-day badge or
suppressed section, OR the field overwrites/mutates any existing meta key.

R5 — inactive_session rail + freshness, keyed on reason-present
inactive_session is True when lineage is COHERENT and (session_type ∈
INACTIVE_SESSION_TYPES OR meta.nyse_session_closed_reason is not None) — the new
condition is OR-ed in, extending the existing branch, never replacing it.
_allowed_freshness_window returns the INACTIVE_SESSION_MAX_AGE_HOURS (72h) window
when EITHER session_type ∈ INACTIVE_SESSION_TYPES OR the reason is present;
SUNDAY_PREMARKET continues to resolve 72h unchanged.
FAIL: a coherent reason-present payload does not collapse Trend Structure, OR
SUNDAY_PREMARKET stops resolving the 72h window, OR the publish-gate applies the
180m live window to a reason-present payload.

R6 — Closed-day badge + holiday-path label; Sunday unchanged
System-state renders a "NY SESSION CLOSED" badge post-title, reading the holiday
name from meta.nyse_session_closed_reason, when the reason is present; the Trend
Structure inactive label uses the closed-day reason only on the holiday path; the
Sunday SUNDAY_PREMARKET INACTIVE_SESSION_LABEL is unchanged.
FAIL: the Sunday path renders the closed-day label, OR a reason-present holiday
renders the unchanged "SESSION INACTIVE" label, OR the badge omits the holiday name.

R7 — Coverage-expired loud consumer
When the run date exceeds the table's covered range (the R2 signal), system-state
renders a "NYSE CALENDAR COVERAGE EXPIRED" badge. The day otherwise renders
normally — no fabricated closure — but the expired state is never silent. R7 is
decoupled from R3: is_session_closed does not change; the badge alone carries the
coverage uncertainty.
FAIL: a run date past the table's last covered date renders a normal open day with
no visible "NYSE CALENDAR COVERAGE EXPIRED" indicator.

DATA FLOW
1. data/nyse_holidays_2026.json (read) + get_now_et().date() →
   nyse_holidays.load_schedule / is_session_closed + holiday_for (runtime) →
   system_state holiday-name field (+ coverage signal).
2. system_state field → payload.meta.nyse_session_closed_reason (nullable) →
   (a) dashboard_renderer.inactive_session (reason-present) + "NY SESSION CLOSED"
   badge/label; (b) _allowed_freshness_window 72h window.
3. loader coverage signal → dashboard_renderer "NYSE CALENDAR COVERAGE EXPIRED"
   badge (decoupled from is_session_closed).

FAIL CONDITIONS
- Loader raises or returns indistinguishable-empty on a missing/malformed table.
- Coverage guard is silent past coverage, OR a past-coverage run renders no
  "NYSE CALENDAR COVERAGE EXPIRED" badge.
- is_session_closed True on a weekend, or wrong on a tabled holiday.
- Key absent OR None yields a false-closed render, or the field mutates an existing
  meta key.
- Coherent reason-present payload fails to collapse the section, gets the 180m
  window, OR SUNDAY_PREMARKET stops resolving 72h.
- Sunday label changes, or the holiday path keeps "SESSION INACTIVE", or the badge
  omits the holiday name.

KNOWN DEPENDENCY
Suppression and the "NY SESSION CLOSED" badge fire only on a fresh COHERENT run
that carries meta.nyse_session_closed_reason (inactive_session requires COHERENT
lineage). The "NYSE CALENDAR COVERAGE EXPIRED" badge (R7) is independent of lineage
— it surfaces whenever the loader reports the run date past coverage. The hourly cron is holiday-blind, so
it runs and emits one such artifact; a skipped or stale run degrades to a
normal-day render — safe failure: no false-closed, no STALE trap. Named here as a
dependency, not a silent assumption.
