WHY NOW
PRD-204's scope-gate run flagged `cuttingboard/delivery/regime_history.py` as a
protected pipeline file because it matches `docs/AGENT_WORKFLOW.md:36`'s
`*regime*.py` glob (Trading-logic category). But `regime_history.py` is the
PRD-175 read-only observational scoreboard sidecar — it imports no
runtime/contract/payload/output layer and casts no vote (enforced by
`tests/test_regime_history.py::test_prd175_module_imports_no_decision_layers`).
The glob is meant to protect the regime DECISION engine (`cuttingboard/regime.py`),
so the match on the sidecar is a false-positive that forces a HIGH-RISK lane
decision (or a documented per-PRD exemption, as PRD-204 took) on every future edit
to a provably read-only sidecar. Fix the guard so the exemption isn't re-litigated.

MAX EXPECTED DELTA
1 changed line in `docs/AGENT_WORKFLOW.md` (the Trading-logic row); ≤5 files in
FILES (the rest bookkeeping). No code change.

GOAL
Narrow the Trading-logic protected glob `*regime*.py` to `regime.py` so the regime
decision engine stays protected while the PRD-175 read-only sidecar
(`regime_history.py`) is no longer falsely caught.

  Trading-logic protected-set row. The other globs in that row (`*gate*.py`,
  `*qualify*.py`, `*signal*.py`) are unchanged.

  this PRD removes the need for the NEXT such exemption.
