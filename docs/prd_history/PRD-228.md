# PRD-228 — Bot-review-thread disposition clause (governance guardrail)

Lane: MICRO / GOVERNANCE
Status: IN PROGRESS

## Problem

Automated PR reviewers (the `chatgpt-codex-connector` GitHub app, and future
connector bots) post inline review threads on every PR. CLAUDE.md already records
that a connector-only ephemeral comment does NOT satisfy the Codex cross-review
gate (not in-tree, not durable, not SHA-pinned). But it does not say what MUST
happen to those threads. Left unstated, substantive bot catches can be silently
ignored, or a real defect can be patched to clear a thread without the normal
PRD/red-test treatment, or a resolved thread can be mistaken for a satisfied
review gate. This PRD codifies the disposition discipline.

## Fix

Add one bullet to CLAUDE.md § "Review and commit discipline", directly after the
connector-only-comment clause: bot threads are advisory INPUT, never
gate-satisfying; every substantive thread is ACTIONED (fix lands, thread resolved
citing the SHA/PRD) or DISMISSED with a one-line in-thread reason; a real defect
gets the normal treatment (PRD-before-build when non-trivial, mutation-verified
red test); the thread never substitutes for the Claude review + durable Codex
cross-review a HIGH-RISK PRD requires. The clause marks itself
MANUAL-MERGE-ONLY, consistent with the existing governance carve-out.

## Why a separate PRD (not folded into the audit cleanup)

Surfaced by the codex-connector review of PR #95, which correctly flagged that
adding this clause inside the PRD-226/227 cleanup PR violated scope-locking
(CLAUDE.md is outside both PRDs' FILES) and the governance carve-out (a
review/merge-policy change must not ride bundled on feature work). Split out here
per Dustin (2026-07-03).

## Acceptance criteria

- CLAUDE.md contains a "Bot-review-thread disposition" bullet with the three
  sub-points (triage/action-or-dismiss, normal-treatment-for-real-defects,
  thread-is-not-the-artifact) and the MANUAL-MERGE-ONLY self-designation.
- The clause sits under § "Review and commit discipline", adjacent to the
  existing connector-only-comment clause it extends.
- No source, test, or non-CLAUDE.md change.

## Non-goals

- Mechanical enforcement (CODEOWNERS / branch-protection). The clause is
  agent-honored policy, like the rest of the governance carve-out; the PRD-186 R4
  CODEOWNERS hardening remains the separate, deferred mechanical path.
- Any change to what satisfies the Codex cross-review gate — that clause is
  unchanged; this one only governs thread DISPOSITION.
