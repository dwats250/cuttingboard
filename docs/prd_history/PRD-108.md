# PRD-108 — Registry Hook Hygiene

STATUS
IN PROGRESS

GOAL
Stop the UserPromptSubmit registry-gap hook from flagging review and
adjudication artifacts as missing PRD registry rows. Per CLAUDE.md
§ Review artifact discipline, `*.review.*.md` and `*.adjudication.md`
files are NOT PRDs and MUST NOT have registry rows. The current hook
treats them as PRDs, producing a false-positive warning on every
prompt and adding noise to every session.

SCOPE
- Modify the hook's prd_history scan to exclude review and adjudication
  artifact filename patterns.
- Add a deterministic unit test that exercises the exclusion on a
  fixture prd_history directory.
- Preserve detection of real un-registered `PRD-NNN.md` files.
- Promote PRD-107 to COMPLETE in the registry as part of this PRD's
  bookkeeping. PRD-107 is already committed at `d10e179`; the current
  IN PROGRESS row is stale and is causing the SEQUENCING GATE in the
  same hook script to fire spuriously on every prompt. Promoting it
  here keeps the hook fix and the registry cleanup in a single
  coherent commit.

OUT OF SCOPE

- No changes to the SEQUENCING GATE block in the same hook script.
- No changes to the substring-match logic on registry rows.

EXCLUSION RULES (binding)
A filename in `docs/prd_history/` is EXCLUDED from the registry-gap
scan if and only if at least one of these matches:
1. `".review."` appears as a literal substring in the basename
   (e.g. `PRD-104.review.claude.md`, `PRD-104.review.codex.md`)
2. The basename ends with `".adjudication.md"`
   (e.g. `PRD-104.adjudication.md`)

All other `PRD-*.md` files remain in scope. The patterns must NOT match:
- `PRD-NNN.md`
- `PRD-NNN-PATCH.md` / `PRD-NNN-PATCH-2.md`
- `PRD-NNN-cleanup.md`
- `PRD-NNN.M.md` (e.g. `PRD-003.4.md`)
- `PRD-NNNA.md` (e.g. `PRD-012A.md`)

DATA FLOW
hook fires → glob `docs/prd_history/PRD-*.md` → filter exclusions →
  for each remaining stem: check substring presence in registry rows →
    emit GAP message only if real un-registered files exist.
