`cuttingboard/runtime.py` (2289 LOC) into a `cuttingboard/runtime/`

REQUIREMENTS

R1 ‚Äî Package conversion preserves every public import path.
Every name importable from `cuttingboard.runtime` before the change is
importable after it (entrypoints, mode/outcome/status constants, path
constants, dataclasses, and the test-bound `_`-prefixed functions).
FAIL: `tests/test_runtime_package_surface.py` is red, or any pre-existing
test that imports from `cuttingboard.runtime` errors on import.

R2 ‚Äî Constants/dataclasses are L0 leaves, re-exported from __init__.
`_constants.py` and `_types.py` import only stdlib + non-runtime
`cuttingboard.*` modules (never `cuttingboard.runtime`); `__init__`
re-exports their names so the `cuttingboard.runtime.<name>` surface is
unchanged.
FAIL: `tests/test_runtime_layering.py` finds a `cuttingboard.runtime`
import (or relative package import) in `_constants.py` or `_types.py`.

R3 ‚Äî Zero function moves.
All top-level `def`s that lived in `runtime.py` remain defined in
`cuttingboard/runtime/__init__.py`.
FAIL: `grep -cE "^def " cuttingboard/runtime/__init__.py` is less than the
pre-change `grep -cE "^def " runtime.py` count (65).

R4 ‚Äî Behavior preserved (suite + artifacts).
The full suite passes at the prior baseline plus the new guard tests, and
a fixture pipeline run produces byte-identical artifacts.
FAIL: `python3 -m pytest tests -q` reports any failure, the passing count
is below 2534 + net-new guard tests, or a fixture render diffs.

R5 ‚Äî Monkeypatch surface preserved.
Tests that patch `cuttingboard.runtime.<name>` (constants like `LOGS_DIR`/
`WATCHLIST_PATH`; collaborators like `fetch_all`/`send_notification`)
still intercept, because those names remain bound in `__init__` and every
function that reads them still lives in `__init__`.
FAIL: any pre-existing monkeypatch/setattr-based test against the
`cuttingboard.runtime` namespace fails.
