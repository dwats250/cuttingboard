PRD-124 — Hourly Telegram Alert Header and Body Quality

STATUS
COMPLETE

LANE
STANDARD

CLASS
CONSUMER

WHY NOW
Hourly Telegram receipts show titles like "STAY FLAT 12:20" while the
operator's local clock reads 9:20 AM PT, and the body collapses to a
single "REGIME | POSTURE | CONFIDENCE" line plus generic trigger
boilerplate ("breakout above resistance", "continuation hold above
trigger") that is detached from any candidate symbol. The alert is the
operator's primary out-of-band signal — its current state degrades
trust and forces the operator to reload the dashboard to derive any
useful read. The fix is a localized formatter rewrite using artifacts
already plumbed into `format_hourly_notification`.

MAX EXPECTED DELTA
≤ 140 production LOC across the two FILES entries below (formatter
helpers + call-site wiring). Test additions are not counted. Exceeding
this ceiling MUST stop implementation and amend the PRD.

GOAL
Replace the ET-formatted title clock with a local-PT clock, expand the
hourly body into a deterministic operator-facing summary, and remove
context-free trigger boilerplate — all using artifacts already passed
into `format_hourly_notification`.

SCOPE
- Hourly path only: `format_hourly_notification` and its private
  helpers in `cuttingboard/notifications/__init__.py`.
- Title timestamp: switch from ET ("%H:%M") to PT 12-hour clock with
  AM/PM (matches `_time_line()` semantics in `formatter.py`).
- Body rewrite for STAY FLAT, WATCHLIST, ACTIVE-NO-SETUP, and
  trade-ready hourly variants.
- Macro tape line: derive from `regime.vix_level` /
  `regime.vix_pct_change`; if `market_map` carries other macro driver
  snapshots, include those as additional tape line(s) (read-only).
- Watch/candidate summary line(s) derived from
  `qualification_summary.qualified_trades`,
  `qualification_summary.watchlist`, and parsed `candidate_lines`.
- Explicit ACTION line (TRADE / MONITOR / STAY FLAT / HALT) derived
  from existing outcome/posture state.
- "Pending confirmation" section replaces the unconditional
  `TRIGGERS:` block; only rendered when at least one candidate symbol
  is attached to it.
- Missing-data labels: every section that lacks its underlying
  artifact MUST render an explicit `n/a` or `unknown` token, not a
  blank line or a misleading default.
- Tests covering timestamp formatting, body sections, missing-data
  fallback labels, generic-trigger suppression, and a regression
  assertion that no decision-logic module is modified.

OUT OF SCOPE
- `cuttingboard/notifications/formatter.py` `_format_hourly` path
  (legacy ntfy formatter; not the Telegram hourly entry).
- Regime, qualification, watch, validation, sizing, contract,
  payload, market_map, or dashboard renderer modules.
- AlertEvent dataclass changes.
- New artifact schema or contract field additions.
- Lifecycle alert block (`_append_lifecycle_alerts`) — preserved
  unchanged.
- Intraday, premarket, midmorning, power-hour, run-summary, halt, or
  failure alert paths.
- Suppression policy (`should_suppress_*`) and audit fields.
- Adding new macro driver fields to `RegimeState`.
- Introducing a new repo-wide canonical operator-timezone constant
  shared between notifier and dashboard surfaces. PRD-124 reuses the
  existing notifier-package constant `LOCAL_TZ` from
  `cuttingboard.notifications.formatter` (see R1). Cross-surface
  canonicalization is deferred to a separate PRD; the dashboard
  renderer is not modified here.
- Any new ranking, weighting, scoring, or prioritization heuristic
  for candidate/watch symbols inside the notifier. See R9.

REQUIREMENTS

R1 — Title timestamp uses local PT clock via the existing notifier-package constant
The hourly title timestamp MUST be derived from `asof_utc` converted
to the existing notifier-package constant `LOCAL_TZ` imported from
`cuttingboard.notifications.formatter` (currently
`ZoneInfo("America/Vancouver")`), and formatted as `%I:%M %p` with
the leading zero stripped (e.g. "9:20 AM"). Equivalent to the
existing `_time_line()` formatting in `formatter.py`. The string
" ET" MUST NOT appear in the hourly title.

A new `ZoneInfo("America/Vancouver")` literal MUST NOT be introduced
in `cuttingboard/notifications/__init__.py`; the existing `_ET_TZ`
literal there SHOULD be replaced with an import from `formatter.py`
to collapse the duplication, but this is not blocking. There MUST
NOT be more than one `LOCAL_TZ` declaration in the notifier package
at end of PRD.

FAIL: any title rendered by `format_hourly_notification` for an
`asof_utc` of `2026-05-11T16:20:00+00:00` (= 09:20 PT) contains the
substring `"12:20"`, `" ET"`, or fails to contain `"9:20 AM"`; OR
`grep -nE "ZoneInfo\\(\"America/Vancouver\"\\)"
cuttingboard/notifications/` returns more than one match at end of
PRD.

R2 — Artifact timestamp is separately labeled when shown
If the body renders the artifact generation timestamp, it MUST be
prefixed with the label `Generated:` and formatted in ET with the
suffix ` ET`. The title's PT clock and the body's generation
timestamp MUST NOT collide on the same line.

FAIL: a rendered body line begins with a bare `HH:MM` token without
a `Generated:` prefix; OR the title contains both a PT clock and an
ET clock; OR the body contains an ET timestamp without the
`Generated:` label.

R3 — Body includes the required operator sections in fixed order
A non-halt hourly body MUST contain, in this order, with the exact
section labels:
  1. `State:` — `<regime> / <posture>` (or `unknown / unknown`)
  2. `Confidence:` — `<0.00–1.00>` (or `unknown`)
  3. `Action:` — one of `TRADE`, `MONITOR`, `STAY FLAT`, `HALT`
  4. `Reason:` — single-line primary no-trade or trade reason
  5. `Blockers:` — comma-separated soft-gate miss tags or `none`
  6. `Macro:` — VIX line and any other available macro driver
     tokens (e.g. `VIX 18.4 (+2.1%)`) or `n/a`
  7. `Focus:` — up to 3 `SYMBOL DIR` tokens from qualified trades
     and watchlist, or `none`
  8. (optional) `Pending confirmation:` — only when at least one
     focus symbol exists; lists trigger conditions attached to a
     specific symbol via `<SYMBOL>: <condition>` format.

FAIL: any of labels 1–7 is missing from a rendered hourly body for a
non-halt event; OR labels appear out of order; OR `Action:` value is
not in the enumerated set.

R4 — Action line is deterministic from existing state
The `Action:` value MUST be computed from existing AlertEvent state:
  - `HALT` iff `validation_summary.system_halted` is true.
  - `TRADE` iff `qualification_summary.symbols_qualified > 0` AND
    `regime.posture != STAY_FLAT` AND not halted.
  - `STAY FLAT` iff `regime is None` OR `regime.posture == STAY_FLAT`
    (and not TRADE/HALT).
  - `MONITOR` otherwise (candidates on watchlist OR forming setups
    AND regime tradable).
No new fields, no new derived metrics, no probabilistic logic.

FAIL: any test event with `regime.posture == STAY_FLAT` renders an
`Action:` value other than `STAY FLAT` (unless halted); OR a halted
event renders an `Action:` other than `HALT`; OR an event with
`qualification_summary.symbols_qualified == 0` and a non-flat regime
renders `Action: TRADE`.

R5 — Generic triggers MUST NOT render without an attached symbol
The current unconditional `TRIGGERS:` block, and the regime-keyed
strings returned by `_trigger_conditions` ("breakdown below support",
"failed reclaim at breakdown level", "breakout above resistance",
"continuation hold above trigger", "range break", "expansion
confirmation", "confirmed direction"), MUST NOT appear in any
rendered hourly body unless attached to a specific candidate symbol
under a `Pending confirmation:` section in `<SYMBOL>: <condition>`
form.

FAIL: any rendered hourly body contains the substring `"TRIGGERS:"`;
OR contains any of the seven phrases above on a line that does not
also contain a ticker symbol from the focus list.

R6 — Missing data degrades with explicit labels
Each section MUST render an explicit fallback token when its
underlying artifact is `None` or empty:
  - regime `None` → `State: unknown / unknown`,
    `Confidence: unknown`, `Macro: n/a`
  - `qualification_summary` `None` or empty → `Focus: none`,
    `Blockers: none`
  - no failure reason and no halt → `Reason: no setups`
No section may be silently omitted, blank, or substituted with a
default that hides the absence of the artifact.

FAIL: an event built with `regime=None,
qualification_summary=None, validation_summary=ValidationSummary(
symbols_attempted=0, ...)` renders a body that is missing any of
labels 1–7 from R3; OR renders the literal string `None`,
`UNKNOWN | UNKNOWN | 0.00`, or an empty value to the right of any
section label.

R7 — Decision logic modules MUST NOT be modified
No file outside `FILES` may be modified. In particular:
`cuttingboard/regime.py`, `cuttingboard/qualification.py`,
`cuttingboard/validation.py`, `cuttingboard/runtime.py`,
`cuttingboard/contract.py`, `cuttingboard/payload.py`,
`cuttingboard/notifications/formatter.py`,
`cuttingboard/notifications/state.py`, and all dashboard renderer
files MUST be byte-identical between the merge base and the PRD-124
HEAD commit.

FAIL: `git diff --name-only <merge-base>..HEAD` includes any path
not listed in this PRD's `FILES` section.

R9 — No new ranking, scoring, or prioritization logic in the notifier
The notifier MAY surface existing candidate/watch symbols sourced
from `qualification_summary.qualified_trades`,
`qualification_summary.watchlist`, `candidate_lines`, and
`market_map`, but MUST preserve the existing deterministic ordering
of those collections. The notifier MUST NOT introduce any new
sorting key, ranking heuristic, weighting function, score field,
priority tier, or discretionary tie-breaker. Slicing (e.g. `[:3]`)
and filtering by `is_tradable_symbol` are permitted because they
already exist in the notifier; any additional sort, key=, max, min,
or custom comparator that depends on artifact values is prohibited.

FAIL: a code-review diff of `cuttingboard/notifications/__init__.py`
introduces any of: `sorted(`, `.sort(`, a new `key=` lambda, a new
`max(`/`min(` over an artifact collection, a new score/weight/rank
field on any helper, or any comparator that orders symbols by a
derived numeric value. (Pre-existing `sorted(...)` calls in
`_watch_lines_from_qualification` are grandfathered and MUST remain
byte-identical.)

R8 — Test coverage MUST exercise all observable behaviors
`tests/test_hourly_alert.py` MUST include explicit tests asserting:
  - R1 PT title format (substring `"9:20 AM"` for a 16:20Z asof).
  - R2 `Generated:` labeling when artifact timestamp is shown.
  - R3 presence and order of all required section labels.
  - R4 each branch of the Action enum.
  - R5 absence of the generic trigger phrases when no symbol is
    attached.
  - R6 missing-data fallback tokens for regime=None and
    qualification_summary=None.
  - R7 a meta-test that imports
    `cuttingboard.notifications.formatter` and asserts its module
    file hash is unchanged within the test run (or, equivalently,
    that the test does not import or monkey-patch any decision-logic
    module).
  - R9 a static-grep assertion (executed inside the test as a string
    scan over `cuttingboard/notifications/__init__.py`) that no new
    `sorted(`, `.sort(`, `key=`, `max(`, or `min(` call has been
    added beyond the pre-existing call in
    `_watch_lines_from_qualification`.

FAIL: `python3 -m pytest tests/test_hourly_alert.py -q` reports any
failure; OR any of the eight required assertions above is missing
from the test file.

  → title PT clock string
regime (RegimeState | None) → State / Confidence / Action / Macro

  Reason line

  Blockers + Action=TRADE/MONITOR branches

  `_parse_candidate_line` → Pending confirmation entries

  (unchanged) → appended lifecycle block
