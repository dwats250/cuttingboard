## GOAL

Surface blocked and near-qualified trade candidates with a deterministic `visibility_status` classification to provide actionable context without modifying trade decisions.

---

## SCOPE

- Create `trade_visibility.py` to classify each decision as `ACTIVE`, `NEAR_MISS`, or `BLOCKED`
- Inject visibility computation in `runtime.py` after `build_market_map` (grade is required from market_map)
- Add visibility fields to trade_candidates in `contract.py`

---

## OUT OF SCOPE

- No changes to trade decision logic or execution policy
- No alerts
- No UI rendering changes
- No mutation of `TradeDecision` fields

## REQUIREMENTS

### R1 â€” Grade source

Grade for each symbol is sourced from `market_map["symbols"][symbol]["grade"]`.

Valid grades: `A+`, `A`, `B`, `C`, `D`, `F` (from `market_map.VALID_GRADES`).

If a symbol is not present in market_map or grade is missing, treat grade as `None`.

FAIL: grade is read from any source other than market_map.
FAIL: missing grade raises an exception instead of defaulting to None.

### R2 â€” Classification rules

```
ACTIVE:
  policy_allowed == True

NEAR_MISS:
  policy_allowed == False
  AND grade in {A+, A, B}
  AND exactly one blocking condition applies
    (single policy gate failed OR only macro_pressure_conflict)

BLOCKED:
  all other policy_allowed == False cases
```

FAIL: a decision with policy_allowed=True is classified as anything other than ACTIVE.
FAIL: a multi-block decision with a high grade is classified as NEAR_MISS.

### R3 â€” Output fields per candidate

```python
{
  "visibility_status": "ACTIVE" | "NEAR_MISS" | "BLOCKED",
  "visibility_reason": str | None,     # primary blocking condition; None if ACTIVE
  "enable_conditions": list[str],       # empty if ACTIVE or BLOCKED; populated for NEAR_MISS
}
```

`enable_conditions` are deterministic fixed strings. Mapping:

| Condition | enable_condition string |
|---|---|
| `macro_pressure_conflict` | `"macro_pressure must align with trade direction"` |
| `orb_inside_range` | `"price must break ORB range"` |
| `cooldown` | `"cooldown period must expire"` |
| `low_confidence` | `"regime confidence must reach 0.60"` |
| Any other single block | `"blocking condition must resolve: {policy_reason}"` |

FAIL: enable_conditions is non-empty for ACTIVE decisions.
FAIL: enable_conditions is non-empty for BLOCKED (multi-block) decisions.
FAIL: any enable_condition string is not deterministic for identical input.

### R4 â€” Function signature

```python
def build_visibility_map(
    decisions: list[TradeDecision],
    market_map: dict,
) -> dict[str, dict]:
    ...
```

Returns `{symbol: {visibility_status, visibility_reason, enable_conditions}}`.

FAIL: function mutates any TradeDecision field.
FAIL: function mutates the market_map argument.

### R5 â€” Runtime injection

In `runtime.py`, call `build_visibility_map(trade_decisions, market_map)` after `build_market_map` completes, before contract assembly. Store result in `PipelineResult` as `visibility_map: dict[str, dict]`.

FAIL: visibility computed before market_map is built (grade would be unavailable).
FAIL: visibility not stored on PipelineResult.

### R6 â€” Contract integration

In `contract.py:_build_trade_candidates`, merge visibility fields into each candidate dict from `pipeline_result.visibility_map.get(symbol, {})`.

Missing or empty visibility map must produce `visibility_status=None`, `visibility_reason=None`, `enable_conditions=[]` â€” not raise.

FAIL: contract builder raises when visibility_map is absent.
FAIL: visibility fields absent from trade_candidates output.

## FAIL CONDITIONS

- Grade read from source other than `market_map["symbols"][symbol]["grade"]`
- Visibility computed before market_map build
- NEAR_MISS classification on a decision with more than one blocking condition
- NEAR_MISS classification on grade C, D, or F
- enable_conditions non-empty for ACTIVE or multi-block BLOCKED
- Contract builder raises on missing visibility_map
