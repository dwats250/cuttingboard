# PRD-084 — Populate market_map current_price

**Status:** IN PROGRESS
**Type:** PATCH (producer-consumer contract fix)

## ROOT CAUSE

missing fail condition — `_build_symbol_record` computes `price` from `NormalizedQuote.price` but never writes `"current_price"` into the returned record dict, so `dashboard_renderer.py` always sees the field absent.

## GOAL

Populate `current_price` in `market_map` symbol records so `dashboard_renderer.py` can consume the field it already expects.

## SCOPE

- `market_map.py` `_build_symbol_record` record dict only.
- Tests for market_map output and dashboard tape consumption.
- PRD registry, PROJECT_STATE.md update.
- Regenerate `ui/dashboard.html` / `ui/index.html` if renderer-facing output changes.

## OUT OF SCOPE

- No new data providers.
- No fetch or normalization changes.
- No dashboard redesign.
- No trading logic changes.
- No candidate qualification changes.
- No macro regime changes.
- No fallback/fake prices.
- No broad schema redesign.

## FILES

- `cuttingboard/market_map.py`
- `tests/test_market_map*.py` (add or update)
- `tests/test_dashboard_renderer.py` (add tape test)
- `docs/prd_history/PRD-084.md`
- `docs/PRD_REGISTRY.md`
- `docs/PROJECT_STATE.md`

## REQUIREMENTS

**R1 — Add current_price field**
In `_build_symbol_record`, include `"current_price": price` in the returned record dict, where `price` is the existing validated optional float derived from `quote.price`.
FAIL: valid `NormalizedQuote.price` exists but `market_map["symbols"][sym]["current_price"]` is absent.

**R2 — Preserve invalid/missing behavior**
If `quote.price` is missing, null, non-numeric, or non-finite, `current_price` must be `None`. No synthetic or fallback values.
FAIL: synthetic or fake prices introduced.

**R3 — Preserve existing fields**
Do not remove or rename existing symbol record keys.
FAIL: existing market_map consumers break due to renamed/removed fields.

**R4 — Dashboard compatibility**
`dashboard_renderer.py` tape path reads `current_price` without schema changes.
FAIL: SPY/QQQ/GDX/GLD/SLV/XLE show DATA_UNAVAILABLE solely because `current_price` is structurally absent.

## DATA FLOW

```
fetch_all() → raw_quotes
normalize_all(raw_quotes) → normalized_quotes[sym].price
build_market_map(normalized_quotes=...) → _build_symbol_record(quote=normalized_quotes.get(sym))
  price = _optional_float(quote.price)          # market_map.py:171 — already computed
  record["current_price"] = price               # ADD THIS LINE
→ market_map["symbols"][sym]["current_price"]
→ dashboard_renderer.py:349 entry.get("current_price")
```

## FAIL CONDITIONS

- Any PRIMARY_SYMBOL with a valid normalized quote lacks `current_price` in the symbol record.
- `current_price` is populated with a non-None value when `quote` is None or price is non-finite.
- Existing symbol record keys are removed or renamed.
- Pipeline crashes on missing quote.
