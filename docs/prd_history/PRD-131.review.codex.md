## Questions and Answers

1. **CLASS / LANE governance** — CLASS: CONSUMER is correct because the only
   production file in scope is `cuttingboard/delivery/dashboard_renderer.py`
   (PRD-131 FILES lines 102-107), a read-only consumer of finalized artifacts
   per `docs/PRD_PROCESS.md` lines 110-114. LANE: STANDARD is not
   governance-compliant. The CONSUMER row lists
   `cuttingboard/delivery/dashboard_renderer.py` under HIGH-RISK FILES
   (`docs/PRD_PROCESS.md` line 154). LANE Axis R11 says any PRD whose FILES
   intersects HIGH-RISK FILES for its CLASS MUST declare LANE: HIGH-RISK
   (`docs/PRD_PROCESS.md` lines 193-201). PRD-131's header comment says
   "No HIGH-RISK FILES touched" and "`dashboard_renderer.py` is a
   CONSUMER-class file, not in the SIDECAR HIGH-RISK list" (lines 8-14), but
   the relevant row is CONSUMER, not SIDECAR.

2. **Predictive / scoring semantics** — Several R1/R2 phrases are not cleanly
   structural. "Established uptrend" implies stability/quality beyond the two
   current SMA comparison tokens (PRD-131 lines 138 and 140). "Established
   uptrend, intraday soft" adds a quality modifier; "soft" can be read as a
   degraded rating (line 139). "Recovery attempt" implies trajectory and
   improvement from a prior weaker state, which is not present in
   `(ABOVE, BELOW, *)` (line 141). "Pullback within uptrend" implies a recent
   retreat from higher prices, which is not present in `(BELOW, ABOVE, *)`
   (line 142). "Structural weakness" and "Structural weakness, intraday firm"
   use quality-loaded "weakness"/"firm" language (lines 143-145).
   "Short-term inflection", "Long-term inflection", and "Dual-MA inflection"
   imply a turn/change event, but AT_LEVEL only means price equals the SMA at
   the current sample (`trend_structure.py` lines 106-117; PRD-131 lines
   146-150). "No intraday confirmation" implies a confirmation signal rather
   than a computation boundary (PRD-131 lines 178-179). "Trend baseline
   forming" is borderline because "forming" implies process over time, but it
   is defensible as a compact rendering of `INSUFFICIENT_HISTORY` (line 180).
   "Structure unavailable" is safe and descriptive (line 181).

3. **Label overlap / ambiguity** — The "Established uptrend" vs
   "Established uptrend, intraday soft" distinction is mechanically
   deterministic but semantically noisy because the suffix is driven solely by
   VWAP comparison in homogeneous ABOVE/ABOVE rows (PRD-131 lines 136-145,
   152-156), while the PRD does not constrain session phase or near-open VWAP
   stability. The inflection triad is not coherent as written: "Short-term
   inflection" covers `AT_LEVEL` on SMA50 with SMA200 either ABOVE or BELOW,
   "Long-term inflection" covers `AT_LEVEL` on SMA200 with SMA50 either ABOVE
   or BELOW, and "Dual-MA inflection" covers both at level (lines 146-150).
   These are equality states, not necessarily inflection events.

4. **R2 precedence completeness** — For realistic tokens emitted by
   `cuttingboard/trend_structure.py`, the precedence is deterministic and
   conflict-free. The module emits comparison tokens through `_cmp()` as
   `ABOVE`, `BELOW`, or `AT_LEVEL` (`trend_structure.py` lines 106-117);
   SMA fields can emit `DATA_UNAVAILABLE` or `INSUFFICIENT_HISTORY`
   (`trend_structure.py` lines 139-149, 165-176); VWAP can emit
   `DATA_UNAVAILABLE` or `NOT_COMPUTED` (`trend_structure.py` lines 120-136,
   152-162). The PRD's precedence order `DATA_UNAVAILABLE` >
   `INSUFFICIENT_HISTORY` > `NOT_COMPUTED` (PRD-131 lines 178-185) resolves
   realistic mixed unknown combinations. No realistic combination is
   underspecified. Non-realistic malformed cases such as VWAP =
   `INSUFFICIENT_HISTORY` or SMA = `NOT_COMPUTED` are outside the current
   emitter vocabulary, though future expansion would need explicit handling.

5. **VWAP suffix boundary** — The asymmetry is under-justified. PRD-131 says
   VWAP distinguishes only homogeneous SMA rows and is ignored for all mixed
   rows (lines 152-156). If VWAP is meaningful enough to distinguish
   ABOVE/ABOVE as "intraday soft" and BELOW/BELOW as "intraday firm" (lines
   138-145), then ignoring VWAP for ABOVE/BELOW and BELOW/ABOVE creates
   inconsistent operator semantics. This could be defended only if the PRD
   explicitly states that the interpretation is SMA-primary and VWAP may only
   add a modifier when SMA50 and SMA200 agree; as drafted, the boundary reads
   arbitrary.

6. **Mixed-state derivability** — The split is mechanically derivable from
   the two SMA tokens, but the chosen labels are not. `ABOVE` SMA50 /
   `BELOW` SMA200 and `BELOW` SMA50 / `ABOVE` SMA200 are current-position
   comparisons only (`trend_structure.py` lines 258-263, 267-280). They do
   not encode recent price trajectory, recovery, pullback, or momentum.
   "Recovery attempt" and "Pullback within uptrend" (PRD-131 lines 141-142,
   279-281) smuggle in directional history that is absent from the input
   record, violating the PRD's own "No inference across symbols, no
   aggregation, no historical comparison" boundary (lines 90-92).

7. **Renderer-only boundary / grep gate** — The intended production boundary
   is clean: PRD-131 scopes the helper to `dashboard_renderer.py` (lines
   53-63, 110-125), the render site is the trend-structure row cell tuple
   (`dashboard_renderer.py` lines 1785-1795), and R5 forbids sidecar,
   payload, contract, notifier, market_map, and qualification changes
   (PRD-131 lines 221-234). The R4 grep gate is necessary but insufficient:
   `grep -RIn "_trend_structure_interpretation" cuttingboard/` (lines
   213-218, 283-284) would miss a copied interpretation string, a renamed
   alias, or a separate mapping imported by notifier/payload code. The FILES
   allowlist and R5 diff check would catch most implementation leaks, but the
   grep itself does not prove the string cannot leak through an indirect
   channel.

8. **Future token stability** — The R2 precedence does not extend cleanly to
   a sixth normalized token such as `STALE` or `RECONCILING`. The current
   table mixes comparison tokens, data-state tokens, and a single
   computation-boundary token in prose bullets (PRD-131 lines 171-185). A new
   normalized state would need an explicit precedence slot and display phrase;
   otherwise the helper could remain total only by adding an ad hoc fallback,
   which conflicts with the closed-vocabulary requirement (lines 156-159,
   187-190).

## Blocking Findings

B1. `LANE: STANDARD` is invalid. Because PRD-131 is CLASS: CONSUMER and its
FILES include `cuttingboard/delivery/dashboard_renderer.py`, it intersects
the CONSUMER HIGH-RISK FILES row in `docs/PRD_PROCESS.md`; R11 requires
`LANE: HIGH-RISK` before implementation.

B2. The mixed-state labels "Recovery attempt" and "Pullback within uptrend"
are not derivable from the normalized tokens alone. They imply direction or
recent trajectory not present in `price_vs_sma_50` and `price_vs_sma_200`.

B3. The AT_LEVEL labels using "inflection" are semantically incorrect for the
available token. `AT_LEVEL` means equality at the current sample, not a turn
event; "Short-term inflection", "Long-term inflection", and
"Dual-MA inflection" should be replaced with equality/at-reference wording.
