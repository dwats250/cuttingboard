PRD-251 — Credit-spread max risk: strategy-aware max-loss arithmetic (Gate 8)

Status: IN PROGRESS

LANE
HIGH-RISK

CLASS
EXECUTION

WHY NOW
Gate 8 sizes credit spreads (BULL_PUT_SPREAD/BEAR_CALL_SPREAD) off the estimated-debit proxy (30% of width) as if it were the max loss, understating true credit-spread risk by ~2.3x (true max loss = width minus credit = 70% of width).

SECOND-MODEL: disposition deferred to close. Dustin has not commissioned
a second model for this slice; default per PRD-242 is the waiver
sentence, to be added verbatim at REVIEW/close alongside the
fresh-context Claude review artifact.

MAX EXPECTED DELTA
<= 60 production lines across cuttingboard/options.py +
cuttingboard/qualification.py combined (a threading + arithmetic change:
one new dataclass field, one new small helper/constant, a handful of
call-site edits — not a rewrite of either module). No new files, no new
external dependencies, no new modules.

GOAL
Make Gate 8 and the final rendered dollar_risk size credit spreads
(BULL_PUT_SPREAD, BEAR_CALL_SPREAD) off their true max loss per share
(width minus estimated credit) instead of the debit-proxy value that
today silently doubles as a wrong "max loss" for credit strategies.

SCOPE
- Strategy-aware max loss for the two credit strategies
  (BULL_PUT_SPREAD, BEAR_CALL_SPREAD): max loss/share = strike width -
  estimated credit (30% of width), i.e. 70% of width.
- Debit strategies (BULL_CALL_SPREAD, BEAR_PUT_SPREAD) keep today's
  behavior unchanged: max loss/share = the estimated debit (unchanged
  30%-of-width proxy).
- A single, explicit max-loss value carried from candidate-build time
  (`cuttingboard/options.py::_build_candidate`/`generate_candidates`)
  through Gate 8 (`cuttingboard/qualification.py::qualify_candidate`,
  the `GATE_MAX_RISK` check) and through the final correlation-adjusted
  resize in `cuttingboard/options.py::build_option_setups`, so the
  qualify/reject decision and the rendered `dollar_risk` never diverge.
- `TradeCandidate.spread_width`'s documented meaning (estimated net
  debit per share) is preserved unchanged — asserted directly by
  `tests/test_phase5.py::test_etf_spread_width` /
  `test_stock_spread_width` — the strategy-aware value is a new,
  separate field, not a redefinition of `spread_width`.

OUT OF SCOPE
- The EXPANSION-regime continuation path
  (`cuttingboard/qualification.py::_qualify_continuation_candidate`,
  lines 630-730, routed from `qualify_all` at lines 230-260 when
  `regime.regime == EXPANSION`). It has the same conceptual bug (its
  ATR-based proxy at line 707, `spread_width = max(0.50,
  dm.atr14 * 0.05)`, can be misapplied as max loss for a credit
  strategy when direction=LONG lands in ELEVATED_IV/HIGH_IV, since
  `_select_strategy` in `build_option_setups` applies uniformly to
  every `QualificationResult` regardless of which qualifier produced
  it) — but its proxy is independent of the strike-distance-based 30%
  convention this PRD fixes, added separately in PRD-008 (commit
  30b583a) with no "width" term to subtract a credit from. Fixing it
  requires inventing a width convention for an ATR-based continuation
  trade — a design decision not yet made. Discovered during this PRD's
  Stage 0 sweep; a fold-in was ruled out once the code showed materially
  different logic (2026-07-10, see docs/DECISIONS.md). Tracked as a
  fast-follow: `docs/prd_history/PRD-251.continuation-path.proposal.md`.
  This PRD does not edit `cuttingboard/qualification.py:630-730` or any
  other continuation-path code.
- PRD-256/A1b's live chain-priced economics (superseding the 30% proxy
  entirely) — this PRD leaves the seam (the new explicit max-loss field)
  for that later substitution; it does not implement live pricing.
- The risk budget cap ($150 -> $400, `cuttingboard/config.py`) — a
  separate, already-decided config-only PRD, never bundled here
  (2026-07-10 decision, docs/DECISIONS.md).
- `cuttingboard/output.py` — no logic change; it already renders
  `setup.dollar_risk` / `setup.max_contracts` (lines 345-350) with no
  further changes needed. Not listed in FILES.

REQUIREMENTS

R1 — Credit-strategy max loss is width minus estimated credit
For BULL_PUT_SPREAD and BEAR_CALL_SPREAD candidates, the per-share value
used for sizing (Gate 8's `spread_cost` and the final `OptionSetup`
resize) equals strike width minus the estimated credit (30% of width),
i.e. 70% of width — not the 30%-of-width value alone.

FAIL: a $5-wide BULL_PUT_SPREAD candidate (LONG direction, HIGH_IV
structure) sizes at $150/contract or qualifies at the current $150
budget — either indicates the debit-proxy value, not width-minus-credit
($350/contract), drove the decision.

R2 — Debit-strategy max loss is unchanged
For BULL_CALL_SPREAD and BEAR_PUT_SPREAD candidates, the per-share value
used for sizing is exactly today's estimated-debit proxy (30% of
width) — byte-identical sizing outcome before and after this PRD.

FAIL: any debit-strategy candidate's `max_contracts` or `dollar_risk`
changes value relative to pre-PRD behavior for the same inputs.

R3 — Gate 8 and the final rendered dollar_risk never diverge
The value Gate 8 uses to qualify/reject a candidate and the value
`build_option_setups` uses to compute the final `dollar_risk` on the
`OptionSetup` are the same max-loss figure for a given candidate — never
two independently-derived numbers that could disagree.

FAIL: a candidate's `QualificationResult.dollar_risk` (Gate 8) and its
resulting `OptionSetup.dollar_risk` (final render) are computed from
different per-share max-loss assumptions for the same strategy.

R4 — Continuation path is untouched
`cuttingboard/qualification.py:630-730` (`_qualify_continuation_candidate`
and its ATR-based proxy) is not modified by this PRD.

FAIL: the merged diff touches any line in
`cuttingboard/qualification.py`'s continuation-path function, or
`tests/test_continuation_mode.py` / `tests/test_continuation_audit.py`
show any behavior change.
