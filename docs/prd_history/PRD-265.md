PRD-265 — Coverage marker: persist RegimeState.total_votes so bounded days are identifiable across runs

Status: IN PROGRESS

WHY NOW
PRD-263 bounds regime confidence under vote dropout, but every DURABLE
cross-run consumer still records only regime/posture/confidence/net_score
— audit.jsonl omits total_votes (audit.py:205-209). So a coverage-bounded
day is indistinguishable from a full-evidence day in postmarket scoring,
regime history, and the scoreboard, which then aggregate a coverage
artifact as if it were market evidence; and the dropout rate PRD-262
proved retroactively unmeasurable stays unmeasurable. This is sol C1 /
connector thread 3611239784.

GATE A (Dustin, 2026-07-19 — build to it)
- Persist RAW total_votes, not a boolean. Consumers derive
  coverage_bounded locally. Rationale: the four coverage STATES must stay
  distinguishable and recoverable from history.
- FOUR-STATE COVERAGE PREDICATE (every consumer distinguishes all four):
    absent (key not in record) = LEGACY  — pre-PRD-265 rows; scored
      exactly as today, NEVER excluded, NEVER shown BOUNDED.
    total_votes == 0            = EXPANSION (genuine verdict) — scored
      normally, not bounded.
    0 < total_votes < 8         = BOUNDED — excluded from postmarket
      scoring; marked BOUNDED in regime_history + scoreboard.
    total_votes == 8            = FULL — scored normally, not marked.
  coverage_bounded(row) := ("total_votes" in row) and
                           (0 < row["total_votes"] < 8).
  Absent is not 0 and not bounded — pinned by a red test, not inferred.
- Scope: TIER 2 — persist the field, EXCLUDE bounded days from postmarket
  scoring, MARK bounded days in regime_history and the scoreboard. The
  coverage-reason surfacing on the human alert (sol C2: output.py:893
  truncation, hourly build-from-RegimeState) is a DIFFERENT surface and
  rides its own fast-follow (recorded in PRD-263 REALIZABILITY; scaffolded
  when this PRD opens).

GOAL
Make coverage-bounded days identifiable and correctly handled in every
durable cross-run consumer: excluded from postmarket regime scoring,
marked in regime history and the scoreboard, and countable so the dropout
rate becomes measurable — while legacy rows behave exactly as today.

SCOPE
1. cuttingboard/audit.py (_build_record, :205-209 Regime block): add
   "total_votes": regime.total_votes if regime else None. The single
   enabling root — _load_run_history and regime_history copy audit rows
   verbatim, so the field is then available downstream.
2. cuttingboard/delivery/regime_history.py (_REGIME_FIELDS, :41): add
   "total_votes" so it propagates into regime_history.jsonl.
3. cuttingboard/reports/postmarket.py: exclude BOUNDED rows from the
   persisted/flipped regime_validation (:118/:135-137) and from the EVR
   classifiers; a bounded CURRENT verdict (read total_votes from the
   contract regime block, contract.py:480, which already carries it)
   yields a coverage-bounded outcome rather than a scored MATCH/MISS.
   Legacy and EXPANSION and full rows score as today.
4. cuttingboard/delivery/dashboard_renderer.py (scoreboard :2770-2787):
   render a BOUNDED marker for coverage-bounded rows read from
   regime_history; legacy/EXPANSION/full render unchanged.

OUT OF SCOPE
- The coverage-reason surfacing on the daily/hourly alert (sol C2):
  its own fast-follow slice (output.py compact reason; hourly build from
  RegimeState). Scaffolded when this PRD opens so it is queued, not
  remembered.
- Run summaries (run_*.json) and the "Changes Since Last Run" block, and
  market_map / market_map_lifecycle coverage-marking: unmarked display
  surfaces, deliberately deferred — none changes a decision, and the
  scoreboard is the trader-facing coverage view. Recorded so they are not
  silently assumed done.
- Any change to compute_regime, qualification, execution, or the contract
  regime block (already coverage-aware).

REQUIREMENTS

R1 — Audit record persists total_votes
_build_record writes "total_votes" from regime.total_votes (None when
regime is None). No other Regime-block field changes.
FAIL: an audit.jsonl row written with a non-None regime lacks a
"total_votes" key, or any of regime/posture/confidence/net_score changes.

R2 — regime_history carries total_votes; forbidden guard flipped
_REGIME_FIELDS includes "total_votes"; total_votes propagates into
regime_history.jsonl summaries; tests/test_regime_history.py's forbidden
tuple no longer lists total_votes and instead asserts its presence.
FAIL: a regime_history summary built from an audit row carrying
total_votes omits it, or the forbidden-tuple guard still bans it.

R3 — Four-state coverage predicate, legacy pinned
Every consumer uses coverage_bounded(row) = "total_votes" in row and
0 < row["total_votes"] < 8. A legacy row (no total_votes) is scored
exactly as pre-PRD-265, never excluded, never marked BOUNDED; an
EXPANSION row (0) and a full row (8) are likewise not bounded.
FAIL: a legacy row is excluded from postmarket scoring or rendered
BOUNDED; or a total_votes==0 or ==8 row is treated as bounded.

R4 — Postmarket excludes bounded days from regime scoring
BOUNDED rows are excluded from persisted/flipped and do not produce a
scored EVR MATCH/PARTIAL/MISS; a bounded current verdict yields a
coverage-bounded (non-scored) outcome.
FAIL: a bounded prior row counts toward persisted/flipped, or a bounded
current verdict is scored as a regime hit/miss.

R5 — Scoreboard marks bounded days
The scoreboard renders a BOUNDED marker for coverage-bounded regime_history
rows; legacy/EXPANSION/full rows render unchanged.
FAIL: a bounded row renders identically to a full-evidence row, or a
legacy/full row renders BOUNDED.

R6 — Behavior-neutral on the existing corpus
Replaying the current logs/audit.jsonl (all legacy rows) through the
changed postmarket and regime_history produces byte-identical scoring and
history output to pre-change (legacy path unchanged).
FAIL: any legacy-row scoring or history output differs pre/post.

- A three-state predicate that conflates absent with 0 or with bounded.
- Any decision-path file (regime/qualification/execution/contract regime
  block) modified.
