## Findings (verbatim disposition)

### P2 — Should-Fix

**P2-1 — Day cache key can be poisoned by a "successful" prefetch that did not warm OHLCV.**
`.github/workflows/cuttingboard.yml` "Save OHLCV cache" + `cuttingboard/runtime/__init__.py:224`
(`execute_prefetch`). `execute_prefetch()` returns `0` even when quote validation
halts (`val.system_halted`), which skips `compute_all_derived()` and therefore the
OHLCV warming, yet the workflow still saves `data/cache` under the immutable day
key. If a transient halt happens at 12:50 and live is healthy at 13:00, live
restores an empty/stale exact key and re-fetches. Fix: only save after a positive
freshness proof (a pre-save check that expected `*_ohlcv.parquet` files exist and
pass `_is_fresh_ohlcv_cache`), or emit an explicit `ohlcv_warmed=true` only after
derive runs and gate the save on it.
**Disposition: ACCEPTED — REMEDIATED.** This is the PRD-198 #2 anti-pattern (save
on the *requested* action, not the *resolved* effect). Added a "Check OHLCV cache
warmed" step that runs `_is_fresh_ohlcv_cache` over `data/cache/*_ohlcv.parquet`
and emits `ohlcv_warmed`; the Save step is now gated on
`job_mode == 'prefetch' && steps.warmed.outputs.ohlcv_warmed == 'true'`. A halted
prefetch (or one that only restored a prior-day cache) no longer saves under
today's key. Authoritative source (the freshness function itself), not a proxy
`ls`.

**P2-2 — actions/cache guard test is substring-only; too weak for PRD-198 #4.**
`tests/test_ci_artifact_hygiene.py` `test_prefetch_warm_cache_is_persisted_via_actions_cache`.
It would pass if restore/save were in the wrong mode, save were not prefetch-only,
the day output were unused, or fallback restore were removed.
**Disposition: ACCEPTED — REMEDIATED.** Rewrote the guard to parse the workflow
YAML and assert structurally: restore `uses` is `actions/cache/restore@`, save is
`actions/cache/save@`; both `path == data/cache`; restore.key == save.key and
carries the `ohlcv-` day key; `restore-keys` prefix present; restore `if` gated to
warming modes; save `if` gated to `prefetch` AND the `ohlcv_warmed` proof (the
red case for "a success-without-warming prefetch must not save").

**P2-3 — Stale PROJECT_STATE text cites the retired `OHLCV_STALE_HOURS` TTL.**
`docs/PROJECT_STATE.md:13` and `:15`.
**Disposition: ACCEPTED — REMEDIATED.** Both bullets annotated that
`OHLCV_STALE_HOURS` was retired by PRD-193 and freshness is now trading-day-keyed.
(Independently flagged by the Claude review, REQUIRED EDIT 2.)
