WHY NOW
A data outage silently LOOSENS qualification (gates 9/10 pass on missing
data with no trace), and a NEUTRAL-direction regime makes every symbol
vanish from all output — not qualified, not watchlisted, not excluded,
not logged. This is a discipline tool; degradation must be visible.

MAX EXPECTED DELTA
~35 production LOC (qualification.py + one render hook in output.py);
no gate outcome changes — visibility only.

GOAL
Every silent path becomes observable: NEUTRAL no-direction symbols land
in `excluded` with reason NEUTRAL_NO_DIRECTION (rendering via the
existing EXCLUDED section and contract rejections); missing-data passes
on gates 9/10 record a `gates_skipped` marker on the QualificationResult,
rendered in the report's setup detail.

SCOPE
- qualification.py: QualificationResult gains `gates_skipped:
  tuple[tuple[str, str], ...] = ()` (additive, defaulted, frozen-safe).
  Gate 9 None-earnings pass and gate 10 missing-metrics pass record
  ("EARNINGS", "no earnings data") / ("EXTENSION", "no ATR/EMA metrics").
  Outcomes UNCHANGED — both gates still pass (the fail-open is a
  documented decision; the silence is the defect).
- qualify_all: when direction_for_regime(regime) is None and candidates
  were provided, each non-CHOP symbol is excluded with
  NEUTRAL_NO_DIRECTION instead of vanishing.
- output.py: the TRADE-branch setup detail renders one line when the
  matching QualificationResult has gates_skipped:
  "Gate skipped (missing data): EARNINGS, EXTENSION".
- Red tests first (three FAIL conditions, one per silent path).

OUT OF SCOPE
- Changing any gate outcome (loosening/tightening) — deciding whether
  gates 9/10 SHOULD fail closed is a separate, regime-affecting PRD.
- The candidates=None Phase-4 legacy path (gates 5-11 skipped wholesale;
  pre-existing, documented).
- Dashboard/notification surfaces (they read the contract; excluded
  already flows to rejections).

REQUIREMENTS

R1 — NEUTRAL symbols are excluded, not vanished
With a regime whose direction_for_regime is None (e.g. NEUTRAL,
net_score 0) and candidates provided, every non-CHOP symbol appears in
qualify_all's excluded with reason NEUTRAL_NO_DIRECTION.
FAIL: such a symbol is absent from qualified, watchlist, AND excluded.

R2 — Missing-data passes are marked
qualify_candidate with has_earnings_soon=None records
("EARNINGS", ...) in gates_skipped; with dm=None (or missing ema21/atr14)
records ("EXTENSION", ...). Gate outcomes unchanged (still passed).
FAIL: a missing-data pass leaves gates_skipped empty, or any gate
outcome flips relative to the pre-change code.

R3 — Report visibility
A TRADE render whose qualified setup carries gates_skipped shows a
"Gate skipped (missing data): ..." line in that setup's detail.
FAIL: the rendered report omits the line for a result with markers.

DATA FLOW
missing inputs -> qualify_candidate markers -> QualificationResult ->
render_report setup detail; NEUTRAL -> excluded -> EXCLUDED section +
contract rejections (existing consumers).

FAIL CONDITIONS
- Any R1-R3 FAIL line. Full suite not green. Any existing gate-outcome
  test modified to pass.
