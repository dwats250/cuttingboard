## RECOMMENDED EDITS
1. **Claude-artifact exclusion is an exact-string compare, not a model parse
   (`tools/validate_prd_registry.py`).** `has_artifact` counts any
   `PRD-NNN.review.*.md` whose `p.name != f"PRD-{number:03d}.review.claude.md"`.
   A misnamed Claude artifact — `PRD-NNN.review.claude2.md`,
   `PRD-NNN.review.claude-fresh.md`, `PRD-NNN.review.claude.v2.md` — would
   satisfy R2 as if it were a second model, defeating R3. Risk is low in
   practice because the `prd-review-claude` skill writes the exact
   `.review.claude.md` slot, so a legitimate Claude review is always correctly
   excluded; the hole requires a deliberately/accidentally misnamed Claude
   file. Suggest parsing the `<model>` segment from the filename and excluding
   any model token that starts with `claude`, rather than matching one literal
   filename. (Confirmed the good cases are clean: `.bak` and
   `PRD-NNN.adjudication.md` do NOT match the `.review.*.md` glob, so those
   cannot wrongly satisfy.)
2. Non-blocking: `_LANE_HIGH_RISK_RE = r"^LANE\b[:\s]*\n?\s*HIGH-RISK"` has a
   redundant `\n?\s*` tail (the preceding `[:\s]*` already consumes newlines).
   Harmless; could be simplified. It correctly matches both the block form
   (`LANE\nHIGH-RISK`, used by every real doc I sampled: PRD-230/238/240) and
   the inline `LANE: HIGH-RISK` form (test_242_inline_lane_form_detected), and
   correctly rejects STANDARD/MICRO (PRD-239/241).
