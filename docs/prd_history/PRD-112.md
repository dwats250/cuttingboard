PRD-112 — Trend Structure Dashboard Panel

STATUS
COMPLETE

GOAL
Expose `logs/trend_structure_snapshot.json` (PRD-107 / PRD-110 sidecar) on the
dashboard as a read-only visibility panel for the curated six-symbol universe
(`SPY, QQQ, GDX, GLD, SLV, XLE`). The renderer consumes the sidecar artifact
directly; the pipeline contract, payload, runtime, sidecar producer, and
notification paths are not modified.

SCOPE
- Add a direct sidecar read in `cuttingboard/delivery/dashboard_renderer.py`
  using a new module-level constant `_TREND_STRUCTURE_PATH =
  Path("logs/trend_structure_snapshot.json")`, mirroring the existing
  `_MACRO_SNAPSHOT_PATH` ingestion pattern (renderer.py:39).
- Render a single read-only HTML section ("Trend Structure") inside the
  existing dashboard body, after the macro-pressure / macro-tape blocks and
  before the candidate board. Section is a flat, compact table that reuses
  existing dashboard CSS scale; no new stylesheet, no new JS, no new
  interactive widgets.
- Display exactly the curated six symbols, in `config.TREND_STRUCTURE_SYMBOLS`
  order (`SPY, QQQ, GDX, GLD, SLV, XLE`).
- Display only fields already present in the sidecar record: `symbol`,
  `current_price`, `price_vs_vwap`, `price_vs_sma_50`, `price_vs_sma_200`,
  `trend_alignment`, `entry_context`, `relative_volume`, `data_status`.
- Surface `generated_at` once at the section header with a stale badge
  computed by the existing `_compute_timestamp_freshness(...)` helper
  (`dashboard_renderer.py:93–100`), the same helper the system-state header
  uses for its timestamp.
- Update `docs/artifact_flow_map.md` row for `logs/trend_structure_snapshot.json`
  to record `delivery/dashboard_renderer.py` as a declared consumer.
- Add unit tests in `tests/test_dashboard_renderer.py` for present, missing,
  malformed, and stale sidecar inputs, and for symbol order / field set.

OUT OF SCOPE
- Any change to `cuttingboard/runtime.py`, `cuttingboard/trend_structure.py`,
  `cuttingboard/contract.py`, `cuttingboard/output.py`,
  `cuttingboard/qualification.py`, `cuttingboard/regime.py`,
  `cuttingboard/execution_policy.py`, `cuttingboard/trade_decision.py`,
  `cuttingboard/trade_policy.py`, or anything in `cuttingboard/notifications/`.
- Any change to the payload schema (`logs/latest_payload.json`), contract
  schema, run-snapshot schema, market_map schema, or sidecar schema.
- Payload mutation, payload adapter layer, sidecar adapter abstraction.
- New computation: the renderer MUST NOT compute SMAs, VWAPs, alignments,
  relative volumes, or any field absent from the sidecar artifact.
- New universe expansion, new symbols, new indicators, new fields, scoring,
  ranking, gating, or strategy behavior.
- Adding the snapshot to the payload, contract, or any other artifact.
- Notification, alert, or Telegram-body changes.
- New CSS files, new JS, charting libraries, or external assets.
- Collapsible widgets (`<details>`, `<summary>`, accordions).
- Card-grid redesign for trend-structure rows.
- Merging trend-structure rows into the candidate-board DOM region.
- Any renderer-generated interpretation labels (see R6).
- `.github/workflows/*.yml` modifications.

R4 — Field set is sidecar-derived only; no derived semantic labels
Rendered cells are formatted directly from the sidecar record fields:
`current_price`, `price_vs_vwap`, `price_vs_sma_50`, `price_vs_sma_200`,
`trend_alignment`, `entry_context`, `relative_volume`, `data_status`. The
renderer applies only:
- numeric rounding (e.g. `round(price, 2)`),
- string formatting (`f"{value:.2f}"`),
- null-to-`—` substitution,
- direct enum-to-CSS-class lookup using a static dict whose keys are
  exactly the sidecar's enum values (e.g. `BULLISH`, `BEARISH`, `MIXED`,
  `UNKNOWN`, `ABOVE`, `BELOW`, `SUPPORTIVE`, `AVOID`, `NEUTRAL`, `OK`,
  `PARTIAL`, `MISSING`).

The renderer MUST NOT generate any new interpretation label. Forbidden
output strings (non-exhaustive list): `ELEVATED`, `STRONG`, `WEAK`,
`BULLISH+`, `BEARISH+`, `HIGH RVOL`, `LOW RVOL`, `OVEREXTENDED`,
`COMPRESSED`, `MOMENTUM`, `FADING`, `BREAKOUT`, `BREAKDOWN`. The renderer
must pass through the sidecar's enum values verbatim.

FAIL: Renderer source contains any arithmetic or boolean comparison on
`current_price`, SMA, VWAP, or volume values beyond `round()` and string
formatting helpers. FAIL: Renderer source contains `if`/`elif`/`match`
branches that emit a displayed label not present verbatim in the
sidecar's enum vocabulary. FAIL: The HTML displays a field name not
listed above. FAIL: The HTML for the trend-structure section contains any
of the forbidden output strings.

R5 — Stale / missing / malformed sidecar degrades gracefully
The renderer's stale evaluation for the trend-structure section MUST use
the existing `_compute_timestamp_freshness(value)` helper at
`dashboard_renderer.py:93–100` applied to the sidecar's own
`generated_at` field (top-level, the field written by
`runtime._write_trend_structure_snapshot`). This is the same helper used
by the dashboard's system-state header (`dashboard_renderer.py:1012`).
The renderer MUST NOT introduce a separate stale-evaluation path,
mtime-based check, or alternate threshold for this section.

State table:

| Sidecar state | Required behavior |
|---|---|
| File missing | Section renders with header + 6 placeholder rows (all `data_status=MISSING`) and a "no trend structure data" notice. No exception. Header badge: `MISSING`. |
| Top-level JSON parse fails | Same as missing. Header badge: `MISSING`. |
| Top-level `symbols` key missing or not a dict | Same as missing. Header badge: `MISSING`. |
| Any required per-record key missing or wrong type for any of the 6 curated symbols | **Entire section** degrades to the missing-state output (6 placeholder rows). Header badge: `MISSING`. |
| `generated_at` missing, naive, or unparseable | Rows render from sidecar values; header badge: `UNKNOWN`. |
| `_compute_timestamp_freshness(generated_at) == "STALE"` | Rows render; header badge: `STALE`. |
| `_compute_timestamp_freshness(generated_at) == "FRESH"` | Rows render; header badge: `FRESH` (or no badge, matching the existing system-state pattern). |

No partial recovery is permitted. The renderer MUST NOT:
- salvage individual symbol rows from a malformed top-level structure,
- render some rows from sidecar data and others as placeholders within a
  single non-degraded render,
- attempt best-effort field coercion (e.g. casting strings to numbers),
- substitute defaults for missing per-record enum values.

If any required per-record key is missing or has the wrong type for any of
the six curated symbols, the entire trend-structure section degrades to
the missing-state output. All-or-nothing.

FAIL: Any state above raises, aborts the dashboard, or causes other
sections to fail. FAIL: stale evaluation uses any helper or threshold
other than `_compute_timestamp_freshness` and `DASHBOARD_STALE_AFTER_SECONDS`.
FAIL: stale evaluation uses file mtime or any timestamp other than the
sidecar's top-level `generated_at`. FAIL: renderer renders a partial
section (some rows from sidecar, others as placeholders) when the
malformed-record condition above is met. FAIL: renderer attempts type
coercion or default substitution on per-record fields.

R6 — No renderer-generated interpretation
This requirement is a separate, standalone failure surface from R4 to
ensure governance visibility. The renderer is forbidden from inventing
any interpretation layer over sidecar values. Specifically:

- No threshold-based label generation (e.g., `relative_volume > 1.5 →
  "ELEVATED"`).
- No multi-field synthesis (e.g., combining `trend_alignment` and
  `relative_volume` into a new badge).
- No directional inference (e.g., upgrading `MIXED` to `BULLISH+` based
  on `price_vs_vwap`).
- No tooltips that reword the sidecar's enum values.
- No color-coding rule that depends on a value not present in the
  sidecar's existing enum vocabulary.

The renderer is a pass-through display surface. Interpretation is the
sidecar's responsibility (and is deliberately conservative there).

FAIL: any of the patterns above appears in renderer source or rendered
HTML for the trend-structure section.
