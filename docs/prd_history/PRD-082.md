PRD-082 — Remove Redundant Dashboard Permission Copy

STATUS
IN PROGRESS

GOAL
Collapse the duplicate Permission + Stay Flat blocks in the System State card into a single "Trade Permission" field that retains the gold posture line.

SCOPE
- Merge the two rendering blocks (lines 730–735 of dashboard_renderer.py) into one
- Label: "Trade Permission" (replaces "Permission" and "Stay Flat")
- Value priority: stay_flat_reason if present, else run["permission"] if present, else nothing rendered
- Preserve `warn` CSS class (gold) when stay_flat_reason is shown
- Regenerate ui/dashboard.html and copy to ui/index.html
- Add/update focused test asserting redundant strings absent when posture detail is present

OUT OF SCOPE
- runtime.py, qualification.py, regime.py, output.py, notifier.py — untouched
- Contract JSON schema — untouched
- Artifact write logic — untouched
- _PERMISSION_LINES dict — untouched (runtime source-of-truth; not a display concern)

REQUIREMENTS

R1 — Single Trade Permission field
The System State card renders at most one permission element with label "Trade Permission".
- If stay_flat_reason is not None: render with `warn` class (gold), value = stay_flat_reason
- Else if run["permission"] is not None: render without `warn` class, value = run["permission"]
- Else: render nothing (no empty element)

FAIL: Both "Permission" label and "Stay Flat" label appear in the same rendered HTML.
FAIL: stay_flat_reason is rendered under any label other than "Trade Permission".
FAIL: Gold (`warn`) class absent when stay_flat_reason is present.

R2 — No redundant strings
When stay_flat_reason is present, "No new trades permitted." and the standalone "STAY FLAT" label
do not appear as separate elements in the System State card.

FAIL: HTML contains both "Stay Flat" as a label AND a separate "Permission" label in the System State block.

R3 — Safe fallback
When both stay_flat_reason and permission are None/missing, the System State card renders without
crashing and without an empty Trade Permission element.

FAIL: Renderer raises when permission and stay_flat_reason are both None.

R4 — Focused test
A test in tests/test_dash_system_state.py (or test_dash_timestamps.py) verifies:
- When stay_flat_reason is present: "Trade Permission" in HTML, "Stay Flat" not as standalone label,
  "No new trades permitted." not present, warn class present on that field.
- When stay_flat_reason absent and permission present: "Trade Permission" in HTML with permission text.

FAIL: Any of the above assertions fail.
