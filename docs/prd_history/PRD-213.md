CLASS
CONSUMER

WHY NOW
The trend-structure table is 8 columns wide and, on a phone, overflows its
container and clips the rightmost columns off-screen (the "Intraday" column
renders as "Intraday Cor…" / "VWAP N/A" truncated). The dashboard is used
primarily on mobile; the wide table is the single biggest legibility loss.

MAX EXPECTED DELTA
45 production LOC (dashboard_renderer.py table block + _CSS media query).

GOAL
Below a narrow-viewport breakpoint, reflow the trend-structure table into one
labeled card per symbol so every value is readable without horizontal scroll;
the desktop wide-table layout is unchanged above the breakpoint.

SCOPE
- Add an `@media (max-width: 640px)` rule to the _CSS block that restyles the
  trend-structure table into a stacked layout (each row → block card; each cell
  labeled by its column header).
- Emit a `data-label="<header>"` attribute on every trend-structure `<td>` so the
  stacked layout can render the header inline (`td::before{content:attr(data-label)}`).
- Give the table a stable class hook (e.g. `class="ts-table"`) for the media query.

OUT OF SCOPE
- Changing the column set, the PRD-165/PRD-208 collapse logic, or any rendered
  value.
- Responsiveness of other panels (macro tape → PRD-214; candidate card → PRD-215).

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dashboard_renderer.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — trend-structure table render block
(~L2219–2314) and the _CSS constant (~L639–744) only: add a class hook, per-cell
`data-label`, and one media-query rule. No change to header identity/order, the
collapsible-column set, or any computed cell value.

REQUIREMENTS

R1 — Stacked reflow below breakpoint
The _CSS block contains an `@media (max-width: 640px)` rule scoping the trend
table to a stacked (one-card-per-row) layout that does not rely on horizontal
overflow to show all columns.
FAIL: rendered HTML has no `@media (max-width: 640px)` rule targeting the trend
table's class.

R2 — Per-cell header labels
Every `<td>` in the rendered trend-structure body carries a `data-label` equal to
its column header text.
FAIL: any trend-structure body `<td>` lacks a `data-label` attribute.

R3 — Desktop layout preserved
Above the breakpoint the 8-column wide table renders unchanged in identity/order.
FAIL: the existing header identity/order and cell-count tests
(test_prd208_r3_*, test_prd132_r6f_*) fail.

DATA FLOW
trend_structure_snapshot → table render (adds class + per-cell data-label) → HTML
_CSS constant → media-query rule → narrow-viewport stacked layout

FAIL CONDITIONS
- No `@media (max-width: 640px)` rule targeting the trend table class.
- Any trend-structure body `<td>` missing `data-label`.
- Existing header identity/order or cell-count tests fail.

VALIDATION
Manual:
- Render the dashboard; at ≤640px width every trend-structure column value is
  visible with no horizontal scroll and each value shows its header label.
- At >640px the table renders identically to the pre-change wide layout.
Automated:
- New test asserts the media query and `data-label` on each body `<td>`.
- Existing PRD-208/PRD-132 table tests remain green.
