CLASS
CONSUMER

WHY NOW
The SYSTEM STATE panel reads like a grep of the engine: REGIME / OUTCOME /
PERMISSION say "no trade" three ways, the permission string leaks engine internals
(regime=NEUTRAL, confidence=0.12), and three separate freshness lines (RUN SNAPSHOT
/ LIVE STATE / SCOREBOARD) crowd it. It should distil to a human-readable verdict.

MAX EXPECTED DELTA
55 production LOC (dashboard_renderer.py system-state block).

GOAL
Distil SYSTEM STATE to a plain-English verdict, one context line, and a single
freshness timestamp — no engine-internal grep, no triple-stated outcome.

SCOPE
- Collapse REGIME / OUTCOME / PERMISSION into one verdict headline plus one context
  line (regime in words + posture), dropping the numeric confidence and the
  redundant restatements.
- Replace the three freshness lines with a single "Updated <token>" line
  (run-snapshot freshness), retaining the scoreboard token compactly.

OUT OF SCOPE
- Changing any decision value, the halt path, the run/payload contract, or the
  freshness-token computation helpers themselves.

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dash_system_state.py
M tests/test_dashboard_renderer.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — the system-state render block only
(the `<div id="system-state">` region). Freshness helpers (_run_snapshot_freshness_
token, _surface_age_token, _scoreboard_age_token) are reused, not changed.

REQUIREMENTS

R1 — Single distilled verdict
The panel renders one verdict headline conveying trade / no-trade / halt, not three
separate REGIME/OUTCOME/PERMISSION restatements.
FAIL: the panel renders the OUTCOME field AND a separate PERMISSION posture field
that restate the same no-trade state.

R2 — No engine-internal leak
The rendered panel does not show the raw "regime=…, confidence=0.NN" internal
string.
FAIL: the system-state block contains "confidence=" in its rendered text.

R3 — Single freshness line
Freshness renders as one timestamp/age line, not three separate RUN SNAPSHOT / LIVE
STATE / SCOREBOARD lines.
FAIL: the block renders more than one distinct freshness field label.

R4 — Halt still unmistakable
A halted system still renders an explicit halt verdict.
FAIL: under system_halted the panel omits an explicit halt indication.

DATA FLOW
market_regime + outcome + permission/halt → one verdict + one context line
freshness tokens → single "Updated <token>" line

FAIL CONDITIONS
- OUTCOME and a separate PERMISSION posture both restate no-trade.
- "confidence=" appears in rendered text.
- More than one freshness field label renders.
- Halt state lacks an explicit indication.

VALIDATION
Manual:
- Render: one verdict line, one context line, one freshness line; no confidence
  internals; halt path shows a clear halt verdict.
Automated:
- Rewrite the system-state field tests to the distilled layout; keep the halt and
  freshness-token boundary tests.
