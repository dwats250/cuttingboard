# PRD-045 ā€” Trade Decision Materialization

**Status:** IN PROGRESS
**Commit:** ā€”
**Author:** Dustin Watson
**Date:** 2026-04-28

---

## GOAL

Convert qualified, chain-validated candidates into explicit `TradeDecision` objects and materialize them into the existing `contract["trade_candidates"]` structure.

Replace implicit runtime inference:

```python
validated_count > 0 ā†’ OUTCOME_TRADE
```

With explicit decision logic:

```python
any(c["decision_status"] == "ALLOW_TRADE" for c in contract["trade_candidates"]) ā†’ OUTCOME_TRADE
```

No new signals. No new gates. No new data sources.

---

## SCOPE

- Introduce `TradeDecision` dataclass in a new `trade_decision.py` module
- Introduce `create_trade_decision()` conversion function
- Populate existing `trade_candidates` contract entries with executable fields and decision status
- Update runtime outcome logic to depend on `TradeDecision`
- Add contract validation for new fields
- Maintain full determinism

---

## OUT OF SCOPE

- New indicators or signals
- Modifying qualification gate logic
- Post-trade evaluation
- UI or dashboard changes
- New data sources
- Order execution
- New top-level contract keys
- A parallel `trade_decisions` list (the existing `trade_candidates` structure is the surface)

## REQUIREMENTS

### R1 ā€” TradeDecision Dataclass

`cuttingboard/trade_decision.py` must define:

```python
@dataclass(frozen=True)
class TradeDecision:
    ticker:       str
    direction:    str            # "LONG" | "SHORT"
    status:       str            # "ALLOW_TRADE" | "BLOCK_TRADE"
    entry:        float
    stop:         float
    target:       float
    r_r:          float          # computed: abs(target - entry) / abs(entry - stop)
    contracts:    int
    dollar_risk:  float
    block_reason: Optional[str]  # required if BLOCK_TRADE, None if ALLOW_TRADE
```

FAIL: dataclass is not frozen.
FAIL: `block_reason` is not None when `status == "ALLOW_TRADE"`.
FAIL: `block_reason` is None when `status == "BLOCK_TRADE"`.
FAIL: any numeric field is non-finite.

---

### R2 ā€” Conversion Function

`trade_decision.py` must define:

```python
def create_trade_decision(
    candidate: TradeCandidate,
    result: QualificationResult,
    setup: OptionSetup,
    chain: ChainValidationResult,
) -> TradeDecision:
```

Decision logic:

```
ALLOW_TRADE if:
    chain.classification == VALIDATED

BLOCK_TRADE if:
    chain.classification != VALIDATED
    block_reason = chain.reason or chain.classification
```

Field mapping:

```
entry       = candidate.entry_price
stop        = candidate.stop_price
target      = candidate.target_price
r_r         = round(abs(target - entry) / abs(entry - stop), 2)
contracts   = setup.max_contracts
dollar_risk = setup.dollar_risk
```

Note: `r_r` is the only value not directly mapped from a field. It is arithmetic on existing fields ā€” no new signals or indicators.

FAIL: `PolicyContext` is referenced or used anywhere in this function.
FAIL: any value is recomputed from raw price data rather than sourced from existing objects.
FAIL: new filtering logic is introduced beyond `chain.classification == VALIDATED`.

### R3 ā€” Runtime Outcome Logic

Replace in `runtime.py`:

```python
validated_count = sum(...)
outcome = OUTCOME_TRADE if validated_count > 0 else OUTCOME_NO_TRADE
```

With:

```python
decisions = [
    create_trade_decision(candidates[s], qual_results[s], setup, chain_results.get(s))
    for s, setup in option_setups_by_symbol.items()
]
outcome = OUTCOME_TRADE if any(d.status == "ALLOW_TRADE" for d in decisions) else OUTCOME_NO_TRADE
```

FAIL: `validated_count` still drives `OUTCOME_TRADE` anywhere in the normal run path.
FAIL: `OUTCOME_TRADE` is produced without at least one `ALLOW_TRADE` decision.

`_build_candidates()` in `contract.py` must populate each candidate dict from the corresponding `TradeDecision`:

```python
{
    "symbol":           result.symbol,
    "direction":        ...,
    "entry_mode":       ...,
    "strategy_tag":     ...,
    "trigger":          None,
    "entry":            decision.entry,          # was None

    "target":           decision.target,         # was None
    "risk_reward":      decision.r_r,            # was None

    "setup_quality":    ...,
    "notes":            ...,
    "decision_status":  decision.status,         # NEW

```

FAIL: `entry`, `stop`, `target`, or `risk_reward` remain `None` for a candidate that has a corresponding `TradeDecision`.
FAIL: `decision_status` or `block_reason` is absent from any candidate dict.
FAIL: A new top-level contract key named `trade_decisions` is added.

---

### R5 ā€” Contract Validation

`assert_valid_contract()` must enforce for each item in `trade_candidates`:

- `decision_status` present and value is `"ALLOW_TRADE"` or `"BLOCK_TRADE"`
- `block_reason` is `None` if `decision_status == "ALLOW_TRADE"`
- `block_reason` is a non-empty string if `decision_status == "BLOCK_TRADE"`
- `entry`, `stop`, `target`, `risk_reward` are finite floats (not None) for `ALLOW_TRADE` candidates

FAIL: `assert_valid_contract()` passes a contract with a candidate missing `decision_status`.
FAIL: `assert_valid_contract()` passes a contract with `ALLOW_TRADE` and non-finite numeric fields.

`audit.py` must serialize `TradeDecision` fields without error. No custom encoder required ā€” all fields are native Python types.

## ACCEPTANCE CRITERIA

**AC1 ā€” One decision per candidate**
Each item in `trade_candidates` has exactly one `TradeDecision` backing it.

**AC2 ā€” Explicit outcome**
`OUTCOME_TRADE` requires at least one candidate with `decision_status == "ALLOW_TRADE"`. `validated_count` no longer drives outcome.

**AC3 ā€” Executable fields populated**
`entry`, `stop`, `target`, `risk_reward` are non-null floats for all `ALLOW_TRADE` candidates.

**AC4 ā€” No new logic**
All values sourced from existing pipeline objects. `r_r` is arithmetic only.

**AC5 ā€” Block reason correctness**
`BLOCK_TRADE` candidates carry exactly one non-empty `block_reason`. `ALLOW_TRADE` candidates carry `block_reason = None`.

**AC6 ā€” Single contract surface**
No new top-level key. All decision data lives in existing `trade_candidates` entries.

**AC7 ā€” Contract validation enforced**
`assert_valid_contract()` rejects contracts with missing or invalid `decision_status` / `block_reason`.

**AC8 ā€” Determinism**
Same inputs produce identical `TradeDecision` list.

**AC9 ā€” No regression**
All existing tests pass. No qualification logic modified.
