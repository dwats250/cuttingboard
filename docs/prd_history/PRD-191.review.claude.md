SUMMARY
PRD-191 fixes the macro-evidence rationale subtitle so it agrees in direction
with the cyclicality-aware vote rendered alongside it. The pre-fix rationale was
a flat payload_key -> string map with no direction term, so a rising 10Y voted
risk-OFF while still reading "easing yields favor risk". The fix keys
MACRO_BIAS_INTERPRETATION by payload_key -> {rising, falling}, selects the form
with the SAME arrow that drives the vote (single source of direction, no second
sign calc), and routes the flat/no-vote case to one shared private neutral
constant. All three requirements (R1/R2/R3) are met; the four new tests are
genuine red->green guards; FILES boundary held. The change is correct and ships.

REQUIRED EDITS
None. R1, R2, R3 are satisfied and the scope contract is respected.

RECOMMENDED EDITS

1. Document the implicit token-substring contract the agreement test depends on.
   The agreement assertion (tests/test_dashboard_renderer.py:3597-3604) decides
   direction by substring: a risk-ON interp must contain "risk" and not
   "caution"; a risk-OFF interp must contain "caution" and not "risk". All eight
   current strings (macro_tape_layout.py:93-96) honor this cleanly, but the
   convention is implicit -- a future driver string that used "risk" inside a
   caution sense (or omitted both tokens) would make the guard mis-pass or
   mis-fail without any structural signal. Non-blocking; the current strings are
   correct. FAIL (if adopted): a code comment at macro_tape_layout.py:91-98 or at
   the test helper (tests/test_dashboard_renderer.py:3592) does not state that
   every rising/falling form must contain exactly one of the tokens {risk,
   caution} matching its vote sign.

2. Consider asserting the neutral string is distinct from every directional form.
   test_prd191_flat_renders_shared_neutral_rationale
   (tests/test_dashboard_renderer.py:3633-3644) verifies the flat case collapses
   to one string carrying neither "risk" nor "caution", which is sufficient
   today. A stronger guard would also assert the neutral string is not equal to
   any value in MACRO_BIAS_INTERPRETATION, pinning the "one shared neutral form,
   not a directional one" intent (PRD-191.md:113-117) against a future edit that
   accidentally reuses a directional phrase. Non-blocking. FAIL (if adopted): the
   flat test passes when _MACRO_BIAS_NEUTRAL_INTERP is set to a string that also
   appears as a rising/falling form.
