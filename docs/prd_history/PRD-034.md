PRD-034 — GitHub Pages Deployment — Remote Read-Only Access

STATUS
IN PROGRESS

OBJECTIVE
Publish the existing static UI to GitHub Pages so the latest run output is
accessible at a public URL without cloning the repo or running anything locally.

After each pipeline run, the live contract is automatically committed to
ui/contract.json and GitHub Pages redeploys within ~1 minute. The URL always
shows the output of the most recent run.

---

SCOPE
- Extend the existing pipeline workflow to publish ui/contract.json after each run
- Add a GitHub Pages deployment workflow triggered by pushes to main
- Add auto-fetch to app.js: silently loads contract.json when on HTTP/HTTPS
- Seed ui/contract.json with mock_contract.json as an initial placeholder

OUT OF SCOPE
- No changes to Python modules (runtime.py, contract.py, delivery/)
- No changes to the contract schema or LATEST_CONTRACT_PATH
- No server, no backend, no authentication
- No changes to the existing file/paste load paths in the UI
- No test suite changes
- Preventing CI from running on contract-only commits (separate concern)

---

ARCHITECTURE

Current state:
  pipeline run → runtime.py → logs/latest_contract.json (committed to main by GHA)

After this PRD:
  pipeline run → runtime.py → logs/latest_contract.json
                                       ↓ (workflow copies)
                               ui/contract.json (committed to main)
                                       ↓ (pages workflow triggers)
                               GitHub Pages redeploys ui/
                                       ↓
                               https://{owner}.github.io/{repo}/

Auto-fetch sequence in browser:
  User opens GitHub Pages URL
  → app.js DOMContentLoaded
  → autoFetch() detects HTTP/HTTPS context
  → fetch('./contract.json')
  → on 200: loadJSON(text) → renders contract
  → on error/404: stays at "NO CONTRACT LOADED" (existing behavior)

file:// open (local dev):
  → autoFetch() detects file:// → exits immediately → existing behavior unchanged

REQUIREMENTS

R1 — Auto-Load on HTTP/HTTPS
When the UI is opened on any HTTP or HTTPS origin, app.js MUST attempt to
fetch './contract.json' and render it automatically if the response is 200.
FAIL: UI on GitHub Pages URL shows "NO CONTRACT LOADED" despite contract.json being present.

R2 — Silent Fallback on file://
When the UI is opened via file://, autoFetch MUST exit immediately without
any fetch attempt, network error, or console noise.
FAIL: Any network error or console warning appears when opened via file://.

R3 — Silent Fallback on Missing Contract
If fetch('./contract.json') returns non-200 or throws, the UI MUST remain at
the "NO CONTRACT LOADED" state. No error message, no console error.
FAIL: UI shows an error or broken state when contract.json is absent.

R4 — Pipeline Publishes Contract
After every pipeline run (live, sunday, all modes), the workflow MUST copy
logs/latest_contract.json → ui/contract.json and include it in the git commit.
FAIL: ui/contract.json is not updated after a pipeline run that produces output.

R5 — Pages Deploys on Push to Main
Every push to main that changes ui/ MUST trigger a GitHub Pages redeploy.
The deployed URL serves the contents of the ui/ directory at the site root.
FAIL: A committed change to ui/contract.json does not result in an updated Pages site.

R6 — No Regression on File/Paste Load
The existing file upload and paste JSON load paths MUST continue to work
identically under all conditions (file://, HTTP/HTTPS, GitHub Pages).
FAIL: Loading a contract via file picker or paste fails after this change.

R7 — No Python Changes
Zero modifications to any .py file under cuttingboard/.
FAIL: Any Python file is modified.

autoFetch() IMPLEMENTATION

  function autoFetch() {
    if (location.protocol === 'file:') return;
    fetch('./contract.json')
      .then(function (r) { return r.ok ? r.text() : null; })
      .then(function (text) { if (text) loadJSON(text); })
      .catch(function () {});
  }

Called at the end of the DOMContentLoaded handler, after all other init.

---

pages.yml SPECIFICATION

  name: Deploy to GitHub Pages
  on:
    push:
      branches: [main]
  permissions:
    pages: write
    id-token: write
    contents: read
  jobs:
    deploy:
      runs-on: ubuntu-latest
      environment:
        name: github-pages
        url: ${{ steps.deployment.outputs.page_url }}
      steps:
        - uses: actions/checkout@v4
        - uses: actions/upload-pages-artifact@v3
          with:
            path: ui/
        - id: deployment
          uses: actions/deploy-pages@v4

FAIL CONDITIONS

- UI on GitHub Pages shows "NO CONTRACT LOADED" when contract.json exists
- autoFetch runs on file:// and produces a network error
- missing contract.json causes any visible error state
- Pipeline run does not update ui/contract.json in the commit
- Push to main does not trigger a Pages redeploy
- File/paste load paths broken after this change
- Any .py file modified
