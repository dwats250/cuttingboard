REQUIRED EDITS

1. Fix the broken `-f`/`-d` short-form deny patterns (PRD-259.md:312, `.claude/settings.json` deny entries `"Bash(git *push* -f:*)"` and `"Bash(git *push* -d:*)"`). Verified live in this review: `git push -f <fake-remote> <ref>` and `git push -d origin refs/heads/<ref>` both execute without a confirmation prompt — the deny patterns never fire for any input, not even the literal-order case the pre-PRD-259 pattern (`"Bash(git push -f:*)"`, a pure literal prefix with no embedded wildcard) used to catch. The root cause, confirmed by contrast-testing every other colon-suffixed pattern in the file (e.g. `"Bash(gh pr merge:*)"`, which correctly denied `gh pr merge --admin 999999` in this review): patterns ending in `:*` treat everything before the colon as a literal prefix, not a glob — embedding `*` characters inside that prefix (as R7's `-f`/`-d` patterns do) makes the pattern require the literal text `git *push* -f` to open the command, which no real invocation ever contains, so the pattern can never match. Every other new/modified pattern in this PRD (R5, R6, R8, and R7's own `--force`/`--delete`/`--mirror`/`--prune`/`--rece` entries) avoids this by using pure-glob form with no colon suffix, which is why only these two are broken.

Fixed once the pattern is corrected to pure-glob form (e.g. `"Bash(git *push* -f *)"` / `"Bash(git *push* -d *)"`, mirroring R6's already-fixed `"Bash(*ruff check* -o*)"` style) and this same live command no longer executes.

2. Substantiate or correct the "37-case matching simulation" claim (PRD-259.md:337-347). The claimed simulation reports "0 failures / 37" including the `-f`/`-d` patterns, but this review's live testing shows both patterns silently match nothing. Either the simulation was not run against actual Bash-tool invocations through the live permission engine (in which case it is a proxy per PRD-198 invariant 3, and the PRD should say so plainly rather than imply engine-verified parity), or it was run against a draft pattern set that diverged from what was actually committed.
