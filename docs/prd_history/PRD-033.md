PRD-033 — UI Theme Layer — Sideloadable CSS Theme System

STATUS
IN PROGRESS

OBJECTIVE
Replace the monolithic styles.css with a two-layer CSS architecture: a structural
base layer and a swappable aesthetic theme layer. Themes are CSS files dropped
into ui/themes/ and loaded at runtime without code changes to app.js or index.html.

Adding a new theme requires only one CSS file and one manifest entry. Nothing else.

---

SCOPE
- Split styles.css into structural base + default aesthetic theme
- Define a CSS custom property contract every theme must satisfy
- Implement a dynamic theme loader in app.js
- Implement a theme switcher UI control with localStorage persistence
- Ship 3 themes: default (current), projection-finance, echofi

OUT OF SCOPE
- No contract schema changes
- No server required — all file:// loads must work
- No Python pipeline changes
- No new rendering logic — themes change appearance only
- No theme editor, no live CSS preview
- No changes to ui/mock_contract.json

---

ARCHITECTURE

Two-layer CSS model:

  Layer 1 — ui/styles.css (structural, always loaded)
    Layout, sizing, flexbox/grid rules, transitions, font-size scale.
    Zero color values. Zero font-family declarations. Zero border-radius values.
    Structural concerns only.

  Layer 2 — ui/themes/{name}.css (aesthetic, swapped at runtime)
    Must define ALL --cb-* custom properties listed in the Theme Contract below.
    May additionally override any selector for aesthetic purposes.

HTML load order (index.html):
  <link rel="stylesheet" href="styles.css">
  <link id="theme-link" rel="stylesheet" href="themes/default.css">

Theme loader in app.js sets #theme-link href on init and on switcher selection.

---

THEME CONTRACT

Every theme file MUST define all of the following CSS custom properties on :root.
No theme may omit any property. Missing properties are a hard fail.

  Surface
  --cb-bg              page background
  --cb-bg-card         card / panel surface
  --cb-bg-elevated     elevated surface (inputs, chips, tooltips)

  Borders
  --cb-border          default border (rgba, subtle)
  --cb-border-hi       bright border (hover, focus)

  Text
  --cb-text            primary text
  --cb-text-2          secondary / label text
  --cb-text-3          muted / placeholder text

  Semantic colors (each requires a full-opacity value and a dim/tinted variant)
  --cb-green           RISK_ON / TRADE / pass
  --cb-green-dim       green at ~10% opacity (backgrounds)
  --cb-red             RISK_OFF / HALT / fail
  --cb-red-dim         red at ~10% opacity
  --cb-amber           NEUTRAL / WATCHLIST / warning
  --cb-amber-dim       amber at ~10% opacity
  --cb-accent          primary UI accent (buttons, links, active states)
  --cb-accent-dim      accent at ~10% opacity

  Typography
  --cb-font-ui         UI font-family stack (sans-serif)
  --cb-font-mono       monospace font-family stack

  Shape
  --cb-radius          default border-radius (cards, buttons)
  --cb-radius-sm       small border-radius (chips, badges)

Total: 20 required properties.

---

THEME MANIFEST

Defined as a JS constant in app.js — NOT a JSON fetch (avoids file:// CORS).

  const THEME_MANIFEST = [
    { id: 'default',            label: 'Default' },
    { id: 'projection-finance', label: 'Projection Finance' },
    { id: 'echofi',             label: 'EchoFi' },
  ];

Theme CSS path convention: ui/themes/{id}.css

---

THEME SWITCHER (rendered by app.js)

- Compact control rendered in the UI footer or header area
- Iterates THEME_MANIFEST to build selector options
- On init: reads localStorage.getItem('cb_theme') ?? 'default', applies it
- On selection: sets #theme-link href, writes localStorage key 'cb_theme'
- No page reload required — CSS hot-swaps via href change

FILES

M  ui/index.html

     No other structural changes

M  ui/app.js
     Add THEME_MANIFEST constant

REQUIREMENTS

R1 — Structural Isolation
styles.css MUST contain zero color values (no hex, rgb, rgba, hsl), zero
font-family declarations, and zero border-radius values after this change.
FAIL: grep for any of the above in styles.css returns a match.

R2 — Theme Contract Completeness
Every shipped theme MUST define all 20 --cb-* custom properties.
FAIL: Any --cb-* property is absent in any shipped theme file.

R3 — Drop-In New Theme
Adding a new theme requires ONLY: (a) create ui/themes/{name}.css with all
--cb-* props defined; (b) add one entry to THEME_MANIFEST in app.js.
No changes to index.html or styles.css are required.
FAIL: Adding a new theme requires editing index.html or styles.css.

R4 — File Protocol Compatible
All theme files MUST load when ui/index.html is opened via file:// with no
server. No fetch() calls for theme loading. No XHR.
FAIL: Any network request fails in file:// mode.

R5 — Theme Persistence
Selected theme MUST persist across page reloads via localStorage key 'cb_theme'.
FAIL: Reloading the page resets the theme.

R6 — Fallback Safety
If #theme-link fires an error event (missing file), app.js MUST:
  - Set href back to themes/default.css
  - Write 'default' to localStorage
  - Log a console.warn identifying the failed theme id
FAIL: UI renders broken or silent on a missing theme file.

R7 — Zero Regression
All contract fields currently rendered by app.js MUST continue to render
correctly under all 3 shipped themes. No field may go missing or unreadable.
FAIL: Any contract field absent or unreadable in any theme.

R8 — Hot-Swap
Switching themes in the switcher MUST re-render the UI without a page reload.
FAIL: Theme change requires a full page reload.

---

DATA FLOW

  ui/index.html
    ↓ loads styles.css (structural base — layout only)
    ↓ loads themes/default.css (initial aesthetic — overridden by initTheme)
    ↓ executes app.js

  app.js init sequence:
    1. initTheme()        — reads localStorage, sets #theme-link href
    2. loadContract()     — fetches/reads contract JSON
    3. renderAll()        — renders contract data using active CSS vars
    4. renderThemeSwitcher() — appends switcher to DOM

  Theme switch event:
    switcher onChange → set #theme-link href → localStorage.setItem → (no reload)

---

FAIL CONDITIONS

- styles.css contains any color value or font-family after migration
- Any --cb-* property missing from any shipped theme
- Theme switch requires page reload
- Missing theme file causes silent broken render
- Any currently-rendered contract field disappears under any theme
- file:// load fails due to fetch/XHR for theme loading

---

VALIDATION

1. Open ui/index.html via file:// — confirm default theme loads, all contract
   fields render, no console errors.

2. Switch to each theme via the switcher — confirm instant palette change,
   all fields still readable, no reload required.

3. Reload after switching — confirm selected theme is restored from localStorage.

4. Rename themes/echofi.css to break it — confirm fallback to default with
   console.warn, no broken render.

5. Create ui/themes/test.css defining only --cb-* properties, add one entry to
   THEME_MANIFEST — confirm it renders with the new palette without touching
   index.html or styles.css.

6. grep -E '(#[0-9a-fA-F]{3,6}|rgba?|hsl|font-family|border-radius)' ui/styles.css
   must return zero matches.
