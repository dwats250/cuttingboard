PRD-022 — Sunday Mode Isolation

STATUS
COMPLETE
Commit: 2b6009a

GOAL
Isolate Sunday mode so it never fetches live data, always produces STAY_FLAT via a synthetic empty regime, and completes the full pipeline offline.

SCOPE
- ingestion.py: add block_live_data() context manager with thread-local guard; fetch_all_quotes and fetch_intraday_bars raise LIVE_DATA_FORBIDDEN_IN_SUNDAY_MODE when guard is active
- runtime.py: Sunday mode without fixture bypasses _load_inputs and validate_quotes entirely; uses synthetic empty ValidationSummary → NEUTRAL/STAY_FLAT with no live fetch
- runtime.py: fix verify_run_summary null check to allow continuation_audit=None
- Add tests/test_sunday_mode.py (13 tests, AC1–AC7)

OUT OF SCOPE
- No changes to fixture mode
- No changes to regime vote model

FILES
M cuttingboard/ingestion.py
M cuttingboard/runtime.py
A tests/test_sunday_mode.py

REQUIREMENTS

R1 — No Live Data
Any call to fetch_all_quotes or fetch_intraday_bars inside block_live_data() MUST raise LIVE_DATA_FORBIDDEN_IN_SUNDAY_MODE.

FAIL: A live data fetch succeeds during Sunday mode execution.

R2 — Synthetic STAY_FLAT
Sunday mode without fixture MUST produce tradable=False via NEUTRAL/STAY_FLAT from an empty ValidationSummary.

FAIL: Sunday mode produces tradable=True.

R3 — Pipeline Completion
Sunday mode MUST complete the full pipeline (through notification and artifact write) without raising an exception.

FAIL: Sunday mode raises an unhandled exception.

R4 — Single Notification
Sunday mode MUST send exactly one notification.

FAIL: Sunday mode sends more than one notification.

R5 — run_status Not ERROR
Sunday mode run_status MUST NOT be ERROR when pipeline completes normally.

FAIL: run_status == ERROR on a successful Sunday mode run.

DATA FLOW
runtime SUNDAY mode → block_live_data() guard → synthetic ValidationSummary → compute_regime({}) → NEUTRAL/STAY_FLAT → pipeline completion

FAIL CONDITIONS
- Live fetch succeeds in Sunday mode
- tradable=True in Sunday mode
- Unhandled exception in Sunday mode
- More than one notification sent
- run_status == ERROR on normal completion

VALIDATION
Run: pytest tests/test_sunday_mode.py -q (13 tests, AC1–AC7)
Manual: python -m cuttingboard --mode sunday — confirm no live fetch, STAY_FLAT output, single Telegram message.
