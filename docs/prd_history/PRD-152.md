# PRD-152 — Batch B: Compatibility Shim Removal

**Status:** COMPLETE
**Lane:** Standard (not HIGH-RISK)
**Date drafted:** 2026-05-22
**Authors:** Claude (project lead) on direction from Dustin
**Predecessor context:** Deferred follow-up from the 2026-05-22 architectural alignment audit (Part B). Scoped in `docs/DECISIONS.md` 2026-05-22 entry under "Deferred follow-ups — Batch B."

---

## GOAL

Remove the `apply_sector_router(...)` pass-through in `cuttingboard/sector_router.py` and the three no-op helpers in `cuttingboard/universe.py` (`filter_execution_dict`, `filter_execution_items`, `log_universe_configuration`) flagged as kill candidates by the 2026-05-22 alignment audit (Flag 2). Update every call site that references these symbols and remove orphaned imports.

Outcome: smaller surface in `runtime.py`, `sector_router.py`, and `universe.py` with no behavior change.

---

## CONTEXT

The 2026-05-22 alignment audit verdict was ALIGNED with four tensions. Flag 2 identified:

- `cuttingboard/sector_router.py:45-50` — `apply_sector_router(...)` returns `(qualification_summary, [])` unchanged. Verified at HEAD.
- `cuttingboard/universe.py:20-37` — `filter_execution_dict`, `filter_execution_items`, `log_universe_configuration` are dict/list pass-throughs and a debug log line. Verified at HEAD.

Flag 2 did **not** flag `SectorRouterState`, `SuppressedCandidate`, or `resolve_sector_router` in `sector_router.py`. Those are out of scope for this PRD.

This PRD is the bounded execution of the deferred follow-up. It is not the runtime monolith refactor (Batch C / runtime Group 6 extraction), which is HIGH-RISK lane.

---

## NON-GOALS

- No `runtime.py` refactor beyond removing the orphaned call sites and imports for the four removed symbols. Runtime monolith debt is re-evaluated 2026-08-15.
- No semantic replacement for `apply_sector_router(...)`. Removal is the change; downstream consumers absorb the empty-list invariant via R5.
- No removal of `SectorRouterState`, `SuppressedCandidate`, or `resolve_sector_router`. These have live consumers (`runtime.py:226, 456, 473, 737, 773, 1720, 1775`; `tests/test_hourly_alert.py:25,109-110`) and were not flagged by Flag 2.
- No removal of `is_tradable_symbol` from `universe.py`. It is the surviving universe boundary.
- No audit record schema change. `suppressed_candidates` continues to emit at `cuttingboard/audit.py:227`. See R5.
- No PRD-151 modification.
- No documentation rewrite beyond the three targeted edits identified in R7. Architectural docs and the sidecar doctrine are not restructured.

## REQUIREMENTS

### R1 — Remove `apply_sector_router` from `sector_router.py`

**Verification:** `grep -rn "apply_sector_router" .` returns zero matches outside this PRD and historical records (DECISIONS, completed PRD files, recon reports).

### R2 — Remove no-op helpers from `universe.py`

**Verification:** `grep -rn "filter_execution_dict\|filter_execution_items\|log_universe_configuration" .` returns zero matches outside historical records.

### R3 — Update call sites in `runtime.py`

Two call-site classes:

**(a) `apply_sector_router` call sites — three known at HEAD:**
- `runtime.py:504-508` — return is `(qual, _)`; delete the assignment, the prior `qualification_summary` value is already correct.
- `runtime.py:533-537` — same shape.
- `runtime.py:821-825` — return is `(qual, suppressed_candidates)`; delete the assignment. `suppressed_candidates` remains the value initialized at `runtime.py:749` (`list[SuppressedCandidate] = []`). See R5.

**(b) Universe-helper call sites — ten known at HEAD:**
- `runtime.py:483, 512, 780` (`log_universe_configuration(logger)`) — delete each line.
- `runtime.py:485-487, 514-516, 781-783` (`filter_execution_dict(x, log=logger)`) — inline as `dict(x)`.
- `runtime.py:799` (`filter_execution_items(items, symbol_getter=…, log=logger)`) — inline as `list(items)`.

Update the import lines at `runtime.py:107-112` (remove `apply_sector_router` from the `sector_router` import tuple) and `runtime.py:114` (remove `filter_execution_dict, filter_execution_items, log_universe_configuration` from the universe import).

Implementation must verify line numbers at the time of work (line numbers drift).

### R4 — Call sites outside `runtime.py`

Known consumers at HEAD that require edits:

- `tools/engine_doctor.py:100` — remove `"apply_sector_router"` from the symbol list in the component tuple. If the row becomes empty of meaningful symbols, leave the row with the remaining `sector_router` component reference (since `resolve_sector_router` is retained).
- `tests/test_runtime_decision.py:147-149, 160` — remove monkeypatches of `runtime.log_universe_configuration`, `runtime.filter_execution_dict`, `runtime.filter_execution_items`, `runtime.apply_sector_router`. - `tests/test_gap_down_permission_integration.py:402-406` — remove the `patch("cuttingboard.runtime.log_universe_configuration")` and `patch("cuttingboard.runtime.filter_execution_dict", …)` entries. Retain the `resolve_sector_router` patch.

**Verification per helper:** `grep -rn "<removed-symbol>" .` returns zero matches outside historical records and this PRD.

### R5 — Preserve audit record `suppressed_candidates` field (recommended option)

`runtime.py:749` already initializes `suppressed_candidates: list[SuppressedCandidate] = []` before any `apply_sector_router` call. The audit call at `runtime.py:1036, 1101` reads this variable and `cuttingboard/audit.py:227` emits `list(suppressed_candidates or [])`. After R3 deletes the line-821 assignment, the line-749 initialization remains the sole writer and the audit emission is unchanged.

**Implementation choice:** Use the existing `runtime.py:749` initialization as the empty-list source. No edit to `cuttingboard/audit.py` is required. Audit record schema unchanged.

**Verification:** Inspect a `logs/audit.jsonl` record after a representative run; `suppressed_candidates` is present and equals `[]`.
