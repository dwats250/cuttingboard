# PRD-171 — Sync PRD templates' status markers to the prd_close.sh flip convention

CLASS
GOVERNANCE

GOAL
`scripts/prd_close.sh` flips a PRD's status with two line-anchored
regexes — `^Status:.*$` → `Status: COMPLETE` and `^STATUS:.*$` →
`STATUS: COMPLETE @ <hash>` — and `scripts/prd_open.sh` scaffolds docs
with exactly those two markers (`Status: IN PROGRESS` header + trailing
`STATUS: IN PROGRESS`). But `docs/PRD_TEMPLATE.md` and
`docs/PRD_MICRO_TEMPLATE.md` both use a block-style `STATUS` / `<value>`
header with NO `Status:` header line and NO trailing `STATUS:` marker.
A PRD hand-authored from either template therefore cannot be flipped by
`prd_close.sh`: at PRD-170 closeout the script emitted
"'Status:' header line not found" / "trailing STATUS line not found"
WARNs and appended a marker, leaving the doc's top block reading
`IN PROGRESS` until it was manually patched. Sync both templates to the
flip convention so future hand-authored PRDs close out cleanly.

SCOPE
- In `docs/PRD_TEMPLATE.md`: replace the block `STATUS` / `IN PROGRESS`
  header (lines 3-4) with a `Status: IN PROGRESS` header line, and add a
  trailing `STATUS: IN PROGRESS` marker as the last line.
- In `docs/PRD_MICRO_TEMPLATE.md`: replace the block `STATUS` /
  `PROPOSED` header (lines 7-8) with a `Status: PROPOSED` header line,
  and add a trailing `STATUS: PROPOSED` marker as the last line.
  (Initial-status values are preserved per template — full=IN PROGRESS,
  micro=PROPOSED; only the marker FORMAT changes.)
- Track this PRD in registry/state per existing convention.

OUT OF SCOPE
- `scripts/prd_open.sh`'s thin scaffold body (Problem / Acceptance /
  Non-goals) diverging from the full template's richer sections
  (WHY NOW / MAX EXPECTED DELTA / DATA FLOW / FAIL CONDITIONS …) — a
  separate concern, not a status-marker fix.
- Any change to `prd_close.sh` / `prd_open.sh` / the validator.
- No executable code; no tests.

REQUIREMENTS

R1 — Full template carries both flippable markers.
After the change, `docs/PRD_TEMPLATE.md` contains a line matching
`^Status: IN PROGRESS$` and a line matching `^STATUS: IN PROGRESS$`, and
no longer contains a bare block-style `^STATUS$` line immediately
followed by `^IN PROGRESS$`.
FAIL: `grep -cE "^Status: IN PROGRESS$" docs/PRD_TEMPLATE.md` == 0, OR
`grep -cE "^STATUS: IN PROGRESS$" docs/PRD_TEMPLATE.md` == 0.

R2 — Micro template carries both flippable markers.
After the change, `docs/PRD_MICRO_TEMPLATE.md` contains a line matching
`^Status: PROPOSED$` and a line matching `^STATUS: PROPOSED$`.
FAIL: `grep -cE "^Status: PROPOSED$" docs/PRD_MICRO_TEMPLATE.md` == 0,
OR `grep -cE "^STATUS: PROPOSED$" docs/PRD_MICRO_TEMPLATE.md` == 0.

R3 — A template-derived doc flips cleanly under prd_close.sh's regexes.
Applying `re.sub(r"^Status:.*$", "Status: COMPLETE")` and
`re.sub(r"^STATUS:.*$", "STATUS: COMPLETE @ HASH")` (the exact
prd_close.sh transforms) to either template yields a doc whose `Status:`
header reads `Status: COMPLETE` and whose trailing marker reads
`STATUS: COMPLETE @ HASH`, with zero residual `IN PROGRESS`/`PROPOSED`
status markers.
FAIL: after the two transforms on a template copy, any line still
matches `^(Status|STATUS):.*(IN PROGRESS|PROPOSED)`, OR no line matches
`^STATUS: COMPLETE @ `.

Clean-flip simulation (must print COMPLETE markers, no residual):
```
python3 - <<'PY'
import re
for f in ("docs/PRD_TEMPLATE.md", "docs/PRD_MICRO_TEMPLATE.md"):
    t = open(f).read()
    t = re.sub(r"^Status:.*$", "Status: COMPLETE", t, count=1, flags=re.M)
    t = re.sub(r"^STATUS:.*$", "STATUS: COMPLETE @ HASH", t, flags=re.M)
    assert re.search(r"^Status: COMPLETE$", t, re.M), f
    assert re.search(r"^STATUS: COMPLETE @ HASH$", t, re.M), f
    assert not re.search(r"^(Status|STATUS):.*(IN PROGRESS|PROPOSED)", t, re.M), f
    print("clean-flip OK:", f)
PY
```
