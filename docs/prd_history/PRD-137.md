ROOT CAUSE
hidden dependency

PRD-136 added optional gold/silver macro drivers to
cuttingboard/contract._MACRO_DRIVER_SYMBOLS and
cuttingboard/contract._OPTIONAL_MACRO_DRIVERS but missed the duplicated
macro-driver validation surface in cuttingboard/delivery/payload.py. The
intentional duplication is documented at payload.py:240–242, which warns
"semantics must not drift." Drift occurred: the live pipeline now fetches
GC=F/SI=F, contract emission populates macro_drivers["gold"] and ["silver"],
and assert_valid_payload raises
`ValueError: macro_drivers has unexpected driver keys: ['gold', 'silver']`
at payload.py:263. _write_payload_artifacts fails, the renderer is never
invoked, and the dashboard does not refresh. Contract artifacts are
unaffected.

GOAL
Realign cuttingboard/delivery/payload.py's macro-driver validator with
PRD-136's contract emission so optional gold and silver drivers are
accepted, without altering any runtime, renderer, regime, qualification,
notifier, market_map, sizing, or decision behavior.

MAX EXPECTED DELTA
~4 LOC of production change in cuttingboard/delivery/payload.py
(_OPTIONAL_MACRO_DRIVERS set + two entries in `expected` dict). Test
deltas: 1 existing assertion repointed, several new assertions added. No
edits to cuttingboard/runtime.py, contract.py, config.py, ingestion.py,
normalization.py, regime.py, qualification.py, execution_policy.py,
market_map.py, notifications/, sidecars, or the dashboard renderer.

SCOPE
- Extend `cuttingboard/delivery/payload._OPTIONAL_MACRO_DRIVERS` to
  `frozenset({"oil", "gold", "silver"})`.
- Extend the `expected` map in
  `cuttingboard/delivery/payload._require_macro_drivers` to include
  `"gold": {"symbol", "level", "change_pct"}` and
  `"silver": {"symbol", "level", "change_pct"}`.
- Repoint the existing unknown-extra-driver test that uses
  `"gold"` as the unknown key
  (`tests/test_payload_macro_drivers.py:165–169`) to use a key that
  is still outside the allowlist, e.g. `"platinum"`.
- Add positive-case tests proving optional `gold` and `silver`
  blocks are accepted, individually and together, and that their
  absence is still accepted.
- Add field-shape failure tests for each new driver mirroring
  `test_require_macro_drivers_invalid_oil_field_shape_raises`.

REQUIREMENTS

R1 — Payload optional-driver set matches PRD-136
`cuttingboard.delivery.payload._OPTIONAL_MACRO_DRIVERS` equals
`frozenset({"oil", "gold", "silver"})`.

FAIL: `gold` or `silver` is absent from
`payload._OPTIONAL_MACRO_DRIVERS`.

R2 — Payload expected-driver schema includes gold and silver
`cuttingboard.delivery.payload._require_macro_drivers`'s internal
`expected` map contains `"gold"` and `"silver"` keys, each mapped to
`{"symbol", "level", "change_pct"}` — identical to the `"oil"` shape.

FAIL: Either key is missing from the `expected` map, or the field set
differs from `{"symbol", "level", "change_pct"}`.

R3 — Optional really is optional
`assert_valid_payload` passes when `macro_drivers` contains:
(a) only the four originally required drivers,
(b) those four plus `gold` only,
(c) those four plus `silver` only,
(d) those four plus both `gold` and `silver`.

FAIL: Any of (a)–(d) raises.

R4 — Unknown keys still rejected
`assert_valid_payload` raises `ValueError` matching
`"unexpected driver keys"` when `macro_drivers` contains a key outside
`{volatility, dollar, rates, bitcoin, oil, gold, silver}` (test with
`"platinum"`).

FAIL: An unrelated key is accepted.

R5 — Existing unknown-key test repointed
`tests/test_payload_macro_drivers.py::test_require_macro_drivers_unknown_extra_driver_raises`
uses a key other than `gold` or `silver` (e.g. `"platinum"`).

FAIL: The test still asserts `gold` is rejected as unknown.

R6 — Field-shape validation extends to new drivers
For each of `gold` and `silver`, a malformed block (missing
`change_pct`) is rejected by `_require_macro_drivers`.

FAIL: A malformed `gold` or `silver` block is accepted.

R7 — No decision-surface or non-target file mutation
No file outside the FILES allowlist is modified. In particular,
`cuttingboard/runtime.py`, `cuttingboard/contract.py`,
`cuttingboard/config.py`, the renderer, regime, qualification,
execution_policy, market_map, notifications/, and all sidecar modules
are byte-identical to HEAD.

FAIL: `git diff --name-only main...HEAD` includes any path outside
FILES.
