PRD-127 — Hourly Alert Action Language Alignment

STATUS
COMPLETE

LANE
STANDARD

CLASS
EXECUTION + PATCH

WHY NOW
A prior read-only review found that hourly Telegram alerts can render
`Action: TRADE` purely because `qualification_summary.symbols_qualified
> 0` and the regime is tradable, while the canonical hourly artifacts
(`logs/latest_hourly_run.json`, `logs/latest_hourly_contract.json`) are
written with `outcome=OUTCOME_NO_TRADE` (hardcoded at
`cuttingboard/runtime.py:560`, `:600`, and inside `_build_hourly_contract`
at `:1752`). This produces an operator-facing semantic mismatch: the
Telegram alert can imply an executable trade while the contract and run
summary report NO_TRADE. The misalignment is in the notifier's
`_action_label` (`cuttingboard/notifications/__init__.py:88-104`), which
infers action from candidate presence rather than from canonical hourly
outcome. PRD-101 set the truth contract for hourly notifications;
PRD-124 reshaped the body but preserved the candidate-driven TRADE
inference. This PRD removes the inference so monitor/review alerts are
labeled as such.

MAX EXPECTED DELTA
≤ 25 production LOC in `cuttingboard/notifications/__init__.py`
(restricted to `_action_label` and the title/action branches in
`format_hourly_notification`). Test additions in
`tests/test_hourly_alert.py` are not counted against this ceiling.
Exceeding the ceiling MUST stop implementation and amend the PRD.

GOAL
Prevent hourly Telegram alerts from using executable trade language
(`Action: TRADE`) when the canonical hourly contract/run outcome is
`OUTCOME_NO_TRADE`; render monitor/review language (`Action: MONITOR
SETUP`) when qualified or watchlist symbols exist but no canonical
TRADES outcome is supplied.

SCOPE
- `_action_label()` in `cuttingboard/notifications/__init__.py`: the
  "qualified and not flat → TRADE" branch is removed; candidate
  presence alone MUST NOT yield `Action: TRADE`.
- Replacement language when qualified or watchlist symbols exist with
  a non-flat tradable regime: `MONITOR SETUP`.
- Title branches in `format_hourly_notification` that currently render
  `f"{first_direction} {first_symbol} {pt}"` or `f"TRADE {pt}"` are
  re-routed through the `MONITOR SETUP` branch in this fallback case.
  A future canonical-TRADES path may reintroduce a TRADE title; see
  R3.
- Tests asserting the mismatch regression and the existing-behavior
  protections in R6/R7.

OUT OF SCOPE
- `cuttingboard/runtime.py`. Direct inspection shows
  `format_hourly_notification` does not receive a canonical-outcome
  argument; the action label is derived inside the notifier from
  `regime` + `validation_summary` + `qualification_summary`. Because
  hourly runtime today hardcodes `OUTCOME_NO_TRADE` at three call
  sites, the canonical-outcome value is a constant at the notifier
  boundary, so no signature change or runtime wiring is required to
  remove the executable-trade inference. Runtime is therefore not
  modified.
- `_build_hourly_contract` and the hourly run summary: no contract
  schema change, no payload field addition, no canonical-outcome
  rewiring.
- Notification transport (Telegram, ntfy), retry, throttling,
  suppression, audit.
- Regime, qualification, validation, sizing, market_map, OHLCV cache,
  fixture mode, dashboard renderer, readiness ordering.
- Banned-text guards or artifact refresh.
- Logs, reports, UI files.
- Intraday, premarket, midmorning, power-hour, market-close,
  run-summary, halt, or failure alert paths (this PRD touches only
  the hourly Telegram path through `format_hourly_notification`).
- The legacy `_format_hourly` path in
  `cuttingboard/notifications/formatter.py`.
- Modifying `_execute_notify_run` to pass `canonical_outcome` at the
  hourly call site. The parameter defaults to `None`; runtime
  rewiring is a future PRD. `runtime.py` remains excluded from
  `FILES`.

ROOT CAUSE
Missing fail condition. PRD-124 R4 defined the action-label rules but
allowed `Action: TRADE` whenever `symbols_qualified > 0` and the regime
was tradable, without referencing the canonical hourly contract/run
outcome. Because hourly runtime hardcodes `OUTCOME_NO_TRADE` for all
hourly summary and contract writes, the notifier's TRADE label is
unreachable in alignment with canonical artifacts.

CHANGE SURFACE
- `cuttingboard/notifications/__init__.py`: edits scoped to
  `_action_label` (logic) and the title-selection branches inside
  `format_hourly_notification`. No public signature changes; no
  changes to `__all__`. No new dependencies. No new fields on
  AlertEvent. No new ranking/scoring.
- `tests/test_hourly_alert.py`: additive regression tests; existing
  assertions that depend on the removed TRADE branch (e.g. line 327
  `assert "Action: TRADE" in body` and line 465
  `assert "Action: TRADE" in body`) MUST be updated in the same
  commit to assert `Action: MONITOR SETUP` instead, matching the
  new semantics. This is a Visible-String Pre-Edit Audit confirmed
  surface — both literals are in this PRD's FILES.

REQUIREMENTS

R1 — Candidate presence alone MUST NOT emit executable trade language
`format_hourly_notification` MUST NOT render `Action: TRADE` solely
because `qualification_summary.symbols_qualified > 0` or because
candidate-derived data exists. The "qualified and not flat → TRADE"
branch in `_action_label` MUST be removed.

FAIL: any rendered hourly body where the inputs satisfy
`qualification_summary.symbols_qualified > 0`,
`regime.posture != STAY_FLAT`, and `validation_summary.system_halted
is False`, AND no explicit canonical TRADES outcome is supplied,
contains the substring `Action: TRADE`.

R2 — NO_TRADE-equivalent hourly path uses monitor/review language
When qualified or watchlist symbols exist, the regime is tradable, and
the system is not halted (the operator-visible "setups present but no
canonical TRADES outcome" case), the rendered body MUST contain the
exact string `Action: MONITOR SETUP`. The title MUST NOT begin with
`TRADE ` or with a `LONG <SYMBOL>` / `SHORT <SYMBOL>` directional
prefix in this case; it MUST begin with `MONITOR SETUP `.

The pre-existing `MONITOR` label returned today when watchlist symbols
exist with no qualified symbols and a tradable regime MUST remain
unchanged. `MONITOR SETUP` is reserved for the previously-TRADE
branch only: qualified symbols exist, tradable regime, not halted,
and `canonical_outcome != OUTCOME_TRADE`. The `MONITOR SETUP` title
and body MUST NOT leak directional executable prefixes (`TRADE `,
`LONG `, `SHORT `) anywhere in the title or in the `Action:` line.

FAIL: a body for the above input shape lacks `Action: MONITOR SETUP`;
OR its title begins with `TRADE `, `LONG `, or `SHORT `; OR the
existing `MONITOR` label is removed from the watchlist-only branch.

R3 — TRADES canonical outcome path MUST remain renderable via an explicit parameter

`format_hourly_notification` MUST add a keyword parameter
`canonical_outcome: Optional[str] = None`, threaded into
`_action_label`. The TRADE branch in `_action_label` MUST be gated
on `canonical_outcome == OUTCOME_TRADE` or the existing equivalent
trade outcome constant in the notification module. No existing caller
in this PRD modifies its call site; the parameter defaults to `None`,
preserving today's behavior at runtime call sites. Future PRDs may
begin passing a canonical outcome from `_execute_notify_run`; that
change is out of scope here.

FAIL: `format_hourly_notification` does not declare a
`canonical_outcome` keyword parameter; OR `_action_label` returns
`"TRADE"` on any branch whose enclosing condition does not include
`canonical_outcome == OUTCOME_TRADE` or the existing equivalent trade
outcome constant; OR the default value of `canonical_outcome` is not
`None`.

R6 — Regression test covers the mismatch
`tests/test_hourly_alert.py` MUST include a deterministic test that
constructs `format_hourly_notification` inputs satisfying
`qualification_summary.symbols_qualified > 0`,
`regime.posture != STAY_FLAT`, and not halted, and asserts both:
  - `"Action: TRADE"` is NOT in the rendered body.
  - `"Action: MONITOR SETUP"` IS in the rendered body.
  - the rendered title does not begin with `TRADE `, `LONG `, or
    `SHORT `.

FAIL: `python3 -m pytest tests/test_hourly_alert.py -q` does not
include a test that asserts all three conditions on a single rendered
hourly event with the above input shape.

R7 — Existing supported outcomes remain green
Existing `tests/test_hourly_alert.py` assertions that depend on the
removed TRADE inference MUST be updated in the same commit to assert
the new `MONITOR SETUP` semantics. No other assertions in the file
may regress. The HALT branch, STAY FLAT branch, and MONITOR branch
(when no qualified or watchlist symbols exist) MUST remain reachable
and tested.

If a true canonical-TRADES hourly path exists today, the test file
MUST include a fixture exercising it and asserting `Action: TRADE`
renders. Direct inspection of `_execute_notify_run` shows hourly
runtime hardcodes `OUTCOME_NO_TRADE` at `cuttingboard/runtime.py:560`
and `:600`, and `_build_hourly_contract` sets
`contract["outcome"] = OUTCOME_NO_TRADE` at `:1752`; there is no
hourly TRADES path today. The test file MUST document this in a
comment alongside the R3 reachability test (which exercises the
formatter directly with a future-style canonical-TRADES signal, not
via runtime).

Additional required test (R3 reachability):
  - A test that calls `format_hourly_notification(...,
    canonical_outcome=OUTCOME_TRADE)` or the existing equivalent
    trade outcome constant with the same qualified inputs and asserts
    `"Action: TRADE"` IS in the body, proving R3 reachability via the
    new parameter.

FAIL: the regression test suite fails; OR existing tests asserting
`Action: TRADE` on candidate-only inputs are not updated to
`Action: MONITOR SETUP`; OR the HALT/STAY FLAT/MONITOR branches lose
test coverage; OR no test exercises
`canonical_outcome=OUTCOME_TRADE` or the existing equivalent trade

R8 — Grep-level proof that candidate presence does not drive trade language
After implementation, a static grep over
`cuttingboard/notifications/__init__.py` MUST show that no code path
returns the string `"TRADE"` (action label) from `_action_label`
solely on the basis of `qualification_summary.symbols_qualified` or
`qualification_summary.symbols_watchlist`. Any remaining return of
`"TRADE"` MUST be gated on a non-derived explicit canonical-outcome
signal (e.g. a parameter, not a computation over
`qualification_summary`).

FAIL: `grep -nE 'return "TRADE"' cuttingboard/notifications/__init__.py`
returns a match on a line not inside an `if`/`elif` block that tests
`canonical_outcome`; OR the formatter file does not contain at least
one `canonical_outcome` reference; OR
`format_hourly_notification`'s signature does not include
`canonical_outcome` as a keyword parameter.

  → `_action_label(...)` → one of {HALT, STAY FLAT, MONITOR,

`logs/latest_hourly_contract.json` and `logs/latest_hourly_run.json`
continue to record `outcome=OUTCOME_NO_TRADE` exactly as today.)
