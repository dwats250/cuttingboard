# PRD-069 - Entry Quality and Chase Filter

\*\*Status:\*\* COMPLETE
**Commit:** --
**Depends on:** PRD-067

---

## GOAL

Prevent impulsive or late entries by classifying candidate entry quality and blocking trades that are deterministically identified as chased, extended, structurally stale, or lacking a valid entry condition.

---

## SCOPE

- Add deterministic entry quality evaluation.
- Classify candidates before final trade allowance.
- Convert blocking-status ALLOW_TRADE candidates to BLOCK_TRADE via dataclasses.replace().
- Preserve existing TradeDecision enum values.
- Preserve existing execution policy and macro pressure contracts.
- Use only existing runtime/candidate/market context fields.
- Directly affect trade avoidance.

---

## OUT OF SCOPE

- No broker integration.
- No account-size logic.
- No position sizing changes.
- No dashboard/UI changes.
- No notifications.
- No new indicators.
- No new price-level generation.
- No LLM commentary.
- No market_map generation changes.
- No discretionary text interpretation.
- No modification of decisions already BLOCK_TRADE before this gate.

---

PRD-067 for thesis confirmation/invalidation fields. Gate must run after PRD-067 apply_thesis_gate. PRD-069 must not be implemented before PRD-067 is COMPLETE.

## REQUIREMENTS

### R1 - Entry quality object shape

Each materialized candidate receives entry_quality with:

- status: one of CLEAN, EXTENDED, STALE, CHASE_RISK, MISSING_ENTRY, UNKNOWN
- action: one of ALLOW, WAIT, AVOID, UNKNOWN
- reason: str or null
- blocking: bool
- source: str or null

FAIL: any key absent.
FAIL: status or action outside allowed sets.
FAIL: blocking is not bool.

### R2 - Blocking statuses

The following statuses convert an ALLOW_TRADE decision to BLOCK_TRADE via dataclasses.replace():

- EXTENDED
- STALE
- CHASE_RISK
- MISSING_ENTRY

Replacement TradeDecision fields:
- status = BLOCK_TRADE
- block_reason = "ENTRY_QUALITY_BLOCK"
- decision_trace = {"stage": "ENTRY_QUALITY_GATE", "source": "entry_quality", "reason": "ENTRY_QUALITY_BLOCK"}
- policy_allowed = False
- policy_reason = "entry_quality_gate_blocked"

Only ALLOW_TRADE decisions are converted. Decisions already BLOCK_TRADE are not modified.

FAIL: gate modifies a decision already BLOCK_TRADE.
FAIL: block_reason != decision_trace["reason"] after gate.
FAIL: blocking == True on a decision that was already BLOCK_TRADE before gate.

### R3 - CLEAN behavior

CLEAN status:

- Decision remains unchanged.
- action = ALLOW
- blocking = False

FAIL: CLEAN status changes any decision field.

### R4 - UNKNOWN behavior

UNKNOWN must not block by itself.

UNKNOWN produces:
- action: UNKNOWN
- blocking: False
- reason: "INSUFFICIENT_DETERMINISTIC_INPUTS"

FAIL: UNKNOWN blocks a trade.

### R5 - Deterministic input sources

Entry quality may only use existing deterministic fields available at runtime:

- QualificationResult.entry_mode if present
- StructureResult.structure for the symbol if available in execution_structure
- thesis.confirmation and thesis.invalidation from PRD-067 thesis_map if available
- TradeDecision.policy_reason if non-default
- Market_map symbol record if available in runtime scope, read-only

Do not invent new price levels, indicators, or thresholds.

FAIL: entry quality uses any field not listed above.

### R6 - Missing entry condition

If no deterministic entry condition, setup state, structure, trigger, or thesis field is available for a candidate, status = MISSING_ENTRY.

If the candidate is already BLOCK_TRADE before this gate, MISSING_ENTRY is attached to entry_quality without changing the decision.

### R7 - Chase risk

CHASE_RISK may only be assigned when an existing deterministic field explicitly shows the candidate is outside an acceptable entry zone, late, or structurally unfavourable. Do not infer from price movement alone unless the required field already exists.

### R8 - Stale setup

STALE may only be assigned when an existing timestamp, lifecycle state, or QualificationResult field indicates the setup is no longer fresh. Do not create new freshness rules.

### R9 - Contract validation

contract.py:_build_trade_candidates merges entry_quality_map.get(symbol, None) into the candidate dict as key "entry_quality". Missing value produces "entry_quality": null, not raise.

Historical candidates without entry_quality remain readable.

FAIL: contract builder raises when entry_quality_map is absent.

### R10 - No enum drift

Do not add new TradeDecision enum values.

Gate function signature:

```python
def apply_entry_quality_gate(
    trade_decisions: list[TradeDecision],
    candidates: dict[str, TradeCandidate],
    qualified_by_symbol: dict[str, QualificationResult],
    execution_structure: dict[str, StructureResult],
    thesis_map: dict[str, dict],
) -> tuple[list[TradeDecision], dict[str, dict]]:
    ...
```

Returns (decisions, entry_quality_map).

Gate runs after apply_thesis_gate from PRD-067 and before build_pipeline_output_contract.

FAIL: gate runs before apply_thesis_gate.
