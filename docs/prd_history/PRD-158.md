## 1. Problem

The dashboard renders internal pipeline state as if it were trade-decision
support. The trader is forced to mentally integrate contradictory signals
instead of receiving a single coherent verdict.

Field-by-field audit (output-translation-auditor, 2026-05-26) classified
52 rendered fields:

- KEEP: 7 — concrete trade-decision inputs
- TRANSLATE: 13 — real information shown as internal state
- CUT: 21 — pipeline status leakage, no trade action follows
- CONTRADICTS: 11 — on-screen conflicts with no resolution

Root causes at the rendering boundary:

1. Each pipeline layer's state is displayed independently with no
   integration step before the renderer.
2. Upstream data-availability failures render as multiple
   DATA UNAVAILABLE / NOT COMPUTED / UNKNOWN rows instead of one
   symbol-level skip reason.

The day-one Cuttingboard dashboard worked because it had fewer fields and
more translation. The fix is at the output surface, not the engine.

---

## 2. Objective

1. Delete 21 noise fields from the renderer.
2. Rewrite 13 internal-state labels into trader-facing actions.
3. Insert one minimal renderer-bound integration function (`dashboard_integrator`)
   between L9 output and the renderer.

## 4. Specification

### 4.1 Deletions (21 fields, canonical)

ACTION line · SOURCE: OK — system block · MACRO SOURCE: OK ·
TREND STRUCTURE SOURCE: OK · MARKET MAP SOURCE · MACRO PRESSURE / Rates ·
Trend Structure header badge: FRESH · TREND SYMBOLS count ·
Status column: PARTIAL · Tier header: D/F — FAILING · Card VALIDATION ·
Changes Since Last Run — Confidence · History — Time · History — Regime ·
History — Posture · History — Conf · Artifact diagnostics — lineage_state ·
Artifact diagnostics — payload path/ts · Artifact diagnostics — generation_ids ·
Artifact diagnostics — run path · Artifact diagnostics — market_map/contract paths

### 4.2 Translations (13 fields)

Regime → `Longs allowed` / `Shorts allowed` / `Stand down`.
Confidence → suppressed (no caution/sizing hook exists).
Run snapshot timestamp → relative freshness.
Macro pressure (Volatility/Dollar/Bitcoin) → decision phrases or cut.
Macro pressure (Overall) → suppressed; integrator resolves conflicts.
Outcome / Permission / Macro Bias → integrator-conditional suppression.
Card GRADE label preserved; value translates to `Tradeable` / `Developing` /
suppressed (low-grade carries FAILURE REASON).
Card level diagram → rendered only when anchor + level context present.
Changes Since Last Run — Regime → `Permission flipped to longs/shorts` or
suppressed.

### 4.3 Decision integrator

`dashboard_integrator(payload: dict) -> dict` — a single renderer-bound
function with four rules and explicit precedence:

- Rule 1 (symbol scope): required-data collapse. Symbols missing
  current_price / setup_direction / setup_type / trigger / invalidation
  render as `<SYMBOL> skipped — required market data unavailable.`
- Rule 2 (screen scope): regime-vs-setup-availability. Emits
  `No qualifying long/short setups currently available.`
- Rule 3 (screen scope): directional conflict collapse. Emits
  `Mixed tape — directional trades require symbol-level confirmation.`
- Rule 4 (screen scope): empty-tier suppression.

Drift guardrail recorded in `docs/DECISIONS.md` (2026-05-27 entry).

  `macro_bias` arrow-counting inversion for VIX/DXY/Rates. The integrator
  correctly suppresses the bad label whenever it conflicts with
  regime/setup direction, so the trader does not see the bug — but the
  Mixed-tape verdict may fire even when the actual macro pressure is
  uniformly risk-on. This is pre-existing tech debt; a follow-up PRD
  should fix the arithmetic in the macro_bias derivation directly. Not
  PRD-158 scope.
- LOW: card-level internal vocabulary (`STATE: STRONG_WATCH`, `STRUCTURE
  TRENDING_UP`, `IF NOW WAIT`, `BIAS BULL/BEAR`) still leaks through.
  Not in PRD-158 § 4.1 or § 4.2 lists. A future "card-level translation"
  PRD could address.
