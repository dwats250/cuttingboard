# PRD-116 — Dashboard Mixed-Artifact Hierarchy Hardening

STATUS: COMPLETE

GOAL
Prevent mixed, stale, or missing dashboard artifacts from rendering as
actionable normal dashboard content.

SCOPE
1. Enforce dashboard section hierarchy when artifact lineage is unhealthy.
2. Gate Sunday context on coherent Sunday artifact state.
3. Render Trend Structure as disabled when artifact lineage is unhealthy.
4. Suppress Market Map setup cards when artifact lineage is unhealthy.
5. Preserve coherent live-session behavior.
6. Preserve coherent Sunday/pre-market behavior except for required section
   ordering.
7. Preserve existing MIXED_ARTIFACTS warning generation-detail diagnostics.

OUT OF SCOPE
1. No trading logic changes.
2. No candidate grading changes.
3. No market_map generation changes.
4. No macro driver calculation changes.
5. No runtime artifact write-ordering changes.
6. No payload contract changes.
7. No CSS changes outside disabled-state styling and section ordering support.

NOTE: `tests/dash_helpers.py` is added to FILES because the default
fixture artifacts must carry coherent `generation_id` values; without
this, every test that does not explicitly target an unhealthy lineage
state would trip the new disabled-state branch. The three additional
Sunday/timestamp test files are added because their fixtures construct
payloads and runs directly (not via dash_helpers) and require coherent
generation_ids to exercise Sunday-context behavior under PRD-116 R3.
This is test infrastructure only; no production module imports these
test files.

TERMINOLOGY
- `artifact_lineage_state` ∈ {COHERENT, MIXED, STALE, MISSING} as returned by
  `cuttingboard.delivery.dashboard_renderer._artifact_lineage_state`.
- "unhealthy lineage" := `artifact_lineage_state` ∈ {MIXED, STALE, MISSING}.
- The rendered headline string "MIXED_ARTIFACTS" is a display token, not a
  state value, and is governed only by R2.

REQUIREMENTS

R1 — Unhealthy artifact hierarchy
When `artifact_lineage_state` is MIXED, STALE, or MISSING, the System State
block must be the first normal dashboard block after the page wrapper. The
MIXED_ARTIFACTS warning banner (`id="artifact-coherence"`) may render before
System State. No Sunday context, Macro Tape, Trend Structure, or Market
Map / Developing Setups section may render before System State.
FAIL: Any block other than the MIXED_ARTIFACTS warning renders before
`id="system-state"` under unhealthy lineage.

R2 — Mixed-generation diagnostics regression
When generation_ids differ across payload/run/market_map, the
`id="artifact-coherence"` warning block must continue to emit a line
containing `payload=`, `run=`, and `market_map=` generation values.
FAIL: `id="artifact-coherence"` renders without payload/run/market_map
generation detail text.

R3 — Sunday context coherence gate
The Sunday pre-market banner (`id="premarket-banner"`) and Sunday macro
context block (`id="sunday-macro-context"`) may render only when all of
the following are true:
- `artifact_lineage_state == "COHERENT"`
- `payload.meta.session_type == "SUNDAY_PREMARKET"`
- `_is_sunday_pt(payload.meta.timestamp)` is True
FAIL: Either Sunday block renders when this predicate is false.

R4 — Trend Structure disabled state
When `artifact_lineage_state` is MIXED, STALE, or MISSING, the
`id="trend-structure"` block must include a `disabled` class marker on its
container element and must not render per-symbol data rows.
FAIL: Under unhealthy lineage, `id="trend-structure"` lacks a `disabled`
class marker, or contains any `<tr>` data rows beyond the table header.

R5 — Market Map setup suppression
When `artifact_lineage_state` is MIXED, STALE, or MISSING, the
`id="candidate-board"` block must include a `disabled` class marker and
must not render candidate cards or grade-tier setup group headers.
FAIL: Under unhealthy lineage, `id="candidate-board"` lacks a `disabled`
class marker, or contains candidate cards / grade-tier setup group headers.

R6 — Diagnostics preserved
Under unhealthy lineage, the `id="artifact-diagnostics"` block must remain
in the HTML and must contain `artifact_lineage_state=`,
`payload_generation_id=`, `run_generation_id=`, and
`market_map_generation_id=` entries.
FAIL: `id="artifact-diagnostics"` is absent, or any of the four required
entries are missing.

R7 — Coherent live regression
When `artifact_lineage_state == "COHERENT"` and `session_type` is not
`SUNDAY_PREMARKET`, Macro Tape, Trend Structure, and Market Map sections
must continue to render without a `disabled` marker.
FAIL: A coherent live-session dashboard suppresses or disables Macro
Tape, Trend Structure, or Market Map.

R8 — Coherent Sunday regression
When the R3 predicate is true, both `id="premarket-banner"` and
`id="sunday-macro-context"` must render.
FAIL: Coherent Sunday/pre-market dashboard omits either block.

R9 — Renderer-only implementation
Implementation must remain confined to
`cuttingboard/delivery/dashboard_renderer.py` and
`tests/test_dashboard_renderer.py`, plus the PRD/registry/state files
listed in FILES.
FAIL: Any file outside FILES is modified.

DATA FLOW
payload, run, market_map → `render_dashboard_html` →
`_artifact_lineage_state` (existing) → unhealthy-lineage branch governs
ordering (R1), Sunday gate (R3), Trend Structure disabled (R4), and
Candidate Board disabled (R5). No new artifacts, no new fields, no new
writers, no new readers.

FAIL CONDITIONS
Each R FAIL line above is binding. Additionally:
- Any modification outside FILES is a scope violation.
- Any change to `_artifact_lineage_state`'s return contract is a scope
  violation.
- Any change to payload/run/market_map artifact shape is a scope violation.
