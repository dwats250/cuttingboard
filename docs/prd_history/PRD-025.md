PRD-025 — Decision Compression Layer

STATUS
COMPLETE
Commit: 3d532cd

GOAL
Restructure the contract UI so the most actionable signal appears immediately at the top, without requiring the user to scan the full contract. Demote metadata. Promote the primary trade and no-trade state to dominant positions.

SCOPE
- Add signal bar (posture, correlation state, risk modifier) as the top-of-page anchor
- Add primary trade block: one prominent setup when tradable == True
- Add no-trade block: explicit STAY_FLAT state with reasons when not tradable and no candidates
- Add watchlist block: intermediate state when candidates exist but not tradable
- Reduce setups table to secondary setups only (candidates[1:])
- De-emphasize header block (moved to bottom, reduced opacity)
- Add correlation block visual states (ALIGNED / CONFLICT color coding)
- Rename page from "Contract View" to "Decision Surface"

OUT OF SCOPE
- No changes to pipeline, contract schema, or any Python module
- No new contract fields
- No backend logic changes

FILES
M ui/app.js
M ui/index.html
M ui/styles.css

REQUIREMENTS

R1 — Signal Bar
A horizontal bar at the top of the contract view MUST show three cells:
- Posture: derived from contract["status"] and contract["system_state"]["tradable"]
- Correlation: contract["correlation"]["state"]
- Risk: contract["correlation"]["risk_modifier"] formatted as Nx

Each cell MUST apply a color class matching its value (TRADE_READY, WATCHLIST, STAY_FLAT, ALIGNED, NEUTRAL, CONFLICT).

R2 — Primary Trade Block
When contract["trade_candidates"].length > 0:
- Show block with candidates[0] only
- Display: symbol (large), direction, entry_mode, strategy_tag, R:R (to 1 decimal)
- Block MUST be hidden when no candidates

R3 — No Trade Block
When derivePosture() == "STAY_FLAT":
- Show block with label "NO TRADE"
- Collect and display reasons from: stay_flat_reason, empty candidates, correlation CONFLICT
- Block MUST be hidden when posture != STAY_FLAT

R4 — Watchlist Block
When derivePosture() == "WATCHLIST":
- Show block with label "WATCHLIST"
- Display reason string based on candidate presence
- Block MUST be hidden when posture != WATCHLIST

R5 — Secondary Setups Table
When contract["trade_candidates"].length > 1:
- Show candidates[1:5] in table
- Block MUST be hidden when <= 1 candidate

R6 — Correlation Block Visual State
When correlation block is visible:
- Apply class corr-ALIGNED or corr-CONFLICT to block element
- ALIGNED: green border/background tint
- CONFLICT: red border/background tint

R7 — Header De-emphasis
Header block (timestamp, regime, status) MUST:
- Render at the bottom of the contract view (after rejections)
- Render at 0.6 opacity

R8 — Render Order
renderContract() MUST call renderers in this order:
renderSignalBar → renderPrimaryTrade → renderNoTrade → renderWatchlist →
renderSecondarySetups → renderCorrelation → renderRejections → renderHeader → renderRouter

DATA FLOW
contract["status"] + contract["system_state"]["tradable"] → derivePosture() → signal bar, no-trade/watchlist block visibility
contract["trade_candidates"][0] → primary trade block
contract["trade_candidates"][1:5] → secondary setups table
contract["correlation"]["state"] + ["risk_modifier"] → signal bar, correlation block class
contract["system_state"]["stay_flat_reason"] → no-trade reasons

FAIL CONDITIONS
- Primary trade block visible when trade_candidates is empty
- No-trade block visible when posture != STAY_FLAT
- Watchlist block visible when posture != WATCHLIST
- candidates[0] appearing in secondary setups table
- Any block consuming contract fields not listed in DATA FLOW above

VALIDATION
Manual: open ui/index.html in browser, load a contract fixture
Confirm:
- Signal bar renders at top with three cells
- TRADE_READY contract: primary trade block visible, no-trade and watchlist blocks hidden
- STAY_FLAT contract: no-trade block visible with reasons, primary trade block hidden
- WATCHLIST contract: watchlist block visible, primary trade block hidden
- Correlation CONFLICT contract: correlation block has red tint
- Header block renders at bottom at reduced opacity
- Setups table shows candidates[1:] only, not candidates[0]
