# PRD-168: Suppress idle screen-verdict above populated candidate cards

Lane: HIGH-RISK / CONSUMER
Status: IN PROGRESS
Author: Dustin
Filed: 2026-06-05

## WHY NOW

The integrator screen-verdict "No qualifying long setups currently available."
(`dashboard_integrator.py:33` `RULE2_LONG_VERDICT`; SHORT variant `:34`) renders
as an `idle-summary` banner at `dashboard_renderer.py:2089-2091`, gated only on
`not unhealthy_lineage`. It is NOT gated against the presence of populated
high-grade candidate cards (rendered below, tier loop `:2144-2163`, gated on
grade ∈ `_HIGH_GRADES = {A+,A,B}` via `has_actionable` `:2136`). The PRD-165
candidate notes observed (fixture mode) the idle verdict rendering *above*
populated actionable cards — two states that read as contradictory to an
operator.

## REALIZABILITY FINDING (must be resolved at review)

The integrator and the renderer key off **different signals**, which is the
root of the juxtaposition and changes what the correct fix is:

- Rule 2 LONG verdict emits iff `regime_permission == "longs"` AND no
  non-skipped symbol has a long `setup_direction`
  (`dashboard_integrator.py:_rule2_verdict :139-147`, `_qualifying_setup_directions
  :123-136`, `_LONG_DIRECTIONS :30`). It is **grade-agnostic**.
- A candidate card renders when a symbol's `grade` ∈ `{A+,A,B}`
  (`dashboard_renderer.py:2136`). It is **direction-agnostic**.

So the verdict and a high-grade card can co-render in two realizable ways:
1. **Different-direction (verdict is accurate):** regime permits longs, the only
   high-grade card is a SHORT/neutral setup. "No qualifying *long* setups" is
   literally true; the card is a short. Suppressing the verdict here would
   *delete accurate information*.
2. **Grade/direction disagreement (data gap):** a high-grade card whose
   `setup_direction` is absent/unrecognized, so it is not counted as a long
   setup even though it renders as an actionable card under longs-permission.

Note: a high-grade card that **is** a long setup already places `"long"` in
`qualifying_directions`, so Rule 2 LONG **does not emit** — the same-direction
case is already mutually exclusive. The bug is therefore narrower than the
candidate implied. The review must pick the gating semantics (D1) with this in
mind; an unconditional "suppress when any card renders" risks deleting a
correct directional verdict (case 1).

## DESIGN DECISIONS FOR REVIEW

- **D1 (gating predicate) — the load-bearing decision.**
  - **A — any-card:** suppress the screen-verdict whenever any high-grade card
    renders (`has_actionable`). Simplest; but per the realizability finding it
    deletes an *accurate* directional verdict when the high-grade card is the
    opposite direction (case 1). Not recommended alone.
  - **B — direction-aware (renderer-only):** suppress the LONG verdict only when
    a high-grade card with a long `setup_direction` renders; SHORT verdict only
    when a high-grade short card renders. Targets the true contradiction
    (case 2) and leaves accurate opposite-direction verdicts intact. Recommended.
  - **C — leave as-is:** declare the juxtaposition correct (both statements are
    accurate) and close PRD-168 as "won't fix / not a defect." Viable if review
    concludes case 2 is unreachable under current routing.
  Recommendation: **B**, contingent on Codex confirming case 2 is realizable;
  fall back to **C** if it is not. Dustin to lock.
- **D2 (Rule 3 MIXED).** Does the same gate apply to `RULE3_MIXED_VERDICT`?
  Recommendation: yes — apply the D1 predicate uniformly to all screen-verdicts.
  Confirm.
