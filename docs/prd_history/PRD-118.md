PRD-118 — Coherent Dashboard Publish Artifact Set

STATUS: COMPLETE (commit 136bbfe)

GOAL
Prevent publication of ui/dashboard.html and ui/index.html built from
mixed, stale, fixture, missing, or mismatched-generation artifacts by
enforcing coherent-artifact validation inside the dashboard renderer
before it writes its output file.

SCOPE
1. Add coherent-artifact validation to the dashboard renderer's publish path.
2. Validate that payload, run, and market_map passed to a given renderer
   invocation share the same generation_id.
3. Reject publication to ui/dashboard.html when validation fails.
4. Reject publication to ui/dashboard.html when any source artifact is a
   fixture artifact (generation_id substring "fixture", or fixture_mode
   active).
5. Preserve PRD-116 unhealthy-lineage rendering behavior.
6. Preserve PRD-117 inactive-session labeling behavior.
7. Preserve all runtime, trading, and decision logic behavior.

OUT OF SCOPE
1. No runtime / trading / qualification / regime / sizing changes.
2. No market_map generation changes.
3. No payload or run schema changes.
4. No generation_id format or semantics changes.
5. No dashboard redesign.
6. No backfill or migration of historical artifacts.
7. No changes to notification logic.
8. No changes to .github/workflows YAML beyond what is required to surface
   non-zero exit from the renderer (workflows already use
   `set -euo pipefail`; no edit is expected).

TERMINOLOGY
- payload_generation_id := payload["meta"]["generation_id"]
- run_generation_id := run["generation_id"]
- market_map_generation_id := market_map["generation_id"]
- coherent artifact set (for a renderer invocation) :=
  * payload, run, market_map are all loaded (none missing, none empty)
  * payload_generation_id, run_generation_id, market_map_generation_id
    are all present (non-None, non-empty strings)
  * all three values compare equal as exact strings
- fixture artifact := any artifact whose generation_id contains the
  substring "fixture", consistent with cuttingboard.runtime._run_id
  output format `{mode}-fixture-{stem}`.
- fixture mode := the renderer's `fixture_mode` kwarg is true OR the
  environment variable FIXTURE_MODE == "1".
- publish output := the renderer's `output_path` argument. The publish
  gate applies when `output_path` resolves under the repository's `ui/`
  directory.
- publish step := the workflow sequence
  `python3 -m cuttingboard.delivery.dashboard_renderer --output ui/dashboard.html`
  followed by `cp ui/dashboard.html ui/index.html`. The renderer owns
  only the first step; the second is shell in
  .github/workflows/cuttingboard.yml and .github/workflows/hourly_alert.yml
  and is protected transitively by the renderer's non-zero exit.

REQUIREMENTS

R1 — Coherent publish validation (renderer)
The dashboard renderer must, before writing `output_path`, validate the
coherent artifact set for the payload, run, and market_map it is about
to render.
FAIL: Renderer writes `output_path` despite a missing artifact or any
mismatched generation_id among payload/run/market_map.

R2 — Fixture publish prevention
The renderer must reject publication when `output_path` is under `ui/`
AND any of the following holds:
  (a) payload_generation_id, run_generation_id, or market_map_generation_id
      contains the substring "fixture";
  (b) `fixture_mode` is true;
  (c) environment variable FIXTURE_MODE == "1".
Rendering to non-`ui/` paths (e.g., reports/output/, CLI overrides) is
unaffected.
FAIL: ui/dashboard.html is written when any of (a)/(b)/(c) holds.

R3 — Publish gating
When validation fails, the renderer must abort by raising a deterministic
exception (causing non-zero exit when invoked as `__main__`) before
writing `output_path`. The renderer must not write a partial file.
FAIL: `output_path` is modified, created, or truncated after a failed
validation. Or: validation failure does not surface as non-zero exit.

R4 — Explicit publish failure reason
On validation failure, the renderer must emit a single deterministic
diagnostic line to stderr identifying exactly which condition failed:
  - missing artifact (and which one)
  - generation_id mismatch (and the three observed values)
  - fixture artifact detected (and which artifact / which trigger)
FAIL: Validation failure produces no reason, or an ambiguous reason that
does not name the failing artifact or condition.

R5 — Coherent publish success path
When all three generation_ids are present, equal, contain no "fixture"
substring, and fixture mode is inactive, the renderer must produce
ui/dashboard.html with the same byte-for-byte content as today's
renderer (modulo the new diagnostics output going to stderr only).
FAIL: A valid coherent set produces no output, an error, or a
content-different ui/dashboard.html.

R6 — PRD-116 regression protection
Existing unhealthy-lineage rendering (mixed/stale/missing-artifact
display behavior) remains exercised by its existing tests and is not
weakened. The new gate fires before that rendering branch only when
the gate's preconditions are met (i.e., output is under `ui/`); for
non-`ui/` outputs (reports/output/, tests, fixtures), PRD-116
rendering behavior is preserved.
FAIL: PRD-116 tests regress, or unhealthy-lineage rendering no longer
runs for non-`ui/` outputs.

R7 — PRD-117 regression protection
Inactive-session labeling behavior remains unchanged.
FAIL: PRD-117 tests regress.

R8 — Runtime artifact contract preservation
No changes to payload, run, market_map schema or generation_id
production. generation_id continues to be written by
`cuttingboard.runtime._generation_id` and attached to payload via
`_attach_generation_id_to_payload` at the same nesting (`meta`).
FAIL: Any module other than dashboard_renderer.py or its tests is
modified; or generation_id storage location for any artifact changes.

R9 — Deterministic generation comparison
The gate compares payload_generation_id, run_generation_id, and
market_map_generation_id using exact string equality after stripping
whitespace. No timestamp parsing, no prefix matching, no substring
matching (except the explicit "fixture" substring check in R2).
FAIL: Validation accepts or rejects based on anything other than exact
string equality of the three extracted generation_ids.

R10 — Renderer publish ordering
Validation must execute before any byte is written to `output_path`. It
is acceptable for `output_path.parent.mkdir(...)` to execute before
validation.
FAIL: `output_path.write_text(...)`, `open(..., "w")`, truncate,
replace, rename, or any equivalent write operation executes before
validation passes.

R11 — Unit-test proof
tests/test_dashboard_renderer.py must add deterministic tests covering:
  - coherent publish success: three matching generation_ids → file written
  - mismatched generation_ids → exception raised, no file written
  - missing payload generation_id (payload["meta"] lacks key) → exception, no file
  - missing market_map → exception, no file
  - fixture substring in any one of the three generation_ids → exception, no file
  - fixture_mode=True with `ui/` output_path → exception, no file
  - fixture_mode=True with non-`ui/` output_path → file written (gate scoped to ui/)
  - silent-fallback prevention: missing artifact is not replaced with a
    default/empty artifact (verifies R12)
FAIL: Any of the eight branches lacks a deterministic test, or tests
rely on filesystem state outside tmp_path.

R12 — No silent fallback
When any required artifact is missing, the renderer must raise the
validation exception. It must not substitute an empty dict, a default
payload, a fixture artifact, or any cached prior artifact.
FAIL: Missing payload/run/market_map is silently replaced by any
default/fixture/cached value at publish time.

DATA FLOW
  CI workflow
    └─ invokes: python3 -m cuttingboard.delivery.dashboard_renderer \
                   [--payload <p>] [--run <r>] --output ui/dashboard.html
        └─ main() loads payload, run, market_map
            └─ validate_coherent_publish(payload, run, market_map,
                                         output_path, fixture_mode)
                ├─ FAIL → raise → non-zero exit → workflow halts
                └─ PASS → write_dashboard() → output_path written
                              └─ workflow `cp ui/dashboard.html ui/index.html`

No new artifact types. No payload/run/market_map schema changes.
No generation_id redesign.

FAIL CONDITIONS (summary, binary)
- ui/dashboard.html written with any missing source artifact.
- ui/dashboard.html written with non-matching generation_ids.
- ui/dashboard.html written when any generation_id contains "fixture".
- ui/dashboard.html written when fixture_mode is true or FIXTURE_MODE=1.
- Validation failure produces zero exit code from CLI invocation.
- Validation failure produces no diagnostic line to stderr.
- Silent substitution of missing artifact with default/fixture value.
- PRD-116 or PRD-117 tests regress.
