## Problem

This session surfaced one recurring failure class: **a check that passes on the
letter while failing on the meaning.** Five instances in a single arc:

1. **Requested ≠ resolved provenance** — the Codex review artifact recorded
   `model=gpt-5-codex` (the *requested* input) while the run warned `Model
   metadata for gpt-5-codex not found` and the model self-reported `gpt-4.1`. The
   one artifact whose entire job is truthful provenance carried a false claim.
2. **Proxy ≠ authoritative source** — the follow-up fix parsed
   `steps.codex.outputs.final-message` (the assistant's *prose*) as if it were the
   `--json` event stream, so it would either fail closed forever or scrape the
   model's *self-report* — the very thing the gate must not trust.
3. **Sandbox ≠ CI baseline** — the suite reports 2768 in this sandbox but 2773+ in
   CI (5 signing tests fail environmentally); a baseline read from the sandbox
   would understate the truth.
4. **Vacuous-green** — `pytest.importorskip` / `pytest.skip` on a missing input
   makes a test pass without testing anything.
5. **Silent fallback** — `WARN`-and-`exit 0`, `|| true`, `|| echo 'unknown'`: a
   step that substitutes-and-continues reports a success it never verified.

Drift here is invisible by construction: the gate is green, so nothing flags it.
The defense is to **name the signature and codify the invariants that close it**,
then inventory where else it already hides.

## Part A — Proposed CLAUDE.md doctrine (HELD diff; not applied)

Proposed new section for `CLAUDE.md`, after "Operational rules". Each invariant
carries a one-line rationale and the incident it generalizes. **This is a
proposal block, not an edit to CLAUDE.md.**

```markdown
## Semantic-failure hardening (PRD-198)

A check is only worth its green if it verifies *correspondence to reality*, not
the *presence of the right words*. Six invariants close the "passes-on-the-letter,
fails-on-the-meaning" failure class. Each names the incident it generalizes.

1. **Fail-loud, never silent-fallback.** A missing dependency, an unresolvable id,
   or an unreachable source must exit non-zero — never substitute-and-continue.
   *Why:* a step that degrades silently reports a success it never verified.
   *Incident:* `codex --version || echo 'unknown'` and `engine_doctor` WARN→exit 0
   record/emit a non-result as if it were a result.

2. **Assert the resolved, not the requested.** Verify the actual effect — the
   resolved model, the executed test, the CI count — never the declared intent.
   *Why:* intent and effect diverge precisely where it matters.
   *Incident:* provenance recorded the requested `gpt-5-codex` while the served
   model self-reported `gpt-4.1`.

3. **Authoritative source, not proxy.** Every check names and reads its source of
   truth; never a proxy that can diverge from it.
   *Why:* a convenient nearby value is not the value you mean to check.
   *Incident:* parsing the assistant's prose `final-message` as if it were the
   `--json` response-metadata stream.

4. **Every guard ships a red test.** A guard merges only with a negative test
   proving it fails when violated. Banned: `importorskip` on a required dep,
   `WARN`-and-`exit 0`, and any test that cannot fail.
   *Why:* a guard with no failing-case test rots to always-green unnoticed.
   *Incident:* hermetic YAML tests asserted the parser's *logic existed* but never
   that it *worked* against real output — which was, in fact, broken.

5. **Verify where truth is determined.** Achieve environment parity with the gate
   (CI); local/sandbox green is unverified until reproduced where the decision is
   made.
   *Why:* the environment that decides is the only one whose result counts.
   *Incident:* the sandbox suite (2768) differs from CI (2773+); a baseline read
   from the sandbox is wrong.

6. **Pin identities that matter.** Model → dated snapshot, action → commit SHA,
   dependency → declared *and* locked. A movable identity changes behavior with no
   diff.
   *Why:* an alias or floating tag can be re-pointed under you, silently.
   *Incident:* `gpt-5-codex` (an alias, not a snapshot) appears to resolve to a
   fallback; `actions/*` are pinned to movable `@vN` tags; deps are floor-pinned
   (`>=`) with no lockfile.
```

### Correspondence-honoring patterns (kept as positive references)
- `.github/workflows/ci.yml` — the merge gate runs `ruff check` + `pytest -q`
  with no `|| true`/skip: fail-loud (invariant 1). *Gap:* it does not fail on
  vacuous skips or a count regression (relates to #6, #8).
- `tests/test_codex_review.py` uses a **hard** `import yaml` (no `importorskip`):
  the suite fails loudly if the parser is missing (invariant 4) — even though its
  *behavioral* coverage is the gap in #1.
- `docs/PROJECT_STATE.md` baseline bullet explicitly records **CI truth** and
  explains the sandbox delta (invariant 5) — the staleness in #6 is the only gap.

### Severity rationale
- **HIGH (#1, #2):** the active, security-relevant Codex gate — a SURFACE check on
  the exact mechanism whose meaning was already shown to diverge from its letter.
- **MED (#3–#7):** reproducibility / supply-chain / bookkeeping-drift surfaces
  where a green can mask a real change; none currently on the CI merge gate's
  critical path.
- **LOW (#8–#10):** localized vacuity / silent degradation with limited blast
  radius today.
