PRD-142 — PATCH PRD-141 Persist hourly slot state across CI runs

STATUS
IN PROGRESS

LANE
MICRO

CLASS
INFRA + PATCH

ROOT CAUSE
hidden dependency — PRD-141 added a runtime dependency on
`logs/last_hourly_slot.json` persisting across CI runs but did not
verify that `.github/workflows/hourly_alert.yml`'s artifact-commit
allowlist included the new file. `logs/` is gitignored at
`.gitignore:48`, and the workflow uses an explicit `git add -f` list
of 10 named log files (`hourly_alert.yml:80-90`) which does not
include `last_hourly_slot.json`. Each scheduled cron firing checks
out from `main`, sees no slot file, and the dedup gate is inert.

GOAL
Make PRD-141's cross-run idempotency gate effective in CI by
persisting `logs/last_hourly_slot.json` to `main` at the end of each
successful hourly_alert run, so the next scheduled cron firing reads
the prior slot and correctly suppresses same-PT-hour duplicates.

SCOPE
- Add a conditional `git add -f logs/last_hourly_slot.json` to the
  "Commit hourly artifacts" step in `hourly_alert.yml`, guarded by a
  file-existence check for the bootstrap case (first post-deploy run,
  and any run where alert_runner exited before save_last_slot was
  reached — premarket, force-slot, or pre-PRD-141 baseline).

REQUIREMENTS

R1 — Conditional stage of slot file
The "Commit hourly artifacts" step adds, immediately before the
existing forced `git add -f \` artifact allowlist:

  if [ -f logs/last_hourly_slot.json ]; then
    git add -f logs/last_hourly_slot.json
  fi

Indentation matches the surrounding `run: |` block.

FAIL: `grep -A1 'logs/last_hourly_slot.json' .github/workflows/hourly_alert.yml`
returns a `git add -f` line inside the "Commit hourly artifacts" step,
guarded by an `[ -f logs/last_hourly_slot.json ]` test.

R2 — Behaviour on first post-deploy scheduled cron pair
After this patch is merged to `main`, the first scheduled
weekday `*/30 14-21` cron pair within the same PT hour (e.g.
`20:00Z` + `20:30Z`) MUST produce, observable in the post-cron
`git log` and `logs/audit.jsonl` on `main`:

  - The first run's CI commit contains
    `logs/last_hourly_slot.json` with a `slot_utc` value matching
    the top of the PT hour for that runtime.
  - The second run's `logs/audit.jsonl` contains a row with
    `status="suppressed"` and `reason="suppressed_same_slot"`.
  - The second run's CI commit does NOT mutate
    `latest_hourly_run.json`, `latest_hourly_contract.json`, or
    `latest_hourly_payload.json` (alert_runner exited before
    `_execute_notify_run`; those artifacts retain the first run's
    bytes).

FAIL: R2 is the gate for flipping PRD-142 to COMPLETE. The
implementation commit lands now; the COMPLETE flip is deferred
until R2 is observed in production CI on a scheduled cron pair.
