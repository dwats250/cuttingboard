Regex edge-case analysis (the over/under-match question): the matcher
`^- \*\*Test baseline:\*\*.*(?:\n(?!\s*-\s|\s*#|\s*$).*)*` (re.MULTILINE) was
tested against eight inputs:
- single-line bullet followed by "- **Active PRD:**" -> matches ONLY the
  baseline line; does NOT swallow the next bullet. CORRECT.
- soft-wrapped bullet (commit ref on an indented next line) -> matches whole
  (label line + continuation). CORRECT.
- multi-sentence single line -> matches whole. CORRECT.
- multi-line continuation (the exact alignment-#4 drift shape) then next bullet
  -> matches the full multi-line bullet, stops before the next bullet. CORRECT.
- bullet at EOF with no trailing newline -> matches. CORRECT.
- blank line after the bullet -> match STOPS at the blank line (does not absorb
  text after a paragraph break). CORRECT.
- live exact line then blank then "**Next step:**" -> matches only the bullet.
  CORRECT.
- nested indented sub-bullet ("  - ...") -> match STOPS at the nested dash
  (under-match toward safety). Acceptable; see Recommended #2.
