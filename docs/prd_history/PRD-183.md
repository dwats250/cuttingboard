PRD-183 — Realign closeout tooling to the new PROJECT_STATE format

Status: IN PROGRESS

LANE
STANDARD

CLASS
INFRA

WHY NOW
The PRD-182 closeout exposed that `scripts/prd_close.sh` and
`scripts/pre_commit_sanity.sh` (and the `prd-closeout-verified` skill
that wraps them) still target the pre-realignment PROJECT_STATE.md
markers. The live doc was restructured to the "Current state / Recent
ships" format without updating the tooling, so every closeout now
silently skips five edits and the scope-lock nudge always reads "no
active PRD." Each future closeout needs the same manual patching until
the tooling is realigned.

MAX EXPECTED DELTA
<= 120 changed lines across the four tooling/test files (regex and
fixture churn dominate). No production engine code. Exceeding this
stops implementation and re-triggers review.

GOAL
Realign the closeout tooling and its tests to the current
PROJECT_STATE.md format so `scripts/prd_close.sh` performs a complete
closeout with zero WARN-skips, and the scope-lock nudge correctly reads
the active PRD.

SCOPE
- prd_close.sh: reset the bulleted `- **Active PRD:**` line; prepend to
  the 3-column "Recent ships" table; update the `- **Test baseline:**`
  line in place (count + commit ref, xfailed preserved); drop the
  removed `Last completed PRD` / `Last work completed` edits and route
  `--summary` into the bookkeeping commit body instead.
- pre_commit_sanity.sh: parse the active PRD from the bulleted
  `- **Active PRD:**` line (first token after the marker).
- test_prd_close.py: rebuild the fixture in the new format; update
  assertions accordingly; keep full coverage of R2/R3/R4/R5 behavior.
- prd-closeout-verified SKILL.md: update V6/V7/V8 (and the inputs note)
  to the new-format records.

OUT OF SCOPE
- No change to the PROJECT_STATE.md format itself (the realignment is
  canonical; this PRD adapts tooling TO it).
- No change to prd_open.sh, prd_eval.sh (not a format consumer), or the
  scope-lock-precommit skill's parse contract.
- No engine/trading/contract/payload/dashboard code.

CHANGE SURFACE
- scripts/*.sh: regex/marker-targeting changes and the --summary
  routing only; the arg interface (--prd/--hash/--title/--tests/--added/
  --summary/--next/--commit) stays stable so the closeout skill keeps
  working.
- test_prd_close.py: fixture + assertions only; no change to the
  test-runner harness (`_run`, fixture tree builder).

REQUIREMENTS

R1 — Active PRD reset on the bulleted line
prd_close.sh resets the live `- **Active PRD:** <...>` line to
`- **Active PRD:** none in progress.` (bullet preserved).

FAIL: after closeout against the real PROJECT_STATE.md, the Active PRD
line still names the just-closed PRD, OR prd_close.sh emits
`WARN: 'Active PRD' marker not found`.

R2 — Recent ships row prepend
prd_close.sh prepends `| PRD-NNN | <title> | <date> |` immediately
under the 3-column `## Recent ships` separator (`|-----|-------|-----------|`).

FAIL: closeout emits `WARN: history table separator not found`, OR no
PRD-NNN row is added to the Recent ships table, OR a 4-column row is
written.

R3 — Test baseline updated in place
prd_close.sh updates the `- **Test baseline:** <N> passing...` line:
new passing count and commit ref, with any `M xfailed` text preserved.

FAIL: closeout emits `WARN: test baseline bullet not found`, OR the
passing count / commit ref is not updated.

R4 — Removed markers dropped; summary preserved in the commit
prd_close.sh no longer attempts the `Last completed PRD` or
`Last work completed` edits (those lines were removed by the
realignment) and emits no WARN for them. `--summary` is written into
the `Close PRD-NNN bookkeeping` commit message body.

FAIL: closeout emits a WARN for either removed marker, OR `--summary`
text appears in neither the commit body nor any artifact.

R5 — Scope-lock nudge reads the bulleted active PRD
pre_commit_sanity.sh extracts the active PRD id from the bulleted
`- **Active PRD:**` line: prints "Active PRD: PRD-NNN" when one is
active and "no active PRD" when the line reads "none".

FAIL: with an active PRD set in the new format, pre_commit_sanity.sh
prints "no active PRD — scope-lock check not required".

R6 — Closeout skill verification matches the new format
prd-closeout-verified SKILL.md V6/V7/V8 (and the Verification Report
shape) reference the new-format records: Recent ships row + bulleted
Active PRD reset, not the removed prose markers.

FAIL: the skill still instructs verifying `Last completed PRD` /
`Last work completed` lines as required closeout outputs.

R7 — Tests realigned and green
test_prd_close.py uses a new-format fixture and asserts the R1-R4
behavior; the full suite passes with no new failures.

FAIL: `python -m pytest tests/test_prd_close.py -q` fails, OR the full
suite regresses from the 2607-passing / 1-xfailed baseline (net of any
intended assertion changes in this file).

DATA FLOW
prd_close.sh --commit -> edits PROJECT_STATE.md (Active PRD, Recent ships, baseline) + registry + index + PRD doc -> single bookkeeping commit (summary in body)
pre_commit_sanity.sh -> reads bulleted Active PRD line -> scope-lock reminder
test_prd_close.py -> new-format fixture -> asserts prd_close.sh output

FAIL CONDITIONS
- Any WARN emitted by prd_close.sh against the real PROJECT_STATE.md.
- Active PRD not reset, or scope-lock nudge blind to an active PRD.
- Recent ships row missing/malformed, or baseline not updated.
- test_prd_close.py red, or full-suite regression beyond this file's
  intended assertion changes.

VALIDATION

- Build a scratch copy of the real PROJECT_STATE.md; run
  `scripts/prd_close.sh --prd 999 --hash deadbee --title T --tests 2607
  --added 0 --summary S` against it; confirm ZERO `WARN:` lines on
  stderr and the Active PRD / Recent ships / baseline edits applied.
- `python -m pytest tests/test_prd_close.py -q` -> green.
- `python -m pytest tests/ -q` -> 2607 passed, 1 xfailed.
- Set a fake active PRD in a scratch PROJECT_STATE; run
  pre_commit_sanity.sh; confirm it prints the PRD id.
- `ruff check cuttingboard tests` -> clean.
