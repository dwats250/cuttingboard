## Problem

`codex-review.yml` leaves `codex-version` **unpinned** (`.github/workflows/codex-review.yml:105`),
so each dispatch installs whatever Codex CLI is "current." The current CLI no longer carries
local metadata for the `gpt-5-codex` model id: it emits a structured `item.error`
("`Model metadata for ``gpt-5-codex`` not found. Defaulting to fallback metadata`") and runs on a
fallback model. The PRD-207 honor gate then correctly **fails closed** (exit 4, no artifact).

This blocks **every** future Codex cross-review, not just PRD-208 — the HIGH-RISK review lane has no
working gate. It is the movable-identity failure class CLAUDE.md §"Pin identities that matter" (#6)
names, and the same family as PRD-207 (served-vs-requested) and PRD-198.

### Diagnostic (this arc — evidence, not inference)
- **Key/access are fine.** Diagnostic run `28467193644` completed a full 1.55M-input-token reasoning
  review; the `review` job concluded `success`. A missing/wrong/unscoped key fails at auth with no
  `turn.completed`. So the cause is **not** key or access — it is alias/CLI drift.
- **The movable identity is the CLI, not a model snapshot.** There is **no** dated
  `gpt-5-codex-YYYY-MM-DD` snapshot anywhere in this toolchain; the only model identity it records is
  the alias `gpt-5-codex` (+ the reviewer's prose self-report `gpt-5.5`). By PRD-207's finding the
  toolchain surfaces no structured served-model id, so there is nothing dated to pin.
- **Last CLI with landed proof of honoring the alias = `codex-cli 0.142.1`.** Two certified artifacts
  record it verbatim:
  `Provenance: resolved-model=``gpt-5-codex`` (requested=``gpt-5-codex``; allowlist-verified), codex-cli=``codex-cli 0.142.1```
  — `codex-review/PRD-210-7e836b9765cd` and `codex-review/PRD-208-b42ec9aa7ebe`.

## The fix — pin the CLI identity

1. **Pin `codex-version: 0.142.1`** in the `openai/codex-action` `with:` block
   (`.github/workflows/codex-review.yml`, at the currently-unpinned site `:105`). `0.142.1` is well
   above `0.2.x`, so it still supports the `--sandbox` flag — `sandbox: read-only` stays enforceable.
2. **Keep `model: gpt-5-codex`** (`:55` default). The alias is deterministic **only** downstream of the
   CLI pin — under `0.142.1` it resolves to real metadata (proven by the two artifacts above). This is
   not a movable bare alias once the CLI is fixed.
3. **`ALLOWED_CODEX_MODELS = gpt-5-codex`** exactly — the one proven-served id, no widening. Set as the
   `vars.ALLOWED_CODEX_MODELS` **repository variable** (human-configured by Dustin; the workflow keeps
   the `${{ vars.ALLOWED_CODEX_MODELS }}` placeholder, never a hardcoded literal — preserves the
   PRD-207 contract test at `tests/test_codex_review.py:185-188`).

  `codex-version: 0.142.1`. It must **fail against the current (unpinned) workflow** before the fix, and
