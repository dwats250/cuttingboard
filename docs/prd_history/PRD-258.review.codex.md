FINDINGS (verbatim from Codex's own report, condensed)

1-2. `git fetch --upload-pack='<cmd>' .` / `git fetch --update-head-ok
   --force origin main:refs/heads/<branch>` — `--upload-pack=<exec>` is
   executed as a shell command when fetching from a local path (confirmed:
   `git fetch --help` documents `--exec=<upload-pack>` is "passed to the
   command run on the other end"), turning an inspection-looking command
   into arbitrary code execution; separately, an explicit destination
   refspec plus `--force`/`--update-head-ok` can overwrite a local branch.
   Gap: the original `"Bash(git fetch *)"` trailing wildcard had no
   restriction on either vector.
3. `git ls-remote --upload-pack='<cmd>' .` — same executable-selecting
   flag, same risk, on the sibling command.
4-6. `git diff/log/show --output=<file>` — `--output=<file>` "output[s]
   to a specific file instead of stdout" (confirmed via `--help`; no short
   synonym), silently overwriting any writable path including a protected
   file. Gap: none of the three trailing-wildcard allow rules excluded it.
7. `python3 -m ruff check --fix --unsafe-fixes` / `ruff format` / `ruff
   clean` — the unscoped `"Bash(python3 -m ruff *)"`/`"Bash(python -m ruff
   *)"` wildcard authorized every Ruff subcommand, not just the read-only
   `check` the rule was intended for.
8. `gh api repos/.../issues/146/comments -F body='...'` and attached-flag
   forms (`-XDELETE`, `--method=DELETE`, `-fbody=x`, `--input=file`) —
   the original deny patterns required a literal space after the flag
   (bypassed by attached/equals forms, confirmed via `gh api --help`'s
   documented flag syntax) and covered only `-f`/`--raw-field`, missing
   the distinct, case-sensitive `-F`/`--field` flag entirely.

Also checked and found CLEAN (Codex's own words): "The allowed repo
scripts themselves are clean for this question: the shell scripts run
only the enumerated read/test commands, the two Python scripts do not
parse arguments or write, and the registry validator's only subprocess is
`git cat-file -e`." No script-shells-out-to-something-gated finding.

DISPOSITION — all 8 ACTIONED in this same PR, none deferred
1-3 (fetch/ls-remote --upload-pack + fetch --force/refspec): fixed by
   narrowing `git fetch`/`git ls-remote` from a trailing wildcard to six
   exact-match forms (`git fetch`, `git fetch origin`, `git fetch --all`,
   `git fetch --prune`, `git ls-remote`, `git ls-remote origin`), plus a
   `"Bash(git * --upload-pack*)"` deny backstop.
4-6 (diff/log/show --output=): fixed by a `"Bash(git * --output*)"` deny.
7 (ruff unscoped): fixed by narrowing to `"Bash(python3 -m ruff check *)"`/
   `"Bash(python -m ruff check *)"`.
8 (gh api flag-form bypass + missing -F/--field): fixed by rewriting all
   seven `gh api` deny patterns to attach the wildcard directly (no
   required space) and adding `"Bash(gh api * -F*)"`/
   `"Bash(gh api * --field*)"`.
