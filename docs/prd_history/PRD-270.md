# PRD-270 — Correct PRD-245 registry provenance (phantom 6017ce9 → merged 8bea09c)

Lane: MICRO
Status: IN PROGRESS

## Problem

PRD-245's recorded commit is `6017ce9` in `docs/PRD_REGISTRY.md` (the
PRD-245 row), `docs/prd_index.json` (the number-245 entry), and cited inline
in `docs/prd_history/PRD-245.md`'s own trailing STATUS block ("tests @
6017ce9"). That SHA does not resolve: `git cat-file -t 6017ce9` →
`Not a valid object name`. PRD-245.md's own in-file RENUMBER NOTE explains
why — the PRD was drafted, implemented, and reviewed as "PRD-244" on a
branch (commits including `6017ce9`) before a numbering collision with a
parallel session forced a renumber to PRD-245 when squash-merged as PR
#122. `main` squash-merges, so PRD-245 landed as a new commit, `8bea09c`
(`PRD-245: FVG swapped-stop Gate 6 floors — fallback-to-DIRECT (HIGH-RISK,
hold for manual merge) (#122)`), which is an ancestor of `main`/HEAD and
resolves. `docs/PROJECT_STATE.md`'s own PRD-245 narrative already cites
`8bea09c` as the real squash commit — the registry/index/doc trio simply
never caught up to it. `tools/validate_prd_registry.py` flags `PRD-245 hash
6017ce9 does not resolve` under commit-resolvability.

Surfaced by a read-only closeout-integrity audit (2026-07-23). Same
provenance-drift class PRD-227 fixed for PRD-221 (phantom pre-squash SHA →
real resolvable squash commit) — the direct precedent for this PRD's shape.

### Why this is NOT covered by the WONTFIX-HISTORICAL ruling

`docs/PROJECT_STATE.md`'s Known-technical-debt entry and the matching
`docs/DECISIONS.md` 2026-07-05 entry both close the broader phantom-SHA
class WONTFIX-HISTORICAL at **29 PRDs / 35 hash tokens**, explicitly scoped
to rows **≤ PRD-222** (pre-dating the PRD-229 `#NNN` closeout convention
that ended new instances of this class). PRD-245 is >222 and closed
2026-07-06 — after both that ruling and the `#NNN` convention were already
in force — so it was never in scope for the waiver and was simply missed.
Unlike the waived rows, the correct value here is known and cheap: the
registry already says COMPLETE, the work landed, and the real commit
resolves cleanly. A waiver is for debt that cannot be paid, not debt that
is tedious to pay — this PRD pays it rather than extending the waiver,
which would blunt exactly what the `#NNN` convention (PRD-229) and the
WONTFIX ruling's own floor logic (PRD-243) exist to prevent.

(Housekeeping note, not a repo defect: an earlier framing of this audit
described the WONTFIX-HISTORICAL set as "23" rows. That number never
appeared in any committed doc — `PROJECT_STATE.md` and `DECISIONS.md`
already correctly say 29 PRDs / 35 tokens throughout. The "23" was a stale
figure from an external, uncommitted session note, not a repo correction
this PRD needs to make. Recorded here so it doesn't resurface as a phantom
follow-up.)

## Fix

Replace `6017ce9` with `8bea09c` in the registry row and index entry.
Amend (not erase) PRD-245.md's trailing STATUS block: its "tests @
6017ce9" citation is itself accurate immutable history (per the doc's own
RENUMBER NOTE, pre-renumber commit messages are preserved verbatim) — add
the resolvable squash-commit pointer alongside it rather than deleting the
historical citation.
