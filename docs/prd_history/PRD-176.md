PRD-176 — Red-folder economic calendar static schedule and loader

Status: COMPLETE

LANE
STANDARD

CLASS
SIDECAR

WHY NOW
The system cannot answer "what matters today" for scheduled macro risk: there is no
calendar source and no event gate. The one adjacent surface (the earnings gate) is
fail-open and fed by a field that is None in every production path, so it silently
does nothing. CPI/FOMC/NFP days are when a discretionary trader gets hurt entering
blind.

MAX EXPECTED DELTA
90 production LOC across 1 new loader module + 1 new data file. Exceeding stops
implementation and re-triggers SIDECAR review.

GOAL
A zero-dependency static schedule file plus a pure loader that returns red-folder
events within a configurable lookahead window relative to a run timestamp, with a
visible expiry warning as the schedule nears its end-of-year. Render is OUT of scope
(absorbed by PRD-177).

SCOPE
- New data file data/red_folder_2026.json: hand-entered CPI/NFP/FOMC/PPI events for
  the 2026 remainder (date, ET time, type, name), entered from published BLS and Fed
  schedules.
- New pure module cuttingboard/red_folder.py: load + validate the schedule, filter to
  a lookahead window (default 48h) relative to a passed-in now_utc using ET window
  math, and expose an expiry signal that fires within 30 days of the last entry.
- Loader fails loud and visible: a missing/malformed file returns a structured
  warning result, never a silent empty pass and never a crash.

OUT OF SCOPE
- Any dashboard render of the section (PRD-177 absorbs the render block).
- Any entry/qualification gate keyed on event windows (deferred:
  red-folder-entry-gate; that needs fail-closed semantics designed separately).
- API integration, scraping, or earnings dates (different problem, different cadence).
- Any notification-layer change.

FILES
A data/red_folder_2026.json
A cuttingboard/red_folder.py
A tests/test_red_folder.py

CHANGE SURFACE
- cuttingboard/red_folder.py is a new pure helper: it reads only its own bundled
  schedule file, takes now_utc as a parameter, and returns plain data. No imports of
  runtime/contract/payload state; no writes.

REQUIREMENTS

R1 — Window filtering relative to run time
load + filter return exactly the events whose ET event datetime falls within
[now_et, now_et + lookahead] (default 48h), given a now_utc argument.
FAIL: an event inside the window is omitted, or an event outside it is returned, in a
table-driven ET-boundary test (including a run timestamp near a UTC date boundary).

R2 — Loud failure on bad data
A missing or malformed schedule file yields a structured warning result with a
machine-checkable flag, not an exception and not a silently empty list.
FAIL: deleting or corrupting data/red_folder_2026.json causes the loader to raise or
to return an indistinguishable empty result.

R3 — Expiry signal
The loader exposes an expiry-warning flag that is true when now_et is within 30 days
of the schedule's last event date.
FAIL: with now_et inside 30 days of the last entry, the expiry flag is false.

R4 — Schema validation
Each event is validated for required keys (date, time_et, type, name) and parseable
ET datetime; an invalid entry triggers R2's warning path.
FAIL: an event missing a required key or with an unparseable date is returned as
valid.

DATA FLOW
data/red_folder_2026.json (read) + now_utc (arg) -> red_folder.events_in_window ->
list[Event] + warning/expiry flags -> (consumed by PRD-177 renderer).

FAIL CONDITIONS
- Window filter includes/excludes the wrong event at an ET boundary.
- Missing/malformed file raises or returns an indistinguishable empty result.
- Expiry flag wrong within 30 days of schedule end.
- Invalid event returned as valid.

VALIDATION
Manual:
- Construct a schedule with events at, just inside, and just outside a 48h window for
  a run timestamp near 23:xx UTC; confirm correct membership.
- Delete and corrupt the file; confirm the warning result (no raise, no silent empty).
- python3 -m pytest tests/test_red_folder.py -q passes; full suite green; ruff clean.

NOTES
- Verified facts: earnings gate GATE_EARNINGS (qualification.py:49) reads
  has_earnings_soon (qualification.py:87), fail-open, default None in production
  (options.py:389, confirmed). No existing calendar/event infrastructure.
- ET handling: reuse cuttingboard/time_utils.py (canonical ET via
  ZoneInfo("America/New_York"); get_now_utc/convert_utc_to_et). Pass now_utc in from
  the caller; do not read wall-clock inside decision-relevant paths.
- Data-file convention: reference via a module-level path constant (mirroring
  config.OHLCV_CACHE_DIR usage); test naming tests/test_<module>.py.
- All other draft assumptions verified accurate with no drift.
