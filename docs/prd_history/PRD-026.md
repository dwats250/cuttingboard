PRD-026 — Alert Visibility Upgrade — Deterministic ASCII Titles and Structured Body

STATUS
COMPLETE
Commit: 442b813

GOAL
Replace the single notification string with a (title, body) tuple where the title is a deterministic ASCII status label and the body uses structured key:value lines.

SCOPE
- output.py: build_notification_message returns (title, body) tuple
- Title MUST be one of: TRADE READY / NO TRADE - SYSTEM ACTIVE / WATCHLIST - SETUPS FORMING / HALT - SYSTEM ERROR
- Body uses structured key:value lines: Time, Regime, Tradable, Setups, Status, Reason
- runtime.py: unpack (title, body) and pass both to send_notification
- Update tests to assert on title and body separately

OUT OF SCOPE
- No changes to Telegram transport
- No changes to contract schema

FILES
M cuttingboard/output.py
M cuttingboard/runtime.py
M tests/test_prd017_notification_stabilization.py
M tests/test_sunday_mode.py

REQUIREMENTS

R1 — Tuple Return
build_notification_message MUST return (title, body) not a single string.

FAIL: build_notification_message returns a string.

R2 — Title Vocabulary
Title MUST be exactly one of: TRADE READY / NO TRADE - SYSTEM ACTIVE / WATCHLIST - SETUPS FORMING / HALT - SYSTEM ERROR.
No other title values MUST be produced.

FAIL: Title is any string outside the four defined values.

R3 — Structured Body
Body MUST use key:value lines.
Required keys: Time, Regime, Tradable, Setups, Status.
Reason MUST appear when tradable=False.

FAIL: Body contains free-form prose instead of key:value lines.

R4 — Runtime Unpacking
runtime.py MUST unpack (title, body) and pass both to send_notification.

FAIL: runtime.py passes a single string to send_notification.

DATA FLOW
contract → build_notification_message() → (title, body) tuple → send_notification(title, body)

FAIL CONDITIONS
- build_notification_message returns string
- Title outside four defined values
- Body is free-form prose
- send_notification receives single string

VALIDATION

Manual: live run — confirm Telegram message title matches exactly one of the four values, body is key:value.
