GOAL
PRD-245 made the FVG swapped-stop percent floor divide by the zone midpoint
(the post-swap entry), deliberately not `candidate.entry_price` — the
wrong-denominator trap the whole audit arc started from. But no fixture
discriminates the two: in both `_valid_long_fvg_df` and
`_wide_gap_long_fvg_df` the entry price and the zone midpoint sit within
~0.6% of each other, so a mutation reverting the denominator to
`candidate.entry_price` passes the suite green (PRD-245 review REC-1). This
MICRO adds one test whose two candidate denominators straddle
`MIN_STOP_PCT`, so the upgrade/fallback outcome flips on the denominator
alone — pinning the axis PRD-245's own mutation bar left uncovered. Test
only; no production behavior changes.

SCOPE

REQUIREMENTS

R1 — Denominator-straddle fixture
One additive test uses a candidate whose `entry_price` and the FVG zone
midpoint straddle `config.MIN_STOP_PCT`, so `risk/midpoint ≥ MIN_STOP_PCT`
(floor passes → PULLBACK_IMBALANCE) while `risk/entry_price < MIN_STOP_PCT`
(floor breach → DIRECT fallback). The straddle is asserted against
`config.MIN_STOP_PCT` (referenced, never hardcoded) as an in-test guard so a
future retune flags the fixture rather than silently passing. The DIRECT
candidate fully qualifies on its own geometry (both Gate 6 legs, Gate 7 R:R,
Gate 10 extension), so only the percent-leg denominator determines the
outcome. Geometry (verified against live config at `8bea09c`): reuse
`_wide_gap_long_fvg_df` (zone 100.0/102.4, midpoint 101.2, post-swap risk
1.2); candidate entry 130.0 / stop 128.4 / target 133.5, atr14 1.0, ema21
130.0. Straddle: 1.2/130.0 = 0.92% < 1.0% ≤ 1.2/101.2 = 1.19%.
FAIL: the test's two denominators do not straddle `config.MIN_STOP_PCT`
(the R2 mutation stays green), or the straddle bound is written as a
hardcoded rendered value instead of referencing `config.MIN_STOP_PCT`.

R2 — Mutation gate (the point of the PRD)
Forcing the percent leg back to `risk / candidate.entry_price` at the swap
site turns the R1 test red. Verified mechanically (git-apply the mutation →
confirm red → restore → confirm green), not by inspection.
FAIL: the mutation does not turn the R1 test red, or the check is asserted
by inspection rather than run.
