# PRD-070 - Manual Trade Journal and Mistake Taxonomy

**Status:** IN PROGRESS
**Commit:** --
**Depends on:** none

---

## GOAL

Create a deterministic manual trade journal format that records actual user trade actions, trade intent, thesis adherence, and mistake labels so real behavior can be reviewed against system output.

---

## SCOPE

- Add append-only manual trade journal support.
- Define a fixed mistake taxonomy.
- Define a fixed trade intent taxonomy.
- Allow manual records to reference system candidates by string ID when available.
- Allow manual records without system candidate linkage.
- Validate journal records deterministically.
- Produce machine-readable JSONL records for later review.
- Do not affect live trade decisions in this PRD.

---

## OUT OF SCOPE

- No broker integration.
- No automatic trade import.
- No account-size logic.
- No P/L calculation unless manually supplied.
- No dashboard/UI changes.
- No notification changes.
- No LLM commentary.
- No changes to execution policy.
- No changes to trade decisions.
- No automated scoring.
- No automatic correction of user input.

## REQUIREMENTS

### R1 - Journal path

Manual trade journal records must be appended to:

```
logs/manual_trades.jsonl
```

Records are append-only. Existing records must not be modified.

### R2 - Record shape

Each record must include exactly the following fields:

- recorded_at_utc: ISO-8601 UTC string, generated at write time by the journal writer
- trade_date: ISO-8601 date string, YYYY-MM-DD
- symbol: str
- action: str, see R3
- direction: str, see R4
- instrument_type: str, see R5
- thesis_adherence: str, see R6
- intent: str, see R7
- mistake_labels: list[str], see R8, must contain at least one element
- system_candidate_id: str or null
- notes: str or null

FAIL: any required field is absent.
FAIL: recorded_at_utc is a naive datetime or non-UTC.
FAIL: mistake_labels is empty list.

### R3 - Allowed action values

CONSIDERED, ENTERED, EXITED, SKIPPED, MISSED, CANCELLED

FAIL: any other value raises ValueError.

### R4 - Allowed direction values

LONG, SHORT, NEUTRAL, UNKNOWN

FAIL: any other value raises ValueError.

### R5 - Allowed instrument_type values

STOCK, ETF, OPTION, OPTION_SPREAD, CASH, UNKNOWN

FAIL: any other value raises ValueError.

### R6 - Allowed thesis_adherence values

FOLLOWED_THESIS, VIOLATED_THESIS, NO_THESIS, THESIS_UNKNOWN

FAIL: any other value raises ValueError.

### R7 - Allowed intent values

PLANNED_TRADE, IMPULSE_TRADE, HEDGE, TEST_SIZE, EXIT_MANAGEMENT, REVIEW_ONLY, UNKNOWN

FAIL: any other value raises ValueError.

### R8 - Mistake taxonomy

Allowed mistake_labels values:

CHASED_ENTRY, OVERSIZED, ENTERED_WITHOUT_THESIS, IGNORED_INVALIDATION, IGNORED_MACRO_CONFLICT, ENTERED_LATE, EXITED_EARLY, HELD_TOO_LONG, TOOK_LOW_QUALITY_SETUP, REVENGE_TRADE, OVERTRADED, BROKE_RULES, NONE

If no mistake occurred, mistake_labels must be ["NONE"].

If "NONE" is present alongside any other label, raise ValueError. "NONE" must be the only element when present.

FAIL: mistake_labels is empty.
FAIL: "NONE" present with any other label.
FAIL: any label outside the allowed set raises ValueError.

### R9 - Validation

Invalid enum values raise ValueError.
Missing required fields raise ValueError.
Empty mistake_labels raises ValueError.
"NONE" combined with any other label raises ValueError.

### R10 - No decision impact

This PRD must not affect runtime trade decisions, execution policy, macro pressure, market_map, notifications, dashboard rendering, or contract validation.

FAIL: any import of manual_journal in runtime.py, contract.py, or any delivery module.

### R11 - Future compatibility

Schema must be usable by PRD-071 review scorecard. No scorecard logic in this PRD.

## ACCEPTANCE CRITERIA

- Valid ENTERED option trade record appends to logs/manual_trades.jsonl.
- Invalid action value raises ValueError.
- Invalid intent value raises ValueError.
- Invalid mistake label raises ValueError.
- Missing required field raises ValueError.
- ["NONE"] accepted as the only no-mistake label.
- Empty mistake_labels raises ValueError.
- ["NONE", "CHASED_ENTRY"] raises ValueError.
- system_candidate_id = null is accepted.
- No runtime decisions change.
- All tests pass.
