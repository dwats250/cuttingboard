PRD-080 — Sunday Report Expansion Layer

STATUS: COMPLETE

GOAL
Extend the Sunday pre-market report into a macro context layer that summarizes
cross-asset conditions before the Monday cash session without approving trades
or mutating decision logic.

SCOPE
- Extend existing SUNDAY_PREMARKET mode created in PRD-077
- Add a Sunday context block rendered in dashboard_renderer.py
- Block is built from data already available at render time:
  - payload (sections, meta, macro_drivers, summary)
  - market_map (already passed to render_dashboard_html)
  - pressure dict (already computed by _build_pressure_snapshot inside renderer)
- Surface the following summary fields:
  - session_type
  - headline
  - macro_posture (sourced from payload["summary"]["market_regime"])
  - dollar_context (sourced from macro_drivers.dollar)
  - rates_context (sourced from macro_drivers.rates)
  - volatility_context (sourced from macro_drivers.volatility)
  - metals_context (sourced from market_map.symbols: GLD, SLV, GDX)
  - risk_context (sourced from macro_drivers.bitcoin)
  - monday_watch (derived from posture + market_map grades)
- Keep output outcome as NO_TRADE
- Keep system_state.stay_flat_reason as PREMARKET_CONTEXT

OUT OF SCOPE
- No trade approvals
- No new hard gates
- No execution policy changes
- No candidate forcing
- No new data sources or external fetches
- No economic calendar integration
- No earnings calendar integration
- No ML summaries
- No web scraping
- No backtesting
- No notification expansion beyond existing Sunday premarket title behavior
- No changes to build_report_payload signature or payload sections schema

FILES
- cuttingboard/delivery/dashboard_renderer.py
- tests/test_sunday_report_expansion.py
