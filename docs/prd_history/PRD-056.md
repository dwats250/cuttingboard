# PRD-056: Candidate Lifecycle Tracking

## GOAL

Inject deterministic lifecycle metadata into market_map.json so the system can
identify how each candidate changed between the previous and current snapshot.

This PRD adds a data-layer patch only. It does not change trade decisions, regime
logic, qualification gates, dashboard behavior, or notification logic.

---

## SCOPE

- Add `lifecycle` block to each symbol entry in market_map["symbols"]
- Add `removed_symbols` list at the top level of market_map
- Read the previous market_map.json before overwriting it (runtime.py owns persistence — confirmed by audit)
- Compute lifecycle transitions deterministically from grade and setup_state
- Raise RuntimeError on malformed previous market_map.json; return None when absent

---

## OUT OF SCOPE

- Notifications
- Dashboard changes
- Execution / paper execution logic
- Trade decision logic
- Qualification or policy logic
- New symbols or data sources
- Price levels, R:R, stops, targets
- AI inference or subjective commentary
- History subsystem or rolling snapshots beyond one previous file
- Direction/setup_type as identity key (symbol only)

## REQUIREMENTS

### R1 — Lifecycle block per symbol

Every symbol in current market_map["symbols"] must contain a `lifecycle` key with
the following structure:

```json
{
  "previous_grade": "str | null",
  "current_grade": "str",
  "grade_transition": "NEW | UNCHANGED | UPGRADED | DOWNGRADED | UNKNOWN",
  "previous_setup_state": "str | null",
  "current_setup_state": "str | null",
  "setup_state_transition": "NEW | UNCHANGED | CHANGED | UNKNOWN",
  "is_new": "bool",
  "is_removed": "bool"
}
```

`is_new` is true only when a previous snapshot exists AND the symbol is absent from it.
`is_new` is false when no previous snapshot exists.
`is_removed` is false for every symbol in current market_map["symbols"].

FAIL: Any symbol in market_map["symbols"] lacks a "lifecycle" key after a fixture run.

### R2 — Grade ordering

Grade order for transition direction: A+ > A > B > C > D > F.
FAIL: A symbol that moved from B to A is not labeled UPGRADED.
FAIL: A symbol that moved from A to C is not labeled DOWNGRADED.

### R3 — Transition rules (grade)

- No previous snapshot → grade_transition = "UNKNOWN", is_new = false
- Symbol absent from previous (snapshot exists) → grade_transition = "NEW", is_new = true
- Grades equal → "UNCHANGED"
- Current grade better than previous → "UPGRADED"
- Current grade worse than previous → "DOWNGRADED"
- Either grade unrecognized or missing → "UNKNOWN"

`is_removed` is always false for symbols present in current market_map["symbols"].

FAIL: Any rule above produces the wrong value.

### R4 — Transition rules (setup_state)

- No previous snapshot → "UNKNOWN"
- Symbol absent from previous (snapshot exists) → "NEW"
- States equal → "UNCHANGED"
- States different, both present → "CHANGED"
- Either missing → "UNKNOWN"

FAIL: Any rule above produces the wrong value.

### R5 — Removed symbols

Symbols in previous_map["symbols"] but absent from current_map["symbols"] must
appear in market_map["removed_symbols"] as:

```json
{"symbol": "str", "previous_grade": "str", "grade_transition": "REMOVED", "is_removed": true}
```

`removed_symbols` must be present as a top-level key (empty list if none removed).
FAIL: removed_symbols key absent from market_map output.
FAIL: A symbol present in previous but absent in current is not in removed_symbols.

### R6 — No mutation

inject_lifecycle must not mutate either input dict.
FAIL: current_map or previous_map is modified after calling inject_lifecycle.

### R7 — No crash when previous absent

When logs/market_map.json does not exist, _load_previous_market_map returns None
and the pipeline completes normally with all lifecycle fields set to UNKNOWN variants.
FAIL: Pipeline crashes or raises when no previous market_map.json exists.

### R8 — Crash on malformed previous

When logs/market_map.json exists but contains invalid JSON, _load_previous_market_map
raises RuntimeError and does not silently ignore the malformed file.
FAIL: Malformed previous market_map.json is silently swallowed or raises any exception
other than RuntimeError.

### R9 — No regression

Existing behavior must not regress. Existing tests must not be loosened.
Fixture updates are allowed only where required to account for added lifecycle and
removed_symbols fields.
FAIL: Any previously passing test fails after this change.
FAIL: Any existing test assertion is weakened or removed.

## DATA FLOW

1. execute_run() calls _run_pipeline() → returns PipelineResult with market_map dict
2. execute_run() calls _load_previous_market_map()
   - MARKET_MAP_PATH exists + valid JSON → return dict
   - MARKET_MAP_PATH exists + invalid JSON → raise RuntimeError
   - MARKET_MAP_PATH absent → return None
3. execute_run() calls inject_lifecycle(pipeline.market_map, previous_market_map)
   - Returns new dict with lifecycle blocks injected into each symbol
   - Appends removed_symbols list at top level
4. execute_run() calls _write_market_map_file(enhanced_market_map)
   - Overwrites logs/market_map.json

inject_lifecycle is a pure function with no file I/O.
_load_previous_market_map has no side effects beyond reading one file.

---

## FAIL CONDITIONS

- Lifecycle block absent from any symbol → R1 FAIL
- is_new true when no previous snapshot exists → R1/R3 FAIL
- is_removed true for any symbol in current market_map["symbols"] → R1 FAIL
- Wrong grade_transition value for any known transition → R2/R3 FAIL
- Wrong setup_state_transition value → R4 FAIL
- removed_symbols key absent from market_map → R5 FAIL
- A removed symbol not listed in removed_symbols → R5 FAIL
- Input mutation → R6 FAIL
- Crash when no previous file exists → R7 FAIL
- Malformed previous JSON silently ignored or raises non-RuntimeError → R8 FAIL
- Any existing test regresses or is weakened → R9 FAIL

import json
m = json.load(open('logs/market_map.json'))
syms = m['symbols']
for s, v in syms.items():
    assert 'lifecycle' in v, f'{s} missing lifecycle'
    lc = v['lifecycle']
    assert 'grade_transition' in lc
    assert 'setup_state_transition' in lc
    assert isinstance(lc['is_new'], bool)
    assert isinstance(lc['is_removed'], bool)
    assert lc['is_removed'] is False
assert 'removed_symbols' in m
print('PASS: all lifecycle fields present')
print('removed_symbols count:', len(m['removed_symbols']))
for s, v in syms.items():
    print(s, v['lifecycle']['grade_transition'], v['lifecycle']['setup_state_transition'])
"
