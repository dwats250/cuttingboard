# PRD-071 - Trading Process Review Scorecard

**Status:** IN PROGRESS
**Commit:** --
**Depends on:** PRD-070

---

## GOAL

Generate a deterministic process-quality scorecard from manual trade journal records so the trader can review discipline, thesis adherence, and rule violations without relying on P/L.

---

## SCOPE

- Read logs/manual_trades.jsonl from PRD-070.
- Generate a daily process scorecard.
- Count rule violations and mistake labels.
- Summarize thesis_adherence and intent distribution.
- Write a machine-readable daily scorecard.
- Do not affect current-day trade decisions.

---

- No broker integration.
- No automatic P/L calculation.
- No account-size logic.
- No dashboard/UI changes.
- No notifications.
- No LLM commentary.
- No trade recommendation.
- No mutation of manual journal records.
- No changes to runtime trade decisions.
- No changes to execution policy.

## REQUIREMENTS

### R1 - Output path

Daily scorecard is written to:

```
logs/review_scorecard_YYYY-MM-DD.json
```

Existing files for the same date are overwritten.

### R2 - Scorecard shape

Scorecard includes exactly:

- trade_date: ISO-8601 date string
- total_records: int
- entered_count: int
- skipped_count: int
- missed_count: int
- exited_count: int
- planned_trade_count: int
- impulse_trade_count: int
- thesis_followed_count: int, count of records where thesis_adherence == FOLLOWED_THESIS
- thesis_violated_count: int, count of records where thesis_adherence == VIOLATED_THESIS
- no_thesis_count: int, count of records where thesis_adherence == NO_THESIS
- mistake_counts: dict[str, int], label to count, excluding NONE
- process_flags: list[str], see R3
- overall_process_grade: str, see R4

### R3 - Process flags

Allowed values, multiple may be present:

NO_TRADES_RECORDED, IMPULSE_TRADE_PRESENT, THESIS_VIOLATION_PRESENT, NO_THESIS_ENTRY_PRESENT, REVENGE_TRADE_PRESENT, OVERTRADING_PRESENT, CLEAN_PROCESS_DAY, INSUFFICIENT_DATA

CLEAN_PROCESS_DAY must be present if and only if overall_process_grade == A.
INSUFFICIENT_DATA must be present if and only if overall_process_grade == INSUFFICIENT_DATA.

FAIL: CLEAN_PROCESS_DAY present on a non-A day.
FAIL: INSUFFICIENT_DATA absent when grade is INSUFFICIENT_DATA.

### R4 - Grade values

A, B, C, D, F, INSUFFICIENT_DATA

### R5 - Grade rules, evaluated in priority order. First matching rule wins.

1. INSUFFICIENT_DATA if total_records == 0, journal file missing, or no records for the date.
2. F if any record has mistake_labels containing REVENGE_TRADE or BROKE_RULES.
3. D if any record with action == ENTERED has thesis_adherence in {NO_THESIS, VIOLATED_THESIS}.
4. C if impulse_trade_count > 0, meaning any record has intent == IMPULSE_TRADE.
5. B if all records with action == ENTERED have thesis_adherence == FOLLOWED_THESIS and mistake_counts excluding NONE is non-empty.
6. A if all records with action == ENTERED have thesis_adherence == FOLLOWED_THESIS and mistake_counts excluding NONE is empty, OR if entered_count == 0 and all records have mistake_labels == ["NONE"].

FAIL: two grade rules produce conflicting results. This cannot happen given priority ordering.
FAIL: CLEAN_PROCESS_DAY is set on a non-A scorecard.

### R6 - Mistake counts

mistake_counts includes only labels observed in records for the date, excluding NONE. If NONE is the only label observed, mistake_counts is {}.

### R7 - No P/L dependency

Scorecard must not require any P/L field. If a journal record includes a P/L field not defined in PRD-070, it must be ignored.

### R8 - Missing journal file

If logs/manual_trades.jsonl does not exist or contains no records for the date:

- Write a valid scorecard with total_records = 0, overall_process_grade = INSUFFICIENT_DATA, process_flags = ["INSUFFICIENT_DATA"].

FAIL: missing journal raises an unhandled exception instead of producing an INSUFFICIENT_DATA scorecard.

### R9 - No decision impact

This PRD must not affect runtime trade decisions, execution policy, contract validation, notifications, or dashboard rendering.

FAIL: any import of review_scorecard in runtime.py, contract.py, or any delivery module.

### R10 - Future compatibility

Scorecard output may be consumed by a future memory compaction PRD. No compaction logic in this PRD.

---

## ACCEPTANCE CRITERIA

- Scorecard writes valid JSON to logs/review_scorecard_YYYY-MM-DD.json.
- Missing journal file produces INSUFFICIENT_DATA grade.
- Any REVENGE_TRADE or BROKE_RULES label produces F.
- ENTERED record with NO_THESIS or VIOLATED_THESIS produces D if no F condition present.
- IMPULSE_TRADE record produces C if no F or D condition present.
- All-followed-thesis day with non-NONE mistakes produces B.
- All-followed-thesis day with only NONE mistakes produces A.
- entered_count == 0 with all-NONE mistakes produces A.
- P/L field, if present in journal, is ignored.
- No runtime decisions change.
- All tests pass.

## VALIDATION

```
python3 -m pytest tests/test_review_scorecard.py -q
python3 -m pytest tests/ -q
```

Required test cases:

- Missing journal produces INSUFFICIENT_DATA grade.
- Records for wrong date only produce INSUFFICIENT_DATA grade.
- Clean trade day with all FOLLOWED_THESIS and NONE mistakes produces A and CLEAN_PROCESS_DAY in flags.
- No-trade discipline day with SKIPPED/MISSED records, NONE mistakes, and entered_count == 0 produces A.
- Minor mistake day with FOLLOWED_THESIS and non-NONE non-critical mistake produces B.
- Impulse trade day with IMPULSE_TRADE intent and no F/D conditions produces C.
- ENTERED record with NO_THESIS adherence and no F condition produces D.
- REVENGE_TRADE label produces F even if thesis followed.
- BROKE_RULES label produces F.
- P/L field in journal record is ignored.
