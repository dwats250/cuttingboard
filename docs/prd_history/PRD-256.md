PRD-256 — Continuation-path ATR proxy max-loss fix or permanent-cap ruling

Status: IN PROGRESS

LANE
HIGH-RISK

CLASS
EXECUTION

WHY NOW
PRD-252 decoupled the continuation path onto
`CONTINUATION_MAX_RISK_PCT_PER_TRADE=0.01` "until its ATR proxy's
max-loss understatement is fixed" (2026-07-10); no PRD carries that fix,
and `docs/PROJECT_STATE.md`'s known-debt entry re-evaluates 2026-08-15.
As it stands, "temporary decouple" is indistinguishable from "two
permanent risk caps" — and a wrong max-loss number is a wrong
invalidation level, one of the four questions this system exists to
pre-answer. Exposure is bounded by the $150 cap today; this is a queued
slot, not an incident.

MAX EXPECTED DELTA
R1 (characterization) is always in scope: 1 file
(`PRD-251.continuation-path.proposal.md`), no code. R2 (the ruling) adds
2 doc files. R3 or R4 (mutually exclusive, chosen at Gate A) each touch
at most 6 further files; the two branches never both land. No branch
touches more than ~120 production/test LOC.

GOAL
Replace the current ambiguity — is the continuation path's frozen $150
budget temporary or permanent? — with a recorded, evidenced ruling: fix
the ATR proxy's max-loss understatement and re-converge onto the raised
~$400 budget, or declare the two-cap split permanent and strike
"temporary" language everywhere it appears. Both outcomes are
acceptable; the current silence is not.

SCOPE
- R1: quantify the understatement — how far `_qualify_continuation_
  candidate`'s ATR proxy diverges from a real max-loss figure, across a
  representative ATR14 range and both CREDIT and DEBIT strategy
  resolutions (only CREDIT strategies carry the width-minus-credit gap;
  DEBIT strategies price at the proxy directly, per the proposal doc).
- R2: Dustin's Gate A ruling, informed by R1, recorded in
  `docs/DECISIONS.md`: FIX (pick a correction, re-converge) or PERMANENT
  (keep two caps, strike "temporary").
- R3 (FIX only): implement the chosen correction, retire
  `CONTINUATION_MAX_RISK_PCT_PER_TRADE`, validate continuation sizing at
  the raised budget.
- R4 (PERMANENT only): strike "temporary" framing from `config.py`,
  `PROJECT_STATE.md`, and the proposal doc; close or reclassify the
  known-debt entry.

OUT OF SCOPE
- Re-deriving PRD-251's direct-path fix (COMPLETE; unrelated code path).
- The correlation-modifier sourcing finding tracked as PRD-253
  (`contract.py`/`audit.py`, a different subsystem from this ATR proxy).
- Picking R2's outcome in this document — that is Gate A's call, informed
  by R1, not a default this PRD asserts.
- Any dev-tooling item from the Fable audit dispatch (PRD-254, PRD-255):
  this is the only real-money item on that board.

REQUIREMENTS

R1 — Characterize the understatement
Produce a quantified comparison (table or worked examples) of
`spread_width = max(0.50, atr14 * 0.05)` treated as max-loss against a
real max-loss figure, across a representative ATR14 range, for both
CREDIT strategies (real max-loss = width - credit; width undefined for
this path today, per the proposal doc's open question) and DEBIT
strategies (proxy already approximates the real number). Append to
`docs/prd_history/PRD-251.continuation-path.proposal.md`.

FAIL: no quantified comparison exists, or it covers only one strategy
type.

R2 — Gate A ruling, recorded
Dustin rules FIX or PERMANENT, informed by R1. FIX: pick one of the
proposal doc's three candidate approaches (or a new one, justified
against the same three) and commit to re-converging
`CONTINUATION_MAX_RISK_PCT_PER_TRADE` onto `MAX_RISK_PCT_PER_TRADE`.
PERMANENT: the two-cap split is intentional; "temporary" language is
retired, not the constant.

FAIL: R3 or R4 work begins before a dated `docs/DECISIONS.md` entry
records which branch was ruled.

R3 — FIX branch (conditional on R2 = FIX)
Implement the chosen correction in `_qualify_continuation_candidate`
(and `options.py` if the approach borrows its strike-distance buckets);
delete `config.CONTINUATION_MAX_RISK_PCT_PER_TRADE`; both sizing sites
(`qualification.py`, `options.py`'s correlation-modifier recompute) read
`MAX_RISK_PCT_PER_TRADE` unconditionally again. Explicitly VALIDATE
continuation sizing at the raised ~$400 budget (new or updated test
asserting the post-fix contract count/dollar-risk at the raised budget)
— not a silent inherit from deleting the interim knob, per the tracked
requirement PRD-252 left on the proposal doc.

FAIL: `CONTINUATION_MAX_RISK_PCT_PER_TRADE` still exists after R3
lands, or no test asserts continuation sizing at the raised budget.

R4 — PERMANENT branch (conditional on R2 = PERMANENT)
Strike every "temporary" / "until its ATR proxy... is fixed" phrase in
`cuttingboard/config.py`'s comment (lines ~72-80),
`docs/PROJECT_STATE.md`'s known-debt entry, and
`docs/prd_history/PRD-251.continuation-path.proposal.md`, replacing each
with the permanent-by-design rationale R2 recorded. Close the
PROJECT_STATE known-debt entry (it is a decision now, not open debt) or
reclassify it as a recorded design constant with no re-evaluation date.

FAIL: any "temporary" framing survives in those three files after R2
rules PERMANENT.
