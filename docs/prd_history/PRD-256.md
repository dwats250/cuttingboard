PRD-256 — Continuation-path ATR proxy max-loss fix or permanent-cap ruling

Status: IN PROGRESS

LANE
HIGH-RISK

CLASS
EXECUTION

WHY NOW
PRD-252 decoupled the continuation path onto
`CONTINUATION_MAX_RISK_PCT_PER_TRADE=0.01` "until its ATR proxy's
max-loss understatement is fixed" (2026-07-10); no PRD carries that fix,
and `docs/PROJECT_STATE.md`'s known-debt entry re-evaluates 2026-08-15.
As it stands, "temporary decouple" is indistinguishable from "two
permanent risk caps" — and a wrong max-loss number is a wrong
invalidation level, one of the four questions this system exists to
pre-answer. Exposure is bounded by the $150 cap today; this is a queued
slot, not an incident.

MAX EXPECTED DELTA
Phase 1 (R1, always in scope): 1 file
(`PRD-251.continuation-path.proposal.md`), no code. R2 (the ruling, a
doc-only gate) adds up to 2 further doc files. Phase 2 (R3 or R4,
mutually exclusive, chosen at R2, never both): each touches at most 6
further files. No branch touches more than ~120 production/test LOC.

GOAL
Replace the current ambiguity — is the continuation path's frozen $150
budget temporary or permanent? — with a recorded, evidenced ruling made
AFTER the evidence exists: characterize the ATR proxy's max-loss
understatement first (Phase 1), rule on the characterization (R2), then
either fix and re-converge onto the raised ~$400 budget or declare the
two-cap split permanent and strike "temporary" language everywhere it
appears (Phase 2). Both outcomes are acceptable; ruling before Phase 1
exists, or staying silent, is not. This PRD's own Gate A approves only
the two-phase shape and Phase 1's scope, NOT the FIX-vs-PERMANENT choice
— that choice is R2, a separate gate that fires after Phase 1's analysis
is actually in hand, not a decision Gate A pre-commits to on no data.

SCOPE
- Phase 1 / R1: quantify the understatement — how far
  `_qualify_continuation_candidate`'s ATR proxy diverges from a real
  max-loss figure, across a representative ATR14 range and both CREDIT
  and DEBIT strategy resolutions (only CREDIT strategies carry the
  width-minus-credit gap; DEBIT strategies price at the proxy directly,
  per the proposal doc).
- Ruling gate / R2: Dustin rules FIX or PERMANENT, informed by R1's
  completed analysis — not Gate A, and not before R1 lands.
- Phase 2 / R3 (FIX only): implement the chosen correction, retire
  `CONTINUATION_MAX_RISK_PCT_PER_TRADE`, validate continuation sizing at
  the raised budget.
- Phase 2 / R4 (PERMANENT only): strike "temporary" framing from
  `config.py`, `PROJECT_STATE.md`, and the proposal doc; close or
  reclassify the known-debt entry.

OUT OF SCOPE
- Re-deriving PRD-251's direct-path fix (COMPLETE; unrelated code path).
- The correlation-modifier sourcing finding tracked as PRD-253
  (`contract.py`/`audit.py`, a different subsystem from this ATR proxy).
- Picking R2's outcome in this document, or at Gate A — R2 is a
  separate, later gate informed by R1's actual output; this PRD asserts
  no default.
- Any dev-tooling item from the Fable audit dispatch (PRD-254, PRD-255):
  this is the only real-money item on that board.

REQUIREMENTS

R1 — Phase 1: characterize the understatement
Produce a quantified comparison (table or worked examples) of
`spread_width = max(0.50, atr14 * 0.05)` treated as max-loss against a
real max-loss figure, across a representative ATR14 range, for both
CREDIT strategies (real max-loss = width - credit; width undefined for
this path today, per the proposal doc's open question) and DEBIT
strategies (proxy already approximates the real number). Append to
`docs/prd_history/PRD-251.continuation-path.proposal.md`. Deliverable is
the analysis only — no recommendation, no pre-committed direction.

FAIL: no quantified comparison exists, it covers only one strategy type,
or it states a FIX/PERMANENT recommendation instead of leaving the
ruling to R2.

R2 — Ruling gate (fires after R1 lands, not at Gate A)
Dustin rules FIX or PERMANENT, reading R1's completed analysis — this
gate cannot fire, and Phase 2 cannot start, until R1's deliverable exists
in the tree. FIX: pick one of the proposal doc's three candidate
approaches (or a new one, justified against the same three) and commit
to re-converging `CONTINUATION_MAX_RISK_PCT_PER_TRADE` onto
`MAX_RISK_PCT_PER_TRADE`. PERMANENT: the two-cap split is intentional;
"temporary" language is retired, not the constant.

FAIL: R3 or R4 work begins before R1's analysis is implemented AND a
dated `docs/DECISIONS.md` entry records which branch was ruled.

R3 — Phase 2, FIX branch (conditional on R2 = FIX)
Implement the chosen correction in `_qualify_continuation_candidate`
(and `options.py` if the approach borrows its strike-distance buckets);
delete `config.CONTINUATION_MAX_RISK_PCT_PER_TRADE`; both sizing sites
(`qualification.py`, `options.py`'s correlation-modifier recompute) read
`MAX_RISK_PCT_PER_TRADE` unconditionally again. Explicitly VALIDATE
continuation sizing at the raised ~$400 budget (new or updated test
asserting the post-fix contract count/dollar-risk at the raised budget)
— not a silent inherit from deleting the interim knob, per the tracked
requirement PRD-252 left on the proposal doc.

FAIL: `CONTINUATION_MAX_RISK_PCT_PER_TRADE` still exists after R3
lands, or no test asserts continuation sizing at the raised budget.

R4 — Phase 2, PERMANENT branch (conditional on R2 = PERMANENT)
Strike every "temporary" / "until its ATR proxy... is fixed" phrase in
`cuttingboard/config.py`'s comment (lines ~72-80),
`docs/PROJECT_STATE.md`'s known-debt entry, and
`docs/prd_history/PRD-251.continuation-path.proposal.md`, replacing each
with the permanent-by-design rationale R2 recorded. Close the
PROJECT_STATE known-debt entry (it is a decision now, not open debt) or
reclassify it as a recorded design constant with no re-evaluation date.

FAIL: any "temporary" framing survives in those three files after R2
rules PERMANENT.

DATA FLOW
R1: code (`qualification.py`, `config.py`) -> analysis -> proposal-doc
table (no runtime change). R2: R1's analysis -> Dustin -> DECISIONS.md
entry (a gate, not a runtime step). R3: DECISIONS ruling ->
`qualification.py`/`options.py` sizing logic ->
`QualificationResult`/`dollar_risk` -> unchanged downstream contract
shape. R4: DECISIONS ruling -> comment/doc text only, no runtime change.

FAIL CONDITIONS
- R1's characterization is missing, single-strategy-only, or asserts a
  ruling instead of leaving it to R2.
- R2's ruling is undocumented, or fires before R1's analysis exists in
  the tree.
- R3 lands without retiring the interim constant or validating the
  raised-budget behavior.
- R4 lands with "temporary" language still present anywhere in scope.
- Both R3 and R4 land (the branches are mutually exclusive).
