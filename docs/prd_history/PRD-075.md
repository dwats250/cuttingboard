PRD-075 — Signal Performance Engine

STATUS: COMPLETE

GOAL

Build a deterministic performance evaluation layer that measures the
real-world effectiveness of trade candidates and validates system edge.

Outputs directly influence:
- trade qualification confidence
- trade avoidance decisions
- future system tuning

-------------------------------------

SCOPE

- Read trade evaluation records from logs/evaluation.jsonl
- Compute performance metrics per symbol
- Generate deterministic performance summary
- Persist results to a new artifact

-------------------------------------

OUT OF SCOPE

- No changes to trade execution
- No broker integration
- No UI/dashboard rendering (handled in later PRD)
- No new market data sources
- No probabilistic modeling
- No ML/AI
- No external APIs
- No bucketing by regime or grade (fields not present in evaluation records;
  defer to a later PRD that extends evaluation.py)

-------------------------------------

FILES

- cuttingboard/performance_engine.py (NEW)
- cuttingboard/runtime.py (MODIFY — append after evaluation step)
- logs/performance_summary.json (NEW artifact)
- tests/test_performance_engine.py (NEW)

-------------------------------------

DATA SOURCES

- logs/evaluation.jsonl (output of cuttingboard/evaluation.py)

Each record structure (actual contract):

Top-level fields:
  symbol               string
  direction            string  (LONG | SHORT)
  entry                float
  stop                 float
  target               float
  decision_run_at_utc  string (ISO UTC)
  evaluated_at_utc     string (ISO UTC)

Nested under "evaluation":
  result               string  (TARGET_HIT | STOP_HIT | NO_HIT)
  R_multiple           float   (signed; positive = favorable exit)
  time_to_resolution   int

All records in the file are implicitly ALLOW_TRADE decisions (filtered at
write time in evaluation.py).

Required fields for processing:
  symbol, direction, evaluation.result, evaluation.R_multiple

If any required field is missing or evaluation.result is not one of the
three valid values:
→ record MUST be skipped (no partial evaluation)

Result mapping:
  TARGET_HIT → WIN
  STOP_HIT   → LOSS
  NO_HIT     → FLAT

-------------------------------------

REQUIREMENTS

R1 — Aggregation buckets

Group performance by symbol.

Bucket key: symbol string (e.g. "SPY")

-------------------------------------

R2 — Metrics calculation

For each bucket compute:

  total_trades
  wins          (evaluation.result == TARGET_HIT)
  losses        (evaluation.result == STOP_HIT)
  flats         (evaluation.result == NO_HIT)

  win_rate      = wins / total_trades
  avg_r_win     = mean(R_multiple) for wins  (0.0 if no wins)
  avg_r_loss    = mean(|R_multiple|) for losses  (0.0 if no losses)

  expectancy    = (win_rate * avg_r_win) - ((1 - win_rate) * avg_r_loss)

All calculations MUST be deterministic and use the full dataset.

-------------------------------------

R3 — Minimum sample rule

If total_trades < 5:

  - mark bucket as: "insufficient_data": true
  - DO NOT compute expectancy
  - DO NOT compute win_rate

-------------------------------------

R4 — Output artifact

Write to: logs/performance_summary.json

Structure:

{
  "buckets": {
    "SPY": {
      "total_trades": 12,
      "wins": 7,
      "losses": 5,
      "flats": 0,
      "win_rate": 0.583,
      "avg_r_win": 1.8,
      "avg_r_loss": 0.9,
      "expectancy": 0.503,
      "insufficient_data": false
    }
  },
  "generated_at": "<UTC ISO timestamp>"
}

-------------------------------------

R5 — Runtime integration

In runtime.py:

- Call performance_engine AFTER the evaluation step completes
  (after run_post_trade_evaluation call at line 930)
- MUST NOT block pipeline if evaluation.jsonl is missing
- If file missing → skip silently (no crash, no log required)

-------------------------------------

R6 — Determinism

Given identical evaluation.jsonl:

- performance_summary.json MUST be identical

No randomness. No sampling. No time-based variation in metrics.
generated_at timestamp is the only non-deterministic field.

-------------------------------------

R7 — No mutation of existing logic

This PRD MUST NOT:

- modify trade decisions
- modify evaluation records
- modify contract or payload
- modify execution policy

-------------------------------------

R8 — Direct trading impact (CRITICAL RULE)

This PRD influences trading indirectly via future PRDs using:

  expectancy, win_rate, per-symbol win/loss counts

This PRD is the foundation source of truth for signal validity.

-------------------------------------

FAIL CONDITIONS

- Missing evaluation.jsonl causes crash
- Buckets computed with < 5 trades produce expectancy or win_rate
- Any metric varies across identical runs (excluding generated_at)
- Any mutation to evaluation.jsonl occurs
- Output file not written when valid records exist
- evaluation.result values outside TARGET_HIT/STOP_HIT/NO_HIT are
  not skipped
- avg_r_loss computed using signed (negative) R_multiple instead of
  absolute value

-------------------------------------

VALIDATION

- Run system with existing evaluation.jsonl
- Confirm performance_summary.json is created
- Verify:
  - metrics match manual calculation against raw records
  - buckets correctly grouped by symbol
  - insufficient_data applied correctly for buckets with < 5 records
  - avg_r_loss uses |R_multiple|, not signed value
- Run twice → bucket metrics must match exactly (generated_at excepted)
