# PRD-201: Canonical read-guard hook (warn on redundant re-read of injected docs)

Status: COMPLETE

LANE
STANDARD

CLASS
INFRA

WHY NOW
The process retrospective found canonical docs re-Read many times per session
(CLAUDE.md 4x in one session). CLAUDE.md and the auto-memory MEMORY.md are
injected into the system prompt at session start, so Reading them via the Read
tool is provably redundant token/context spend. A PreToolUse(Read) guard makes
that redundancy visible mechanically rather than relying on honor-system
guidance (which PRD-198 distrusts).

MAX EXPECTED DELTA
<= 40 LOC for the hook script + <= 60 LOC for its test, plus additive edits to
.claude/settings.json and docs/CLAUDE_HOOKS.md. No production-pipeline code.
Exceeding this stops implementation and re-triggers review.

GOAL
Add a non-blocking PreToolUse(Read) hook that, when the Read tool targets the
repo-root CLAUDE.md or the session auto-memory MEMORY.md, surfaces a
model-visible reminder that the file is already in context, while always
allowing the Read.

SCOPE
- New .claude/hooks/canonical_read_guard.sh: reads the PreToolUse stdin JSON,
  extracts tool_input.file_path, and when it is the repo-root CLAUDE.md or the
  auto-memory MEMORY.md, emits JSON
  {"hookSpecificOutput":{"hookEventName":"PreToolUse","permissionDecision":"allow",
  "additionalContext":"<reminder>"}} and exits 0. For any other path it emits
  nothing and exits 0.
- Wire it in .claude/settings.json under hooks.PreToolUse, matcher "Read".
- Document it in docs/CLAUDE_HOOKS.md (Wired-hooks table + a section).
- Ship a red test.

OUT OF SCOPE
- PROJECT_STATE.md, DECISIONS.md, PRD_REGISTRY.md, prd_index.json -- they
  legitimately change mid-session, so re-reads are not flagged.
- Code-file re-reads (e.g. runtime/__init__.py) -- not canonical docs.
- Any blocking/deny behavior, and any per-session "already read" state tracking
  (unnecessary: these two docs are always injected, so any Read is redundant).
- Any change to the existing hooks (protect_files.sh, prd_eval.sh).

FILES
A .claude/hooks/canonical_read_guard.sh
M .claude/settings.json
M docs/CLAUDE_HOOKS.md
A tests/test_canonical_read_guard_hook.py

REQUIREMENTS

R1 -- Warn for injected canonical docs
When PreToolUse(Read) targets the repo-root CLAUDE.md or the auto-memory
MEMORY.md, the hook's stdout is JSON whose
hookSpecificOutput.additionalContext is a non-empty reminder and whose
hookSpecificOutput.permissionDecision is "allow"; exit 0.

FAIL: a Read of repo-root CLAUDE.md or auto-memory MEMORY.md yields empty or
missing hookSpecificOutput.additionalContext.

R2 -- Never blocks
For every input the hook exits 0 and never emits permissionDecision "deny" or
"ask" (the Read always proceeds).

FAIL: the hook exits non-zero, or emits a permissionDecision other than
"allow", for any input.

R3 -- Silent for non-canonical paths
For a Read whose file_path is not the repo-root CLAUDE.md or auto-memory
MEMORY.md, the hook emits empty stdout and exit 0.

FAIL: a Read of a non-canonical path (e.g. a .py file) emits non-empty
additionalContext.

R4 -- Wired on PreToolUse(Read)
.claude/settings.json registers canonical_read_guard.sh under hooks.PreToolUse
with matcher "Read"; the file remains valid JSON.

FAIL: .claude/settings.json has no PreToolUse entry invoking
canonical_read_guard.sh with matcher "Read", or is not valid JSON.

R5 -- Documented
docs/CLAUDE_HOOKS.md Wired-hooks table lists the hook (Event PreToolUse/Read,
Blocks? No) and has a describing section.

FAIL: docs/CLAUDE_HOOKS.md does not mention canonical_read_guard.sh.

R6 -- Red test proves the guard is real and narrow (PRD-198 invariant 4)
tests/test_canonical_read_guard_hook.py asserts: (a) a Read payload for
repo-root CLAUDE.md and one for auto-memory MEMORY.md each yield non-empty
additionalContext; (b) a Read payload for a code file yields empty stdout;
(c) returncode 0 in all cases.

FAIL: no test asserts both the canonical-doc warning and the non-canonical
silence, or the test cannot fail when the guard regresses.

DATA FLOW
Read tool call -> PreToolUse canonical_read_guard.sh (stdin JSON) -> read
tool_input.file_path -> if repo-root CLAUDE.md / auto-memory MEMORY.md: emit
allow + additionalContext (-> agent context); else emit nothing -> Read proceeds

FAIL CONDITIONS
- A Read of repo-root CLAUDE.md or auto-memory MEMORY.md yields no
  additionalContext.
- The hook exits non-zero or emits permissionDecision != "allow" for any input.
- A Read of a non-canonical path emits additionalContext.
- .claude/settings.json lacks the PreToolUse/Read wiring or is invalid JSON.
- docs/CLAUDE_HOOKS.md does not document the hook.
- No red test exercises both the warn and the silence cases.

VALIDATION
Manual:
- Pipe a PreToolUse payload {"tool_input":{"file_path":"<repo>/CLAUDE.md"}} into
  the hook; expect JSON with permissionDecision "allow" + non-empty
  additionalContext; exit 0. Repeat for the auto-memory MEMORY.md.
- Pipe a payload for a .py file; expect empty stdout, exit 0.
- Run tests/test_canonical_read_guard_hook.py; expect pass.
- Negative check: make the guard always-silent; the test fails. Restore.
- Validate .claude/settings.json parses and contains the PreToolUse/Read entry.
