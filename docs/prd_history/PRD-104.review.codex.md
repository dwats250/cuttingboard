## MISSED RISKS

**Risk 1 — `logs/latest_run.json` vs `logs/latest_hourly_run.json` path split is not documented in AC or VALIDATION.**
Claude added `logs/latest_run.json` to AC #5's artifact inventory list, which is correct. However, the revised PRD does not add an explicit AC or VALIDATION grep requiring the document to explain the run-path split analogously to the payload-path split (new AC #8). `runtime.py:165` defines `LATEST_RUN_PATH = LOGS_DIR / "latest_run.json"` and `runtime.py:166` defines `LATEST_HOURLY_RUN_PATH = LOGS_DIR / "latest_hourly_run.json"`. The dashboard renderer reads `logs/latest_run.json` by default (`dashboard_renderer.py:37: _RUN_PATH = Path("logs/latest_run.json")`), and the hourly workflow passes `--run logs/latest_hourly_run.json` to override. This is structurally identical to the payload split Claude documented. An implementer who documents the payload split but not the run split will produce a document with a systematic gap. **Fix:** Add AC analogous to revised AC #8: "docs/artifact_flow_map.md documents that logs/latest_run.json is the non-hourly path (renderer default) and logs/latest_hourly_run.json is the hourly path, and that the dashboard renderer defaults to the former but is overridden in the hourly workflow."

**Risk 2 — `reports/output/report.html` listed in revised inventory but absent from AC #5 and VALIDATION greps.**
Claude's revised PRD adds `reports/output/report.html` (transport default HTML, `transport.py:15: _DEFAULT_HTML_PATH = "reports/output/report.html"`) to the inventory list prose but does not add it to AC #5's grep-verifiable artifact list, and does not add a VALIDATION grep command for it. An implementer who passes all ACs and VALIDATION commands could still omit this artifact from the document. **Fix:** Add `reports/output/report.html` to AC #5's artifact list and to the VALIDATION grep block.
