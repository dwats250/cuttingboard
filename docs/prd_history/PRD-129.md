PRD-129 — CI Artifact Hygiene and Push-Guard Stability

STATUS
IN PROGRESS

LANE
MICRO

CLASS
INFRA + PATCH

WHY NOW
Every recent `Cuttingboard Pipeline` run has failed silently for hours
— first on a ruff lint regression introduced by PRD-128, then on the
`tools/ci_push_artifacts.sh` "dirty tree before rebase" guard tripping
on untracked `engine_doctor.txt` / `engine_doctor.json` left in the
working tree by the Engine health check step (`PRD-020`). The lint
break was patched in commit `5d9be6d` to unblock today's dashboard
refresh attempt, which surfaced the push-guard defect. This PRD
canonicalizes both fixes and adds a guardrail so transient CI outputs
cannot dirty the tree again.

MAX EXPECTED DELTA
≤ 10 production-code lines, confined to `.gitignore` (append-only) and
`tests/test_ci_artifact_hygiene.py` (new file). No edits permitted to
`tools/ci_push_artifacts.sh`, `.github/workflows/cuttingboard.yml`,
`cuttingboard/`, contracts, payloads, dashboard renderer, notifications,
market_map, OHLCV/cache/fixture code, or trading logic.

GOAL
Restore green `Cuttingboard Pipeline` runs end-to-end (lint → tests → render
→ commit → push) by ensuring generated/transient CI artifacts cannot
leave the tree in a state the push guard treats as dirty.

SCOPE
- Add `engine_doctor.txt` to `.gitignore`. `engine_doctor.json` is
  already covered transitively by the existing `*.json` rule
  (`.gitignore` line 51), so no new entry is required for it; R1's
  `check-ignore` assertion will still succeed for both filenames.
- Retain `5d9be6d`'s test-only lint fixes (E402 `# noqa` on intentional
  section-divider imports in `tests/test_dashboard_renderer.py`,
  removed unused `pytest` in `tests/test_runtime_trend_structure_refresh.py`)
  by reference; no further changes required to those files.
- Add a deterministic regression test asserting `engine_doctor.json`
  and `engine_doctor.txt` are matched by `.gitignore` patterns.

OUT OF SCOPE
- No changes to `cuttingboard/runtime.py` or any module imported by it.
- No changes to artifact contracts, payload schema, dashboard renderer,
  notification formatters, market_map, OHLCV/cache/fixture code, or
  decision logic.
- No changes to `tools/engine_doctor.py` itself.
- No changes to CI workflow YAML beyond what is required by the FILES
  list below (currently: none).
- No "while-we're-here" cleanup of unrelated ruff warnings, dead
  imports, or other test hygiene issues.

FILES
M .gitignore
A tests/test_ci_artifact_hygiene.py

ROOT CAUSE
Hidden dependency. `tools/ci_push_artifacts.sh` (introduced by
PRD-100-PATCH, commit `9403428`) treats any non-empty `git status --short`
output as dirty, on the assumption that all upstream CI steps either
commit or remove their working-tree outputs. The Engine health check
step (PRD-020, commit `0472cfd`) writes `engine_doctor.json` and
`engine_doctor.txt` to the repo root for artifact upload but never
cleans them up and they are not gitignored, leaving the tree dirty by
the push guard's definition. Neither PRD made the dependency on the
other explicit, and the guard's behavior was masked for weeks by the
PRD-128 lint regression failing earlier in the job.

CHANGE SURFACE
- `.gitignore`: append-only; add two literal filename entries under a
  clearly labeled CI section. No reordering, no removals.
- `tests/test_ci_artifact_hygiene.py`: new file; one deterministic test
  asserting `.gitignore` patterns match both filenames using
  `git check-ignore`.

REQUIREMENTS

R1 — Engine doctor outputs are gitignored
`.gitignore` MUST contain entries that cause `git check-ignore
engine_doctor.json` and `git check-ignore engine_doctor.txt` to exit
with status 0 (matched).

FAIL: `git check-ignore -q engine_doctor.json` exits non-zero, OR
`git check-ignore -q engine_doctor.txt` exits non-zero.

R2 — Push guard tolerates engine doctor outputs
After running `tools/engine_doctor.py --json --baseline tools/baseline.json
> engine_doctor.json` and `tools/engine_doctor.py --baseline tools/baseline.json
> engine_doctor.txt 2>&1` in a clean working tree, `git status --short`
MUST produce empty output (the exact predicate evaluated by
`tools/ci_push_artifacts.sh`).

FAIL: `git status --short` returns any non-empty output after the above
commands run in an otherwise clean tree.

R3 — Regression test exists
A new test file `tests/test_ci_artifact_hygiene.py` MUST exist and MUST
contain tests that programmatically verify BOTH:
  (a) `git check-ignore` returns zero exit status for `engine_doctor.json`
      AND `engine_doctor.txt` (pattern-coverage assertion); and
  (b) after creating both files in the repo root (within a tmp_path-style
      cleanup contract that restores the tree), `git status --porcelain`
      produces empty output for those paths (push-guard-predicate
      assertion).

FAIL: `tests/test_ci_artifact_hygiene.py` does not exist, OR pytest
collection finds zero tests in that file, OR the file is missing either
the check-ignore assertion or the status-porcelain assertion described
above.

R4 — No scope creep
The diff for this PRD's implementation commit MUST modify only the two
files listed in `FILES`. No edits to `cuttingboard/`, workflows,
`tools/ci_push_artifacts.sh`, `tools/engine_doctor.py`, contracts,
payloads, renderers, notifications, or any other path.

FAIL: `git show --name-only <impl-commit>` lists any path outside
`{.gitignore, tests/test_ci_artifact_hygiene.py}`.

DATA FLOW
Engine health check step → writes `engine_doctor.{json,txt}` →
`.gitignore` excludes them from `git status --short` →
`tools/ci_push_artifacts.sh` sees clean tree → rebase + push succeed.

FAIL CONDITIONS
- `git check-ignore -q engine_doctor.json` exits non-zero.
- `git check-ignore -q engine_doctor.txt` exits non-zero.
- `git status --short` reports `engine_doctor.*` as untracked after the
  engine doctor step runs in an otherwise clean tree.
- `tests/test_ci_artifact_hygiene.py` is absent or fails.
- The implementation commit modifies any file outside `FILES`.

VALIDATION
Manual:
1. Run `git check-ignore -v engine_doctor.json && git check-ignore -v
   engine_doctor.txt`. Expected: both print a `.gitignore` match line
   and exit 0.
2. Run `touch engine_doctor.json engine_doctor.txt && git status --short`.
   Expected: empty output.
3. Run `python3 -m pytest tests/test_ci_artifact_hygiene.py -q`.
   Expected: all tests pass.
4. Run `python3 -m pytest tests -q`. Expected: 2268 passing + however
   many tests R3 adds (≥ 1), 0 failures.
5. Dispatch `gh workflow run "Cuttingboard Pipeline" --ref main`,
   wait for completion. Expected: all steps green including the final
   Push step.
6. `rm engine_doctor.json engine_doctor.txt` to clean up post-validation.
