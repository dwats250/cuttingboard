PRD-010 — Continuation Rejection Audit and Threshold Calibration

STATUS
COMPLETE
Commit: de6c0a6a

GOAL
Enforce single-reason rejection taxonomy, remove non-deterministic None returns, and add continuation audit aggregation to QualificationSummary.

SCOPE
- Enforce first-failure semantics for continuation gates inline
- Remove None returns; all paths return deterministic outcomes
- Add continuation_audit aggregation to QualificationSummary
- Add DATA_INCOMPLETE tracking for missing OHLCV
- Add CONTINUATION_HOLD_CANDLES and CONTINUATION_MAX_EXTENSION_ATR constants
- Add runtime audit logging and output integration for EXPANSION mode

OUT OF SCOPE
- No changes to the vote model or regime detection
- No changes to hard gate logic (gates 1–4)

FILES
M cuttingboard/config.py
M cuttingboard/qualification.py
M cuttingboard/output.py
M cuttingboard/runtime.py

REQUIREMENTS

R1 — Single Rejection Reason
Each continuation candidate MUST be rejected with exactly one reason from the canonical taxonomy.
No candidate path MAY return None.

FAIL: A continuation candidate produces no rejection reason or returns None.

R2 — First-Failure Semantics
Continuation gates MUST evaluate in order and stop at first failure.
No gate MAY run after a prior gate has failed.

FAIL: A candidate accumulates multiple rejection reasons in one evaluation pass.

R3 — Continuation Audit
QualificationSummary MUST include continuation_audit: dict mapping rejection reason → count.
EXPANSION mode MUST always surface the audit block in output.

FAIL: EXPANSION run output contains no continuation_audit block.

R4 — DATA_INCOMPLETE Tracking
Missing OHLCV MUST record DATA_INCOMPLETE in continuation_audit, not raise an exception.

FAIL: Missing OHLCV raises an unhandled exception instead of recording DATA_INCOMPLETE.

DATA FLOW
continuation candidate → first-failure gate evaluation → single rejection reason → continuation_audit aggregation → QualificationSummary → output block

FAIL CONDITIONS
- None returned by any continuation path
- Multiple reasons assigned to one candidate
- EXPANSION output missing audit block
- Missing OHLCV raises exception

VALIDATION
Run: pytest -q
Manual: EXPANSION fixture with OHLCV missing — confirm DATA_INCOMPLETE in audit block, no exception.
