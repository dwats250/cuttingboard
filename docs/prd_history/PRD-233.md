CONTRACT

WHY NOW
The strongest invariant surface in the codebase (assert_valid_contract,
contract.py) is called only by tests, and runtime mutates the contract
after build (outcome, system_state.{outcome,permission,reason,
session_type}, artifacts.notification_sent) with no guard — mentor-review
bug #2. Every later typing step (J1/J2) leans on this being live.

MAX EXPECTED DELTA
~35 production LOC across contract.py + runtime/__init__.py; two test
files. No behavior change for valid contracts.

GOAL
Every runtime injection becomes part of the validator's declared schema,
and the validator runs at contract finalization in _run_pipeline — a
corrupt contract raises before any artifact is written.

SCOPE
- contract.py: assert_valid_contract gains (a) a system_state key
  whitelist — 7 builder keys (router_mode, market_regime, intraday_state,
  time_gate_open, tradable, stay_flat_reason, confidence) + 4 declared
  runtime injections (outcome, permission, reason, session_type) — any
  other key fails; (b) a `finalized: bool = False` keyword — when True,
  additionally require top-level `outcome`, system_state.{outcome,
  permission, reason} present, and artifacts.notification_sent present.
- runtime/__init__.py: call `assert_valid_contract(contract,
  finalized=True)` in _run_pipeline immediately after the last mutation
  (artifacts.notification_sent) and BEFORE write_audit_record / report
  builders / PipelineResult return. A failure propagates to execute_run's
  handler, which writes the minimal ERROR contract (fail-loud HALT) — the
  corrupt contract never reaches latest_contract.json.
- Red tests first: unit (missing permission under finalized=True fails;
  undeclared system_state key fails in both modes) + pipeline-level
  (corrupted system_state in a mocked _run_pipeline raises AssertionError
  and writes no audit record).

OUT OF SCOPE
- The hourly contract path (_build_hourly_contract) — separate finalize
  surface with its own injections; follow-up PRD after this pattern
  proves out (declared deferral, not an oversight).
- The `macro_drivers == {}` early-return semantics inside the validator
  (pre-existing; system_state checks are placed BEFORE it so they always
  run, but macro/regime/candidate checks keep their current skip).
- TypedDict adoption (J1) and any contract shape changes.

REQUIREMENTS

R1 — system_state key whitelist
assert_valid_contract fails on any system_state key outside the declared
11, in every mode, for every contract status (checked before the ERROR /
empty-macro early returns).
FAIL: a contract carrying system_state key "wibble" passes
assert_valid_contract.

R2 — finalized mode
With finalized=True the validator additionally requires top-level
`outcome`, system_state outcome/permission/reason, and
artifacts.notification_sent. Default (finalized=False) keeps current
builder-level semantics so existing builder tests stay meaningful.
FAIL: a finalized contract missing system_state.permission passes; or an
un-finalized builder contract (no runtime injections) fails under the
default mode.

R3 — live in _run_pipeline before artifact writes
_run_pipeline validates the finalized contract before write_audit_record
and before returning; a corrupt contract raises, no audit record is
appended, and execute_run's existing handler produces the ERROR contract.
FAIL: a pipeline run whose contract carries an undeclared system_state
key completes and appends an audit record.

assert_valid_contract(finalized=True) → write_audit_record → reports →
