# PRD-072 — Macro Drivers Snapshot Fallback

**Status:** COMPLETE
**Commit:** 7e14a4e
**Depends on:** PRD-044 (macro_drivers in contract), PRD-027 (dashboard renderer)

---

## GOAL

Persist the most recent non-empty `macro_drivers` dict to `logs/macro_drivers_snapshot.json` after each run that produces live data, so the dashboard renderer can display current macro readings even when the active contract has no data (Sunday mode, off-peak, STAY_FLAT without quotes).

---

## SCOPE

- Write `logs/macro_drivers_snapshot.json` in `runtime.py` whenever `contract["macro_drivers"]` is non-empty
- Dashboard renderer reads snapshot as fallback when `payload["macro_drivers"]` is empty
- Snapshot file format: `{"macro_drivers": {...}, "generated_at": "<iso>"}` 
- Renderer CLI accepts `--macro-snapshot` path (default: `logs/macro_drivers_snapshot.json`)
- Snapshot fallback is silent — no error if file is absent

---

## OUT OF SCOPE

- No changes to contract schema or validation
- No changes to payload structure or `assert_valid_payload`
- No staleness badge or "as-of" annotation on the dashboard
- No changes to notification routing
- No changes to regime, qualification, or execution pipeline

## REQUIREMENTS

### R1 — Snapshot write

In `runtime.py`, after `_write_payload_artifacts(contract)` is called, if `contract["macro_drivers"]` is non-empty, write `logs/macro_drivers_snapshot.json` with:

```json
{
  "macro_drivers": { ...contract["macro_drivers"]... },
  "generated_at": "<contract generated_at ISO string>"
}
```

Write is atomic (write to `.tmp` then rename) to prevent a partial file from being read by the renderer.

FAIL: snapshot file written when `contract["macro_drivers"]` is `{}`.
FAIL: snapshot file not written when `contract["macro_drivers"]` is non-empty.
FAIL: write failure raises to caller — must be caught and logged, never propagated.

### R2 — Snapshot load

In `dashboard_renderer.py`, define `_load_macro_snapshot(path: Path) -> dict` that:

- Returns `{}` if file does not exist
- Returns `{}` if file is not valid JSON
- Returns `{}` if parsed content lacks a `"macro_drivers"` key
- Returns `snapshot["macro_drivers"]` (a dict) on success

FAIL: function raises on missing file.
FAIL: function raises on malformed JSON.

### R3 — Renderer fallback

In `render_dashboard_html` (or the renderer `main`), after reading `macro_drivers` from the payload:

```python
if not macro_drivers:
    macro_drivers = _load_macro_snapshot(snapshot_path)
```

`snapshot_path` defaults to `Path("logs/macro_drivers_snapshot.json")`. When the renderer is invoked via CLI, it accepts `--macro-snapshot <path>` to override the default.

FAIL: fallback applied when payload macro_drivers is non-empty.
FAIL: renderer raises when snapshot file is absent.
FAIL: renderer raises when snapshot contains malformed data.

### R4 — Snapshot not committed by CI

The snapshot path `logs/macro_drivers_snapshot.json` must be excluded from git tracking. Add it to `.gitignore` if not already covered by the `logs/` entry.

FAIL: snapshot file appears as a tracked git artifact.

## DATA FLOW

```
runtime.py
  contract["macro_drivers"] non-empty
    → _write_macro_snapshot(contract, LOGS_DIR / "macro_drivers_snapshot.json")
    → logs/macro_drivers_snapshot.json written atomically

dashboard_renderer.py (CI render step)
  payload["macro_drivers"] == {}  (Sunday / off-peak run)
    → _load_macro_snapshot(snapshot_path)
    → macro_drivers = last known good data
    → tape slots and pressure block render with real values
```

---

## FAIL CONDITIONS

- Snapshot written when macro_drivers is empty
- Snapshot not written when macro_drivers is non-empty
- Write failure propagates to caller instead of being caught and logged
- Renderer raises on missing snapshot file
- Renderer raises on malformed snapshot JSON
- Fallback applied when current payload has non-empty macro_drivers
- Snapshot appears as a tracked git file
