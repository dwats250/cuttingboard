PRD-011 — Freeze Canonical Pipeline Output Contract

STATUS
COMPLETE
Commit: e6896f39

GOAL
Introduce a frozen contract schema that all pipeline outputs must conform to, enforced by a dedicated contract module and test suite.

SCOPE
- Add cuttingboard/contract.py: frozen dataclass schema for the pipeline output contract
- Update output.py to produce contract-conformant output
- Update runtime.py to validate contract on every run
- Add tests/test_contract.py with full schema coverage

OUT OF SCOPE
- No changes to pipeline logic
- No changes to ingestion or validation layers

FILES
A cuttingboard/contract.py
M cuttingboard/output.py
M cuttingboard/runtime.py
A tests/test_contract.py

REQUIREMENTS

R1 — Contract Schema
contract.py MUST define a frozen dataclass covering all pipeline output fields.
No field MUST be added to the contract without a corresponding test.

FAIL: contract.py contains a non-frozen dataclass.

R2 — Runtime Validation
runtime.py MUST validate the contract on every live run.
Contract validation failure MUST halt the run with a HALT output.

FAIL: A live run completes with an invalid contract and no HALT is issued.

R3 — Test Coverage
tests/test_contract.py MUST cover all contract fields with at least one passing and one failing case.

FAIL: A contract field has no test.

DATA FLOW
pipeline output → contract.py schema → runtime validation → HALT on violation | continue on pass

FAIL CONDITIONS
- Non-frozen contract dataclass
- Live run accepts invalid contract
- Contract field untested

VALIDATION
Run: pytest tests/test_contract.py -q
Confirm: 570 tests in test_contract.py, all passing.
