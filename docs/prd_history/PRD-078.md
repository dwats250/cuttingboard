# PRD-078 ‚Äî Dashboard Demo Candidate Fixture Mode

**STATUS:** COMPLETE

---

## GOAL

Enable deterministic population of A+/A/B candidate cards for dashboard
visualization without affecting real trading logic or contract outputs.

---

## SCOPE

- Add optional fixture mode toggle (env var `FIXTURE_MODE=1`)
- When enabled:
  - substitute fixture symbol entries for `market_map.symbols` in renderer
  - `payload.meta.fixture_mode = true`
  - dashboard shows `"DEMO MODE ‚Äî FIXTURE DATA"` banner
- Fixture data defined as a hardcoded dict in a new module

---

## OUT OF SCOPE

- No randomness
- No modification of contract
- No modification of decision logic
- No evaluation logging
- No persistence of fake data to disk
- No notification changes
- No modification of `runtime.py` pipeline

---

## FILES

- `cuttingboard/delivery/fixtures.py` (new ‚Äî fixture symbol data)
- `cuttingboard/delivery/payload.py`
- `cuttingboard/delivery/dashboard_renderer.py`
- `tests/test_fixture_mode.py`

---

## REQUIREMENTS

### R1 ‚Äî Toggle

- Env var `FIXTURE_MODE=1` activates fixture mode
- Read at the call site in `runtime.py` where `_write_payload_artifacts` and
  `_write_hourly_artifacts` are called: pass `fixture_mode: bool` as a
  parameter down to `build_report_payload` and `render_dashboard_html`
- Default: OFF (`FIXTURE_MODE` unset or `0`)
- **FAIL:** fixture mode activates without `FIXTURE_MODE=1`

---

### R2 ‚Äî Payload Flag Only

In `build_report_payload` (`payload.py`):
- Accept `fixture_mode: bool = False` parameter
- If `fixture_mode`: `payload["meta"]["fixture_mode"] = True`
- If not `fixture_mode`: do not add `fixture_mode` key to meta

Must NOT modify: contract, `market_map`, evaluation pipeline.

- **FAIL:** contract contents differ when fixture mode is enabled

---

### R3 ‚Äî Deterministic Fixture Data

Define `FIXTURE_SYMBOLS: dict[str, dict]` in `cuttingboard/delivery/fixtures.py`.
Minimum 3 entries with grades `A+`, `A`, `B`.

Each entry must conform to market_map symbol schema:
```python
{
    "symbol": str,
    "grade": "A+" | "A" | "B",
    "bias": "LONG" | "SHORT",
    "structure": str,
    "setup_state": str,
    "trade_framing": {
        "entry": str,
        "if_now": str,
        "downgrade": str,
    },
    "invalidation": [str],
    "reason_for_grade": str,
}
```

Values hardcoded ‚Äî no runtime computation.

- **FAIL:** fixture output changes between identical runs

---

### R4 ‚Äî Renderer Substitution

In `render_dashboard_html` (`dashboard_renderer.py`):
- Accept `fixture_mode: bool = False` parameter
- If `fixture_mode`:
  - replace `market_map["symbols"]` with `FIXTURE_SYMBOLS` before candidate
    board rendering (in-memory only ‚Äî `market_map.json` on disk unchanged)
  - render banner at top of candidate board section:
    `"DEMO MODE ‚Äî FIXTURE DATA"`
- If not `fixture_mode`: no change

- **FAIL:** banner absent when `fixture_mode=True`
- **FAIL:** banner present when `fixture_mode=False`
- **FAIL:** live `market_map.symbols` visible when `fixture_mode=True`

---

### R5 ‚Äî Isolation Guarantee

- `logs/latest_hourly_contract.json`: unchanged
- `logs/evaluation.jsonl`: unchanged
- `market_map.json` on disk: unchanged

- **FAIL:** fixture symbols appear in any artifact file

---

### R6 ‚Äî Determinism

Same env ‚Üí same fixture data rendered.

- **FAIL:** fixture output differs between runs

---

## DATA FLOW

```
FIXTURE_MODE env var
  ‚Üí read once at runtime call site
  ‚Üí passed as fixture_mode bool to build_report_payload
      ‚Üí payload["meta"]["fixture_mode"] = True
  ‚Üí passed as fixture_mode bool to render_dashboard_html
      ‚Üí market_map["symbols"] substituted with FIXTURE_SYMBOLS (in-memory only)
      ‚Üí "DEMO MODE" banner injected
  ‚Üí contract, market_map.json, evaluation.jsonl: untouched
```

---

## ACCEPTANCE CRITERIA

- `FIXTURE_MODE=1`: dashboard shows 3 fixture candidate cards + DEMO banner
- `FIXTURE_MODE` unset: normal behavior, no banner
- Contract file identical in both modes
- `market_map.json` identical in both modes

---

## VALIDATION

- Unit: `build_report_payload(contract, fixture_mode=True)` ‚Üí `meta.fixture_mode == True`
- Unit: `build_report_payload(contract, fixture_mode=False)` ‚Üí `"fixture_mode" not in meta`
- Unit: `render_dashboard_html(payload, run, fixture_mode=True, ...)` contains `"DEMO MODE"`
- Unit: `render_dashboard_html(payload, run, fixture_mode=False, ...)` does not contain `"DEMO MODE"`
- Unit: contract dict identical before/after fixture mode toggle
- Snapshot: fixture candidate cards render with correct grades
