# PRD-068 - Invalidation and Exit Guidance Layer

**Status:** IN PROGRESS
**Commit:** --
**Depends on:** PRD-067

---

## GOAL

Use the trade thesis invalidation field from PRD-067 to produce deterministic exit or avoidance guidance when a candidate trade thesis is weakened, conflicted, or invalidated before entry.

---

## SCOPE

- Add deterministic invalidation evaluation for trade candidates.
- Use existing thesis data from PRD-067.
- Produce invalidation_guidance fields in the contract-level candidate dict.
- Convert ALLOW_TRADE decisions with triggered invalidation to BLOCK_TRADE via dataclasses.replace().
- Preserve existing TradeDecision enum values.
- Preserve execution policy structure.
- Keep output deterministic and testable.

---

## OUT OF SCOPE

- No broker integration.
- No live position management.
- No P/L calculation.
- No account-size logic.
- No dashboard/UI changes.
- No notification changes.
- No new market data sources.
- No LLM commentary.
- No automatic order generation.
- No mutation of market_map generation.
- No modification of decisions that are already BLOCK_TRADE before this gate.

PRD-067. Thesis data and apply_thesis_gate must be complete.

## REQUIREMENTS

### R1 - Exit guidance object shape

Each candidate with a thesis receives invalidation_guidance with:

- status: one of NOT_TRIGGERED, WARNING, TRIGGERED, UNKNOWN
- action: one of HOLD_OK, AVOID_ENTRY, REDUCE_OR_EXIT, NO_ACTION, UNKNOWN
- reason: str or null
- triggered_by: str or null
- thesis_status: str, copied from thesis.status

FAIL: any key absent.
FAIL: status or action outside allowed sets.

### R2 - Candidate trade avoidance

If a candidate would be ALLOW_TRADE and invalidation_guidance.status == TRIGGERED:

- Produce replacement TradeDecision via dataclasses.replace() with:
  - status = BLOCK_TRADE
  - block_reason = "INVALIDATION_TRIGGERED"
  - decision_trace = {"stage": "INVALIDATION_GATE", "source": "invalidation", "reason": "INVALIDATION_TRIGGERED"}
  - policy_allowed = False
  - policy_reason = "invalidation_gate_blocked"

Only ALLOW_TRADE decisions are converted. Decisions already BLOCK_TRADE are not modified.

FAIL: gate modifies a decision that was already BLOCK_TRADE.
FAIL: block_reason != decision_trace["reason"] after gate.

### R3 - Exit guidance for already-allowed candidates

If a candidate remains ALLOW_TRADE and invalidation_guidance.status == WARNING:

- Decision remains unchanged.
- action is REDUCE_OR_EXIT or NO_ACTION based on deterministic rule.
- reason is populated.

FAIL: WARNING status changes a decision to BLOCK_TRADE.

### R4 - No active-position assumption

Do not infer that an ALLOW_TRADE candidate is an open position.

If no active position source exists, output is guidance only.

FAIL: guidance assumes or infers an open position.

### R5 - Deterministic trigger sources

Invalidation status is derived from these existing fields only:

- thesis.status in {INCOMPLETE, CONFLICTED} -> TRIGGERED
- thesis.block_reason set and non-null -> WARNING
- Candidate block_reason contains a risk/stop failure reason if already blocked -> informational only, not a re-block
- overall_pressure conflicts with candidate direction -> WARNING

If none of these inputs are present, status is UNKNOWN.

FAIL: INVALIDATED used as a thesis status check. INVALIDATED is not a valid PRD-067 status value.
FAIL: trigger derived from any source not listed above.

### R6 - UNKNOWN handling

UNKNOWN status must not block a trade.

UNKNOWN produces:
- action: UNKNOWN
- reason: "INSUFFICIENT_DETERMINISTIC_INPUTS"
- triggered_by: null

FAIL: UNKNOWN invalidation status blocks a trade.

### R7 - Block reason exclusivity

Gate blocking sets block_reason = "INVALIDATION_TRIGGERED" as the sole reason for newly blocked decisions. Decisions already BLOCK_TRADE before this gate are not touched.

FAIL: gate overwrites block_reason on a decision that was already BLOCK_TRADE.

### R8 - Contract validation

contract.py:_build_trade_candidates merges invalidation_guidance_map.get(symbol, None) into the candidate dict as key "invalidation_guidance". Missing value must produce "invalidation_guidance": null, not raise.

Historical candidates without invalidation_guidance remain readable.

FAIL: contract builder raises when invalidation_guidance_map is absent or a symbol is missing.

### R9 - Runtime ordering

This gate runs after PRD-067 apply_thesis_gate and before build_pipeline_output_contract.

Gate function signature:

```python
def apply_invalidation_gate(
    trade_decisions: list[TradeDecision],
    thesis_map: dict[str, dict],
    overall_pressure: str,
) -> tuple[list[TradeDecision], dict[str, dict]]:
    ...
```

Returns (decisions, invalidation_guidance_map).

FAIL: gate runs before apply_thesis_gate.

### R10 - No enum drift

Do not add new TradeDecision enum values.

## DATA FLOW

```
runtime.py
  trade_decisions post apply_thesis_gate
  thesis_map from apply_thesis_gate
  overall_pressure
    -> apply_invalidation_gate(...)
    -> trade_decisions, invalidation_guidance_map

contract.py
  _build_trade_candidates(...)
    -> per candidate: candidate["invalidation_guidance"] = invalidation_guidance_map.get(symbol)
```

## FAIL CONDITIONS

- Gate modifies a decision already BLOCK_TRADE.
- block_reason != decision_trace["reason"] after gate.
- INVALIDATED used as a thesis status check.
- UNKNOWN invalidation blocks a trade.
- Historical artifact read fails due to missing invalidation_guidance.
