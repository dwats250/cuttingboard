WHY NOW
Master-plan J2 + M-map design: the contract consumers still declare dict; SCHEMA_MAP still poses as the contract schema (contract_types.py is now the artifact); the renderer decomposition needs its ordered seam map before Opus executes K/L/M.

MAX EXPECTED DELTA
~15 annotation lines (imports + signature/local types) across six
consumer modules. Zero executable-behavior delta. Plus two docs: the
SCHEMA_MAP contract-section rewrite and the NEW renderer decomposition
map (design only, no renderer restructuring).

GOAL
Every module that consumes the contract declares PipelineContract at its
seam (completing J1's producer-side adoption), SCHEMA_MAP stops
duplicating the now-typed schema, and the renderer decomposition has an
ordered, reviewable seam map for the post-window K/L/M work.

SCOPE
- Adopt PipelineContract at every remaining contract-consuming seam:
  notifications/state.py (notification_state_key,
  classify_notification_priority), output.py
  (build_notification_message), overnight_policy.py
  (apply_overnight_policy param/return), reports/premarket.py
  (build_premarket_report), reports/postmarket.py
  (build_postmarket_report), and the renderer's single direct contract
  read (dashboard_renderer._load_contract_entry_context's parsed local).
- Retire SCHEMA_MAP.md's contract field-lookup role: the contract
  section becomes a pointer at cuttingboard/contract_types.py, KEEPING
  only what the TypedDicts intentionally leave dict[str, Any]
  (macro_drivers per-driver internals) and consumer-semantics notes.
  The payload / market_map / run sections stay — those artifacts are
  not typed.
- A docs/renderer_decomposition_map.md design doc (M-map): RenderContext
  shape, kwarg->section matrix for render_dashboard_html, and the
  ordered extraction seams sized for post-window execution. Design
  only — no renderer code is restructured under this PRD.

OUT OF SCOPE
- Any renderer decomposition EXECUTION (K/L/M) — the map is the
  deliverable, the cuts are post-window Opus work under their own PRDs.
- Typing the payload / market_map / run artifacts (SCHEMA_MAP keeps
  those sections precisely because they remain untyped).
- A mypy CI gate (unchanged follow-up option from PRD-237).
