PRD-113 — PRD Governance Hardening

STATUS
COMPLETE

GOAL
Formalize the PRD process — class-driven inference, forbidden patterns,
change-surface limits, failure taxonomy — so future PRDs are evaluated
against a deterministic governance contract. Governance-only.

GUIDING PRINCIPLE
Governance must minimize reviewer cognitive load while maximizing drift
resistance. Any field that becomes routinely repetitive, mechanically
inferable, or cargo-culted MUST migrate from per-PRD declaration into
process-level convention (the CLASS matrix in `docs/PRD_PROCESS.md`).
Author assertions are reserved for decisions that cannot be inferred
from CLASS, FILES, or existing governance docs. Prospective only;
existing template fields are not retroactively trimmed by this PRD.

SCOPE
- Extend `docs/PRD_TEMPLATE.md` with three mandatory header fields and
  an optional CHANGE SURFACE section.
- Extend `docs/PRD_PROCESS.md` with: six-class base + PATCH overlay,
  CLASS matrix (incl. HIGH-RISK FILES column), forbidden patterns,
  stability tiers (process-level only), failure taxonomy, governance
  compression rule, adjudication trigger.

OUT OF SCOPE
- Any runtime, contract, payload, schema, sidecar, dashboard, or
  notification behavior change.
- Retroactive re-classification of PRD-001..PRD-112.
- Tooling/lint to enforce new fields.
- Modifications to `docs/PRD_MICRO_TEMPLATE.md`.
- Retroactive trimming of existing template fields.

FILES
M docs/PRD_TEMPLATE.md
M docs/PRD_PROCESS.md

REQUIREMENTS

R1 — Six Base Classes + PATCH Overlay
Closed base set: `GOVERNANCE | SIDECAR | CONSUMER | EXECUTION | CONTRACT
| INFRA`. PATCH is an overlay carrying `ROOT CAUSE` ∈ {missing fail
condition, ambiguous requirement, hidden dependency}; not a peer class.
FAIL: PRD_PROCESS.md lists PATCH as a base class, omits any of the six
base classes, or omits ROOT CAUSE overlay metadata.

R2 — Mandatory Header Fields (Three Only)
PRD_TEMPLATE.md mandatory header between STATUS and GOAL: exactly
`CLASS`, `WHY NOW`, `MAX EXPECTED DELTA`. STABILITY TIER, ARCHITECTURAL
OWNER, REQUIRED REVIEWERS MUST NOT appear as mandatory header fields.
FAIL: Mandatory header is not exactly these three fields.

R3 — CLASS Matrix with HIGH-RISK FILES
PRD_PROCESS.md CLASS matrix MUST have five columns per base class:
(a) default stability tier, (b) required reviewers, (c) validation
depth, (d) forbidden mutation surfaces, (e) HIGH-RISK FILES.
FAIL: Matrix omits any base class or any column.

R4 — CHANGE SURFACE (Conditional)
Optional `CHANGE SURFACE` section after FILES in template. Mandatory
iff (i) CLASS default tier is T0/T1, OR (ii) any FILES entry matches
HIGH-RISK FILES for that class.
FAIL: CHANGE SURFACE missing, or the (i)/(ii) trigger absent.

R5 — Stability Tiers (Process-Level; Override Tagged)
Define exactly four tiers: `T0 core contracts/runtime | T1 sidecars/
evaluation | T2 consumers/dashboard | T3 docs/tests/process`. Default
per CLASS is authoritative; reviewer override only in review artifact,
tagged `factual drift`. NOT a mandatory header field.
FAIL: Fewer than four tiers, template introduces STABILITY TIER as
mandatory header, or override mechanism unspecified.

R6 — Forbidden Patterns Catalog
Closed set with one-line definitions: `hidden coupling`, `payload
mutation`, `renderer-derived semantics`, `threshold→label synthesis`,
`wall-clock dependence`, `partial malformed recovery`, `silent
fallback`, `sidecar mutation`, `while-we're-here mutations`.
FAIL: Any of the nine missing or undefined.

R7 — Review Failure Taxonomy
Closed taxonomy for `*.review.*.md`: `hidden coupling | schema
ambiguity | consumer contamination | ownership violation |
non-determinism | derived semantics | stale-data ambiguity | scope
creep | factual drift`.
FAIL: Taxonomy missing or under nine labels.

R8 — Binding MAX EXPECTED DELTA
Implementation exceeding the declared ceiling MUST stop, amend the PRD
(revising ceiling + reason), and re-trigger review per matrix before
resuming.
FAIL: PRD_PROCESS.md describes MAX EXPECTED DELTA as advisory or omits
stop-and-amend rule.

R9 — Adjudication Trigger
CONTRACT-class PRDs require adjudication artifact on any reviewer
disagreement. All other classes follow CLAUDE.md cross-review gate
(adjudication only when unresolved disagreement remains).
FAIL: Adjudication rule omits CONTRACT-stricter clause or contradicts
CLAUDE.md.

R10 — Governance Compression Rule (Prospective)
PRD_PROCESS.md MUST include the GUIDING PRINCIPLE text verbatim and
state that any future proposal to add a per-PRD mandatory field MUST
first justify why it cannot be inferred from CLASS, FILES, or the
matrix. Prospective only.
FAIL: Compression rule, inference-first gate, or prospective clause
missing.

R11 — Governance-Only Guarantee
Diff modifies only the two FILES (registry/state on close excepted).
FAIL: `git diff main...HEAD --name-only` shows any path outside FILES
beyond registry/state on close.

DATA FLOW
Author declares CLASS + WHY NOW + MAX EXPECTED DELTA → reviewer applies
CLASS matrix to derive reviewers, validation depth, forbidden surfaces,
leakage guards → CHANGE SURFACE supplied iff R4 trigger fires →
findings tagged with failure taxonomy → adjudication per R9 → registry
close.

FAIL CONDITIONS
See per-requirement FAIL lines (R1–R11). No additional conditions.

VALIDATION
Manual:
- PRD_TEMPLATE.md: exactly three mandatory header fields between STATUS
  and GOAL; CHANGE SURFACE section after FILES; no STABILITY TIER /
  ARCHITECTURAL OWNER / REQUIRED REVIEWERS as mandatory.
- PRD_PROCESS.md: six base classes + PATCH overlay; matrix with all
  five columns including HIGH-RISK FILES; four-tier list with override
  clause; nine forbidden patterns; nine-label taxonomy; binding MAX
  EXPECTED DELTA; CONTRACT adjudication clause; prospective compression
  rule.
- Matrix reviewer column does not contradict CLAUDE.md cross-review gate.
- `git diff main...HEAD --name-only` shows only the two FILES (plus
  registry/state on close).
- Draft throwaway skeleton: confirm CLASS + WHY NOW + MAX EXPECTED
  DELTA is sufficient author input; reviewers and forbidden surfaces
  fall out of the matrix.
