# PRD-088 — Candidate Board Level Diagram Price Fallback

**Status:** IN PROGRESS
**Owner:** Codex

---

## GOAL

Fix Candidate Board cards that show "Chart unavailable — no price data" even when the
same tradable symbol has a valid current_price in the Macro Tape.

The Candidate Board level diagram currently depends only on
latest_hourly_contract.json trade_candidates[*].entry. Market map symbols with valid
current_price but no contract entry render as unavailable. This PRD adds a
renderer-only fallback so the level diagram can use market_map.symbols[sym].current_price
when contract_entry is missing.

---

## SCOPE

1. Update cuttingboard/delivery/dashboard_renderer.py only.
2. In _render_candidate_card(), keep contract_entry as the preferred level diagram anchor.
3. If contract_entry is None, use entry.get("current_price") as the fallback anchor.
4. Pass the resolved anchor into _render_level_diagram().
5. Preserve _render_level_diagram() behavior: if the resolved anchor is None or <= 0,
   the existing unavailable message remains.
6. Preserve existing Candidate Board layout, grade rendering, lifecycle rendering, bias,
   structure, reason, watch zones, and fib level rendering.
7. Preserve Macro Tape rendering unchanged.
8. Preserve _build_contract_entry_map() unchanged.
9. Add or update tests so:
   - a candidate card with no contract_entry but valid current_price renders a level
     diagram instead of "Chart unavailable — no price data"
   - a candidate card with no contract_entry and no valid current_price still renders
     the unavailable message
   - contract_entry remains preferred over current_price when both exist

## REQUIREMENTS

### R1 — Anchor resolution in _render_candidate_card()

Resolve the level diagram anchor using this precedence:
- contract_entry if contract_entry is not None
- entry.get("current_price") otherwise

Pass the resolved anchor to _render_level_diagram().

FAIL: _render_level_diagram() is called with raw contract_entry instead of the resolved anchor.

### R2 — contract_entry remains preferred

If contract_entry is not None, current_price must not override it.

FAIL: A card with both contract_entry and current_price uses current_price as the anchor.

### R3 — current_price fallback renders diagram

If contract_entry is None and current_price is a positive number, the card must not
render "Chart unavailable — no price data".

FAIL: "Chart unavailable" appears in a card that has a valid current_price fallback.

### R4 — Unavailable message preserved when both anchors invalid

If contract_entry is None and current_price is None or <= 0, the card must still render
the existing unavailable message.

FAIL: "Chart unavailable" does not appear when both anchors are absent or non-positive.

### R5 — No side effects

Macro Tape output, _build_contract_entry_map(), market_map schema, and all unrelated
renderer sections must be unchanged.

FAIL: Any diff appears in Macro Tape output, contract map logic, or unrelated renderer sections.

## IMPLEMENTATION NOTES

This PRD repairs a renderer data-source mismatch only.

**Before:**
- Macro Tape showed tradable prices from market_map.symbols[sym].current_price.
- Candidate Board level diagrams ignored current_price and required
  latest_hourly_contract.json trade_candidates[*].entry.
- Market-map candidates without contract entries displayed "Chart unavailable — no price data."

**After:**
- Contract entry remains authoritative when present.
- current_price becomes a renderer-only fallback anchor.
- Candidate Board cards with valid market_map prices can render level diagrams.
- Cards with no valid contract_entry or current_price still show the unavailable message.

**Exact change location:**
- File: cuttingboard/delivery/dashboard_renderer.py
- Function: _render_candidate_card() — lines 683–688
- Change: replace `contract_entry` argument with
  `contract_entry if contract_entry is not None else entry.get("current_price")`
