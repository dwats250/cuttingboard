PRD-114 — Watchlist Snapshot Sidecar

STATUS
IN PROGRESS

CLASS
SIDECAR

WHY NOW
PRD-107 established the trend_structure sidecar pattern; PRD-110/111
hardened curated-symbol governance and sidecar doctrine; PRD-112/113
validated read-only consumption and governance inference. PRD-114 is
the first deliberate exercise of the SIDECAR class governance model
under PRD-113 (CLASS inference, CHANGE SURFACE, MAX EXPECTED DELTA,
forbidden-pattern inheritance, failure taxonomy).

MAX EXPECTED DELTA
- ≤ 5 production files modified or added.
- ≤ 150 production LOC across builder + runtime hook (excluding tests
  and docs).
- 0 lines changed in: contract.py, output.py, qualification.py,
  trade_decision.py, trade_policy.py, regime.py, execution_policy.py,
  delivery/dashboard_renderer.py, notifications/, ui/.

GOAL
Add a deterministic observe-only Watchlist Snapshot sidecar that
persists a curated, discretionary symbol index to
`logs/watchlist_snapshot.json` for downstream visibility, without
mutating decision logic, payload, contract, dashboard, notifications,
or any pipeline-owned artifact.

SCOPE
- New pure builder module: `cuttingboard/watchlist_sidecar.py`.
- One deterministic record per curated watchlist symbol.
- Frozen curated universe defined as a module-level constant inside
  the builder (not config.py).
- Persist to `logs/watchlist_snapshot.json` via a runtime helper
  mirroring `_write_trend_structure_snapshot()` (atomic
  `.tmp → replace`).
- Pass-through `current_price` from existing `normalized_quotes`
  produced upstream of qualification; null when absent.
- Static metadata fields (`sector_theme`, `watch_reason`) sourced
  exclusively from the frozen curated universe constant.
- Unit tests for: deterministic output, missing-quote handling, schema
  shape, frozen-universe equality, no-I/O behavior, no-now() behavior.
- Add producer/consumer row to `docs/artifact_flow_map.md`.

OUT OF SCOPE
- Any derived/dynamic field (vwap relation, sma relations,
  relative_volume, trend_alignment, entry_context, event_risk).
  These duplicate `trend_structure_snapshot` and are deferred to a
  future CONSUMER-class PRD that joins watchlist ⨝ trend_structure
  read-only.
- Earnings calendar integration, news ingestion, automatic symbol
  discovery, ranking, scoring, AI-generated commentary.
- Any change to dashboard, notifications, payload, contract, or
  market_map.
- Expansion of `config.ALL_SYMBOLS` or any tradable universe constant.
- Any consumer of `watchlist_snapshot.json` in this PRD.

REQUIREMENTS

R1 — Frozen Curated Universe
The watchlist universe is a module-level immutable constant in
`watchlist_sidecar.py` of the form
`tuple[tuple[str, str, str], ...]` (symbol, sector_theme,
watch_reason). Initial set:
SPY (Index, broad market reference);
QQQ (Index, tech-heavy reference);
GDX (Commodities, gold miners exposure);
GLD (Commodities, spot gold ETF);
SLV (Commodities, spot silver ETF);
XLE (Commodities, energy sector);
NVDA (High beta, AI/semis bellwether);
TSLA (High beta, retail-flow signal);
META (High beta, large-cap tech);
AMZN (High beta, large-cap tech);
AAPL (High beta, large-cap tech).
FAIL: Constant is mutable, missing any of these eleven symbols, or
contains any symbol not in this list.

R2 — Pass-Through Current Price Only
For each watchlist symbol, `current_price = normalized_quotes[symbol]
.price` if present, else `null`. No fetching, no fallback,
no synthesis from OHLCV.
FAIL: Builder reads any input other than `normalized_quotes` and the
frozen universe constant; or substitutes a non-null default when a
quote is missing.

R3 — No Derived Semantics
Output record contains exactly these keys:
`symbol, sector_theme, watch_reason, current_price`.
No `vwap_relation`, `sma_*_relation`, `trend_alignment`,
`entry_context`, `relative_volume`, `event_risk`, `data_status`,
`reason`, or any label synthesized from thresholds.
FAIL: Output record contains any key outside the four listed.

R4 — Artifact Schema (Top-Level)
Top-level keys exactly: `schema_version` (integer literal `1`),
`source` (string literal `"watchlist"`), `generated_at` (ISO-8601 UTC
string with tzinfo or `null`), `symbols` (object keyed by symbol).
Set of `symbols` keys equals the frozen universe set.
FAIL: Any top-level key missing or extra; `symbols` key set differs
from the frozen universe; `schema_version != 1`; `source !=
"watchlist"`.

R5 — Determinism and No Wall-Clock Dependence
Builder accepts `generated_at: datetime | None` and never calls
`datetime.now()`, `time.time()`, or equivalent. Naive datetime input
raises `ValueError`. Symbol order in `symbols` follows insertion order
of the frozen constant.
FAIL: Source contains `datetime.now`, `time.time`, or `time.monotonic`;
naive datetime does not raise; two calls with identical inputs produce
non-identical output bytes after stable JSON serialization.

R6 — No I/O in Builder
Builder performs no file reads, no network calls, no logging beyond
in-memory return value. All persistence is performed by the runtime
helper.
FAIL: Builder imports `requests`, `urllib`, `httpx`, `pathlib.Path
.write_*`, `pathlib.Path.read_*`, `open(`, or any module under
`cuttingboard.transport` / `cuttingboard.delivery`.

R7 — Runtime Ordering and Isolation
On non-HALT hourly runs, `_write_watchlist_snapshot()` is invoked
immediately after `_write_trend_structure_snapshot()` and before
returning success. On HALT runs, behavior is governed by R11. The
snapshot is NOT read by qualification, contract, payload, dashboard,
notifications, or any other sidecar in this PRD.
FAIL: Any reference to `watchlist_snapshot` or `watchlist_sidecar`
in `cuttingboard/contract.py`, `cuttingboard/output.py`,
`cuttingboard/qualification.py`, `cuttingboard/trade_decision.py`,
`cuttingboard/trade_policy.py`, `cuttingboard/execution_policy.py`,
`cuttingboard/regime.py`, `cuttingboard/delivery/`,
`cuttingboard/notifications/`, or `ui/`.

R8 — Artifact Flow Map Row
`docs/artifact_flow_map.md` gains exactly one new row declaring
`cuttingboard/watchlist_sidecar.py` as the sole producer of
`logs/watchlist_snapshot.json` with consumer set `(none)` for v1.
FAIL: Row missing, or producer/consumer columns disagree with this
PRD.

R9 — Forbidden Pattern Adherence (PRD-113 Inheritance)

FAIL: Implementation introduces any pattern from the catalog;

R10 — Scope Lock

R11 — HALT Skip
If `validation_summary.system_halted` is true on the hourly runtime
path, `_write_watchlist_snapshot()` MUST NOT execute. Any existing
`logs/watchlist_snapshot.json` MUST be left byte-for-byte untouched
(no write, no truncate, no replace, no delete, no `.tmp` sidecar
file).

Placement and guard semantics (binding): direct inspection of
`cuttingboard/runtime.py::_run_pipeline()` confirms that
`_write_trend_structure_snapshot()` sits inside the hourly artifact
block (`if notify_mode in _HOURLY_MODES:`), which is a SIBLING of the
`if not validation_summary.system_halted:` branch — NOT nested inside
it. The hourly artifact block therefore executes on HALT runs.

Because branch inheritance is unavailable, the watchlist call site
MUST be guarded by an explicit `if not validation_summary.system_halted:`
check at the call site. This guard is load-bearing, not defensive
noise: it is the only mechanism preventing a HALT-run watchlist
write. Removal or weakening of the guard is a direct R11 violation.

The watchlist call site MUST be located immediately after
`_write_trend_structure_snapshot()` inside the same hourly artifact
block, wrapped by the explicit HALT guard.

Authoritative pass/fail evidence is a content-hash + size + existence
check, not filesystem mtime. mtime MAY be recorded as supplementary
diagnostic but MUST NOT be the gating signal.

FAIL (any one of the following on a run with
`validation_summary.system_halted == True`):
- `logs/watchlist_snapshot.json` is created when it was absent before
  the run; OR
- the SHA-256 content hash of `logs/watchlist_snapshot.json` differs
  before vs. after the run; OR
- the byte size of `logs/watchlist_snapshot.json` differs before vs.
  after the run; OR
- `logs/watchlist_snapshot.json.tmp` exists at any point during or
  after the run; OR
- `logs/watchlist_snapshot.json` is deleted by the run.

R12 — Static-Only `watch_reason` Fence
`watch_reason` is repository-defined static curation metadata sourced
exclusively from the third element of the frozen
`WATCHLIST_SYMBOLS` tuple. It MUST NOT be computed, derived, or
otherwise influenced by price, volume, trend, calendar, news,
thresholds, market state, sidecar outputs, model output, or any
runtime observation. If this guarantee cannot be cleanly enforced
in implementation, `watch_reason` MUST be removed from v1 and
deferred to PRD-115+.
FAIL: `watch_reason` value for any symbol differs across runs with
identical frozen-constant input, OR `watchlist_sidecar.py` reads any
symbol attribute beyond the frozen constant when constructing the
field, OR the field's value at runtime is not byte-equal to the
constant's third tuple element.

R13 — No Trend Read Coupling
`_write_watchlist_snapshot()` and `build_watchlist_snapshot()` MUST
NOT read `logs/trend_structure_snapshot.json`, MUST NOT reference
`TREND_STRUCTURE_PATH`, and MUST NOT consume the trend builder's
output (in-memory dict, return value, module attribute, or any
intermediary). Temporal adjacency after the trend_structure write is
allowed; data dependency is not.
FAIL: `cuttingboard/watchlist_sidecar.py` or the watchlist runtime
helper contains the substrings `trend_structure`, `TREND_STRUCTURE`,
`trend_structure_snapshot`, or imports any symbol from
`cuttingboard.trend_structure`.

R14 — Serialization-Only Ordering
The insertion order of `WATCHLIST_SYMBOLS` is serialization-only. It
MUST NOT imply rank, priority, conviction, trade preference, alert
order, or execution preference, in this PRD or in any future
consumer. The PRD docstring at the top of `watchlist_sidecar.py`
MUST state this explicitly.
FAIL: Source comments, docstrings, field names, or test fixtures in
the FILES set describe the order as rank, priority, conviction,
preference, or any synonym; OR a future consumer PRD cites
PRD-114's order as semantic without amending PRD-114 first.
