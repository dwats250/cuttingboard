   any realpath that contains `/.claude/` AND ends with `/memory/MEMORY.md`
   (verified: `/some/repo/.claude/memory/MEMORY.md` warns even though it is not
   under `.claude/projects/<slug>/memory/`, which is the real injected path).
   Impact is negligible and bounded - the worst case is one redundant,
   non-blocking, allow-decision reminder on an unusual MEMORY.md - so this is not
   blocking. If tightening is ever wanted, anchor on `{os.sep}.claude{os.sep}
   projects{os.sep}` rather than just `{os.sep}.claude{os.sep}`. Documenting the
   heuristic nature in CLAUDE_HOOKS.md would also suffice.
2. Optional: the relative-path case (`file_path: "CLAUDE.md"`) correctly warns
   because the hook is invoked with cwd == repo root via the settings.json
   `cd "${CLAUDE_PROJECT_DIR:-.}"` wrapper, and realpath resolves against cwd.
   This is fine in practice; no change needed, just noted as load-bearing on the
   wrapper's cd.
