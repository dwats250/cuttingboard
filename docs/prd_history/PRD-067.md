# PRD-067 - Trade Thesis Gate

**Status:** IN PROGRESS
**Commit:** --
**Depends on:** PRD-063

---

## GOAL

Require every candidate that would be ALLOW_TRADE to have a deterministic trade thesis with explicit catalyst, direction, confirmation, and invalidation fields before the trade remains allowed.

---

## SCOPE

- Add a deterministic trade thesis builder.
- Build thesis data only from existing pipeline inputs already available at runtime.
- Attach thesis data to the contract-level trade candidate dict.
- Convert ALLOW_TRADE decisions with incomplete or conflicted thesis to BLOCK_TRADE via dataclasses.replace().
- Preserve HALT behavior.
- Preserve existing TradeDecision enum values.
- Preserve existing execution policy semantics for decisions that would be BLOCK_TRADE before this gate runs.

---

## OUT OF SCOPE

- No broker integration.
- No account-size logic.
- No position sizing changes.
- No dashboard/UI changes.
- No LLM-generated commentary.
- No narrative trade recommendations.
- No new market data sources.
- No changes to market_map generation.
- No changes to macro_pressure.py output contract.
- No grade-based gating unless grade is already present in the candidate scope.
- No new TradeDecision enum values.
- No changes to existing block reasons on decisions that are already BLOCK_TRADE before this gate.

---

PRD-063, macro pressure available in runtime.

## REQUIREMENTS

### R1 - Thesis object shape

Each ALLOW_TRADE candidate receives a thesis dict with:

- symbol: str
- direction: str
- catalyst: str or null
- confirmation: str
- invalidation: str or null
- status: one of VALID, INCOMPLETE, CONFLICTED, UNKNOWN
- block_reason: str or null

FAIL: any key is absent from the thesis dict.
FAIL: status contains any value outside the allowed set.

### R2 - Direction source

direction is sourced from TradeDecision.direction, which is already set before this gate runs.

If direction is missing or empty, status must be INCOMPLETE.

FAIL: direction derived from any source other than the existing TradeDecision or QualificationResult field.

### R3 - Catalyst source

Catalyst is derived from existing deterministic context only. Allowed sources:

- overall_pressure string already computed in runtime
- QualificationResult.entry_mode if present
- StructureResult.structure for the candidate symbol if available in execution_structure
- TradeDecision.policy_reason if it contains a non-default, non-empty reason

If no deterministic catalyst can be derived from these sources, catalyst is null and status is INCOMPLETE.

FAIL: catalyst invented from any source not listed above.

### R4 - Confirmation source

confirmation is a deterministic string derived from existing candidate context. Use the first applicable:

- MACRO_CONFIRMED if overall_pressure aligns with candidate direction
- STRUCTURE_CONFIRMED if structure classification is not CHOP
- POLICY_CONFIRMED if policy_reason indicates an affirmative policy pass
- UNKNOWN if none of the above apply

FAIL: confirmation is null. It must always have a value.
FAIL: UNKNOWN confirmation alone blocks the trade.

### R5 - Invalidation required

invalidation is derived from:

- TradeDecision.stop if non-zero, preferred source formatted as "stop at {stop:.2f}"
- TradeCandidate.stop_price if the candidate is accessible
- TradeDecision.block_reason if already blocked for a risk-related reason

If no deterministic invalidation can be derived, invalidation is null and status is INCOMPLETE.

FAIL: ALLOW_TRADE decision has invalidation == null.

### R6 - Thesis blocking rule

This gate runs after apply_execution_policy_to_decisions and before contract assembly. It only converts decisions that would be ALLOW_TRADE.

If a decision is ALLOW_TRADE and thesis.status is INCOMPLETE or CONFLICTED:

- Produce a replacement TradeDecision via dataclasses.replace() with:
  - status = BLOCK_TRADE
  - block_reason = "THESIS_INCOMPLETE" for INCOMPLETE or "THESIS_CONFLICTED" for CONFLICTED
  - decision_trace = {"stage": "THESIS_GATE", "source": "trade_thesis", "reason": same as block_reason}
  - policy_allowed = False
  - policy_reason = "thesis_gate_blocked"

Decisions already BLOCK_TRADE before this gate are not modified. Only ALLOW_TRADE to BLOCK_TRADE conversion is performed here.

FAIL: an ALLOW_TRADE decision with INCOMPLETE thesis passes contract validation.
FAIL: the gate modifies any decision that was already BLOCK_TRADE.
FAIL: block_reason != decision_trace["reason"] after gate application.

### R7 - UNKNOWN handling

UNKNOWN status is allowed only when:

- symbol exists and is non-empty
- direction exists and is non-empty
- invalidation is non-null
- no deterministic conflict is detected

UNKNOWN must not block the trade.
UNKNOWN must not be silently upgraded to VALID.

FAIL: UNKNOWN thesis blocks an ALLOW_TRADE decision.

### R8 - Historical artifact compatibility

assert_valid_contract runs on newly built contracts only. Historical JSONL records that lack thesis on trade candidates are not re-validated and remain readable.

Contract validation must not require thesis on candidates produced before this PRD path.

FAIL: reading a historical audit record raises due to missing thesis field.

### R9 - Runtime ordering

Gate function signature:

```python
def apply_thesis_gate(
    trade_decisions: list[TradeDecision],
    candidates: dict[str, TradeCandidate],
    qualified_by_symbol: dict[str, QualificationResult],
    execution_structure: dict[str, StructureResult],
    overall_pressure: str,
) -> tuple[list[TradeDecision], dict[str, dict]]:
    ...
```

Returns (decisions, thesis_map) where thesis_map is {symbol: thesis_dict}.

Called in runtime.py after apply_execution_policy_to_decisions and before build_pipeline_output_contract.

If overall_pressure is "UNKNOWN", for example macro pressure failed, thesis uses UNKNOWN macro context without halting.

FAIL: thesis gate runs before apply_execution_policy_to_decisions.
FAIL: macro pressure unavailability causes a run halt.

### R10 - Consumer audit

contract.py:_build_trade_candidates merges thesis_map.get(symbol, None) into the candidate dict as key "thesis". Missing thesis must produce "thesis": null, not raise.

No dashboard or notification consumer changes are required.

FAIL: contract builder raises when thesis_map is absent or a symbol is not in thesis_map.

---

## DATA FLOW

```
runtime.py
  trade_decisions post apply_execution_policy_to_decisions
  candidates, qualified_by_symbol, execution_structure, overall_pressure
    -> apply_thesis_gate(...)
    -> trade_decisions, thesis_map
  trade_decisions = replacement decisions, some may be newly BLOCK_TRADE

contract.py
  _build_trade_candidates(...)
    -> per candidate: candidate["thesis"] = thesis_map.get(symbol)
```

---

## FAIL CONDITIONS

- ALLOW_TRADE candidate with null invalidation passes contract build.
- Gate modifies a decision that was already BLOCK_TRADE.
- block_reason != decision_trace["reason"] after gate.
- Thesis gate runs before execution policy.
- THESIS_INVALID used as block_reason. Use THESIS_INCOMPLETE or THESIS_CONFLICTED only.
- Historical audit record read fails due to missing thesis.
