PRD-009 — Canonical Timezone Handling and Time Gate Validation

STATUS
COMPLETE
Commit: f4ddb677

GOAL
Centralize all timezone operations into a single module and wire ET-aware time gate validation through the qualification pipeline.

SCOPE
- Add cuttingboard/time_utils.py with canonical time functions
- Replace LATE_SESSION_CUTOFF tuple with ENTRY_CUTOFF_ET time(15,30) in config.py
- Refactor _is_late_session to use time_utils
- Thread now_et through qualify_all → qualify_candidate
- Convert report timestamps from UTC to ET in output.py
- Add tests/test_time_utils.py and tests/test_time_gate.py

OUT OF SCOPE
- No changes to regime or ingestion layers
- No new timezone libraries

FILES
A cuttingboard/time_utils.py
M cuttingboard/config.py
M cuttingboard/qualification.py
M cuttingboard/output.py
A tests/test_time_utils.py
A tests/test_time_gate.py

REQUIREMENTS

R1 — Time Utils Module
time_utils.py MUST export: get_now_utc, get_now_et, convert_utc_to_et, is_after_entry_cutoff, is_market_open.
No other module MAY contain inline ZoneInfo or timezone arithmetic.

FAIL: Any cuttingboard module other than time_utils.py contains ZoneInfo instantiation.

R2 — Entry Cutoff Config
LATE_SESSION_CUTOFF tuple MUST be replaced by ENTRY_CUTOFF_ET = time(15, 30) in config.py.

FAIL: LATE_SESSION_CUTOFF tuple exists in config.py after merge.

R3 — now_et Threading
qualify_all() MUST accept now_et as a parameter and pass it to qualify_candidate.
qualify_candidate MUST use the passed now_et for the TIME gate, not call time_utils directly.

FAIL: qualify_candidate calls get_now_et() internally instead of using the passed argument.

R4 — Report Timestamp
output.py MUST render report timestamps in ET, not UTC.

FAIL: Report timestamp displays UTC offset.

DATA FLOW
get_now_et() → qualify_all(now_et=) → qualify_candidate(now_et) → TIME gate
UTC timestamp → convert_utc_to_et() → ET display in report

FAIL CONDITIONS
- Inline ZoneInfo in any module besides time_utils.py
- TIME gate uses wrong timezone
- Report shows UTC timestamp

VALIDATION
Run: pytest tests/test_time_utils.py tests/test_time_gate.py -q (8 cases each)
Manual: confirm report timestamp shows ET offset.
