GOAL
Establish the binding cut-line design, the mechanically-derived
public-surface contract, the two-class monkeypatch-preservation rule
(covering collaborators AND path constants), an ordered-sublayer
(acyclic) package model, and a dependency-valid leaf-first roadmap of
HIGH-RISK extraction PRDs that will split cuttingboard/runtime.py (2289
LOC) into a runtime/ package without changing any runtime behavior — so
future notify-path PRDs edit a focused module instead of the monolith.

REQUIREMENTS

R1 — Ordered-sublayer package shape (acyclic, cluster→cluster forbidden)
The end-state is a `cuttingboard/runtime/` package whose modules are
assigned to ordered sublayers; a module may import (intra-package) only
from STRICTLY-LOWER sublayers. Same-sublayer (cluster→cluster) and
upward imports are forbidden.

  L0  _constants (consts + `_is_fixture_backed`), _types, _timeutil
      (leaf; no intra-pkg import)
  L1  ids_and_time, intraday_permission, fixtures        (import L0 only)
  L2  artifacts (→ ids_and_time, L0), summary (→ ids_and_time, L0),
      failure_handling (→ _timeutil, L0)
  L3  hourly (→ summary, _timeutil, L0)
  L4  __init__  (orchestrators + externally-imported collaborators +
                 module path constants + the re-export facade)

Cycle resolution (r2): `_is_fixture_backed` is a pure predicate over the
MODE_* constants with 7 callers across every layer (runtime.py:705, 1499,
1559, 1598, 2016, 2127, 2140) — it is placed in L0 `_constants`, not L1.
`_deterministic_run_at` (runtime.py:2126) calls `_load_fixture_quotes`
(fixtures), so it is placed in fixtures, not ids_and_time. With those two
moves the only ids_and_time↔fixtures edge disappears, so both sit at L1
importing L0 only. Verified L2/L3 edges driving the order:
`summary._data_status` → `ids_and_time._quote_age_seconds`
(runtime.py:2022); `artifacts` → `ids_and_time._attach_generation_id_to_payload`
(runtime.py:1883, 2002); `hourly._build_hourly_run_summary` →
`summary._summary_regime_fields` / `summary._data_status` +
`_timeutil._iso_z` (runtime.py:1791, 1847, 1821-1824);
`failure_handling._failure_summary` → `_timeutil._iso_z`
(runtime.py:2186-2187).

FAIL: a layering guard test (maps each runtime submodule to its sublayer
and scans its intra-package imports) finds any import from a module in an
equal or higher sublayer — including any cluster→cluster edge.

R2 — Mechanically-derived public-surface contract
The contract set S is every name referenced as a `cuttingboard.runtime`
attribute anywhere in `cuttingboard/` or `tests/` — via
`from cuttingboard.runtime import <name>`, `runtime.<name>`/`rt.<name>`,
or `setattr(runtime|rt, "<name>", …)` / `monkeypatch.setattr(
"cuttingboard.runtime.<name>", …)`. PRD-A adds a guard test that computes
S by static scan and asserts every name in S is an attribute of
`cuttingboard.runtime`. S is NOT a hand-maintained list. (S includes
names omitted from the first draft, e.g. `_hourly_rr` imported at
tests/test_hourly_alert.py:23, plus `execute_run`, `verify_run_summary`,
`safe_write_latest`, `_failure_summary`, the `_write_*` writers, and the
path constants `LOGS_DIR`/`REPORTS_DIR`/`MARKET_MAP_PATH`/`WATCHLIST_PATH`/
`LATEST_*`.)

FAIL: the guard test encodes a hand-maintained list instead of the
scanned set, or any name in S is not importable as an attribute of
`cuttingboard.runtime` after an extraction.

R3 — Two-class patch-preservation rule (functions AND constants) + audit
The suite patches the runtime namespace in two ways; a re-export facade
preserves only the first.
  Class (a) — import-time `from cuttingboard.runtime import X`. The import
    binds the object; the `__init__` re-export preserves it with zero
    churn regardless of which submodule defines X.
  Class (b) — attribute patch `setattr(cuttingboard.runtime, "X", …)`
    (incl. monkeypatch.setattr / mock.patch on that dotted path), for X a
    function OR a module path constant. This intercepts a read/call ONLY
    if the consuming code resolves X in the `__init__` namespace at
    access time. A re-export does NOT make this true; module-reference
    calling (`artifacts.X()`) does NOT either — `artifacts.X` is a
    different binding from `cuttingboard.runtime.X`.
The doctrine:
  (i) The orchestrators (`_run_pipeline`, `_execute_notify_run`,
      `execute_run`, `cli_main`, `execute_prefetch`) and every collaborator
      and constant THEY look up — external imports (`fetch_all`,
      `send_notification`, `compute_regime`, `generate_candidates`,
      `qualify_all`, `create_trade_decision`, `write_audit_record`,
      `should_send`, `save_last_state`, …) and path constants read by
      __init__-resident code — REMAIN bound in `__init__`. Their class-(b)
      patches are preserved with zero churn (verified:
      tests/test_hourly_alert.py:976 `fetch_all`, :983 `send_notification`,
      :847 `LOGS_DIR`).
  (ii) For a function OR constant patched class-(b) via the runtime facade
      that is consumed by RELOCATED cluster code (so the facade patch
      cannot intercept the internal access), the extraction PRD MUST
      migrate that test's patch target to the definition module
      (`cuttingboard.runtime.<cluster>.X`, or
      `cuttingboard.runtime._constants.<CONST>` for a path constant) and
      list the migration. Relocated cluster modules read path constants
      via `_constants` module-reference so a single migrated patch
      intercepts. (Verified at-risk: functions `_load_inputs` /
      `_fixture_validation_clock` tests/test_runtime_decision.py:116-118,
      `_apply_intraday_short_permission`
      tests/test_gap_down_permission_integration.py:216; constant
      `WATCHLIST_PATH` read by `_write_watchlist_snapshot`
      runtime.py:1967,1969 and patched at
      tests/test_watchlist_sidecar.py:216.)
Each extraction PRD records a monkeypatch-target audit: for every moved
symbol AND every path constant the moved code reads, classify each patch
site (a) or (b), and for (b) state "preserved-in-__init__" or
"migrated-to-<module>".

FAIL: an extraction relocates a facade-patched function/constant consumed
by relocated code without migrating that patch target (suite red), or
omits the monkeypatch-target audit.

R4 — Executable per-extraction behavior-preservation gate
Every extraction PRD's acceptance gate is:
  (i)  `python3 -m pytest tests -q` pass-count equals the pre-extraction
       baseline recorded in that PRD.
  (ii) the R2 surface guard test and the R1 layering guard test are green.
  (iii) Artifact byte-identity: with the extraction PRD's pinned fixture
       (default `tests/fixtures/2026-04-12.json`), run
       `python3 -m cuttingboard --mode fixture --fixture-file <F>` then
       `python3 -m cuttingboard.delivery.dashboard_renderer --output
       ui/dashboard.html && cp ui/dashboard.html ui/index.html`, captured
       once before and once after the extraction; `git diff --exit-code`
       must be empty for the set {ui/dashboard.html, ui/index.html,
       logs/latest_payload.json, logs/latest_contract.json,
       logs/latest_run.json, logs/market_map.json}.
  (iv) the R3 monkeypatch-target audit is present in the PRD.

FAIL: pass-count differs from the recorded baseline, either guard test is
red, `git diff --exit-code` is non-empty for any listed artifact under
the pinned fixture, or the audit is absent.

R5 — Complete cut-line map (every symbol assigned exactly once)
All 67 top-level symbols are assigned to exactly one sublayer destination
in the CUT-LINE MAP under DATA FLOW; none is unassigned or double-listed.

FAIL: any top-level def/class in cuttingboard/runtime.py is absent from

R6 — Dependency-valid leaf-first roadmap
The roadmap places the zero-move package-skeleton PRD first, then extracts
in an order where no module is extracted before a module it imports from
(a topological order over R1's sublayers).

FAIL: the skeleton PRD is not first, or a module is sequenced before any
module it imports from.

R7 — Design-only scope guard
No commit attributed to PRD-170 modifies any cuttingboard/*.py file.

FAIL: `git show` for any PRD-170 commit includes a change to a file under
cuttingboard/.
