WHY NOW
The DECISIONS 2026-07-02 deferral gated the risk band on "the engine emitting a
numeric stop for an independent reason"; recon found that condition ALREADY MET
(`trade_candidates[].stop` from `TradeDecision`, sizing-grade, validated finite
for ALLOW_TRADE) — the band is now the "near-free renderer addition" the
deferral predicted, and Dustin has commissioned it as PRD-223.

MAX EXPECTED DELTA
45 production LOC, all in cuttingboard/delivery/dashboard_renderer.py.

GOAL
The candidate card's level ladder shades the zone between the contract's numeric
entry and stop so a trader reads how much room there is before the thesis is
wrong as a to-scale shape, not prose — descriptive only (band = entry→stop; no
targets, no R:R).

SCOPE
- `_load_contract_entry_context` additionally captures `cand["stop"]` (finite,
  > 0) into a `stop_map`, parallel to the existing entry map; returned alongside
  the existing tuple members.
- `render_dashboard_html` gains keyword-only `contract_stop_map: dict | None =
  None`; `write_dashboard` and `main` gain the same pass-through
  parameter/threading. Contract staleness is carried by the card-level pair
  gate (below), NOT by a parallel stop-map nulling: the existing
  `contract_stale_for_run` branch nulls the entry map, so a stale stop can
  never pair up — a separate nulling line proved unobservable (no red test
  could fail on it) and was cut per invariant 4.
- `_render_candidate_card` gains `contract_stop: float | None = None` and passes
  it to the diagram ONLY when the anchor is the contract entry (a stop without
  its entry never draws).
- `_render_level_diagram` gains `contract_stop: float | None = None`. When
  present: the stop joins the y-scale domain; a soft translucent rect (the risk
  zone) spans anchor-y→stop-y behind the level lines; the stop edge renders as a
  dashed red line with a `STOP <price> <±pct>` label routed through the existing
  declutter pass. Zone shading, not a crisp hairline, per the DECISIONS
  second-order caution.
- Guards: stop must be non-boolean (rejected BEFORE coercion — float(True) is
  1.0, a masquerading "price"; Codex P2 on PR #89), float-coercible, finite,
  > 0, and distinct from entry (equal prices → no band, no STOP label).
  Enforced at both the loader (file path) and the diagram (direct-map path).

OUT OF SCOPE
- Emitting numeric `entry_level` / `invalidation_level` companions in
  `market_map.py` next to the prose. Recon killed this on realizability: BOTH
  the entry prose (`_trade_framing`, "hold above {nearest}") and the
  invalidation prose (`_invalidation`, "loses {nearest} with weak recovery")
  reference the SAME `watch_zones[0]` zone, so the numeric pair is identical on
  every directional card — a zero-height band, the exact "always-empty band"
  channel PRD-222 removed. A dead channel with extra steps; excluded per the
  realizability check. Reopen only if the engine ever distinguishes the entry
  referent zone from the invalidation referent zone.
- Profit targets, R:R ratios, or any use of `trade_candidates[].target` /
  `risk_reward` (VISION description-not-prediction line).
- Any change to `cuttingboard/contract.py`, `cuttingboard/market_map.py`, or
  any payload/contract key. The stop already exists and is already validated
  (`_assert_trade_candidates_valid`: float + finite for ALLOW_TRADE).
- Bands for cards without a contract candidate (no honest numeric stop exists
  there; prose stays prose).

REQUIREMENTS

R1 — Stop threads from the contract artifact
`_load_contract_entry_context` returns a stop map built from
`trade_candidates[].stop` under the same symbol/coercibility guards as the entry
map, plus finite/positive checks; the `main` → `write_dashboard` →
`render_dashboard_html` chain threads it as `contract_stop_map`.
FAIL: a contract candidate `{"symbol": "SPY", "entry": 510.0, "stop": 505.0}`
renders a card whose diagram has no `STOP 505.00` label.

R2 — Risk zone renders between entry and stop
When a card has a contract entry (the anchor) and a valid distinct contract
stop, the diagram contains a translucent rect spanning exactly
min(y_entry, y_stop)→max(y_entry, y_stop) at full LINE_W width, rendered before
(behind) the level lines, and a dashed stop line + `STOP <price> <signed pct>`
label at y_stop.
FAIL: entry 510.0 / stop 505.0 produces no rect with `opacity="0.08"`, or the
rect's y/height do not equal the mapped span, or no `STOP 505.00 -1.0%` label
appears.

R3 — Stop participates in the y-scale
A stop outside the min/max of the other levels widens the scale so its y stays
within 0..SVG_H (no off-canvas stop).
FAIL: a stop below every watch zone maps outside 0..110.

R4 — No band without an honest pair
No rect, dashed stop line, or STOP label renders when ANY of: stop is absent /
non-coercible / non-finite / ≤ 0; stop equals entry; the contract entry is
absent (anchor fell back to current_price); the contract is stale for the run
(`contract_stale_for_run` nulls the entry map, and the card-level pair gate —
stop only draws against its own entry — therefore blocks the stale stop).
FAIL: any of those inputs produces `opacity="0.08"`, a `#e05252` stroke, or a
`STOP ` label in the card.

R5 — Pinned geometry undisturbed
All existing diagram assertions hold unchanged: anchor line
`x1="0" ... x2="160" ... stroke="#f5c518"` at its unchanged y, labels on-canvas
(0..110), `NOW <price>` anchor label, % suffixes, and PRD-222's absence
assertions (`stroke="#ffffff"`, `opacity="0.06"`).
FAIL: any pre-PRD-223 test in tests/test_dash_level_diagram.py or
tests/test_dashboard_renderer.py fails.

DATA FLOW
logs/latest_hourly_contract.json trade_candidates[].stop →
_load_contract_entry_context stop_map → render_dashboard_html
contract_stop_map → _render_candidate_card contract_stop (pair-gated on the
contract entry, which staleness nulls) → _render_level_diagram → risk-zone
rect + dashed stop line + STOP label

FAIL CONDITIONS
- Contract stop present+valid+distinct but no rect/stop-line/STOP label (R1, R2).
- Rect span ≠ mapped |y_entry−y_stop| span (R2).
- Stop outside the y-scale maps off-canvas (R3).
- Band artifacts render with absent/invalid/equal stop, anchor fallback, or a
  stale contract (R4).
- Any pre-existing diagram/renderer test fails (R5).
