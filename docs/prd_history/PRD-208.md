PROVENANCE
Recon F09 (audits/recon-2026-06-24/dashboard-coherence.md), presentation-only.
Carries the SETTLED INPUT from that recon: the unavailable token must be
"UNAVAILABLE" / "Structure unavailable", NEVER "NULL" — a bare "NULL" cuts against
the decision-language principle (PRD-158). The live QQQ row that surfaced this is
diagnosed separately in audits/diag-2026-06-24/qqq-sma-history.md (a data-path
cache-truncation bug, OUT OF SCOPE here — this PRD changes only how the cell is
rendered, not whether the data resolves).

WHY NOW
The trend-structure SMA Composite cell renders verbose prose — "Above SMA50 and
SMA200" / "Below SMA50, above SMA200" (dashboard_renderer.py:93-104,
_trend_structure_composite_display:110-120). The four-questions / mobile
cognitive-compression goal is better served by a compressed arrow cell
(↑50 ↓200), and the granular per-SMA columns ("vs SMA50", "vs SMA200") then
duplicate the composite. This is the smallest presentation cut that makes the
section scannable. The unavailable rows already read "Structure unavailable"
(:105) — keep that; this PRD guards it (and the new compression path) against ever
emitting a bare "NULL"/"None".

MAX EXPECTED DELTA
One pure-function rewrite of _trend_structure_composite_display (prose → arrow
vocabulary; unavailable/insufficient/not-computed strings unchanged), plus a
render-loop edit to drop the two now-redundant granular SMA columns and update the
collapse-index set — all in dashboard_renderer.py. Tests updated to assert the
arrow cells + a red test for the NULL/old-prose guard. No data, schema, gate,
count, or regime change; no snapshot-field change.

GOAL
The trend-structure table renders SMA position as a single compressed arrow cell
per symbol (↑/↓/= against SMA50 and SMA200), the redundant granular SMA columns
are cut, and no unavailable state ever renders as "NULL"/"None" — only
"Structure unavailable" (and the existing insufficient/not-computed strings).

SCOPE
- Rewrite `_trend_structure_composite_display` (dashboard_renderer.py:110-120) and
  its display table `_TREND_STRUCTURE_COMPOSITE_DISPLAY` (:93-104) so each resolved
  (price_vs_sma_50, price_vs_sma_200) pair maps to a compressed arrow cell:
    (ABOVE, ABOVE)       → "↑50 ↑200"
    (ABOVE, BELOW)       → "↑50 ↓200"
    (BELOW, ABOVE)       → "↓50 ↑200"
    (BELOW, BELOW)       → "↓50 ↓200"
    (AT_LEVEL, ABOVE)    → "=50 ↑200"
    (AT_LEVEL, BELOW)    → "=50 ↓200"
    (ABOVE, AT_LEVEL)    → "↑50 =200"
    (BELOW, AT_LEVEL)    → "↓50 =200"
    (AT_LEVEL, AT_LEVEL) → "=50 =200"
  Unavailable precedence and strings UNCHANGED: DATA_UNAVAILABLE in pair →
  "Structure unavailable" (:105); INSUFFICIENT_HISTORY → "SMA history insufficient"
  (:106); NOT_COMPUTED → "Structure not computed" (:107). The closed-vocabulary,
  pure-function contract (PRD-131) is preserved.
- Cut the now-redundant granular columns "vs SMA50" and "vs SMA200" from the
  trend-structure table: remove them from `_ts_headers` (:2217-2221) and from the
  per-row `_cells` tuple (:2241-2252 and the None-record placeholder :2236-2239),
  and update `_ts_collapsible_cols` (:2226) and any index-based collapse logic
  (:2259-2280) to the new column positions. The "Alignment" verdict column
  (trend_alignment: BULLISH/BEARISH/MIXED) is RETAINED — it is the semantic
  summary that complements the arrow detail.
- Keep "SMA Composite" as the (now arrow) column; the header may read "SMA 50/200"
  for clarity (implementer's choice within the closed vocabulary).

DESIGN NOTES
- Arrow glyphs reuse the dashboard's existing ↑/↓ convention (monochrome); "="
  denotes AT_LEVEL. No color (color stays reserved for macro-driver rows, per
  PRD-199 precedent).
- The column-cut (R3) is separable from the compression (R1/R2): if review prefers
  to minimize render-loop risk, R3 may be deferred to a follow-up and R1/R2 land
  alone. Recorded here as the recommended full design (VISION "cuts before
  additions"); the reviewer holds that seam.
- The unavailable token "UNAVAILABLE" alternative for the Alignment column's
  DATA_UNAVAILABLE display (`_TREND_STRUCTURE_STATE_DISPLAY`:80, currently "DATA
  UNAVAILABLE") is intentionally LEFT ALONE — it is shared by other columns and is
  already not "NULL". Renaming it is out of scope (see NON-GOALS).

NON-GOALS
- No change to cuttingboard/trend_structure.py: sma_50/sma_200, price_vs_sma_*,
  trend_alignment, data_status, relative_volume, vwap fields are untouched. The
  builder still emits sma_*=null when history is short; THIS IS A RENDER CHANGE.
- No fix to the QQQ cache-truncation data bug (diag-2026-06-24/qqq-sma-history.md);
  that is a separate data-path PRD.
- No change to the global `_ts_display` / `_TREND_STRUCTURE_STATE_DISPLAY` map, the
  Intraday Context cell, the vs VWAP column, RVOL, or any non-SMA column.
- No change to gates, counts, regime logic, payload, or snapshot schema.
- No new data, no new fetch.

REQUIREMENTS

R1 — Resolved SMA position renders as a compressed arrow cell
For every resolved (price_vs_sma_50, price_vs_sma_200) pair, the SMA Composite
cell renders the arrow form in the SCOPE table (↑/↓/= + 50/200), not prose.
FAIL: the SMA Composite cell for (ABOVE, ABOVE) renders "Above SMA50 and SMA200"
(or any prose form) instead of "↑50 ↑200".

R2 — Unavailable/insufficient/not-computed strings preserved; never "NULL"/"None"
DATA_UNAVAILABLE→"Structure unavailable", INSUFFICIENT_HISTORY→"SMA history
insufficient", NOT_COMPUTED→"Structure not computed". The cell NEVER renders
"NULL", "None", "", or a prose alignment string.
FAIL (red test): a record with price_vs_sma_50/200 == "DATA_UNAVAILABLE" (sma_*
null) renders a cell containing "NULL"/"None"/""/a prose form instead of
"Structure unavailable".

R3 — Redundant granular SMA columns cut; table is internally consistent
The rendered trend-structure table no longer contains "vs SMA50"/"vs SMA200"
header columns; every body row has exactly one cell per remaining header; the
"Alignment" verdict column is retained.
FAIL: the rendered table header contains "vs SMA50" or "vs SMA200", OR a body
row's cell count differs from the header column count.

R4 — No regression outside the SMA columns
Price, Alignment, RVOL, SMA Composite (now arrows), Intraday Context, the
collapse behavior of still-present collapsible columns, and the PRD-112
all-or-nothing degrade path are unchanged for identical input.
FAIL: for identical input, any non-SMA-granular cell, the section-degrade
placeholder row, or the column-collapse outcome differs from the pre-PRD render.
