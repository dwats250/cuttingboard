# PRD-063 — Macro Pressure Execution Policy Integration

**Status:** COMPLETE
**Commit:** d47ad79

---

## GOAL

Integrate `overall_pressure` from the macro pressure snapshot into the execution policy so that macro conditions directly affect trade allowance and sizing.

---

## SCOPE

- Extend `evaluate_execution_policy` to accept and apply `overall_pressure`
- Wire `build_macro_pressure` in `runtime.py` before execution policy is called
- Apply deterministic rules to `policy_allowed`, `size_multiplier`, and `block_reason`

---

## OUT OF SCOPE

- No changes to `macro_pressure.py` or its output contract
- No changes to market_map or lifecycle
- No dashboard or UI changes
- No new signals or indicators
- No changes to `TradeDecision` enum values or fields
- No grade-based gating (grade lives in market_map, not accessible in execution policy scope)

---

## FILES

- `cuttingboard/execution_policy.py`
- `cuttingboard/runtime.py`
- `cuttingboard/contract.py`
- `tests/test_execution_policy.py`

## REQUIREMENTS

### R1 — Pressure vocab

`overall_pressure` is one of: `RISK_ON`, `RISK_OFF`, `MIXED`, `NEUTRAL`, `UNKNOWN`.
These are the only valid inputs. Any other value must raise `ValueError`.

FAIL: function accepts an unrecognized pressure string without raising.

### R2 — Pressure rules

| overall_pressure | direction | effect |
|---|---|---|
| `UNKNOWN` | any | no change |
| `NEUTRAL` | any | no change |
| `MIXED` | any | size_multiplier × 0.75 |
| `RISK_OFF` | LONG | block; reason = `macro_pressure_conflict` |
| `RISK_OFF` | SHORT | size_multiplier × 0.5 |
| `RISK_ON` | SHORT | block; reason = `macro_pressure_conflict` |
| `RISK_ON` | LONG | size_multiplier × 0.5 |

FAIL: any cell in the table produces a different output than specified.

### R3 — Composition

Macro pressure rules apply **after** all existing policy gates. If a decision is already blocked (status != ALLOW_TRADE), macro pressure rules do not run — the existing block reason is preserved.

The size_multiplier reduction from macro pressure multiplies the existing size_multiplier (i.e., `new = existing × factor`). It does not replace it.

FAIL: macro pressure check runs on a pre-blocked decision.
FAIL: size reduction replaces rather than multiplies existing multiplier.

### R4 — Function signatures

`evaluate_execution_policy` gains a new keyword-only parameter:
```python
overall_pressure: str = "UNKNOWN"
```
Default `"UNKNOWN"` ensures no behavior change when not supplied.

`apply_execution_policy` and `apply_execution_policy_to_decisions` pass `overall_pressure` through.

FAIL: existing callers that omit `overall_pressure` change behavior.

### R5 — Runtime wiring

In `runtime.py`, call `build_macro_pressure(macro_drivers=_build_macro_drivers(normalized_quotes))` before `apply_execution_policy_to_decisions`. Extract `overall_pressure` and pass it through.

If `build_macro_pressure` raises, log the error and default `overall_pressure` to `"UNKNOWN"` (do not halt the run).

FAIL: overall_pressure is not passed to execution policy.
FAIL: an exception from build_macro_pressure halts the run.

---

## DATA FLOW

```
runtime.py
  normalized_quotes
    → _build_macro_drivers()
    → build_macro_pressure()
    → overall_pressure: str
  apply_execution_policy_to_decisions(
    decisions, ..., overall_pressure=overall_pressure
  )
    → evaluate_execution_policy(decision, ..., overall_pressure=overall_pressure)
      → if already blocked: return existing block
      → apply existing gates (confidence, regime, posture, session, ORB)
      → apply macro pressure rule (R2)
      → return PolicyDecision
```

---

## FAIL CONDITIONS

- `overall_pressure` not forwarded from runtime to evaluate_execution_policy
- Macro pressure check runs on a pre-blocked decision
- `UNKNOWN` or `NEUTRAL` pressure alters size_multiplier or policy_allowed
- `MIXED` pressure blocks a trade instead of reducing size
- Size reduction replaces existing multiplier instead of multiplying it
- `build_macro_pressure` exception propagates and halts pipeline run
