WHY NOW
The macro tape shows the symptom of a shock, the red folder covers scheduled releases,
and the kill switch (PRD-180) fires only at Q4 after a price move breaches thresholds.
None of them names the cause of an unscheduled structural event (BoJ-class) at Q2, which
can matter pre-market, before or without any US move. Named failure case: a benign /
tradeable domestic tape read colliding with an overnight cataclysmic structural event.

MAX EXPECTED DELTA
One isolated producer, one dispatch-only workflow, one eval harness + corpus scaffold,
tests, and an artifact-flow registration. Zero modification to existing pipeline modules
or the renderer. Exceeding this stops implementation and re-triggers SIDECAR review.

GOAL
On manual dispatch (scheduled activation is deferred to PRD-188 so the producer never runs
on a live cadence without its consumer), produce a source-cited classification of whether
an unscheduled structural regime shock is underway (at most one event, QUIET by default),
and prove its false-positive discipline on a labeled corpus before any surface consumes
the artifact.

SCOPE
- tools/macro_awareness_collector.py: isolated producer; lazy-imports anthropic /
  feedparser; no cuttingboard import. The classifier (default claude-opus-4-8) classifies
  and selects only: it picks at most one source item and assigns an event_type; it authors
  no narrative.
- Controlled-schema artifact (see DATA FLOW): no LLM-authored label/evidence, no market
  transmission, no bias/confidence/forecast/posture.
- Deterministic novelty gate (logs/macro_awareness_state.json) and a structural-materiality
  gate; QUIET by default; fail-closed; suppression reasons logged (no silent fallback).
- Source allowlist = central-bank official channels (Fed / ECB / BoJ / BoE / PBoC) for v1;
  source_url domain must be in the allowlist. Wires are not auto-authoritative; add later
  behind the same guard.
- A dispatch-only workflow (.github/workflows/macro_awareness.yml, on: workflow_dispatch)
  that runs the producer for manual/eval runs, validates the on-disk snapshot against the
  full R3 schema before staging, and persists snapshot + state via the existing sidecar
  mechanism: force-add under logs/ and push via tools/ci_push_artifacts.sh (the same path
  used by logs/macro_drivers_snapshot.json and logs/trend_structure_snapshot.json).
- generated_at stamp carried for PRD-188's freshness bound.
- Materiality eval harness: a labeled positive/negative corpus of historical mornings, a
  pinned classifier (claude-opus-4-8), an explicit false-positive threshold T (set by
  Dustin and recorded in PRD-188's GATE before build), emitting an eval result artifact
  under audits/. This is the PRD-188 gate.

REQUIREMENTS

R1 — Producer isolation (AST-enforced)
tools/macro_awareness_collector.py imports no module from cuttingboard/, and no module
under cuttingboard/ imports it. Enforced by an AST import check, not regex alone. The tool
may read data/red_folder_2026.json as a raw file (no code import) to suppress events
already covered as scheduled releases.

FAIL: an AST scan of tools/macro_awareness_collector.py finds any cuttingboard import, OR
grep -rEn "macro_awareness_collector" cuttingboard/ returns any match.

R2 — Artifact-path isolation
The snapshot and state path strings (logs/macro_awareness_snapshot.json,
logs/macro_awareness_state.json) appear only in the producer, the workflow, the tests, and
docs/artifact_flow_map.md. No cuttingboard decision module opens either path.

FAIL: either path string appears in any cuttingboard module other than (in PRD-188) the
renderer; OR a decision-surface module reads the snapshot path.

R3 — Controlled schema (semantic-safe)
The artifact is { schema_version (int), generated_at (UTC ISO-8601, tz suffix), status
("QUIET" | "SHOCK"), shock }. When SHOCK, shock is an object with exactly: event_type (in a
fixed ENUM), source_name (in allowlist), source_url (domain in allowlist), published_at,
source_excerpt (verbatim from the source, length-capped, HTML-escaped), detected_at. When
QUIET, shock is null. Extra keys are rejected inside shock, not only at top level. The
artifact MUST NOT contain, at any nesting level: macro_bias, confidence_score,
forecast/projection, recommended posture/direction, sentiment scalar, market transmission,
or any LLM-authored label or free-text evidence field.

FAIL: status SHOCK with null shock, OR status QUIET with non-null shock; OR any forbidden
field name appears anywhere; OR an extra key appears inside shock; OR source_excerpt exceeds
the length cap or is not HTML-escaped; OR a timestamp lacks a tz suffix.

R4 — Fail-closed and coherent generation
Ambiguity (all feeds read, nothing material) resolves to a written QUIET generation. All
allowlisted feeds are REQUIRED (all-or-preserve): if any required feed is unreachable after
retries, the run preserves the prior snapshot/state and writes nothing - it MUST NOT classify
or commit from an incomplete source set. This source-failure case (preserve prior, no write)
is DISTINCT from genuine QUIET (all feeds read, nothing material -> write QUIET). Any parse,
validation, or credential failure likewise preserves the prior generation and never emits a
populated SHOCK. Snapshot and novelty state publish as one coherent generation or neither
(both written to .tmp then os.replace; on any failure the prior generation is left intact).
Every preserve-prior and novelty/staleness suppression logs an explicit reason (no silent
fallback, per sidecar doctrine).

FAIL: a run emits status SHOCK on malformed model output / a schema violation / a missing
ANTHROPIC_API_KEY; OR a required-feed source failure causes a fresh generation to be written
(classifying or committing from an incomplete source set) rather than preserving the prior;
OR the snapshot is updated while the state is not (or vice versa); OR a suppression occurs
with no logged reason.

R5 — Deterministic novelty
Event identity = (event_type, source_entity, normalized_event_date). The same identity is
suppressed for TTL = 5 trading days unless event_type changes (a material update). State
persists across runs in logs/macro_awareness_state.json.

FAIL: the same event identity emits SHOCK on two runs inside the TTL without an event_type
change; OR novelty state does not persist across runs.

R6 — Source authority
source_url domain must be in the configured allowlist; "trusted wire" status is never
assumed for inclusion.

FAIL: a SHOCK is emitted whose source_url domain is not in the configured allowlist.

R7 — Realizable persistence with a pre-commit validation gate (HIGH-RISK scrutiny)
The workflow_dispatch job MUST, before staging anything: re-validate the on-disk snapshot
against the full R3 contract (exact top-level keys, status/shock consistency, shock exact
keys, event_type enum, source_url domain in allowlist, tz-valid timestamps, source_excerpt
length cap + HTML escaping); a failed validation aborts the run with no commit and no push.
On pass it force-adds ONLY logs/macro_awareness_snapshot.json and
logs/macro_awareness_state.json and pushes via tools/ci_push_artifacts.sh, under a named
concurrency group with cancel-in-progress: false, using the script's rebase/retry. An
artifact-push dry-run proves: only the two paths are staged; a simulated upstream-main
advance exercises rebase/retry; and a conflict or exhausted retry leaves main unchanged.

FAIL: a malformed (R3-invalid) snapshot is committed or pushed; OR the workflow has no
named concurrency group or sets cancel-in-progress: true; OR staging includes any path
other than the two artifacts; OR the dry-run does not exercise the upstream-advance
rebase/retry and the unchanged-main-on-failure case.

R8 — Materiality is not claimed by CI
CI proves R1-R7 plus schema/escaping/length only. The eval harness (separate, pinned
claude-opus-4-8, labeled corpus, explicit false-positive threshold T, result artifact under
audits/) is the only thing that speaks to materiality, and is the PRD-188 gate. T is set by
Dustin and recorded in PRD-188's GATE before any build.

FAIL: a CI test asserts the classifier's materiality/false-positive discipline; OR no eval
harness + corpus + threshold + result artifact exists.

DATA FLOW
source allowlist feeds -> producer (claude-opus-4-8 classify + select) -> novelty +
materiality gates -> { schema_version, generated_at, status, shock } snapshot +
logs/macro_awareness_state.json (both .tmp then os.replace as one generation) -> workflow
full-R3 validation gate -> force-add (two paths only) + tools/ci_push_artifacts.sh push to
main -> (PRD-188 renderer reads + re-validates the snapshot). State is read/written by the
producer only. generated_at is carried for the consumer freshness bound. No cuttingboard
decision module reads or writes any node in this flow.

FAIL CONDITIONS
- Any of R1-R8 violated.
- A successful run never yields a schema-valid artifact.
- status SHOCK with null shock, or status QUIET with non-null shock.
- Any forbidden field present, or an extra key inside shock.
- The producer reads or writes any decision surface.
- The snapshot/state are not persisted to main, land outside the two-path allowlist, or a
  malformed snapshot reaches main.
- The producer runs on a live cron cadence (that belongs to PRD-188).
