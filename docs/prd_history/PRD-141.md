PRD-141 — Hourly Alert Canonical Slot + Cross-Run Idempotency

STATUS
COMPLETE

LANE
STANDARD

CLASS
CONSUMER

WHY NOW
Telegram alerts labeled "hourly" are landing at drift-stamped wall-clock
minutes (e.g. 1:27 PM PT, 2:48 PM PT) and firing multiple times per PT
hour after midday. Root cause is two-fold: the alert header uses
`regime.computed_at_utc` (wall clock at run time) instead of the
intended scheduled slot, and the hourly notify branch in `runtime.py`
has no cross-run idempotency — PRD-018's `should_send` gate is wired
only into the daily/Sunday path at `runtime.py:982`. Fixing both is a
precondition for any operator trust in the hourly cadence.

MAX EXPECTED DELTA
≤ 180 production LOC across the listed FILES (excluding tests, the new
state file `logs/last_hourly_slot.json`, and PRD docs). Exceeding this
stops implementation and re-triggers review.

Revision history:
- 2026-05-18: initial ceiling 120 LOC.
- 2026-05-18: raised to 180 LOC during implementation. Reason: the new
  module `cuttingboard/notifications/hourly_slot.py` is ~80 LOC on its
  own (slot computation + premarket helper + load/save with input
  validation and parent-dir creation); the runtime.py + alert_runner.py
  + formatter.py edits net ~70 LOC. The original 120 estimate did not
  budget for `hourly_slot.py`'s safety surface (tz-aware checks,
  missing/malformed-JSON tolerance, parent-dir creation). Scope and

GOAL
Stamp every hourly Telegram alert with the canonical top-of-PT-hour
slot it represents, and guarantee at most one intraday hourly send per
(date, PT-hour) pair across CI reruns and cron drift, while preserving
the three declared premarket warmup alerts.

SCOPE
- Compute a canonical UTC slot as the top of the PT hour containing
  `now_utc`, using `America/Vancouver` (DST-correct year-round).
- Identify premarket cron firings (declared at UTC minutes 12:50,
  13:00, 13:50) as exempt from the cross-run gate so each premarket
  warmup still sends.
- Pass the slot through to `_format_hourly`; stamp the alert header as
  the canonical PT hour and ET equivalent rather than wall-clock
  runtime.
- Persist last successful intraday `(date, slot_utc)` in
  `logs/last_hourly_slot.json`.
- Gate the intraday hourly notify path on slot freshness; suppress
  send AND the hourly artifact writes when the slot is unchanged.
- Provide `--force-slot` CLI flag on `cuttingboard.alert_runner` and
  auto-set it for `workflow_dispatch` invocations in the workflow YAML.
- Keep the existing `*/30 14-21 * * 1-5` cron — intermediate `:30`
  firings are absorbed by the gate, not removed from the schedule.

OUT OF SCOPE
- Decision logic, regime scoring, qualification gates, sizing.
- Contract schema (beyond one additive `hourly_slot_utc` field on the
  hourly run summary — see R5).
- Dashboard renderer output.
- PRD-018 daily-run state suppression at `runtime.py:982-993` (untouched).
- The default-pipeline path (`cuttingboard.yml` workflow / non-hourly
  notify modes).
- Changing cron cadence in `hourly_alert.yml` schedule entries (the
  gate, not the cron, enforces the hourly cadence).

REQUIREMENTS

R1 — Canonical slot computation
A new module `cuttingboard/notifications/hourly_slot.py` exposes:

- `canonical_slot_utc(now_utc: datetime) -> datetime` — returns the
  UTC datetime of the top of the `America/Vancouver` hour containing
  `now_utc`. Implementation: convert `now_utc` to
  `ZoneInfo("America/Vancouver")`, replace `minute`, `second`,
  `microsecond` with 0, convert back to UTC. Result is tz-aware UTC.
  The function is timezone-pure and DST-correct year-round.
- `is_premarket_slot(now_utc: datetime, tolerance_minutes: int = 5) -> bool`
  — returns `True` iff `now_utc` falls within `±tolerance_minutes`
  of any of the three declared premarket UTC cron minutes:
  `12:50`, `13:00`, `13:50`. Comparison is on `(hour, minute)` of the
  UTC time only; date and seconds are ignored.

FAIL:
- `canonical_slot_utc(2026-05-18T20:27:14Z)` returns
  `2026-05-18T20:00:00+00:00` (top of 1 PM PDT).
- `canonical_slot_utc(2026-05-18T20:48:00Z)` returns
  `2026-05-18T20:00:00+00:00` (still inside 1 PM PDT).
- `canonical_slot_utc(2026-05-18T21:00:00Z)` returns
  `2026-05-18T21:00:00+00:00` (top of 2 PM PDT).
- `is_premarket_slot(2026-05-18T13:02:00Z)` returns `True`;
  `is_premarket_slot(2026-05-18T14:00:00Z)` returns `False`.

R2 — Header stamps canonical PT hour
`_format_hourly` reads `event.asof_utc`, which the runtime now
populates with the canonical `slot_utc` (not `regime.computed_at_utc`).
The header is:

- Line 1: `<slot_utc → America/Vancouver as "%-I:%M %p PT">`
  (e.g. `1:00 PM PT`).
- Line 2: `<slot_utc → America/New_York as "%H:%M ET">`
  (e.g. `16:00 ET`).

For scheduled intraday slots the PT minute MUST be `00`. Premarket
slots stamp the actual premarket UTC minute converted to PT (the three
declared premarket minutes 12:50, 13:00, 13:50 UTC convert to fixed PT
times under DST and are accepted).

FAIL: For any wall-clock runtime between `20:00:00Z` and `20:59:59Z`,
the header line 1 equals `1:00 PM PT` exactly and line 2 equals
`16:00 ET`.

R3 — Cross-run idempotency store
`cuttingboard/notifications/hourly_slot.py` exposes:

- `load_last_slot(path: str = "logs/last_hourly_slot.json") -> Optional[dict]`
- `save_last_slot(slot_utc: datetime, path: str = "logs/last_hourly_slot.json") -> None`

Persisted shape: `{"slot_utc": "<ISO-8601 with +00:00>", "saved_at_utc": "<ISO-8601 with +00:00>"}`.
The file is created on first successful send. Loads of a missing file,
empty file, or malformed JSON return `None` (no raise; no log spam
beyond a single debug line).

FAIL: After `save_last_slot(datetime(2026,5,18,20,0,tzinfo=UTC))`,
`load_last_slot()` returns a dict whose `slot_utc` parses back to that
exact UTC datetime.

R4 — Hourly notify gate
`alert_runner.main()` computes:

```
now_utc   = datetime.now(timezone.utc)
slot_utc  = canonical_slot_utc(now_utc)
premarket = is_premarket_slot(now_utc)
force     = args.force_slot or os.environ.get("CUTTINGBOARD_FORCE_SLOT") == "1"
last      = load_last_slot()
```

Decision:

- `(a)` if `force` or `premarket`: proceed without gate.
- `(b)` else if `last is not None and last["slot_utc"] == slot_utc.isoformat()`:
  write exactly one audit row via `write_notification_audit(
  transport="telegram", status="suppressed", alert_title="hourly",
  attempted=False, success=False, reason="suppressed_same_slot",
  state_key=slot_utc.isoformat())`, return exit 0 WITHOUT invoking
  `_execute_notify_run`. No hourly artifact files are written or
  modified.
- `(c)` else: invoke `_execute_notify_run(mode=MODE_LIVE,
  run_date=now_utc.date(), notify_mode=NOTIFY_HOURLY,
  slot_utc=slot_utc)`. On confirmed Telegram success
  (`alert_sent=True`), the runtime calls
  `save_last_slot(slot_utc, ...)` AFTER `send_notification` returns
  True. Failed sends MUST NOT update the store. Premarket invocations
  also save the slot (so a drifted post-premarket scheduled run at
  the same PT hour is correctly absorbed).

FAIL: Two consecutive scheduled invocations of
`python -m cuttingboard.alert_runner` inside the same intraday PT hour
(with `send_notification` patched to return True and `is_premarket_slot`
False) produce exactly:
  - one audit row with `status=success`, and
  - one audit row with `status=suppressed` and
    `reason=suppressed_same_slot`.

`logs/last_hourly_slot.json` after both invocations equals the value
written after the first.

R5 — Contract additive field
The hourly run summary produced by `_build_hourly_run_summary` gains
one top-level key `hourly_slot_utc` (ISO-8601 UTC with `+00:00`) sourced
from the `slot_utc` used to build the alert. The hourly contract
(`logs/latest_hourly_contract.json`) and hourly payload
(`logs/latest_hourly_payload.json`) schemas are otherwise unchanged.
No daily contract or payload is touched.

FAIL: `logs/latest_hourly_run.json` produced by a live run contains a
top-level `hourly_slot_utc` key whose value parses to a UTC datetime
equal to the value displayed by the alert header line 1 converted from
PT to UTC.

R6 — workflow_dispatch always sends
`.github/workflows/hourly_alert.yml` invokes the runner as:

```
- name: Run hourly alert
  run: |
    if [ "${{ github.event_name }}" = "workflow_dispatch" ]; then
      python -m cuttingboard.alert_runner --force-slot
    else
      python -m cuttingboard.alert_runner
    fi
```

YAML cron comments either drop the `EDT`/`EST` suffix in favor of
UTC-only labels, or annotate both DST states explicitly. The schedule
entries themselves are unchanged.

FAIL:
- A manual `gh workflow run "Cuttingboard Hourly Alert"` inside an
  already-fired intraday PT hour produces a new audit row with
  `status=success` (force path).
- A scheduled `*/30` rerun inside the same PT hour produces an audit
  row with `status=suppressed` and `reason=suppressed_same_slot`.

DATA FLOW
GitHub Actions cron (scheduled or workflow_dispatch)
  → `python -m cuttingboard.alert_runner [--force-slot]`
  → `now_utc = datetime.now(UTC)`
  → `slot_utc  = canonical_slot_utc(now_utc)`
  → `premarket = is_premarket_slot(now_utc)`
  → `last = load_last_slot("logs/last_hourly_slot.json")`
  → if not (force or premarket) and last["slot_utc"] == slot_utc.isoformat():
       write_notification_audit(status="suppressed",
                                reason="suppressed_same_slot",
                                state_key=slot_utc.isoformat())
       exit 0
  → else:
       _execute_notify_run(mode=LIVE, run_date=now_utc.date(),
                           notify_mode=NOTIFY_HOURLY,
                           slot_utc=slot_utc)
       → hourly notify branch builds AlertEvent(asof_utc=slot_utc, ...)
       → _format_hourly stamps canonical PT hour header
       → send_notification(title, body)
       → on alert_sent=True: save_last_slot(slot_utc, ...),
                              write hourly_slot_utc into
                              logs/latest_hourly_run.json

FAIL CONDITIONS
- Alert header line 1 PT minute is anything other than `00` for a
  non-premarket scheduled invocation.
- Two `status=success` audit rows for hourly sends within the same
  `(date, slot_utc)` for a non-`--force-slot`, non-premarket
  invocation.
- `logs/last_hourly_slot.json` updated when `alert_sent=False`.
- PRD-018 daily suppression at `runtime.py:982` changes (suppression
  decision, audit reason text, or `LAST_STATE_PATH` value).
- Daily contract or daily payload (`logs/latest_contract.json`,
  `logs/latest_payload.json`) gains, loses, or renames any field.
- Dashboard HTML rendered from a given hourly payload differs
  byte-for-byte before vs. after this PRD when inputs are identical
  (the slot stamp lives in the alert text, not the dashboard).
- Any change in: ingestion, validation, regime scoring, qualification
  gates, sizing, sector router, overnight policy, market_map writer,
  trend_structure, macro_drivers writer.
- A `workflow_dispatch` invocation suppressed by the slot gate.
- A premarket cron firing (12:50, 13:00, or 13:50 UTC) suppressed by
  the slot gate.

ROLLBACK PLAN
Revert the implementation commit. The new `logs/last_hourly_slot.json`
is data-only; deleting it has no effect on any other path. The
`hourly_slot_utc` field on the hourly run summary is additive and
unread by the renderer.
