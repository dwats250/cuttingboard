# PRD-119 — Dashboard Publish Freshness Gate

STATUS: COMPLETE

## GOAL
Prevent publication of coherent-but-stale dashboard artifacts to live `ui/`
outputs by enforcing deterministic freshness validation, executed after the
PRD-118 coherent-generation gate and before any output bytes are written.

## SCOPE
1. Add freshness validation to the renderer publish gate introduced in PRD-118.
2. Block publication of coherent artifact sets whose payload timestamp
   exceeds the allowed age window for the session context.
3. Preserve PRD-118 coherent-generation validation behavior.
4. Preserve PRD-116 mixed/stale/missing lineage rendering behavior.
5. Preserve PRD-117 inactive-session labeling behavior.
6. Restrict all changes to renderer-side publish validation and renderer tests.

## OUT OF SCOPE
1. No trading logic changes.
2. No `runtime.py` changes.
3. No payload schema changes.
4. No `generation_id` format changes.
5. No market_map generation changes.
6. No scheduler redesign.
7. No dashboard layout changes.
8. No notification logic changes.
9. No historical artifact migration/backfill.
10. No new committed fixture files; staleness is manufactured in tests only.

## TERMINOLOGY
- **payload_timestamp** := `payload["meta"]["timestamp"]` — authoritative;
  the only parsed artifact timestamp.
- **run_timestamp** := `run["timestamp"]` — diagnostic only; never used for
  age computation.
- **freshness_reference_time** := `datetime.now(timezone.utc)`, read exactly
  once per `validate_coherent_publish` invocation.
- **artifact_age** := `freshness_reference_time − parsed payload_timestamp`.
- **coherent artifact set** := PRD-118 `validate_coherent_publish` passes.
- **inactive session** := `payload.meta.session_type == "SUNDAY_PREMARKET"`.
- **live session** := `payload.meta.session_type is None`.
- **ui publish** := `output_path` resolves under a directory whose parts
  contain `"ui"` (matches existing `_output_under_ui` in
  `dashboard_renderer.py`).
- **non-ui render** := any `output_path` whose resolved parts do not contain
  a `"ui"` segment (reports/output/, tmp_path, fixtures, etc.).
- **StalePublishError** := new `RuntimeError` subclass raised by the
  freshness gate; sibling to PRD-118's `CoherentPublishError`.

## FRESHNESS WINDOWS
- `LIVE_SESSION_MAX_AGE_MINUTES` = 180
- `INACTIVE_SESSION_MAX_AGE_HOURS` = 72

## REQUIREMENTS

**R1 — Freshness validation gate.** When `output_path` is a ui publish,
freshness validation runs after PRD-118 coherent-generation validation
succeeds and before any output bytes are written.
FAIL: `ui/dashboard.html` is written despite `artifact_age` exceeding the
applicable window.

**R2 — Live-session freshness enforcement.** When
`payload.meta.session_type is None`, `artifact_age` must not exceed
`LIVE_SESSION_MAX_AGE_MINUTES`.
FAIL: Live-session ui publish succeeds with `artifact_age` greater than
`LIVE_SESSION_MAX_AGE_MINUTES`.

**R3 — Inactive-session freshness enforcement.** When
`payload.meta.session_type == "SUNDAY_PREMARKET"`, `artifact_age` must not
exceed `INACTIVE_SESSION_MAX_AGE_HOURS`.
FAIL: Inactive-session ui publish succeeds with `artifact_age` greater than
`INACTIVE_SESSION_MAX_AGE_HOURS`.

**R4 — Authoritative timestamp source.** Freshness validation uses
`payload["meta"]["timestamp"]` as the sole age input. `run["timestamp"]`
may appear in diagnostics only.
FAIL: Validation reads `generation_id`, filesystem mtime, market_map
timestamp, `run.timestamp`, or any runtime heuristic for age computation.

**R5 — Deterministic timestamp parsing.** `payload["meta"]["timestamp"]`
must parse as an ISO-8601 UTC timestamp with trailing `Z` (the runtime
contract format, e.g. `"2026-05-10T23:32:41Z"`). Parsing failure raises
`StalePublishError` immediately.
FAIL: Invalid or missing `payload.meta.timestamp` falls back to current
time, filesystem metadata, `generation_id` parsing, `run.timestamp`, or any
inferred value.

**R6 — Single freshness_reference_time read.**
`validate_coherent_publish` reads `datetime.now(timezone.utc)` exactly once
per invocation. That single value is used for both age computation and
stderr diagnostics.
FAIL: Multiple `now()` reads, or different reference times used for age and
diagnostics within the same call.

**R7 — Explicit freshness failure diagnostics.** On freshness failure,
stderr must emit:
- observed payload timestamp (raw string)
- computed `artifact_age` (seconds, integer)
- allowed window (e.g. `"180m"` or `"72h"`)
- session_type (`"SUNDAY_PREMARKET"` or `"None"`)

FAIL: Any required diagnostic field is omitted or empty.

**R8 — Distinct exception type.** Freshness rejection raises
`StalePublishError(RuntimeError)`. Coherent-generation rejection continues
to raise `CoherentPublishError`.
FAIL: Freshness path raises `CoherentPublishError`, generic `RuntimeError`,
or any other exception type.

**R9 — PRD-118 regression protection.** `validate_coherent_publish`
executes its existing coherent-generation checks first; freshness logic
runs only after those checks pass.
FAIL: Freshness path bypasses or weakens coherent-generation validation, or
runs before it.

**R10 — PRD-116 regression protection.** Mixed/stale/missing lineage
rendering behavior remains unchanged for non-ui renders and test fixtures.
FAIL: PRD-116 rendering behavior regresses.

**R11 — PRD-117 regression protection.** Inactive-session labeling behavior
remains unchanged.
FAIL: PRD-117 inactive-session labeling regresses.

**R12 — Non-ui render exemption.** Freshness gating applies only when
`_output_under_ui(output_path)` is True. Non-ui renders bypass freshness
validation entirely.
FAIL: A non-ui render (`reports/output/`, `tmp_path` without a `"ui"`
segment) is blocked by freshness validation.

**R13 — No silent freshness fallback.** The renderer must not refresh,
mutate, rewrite, or substitute stale artifacts during validation.
FAIL: Fresh artifacts are synthesized, refetched, or substituted
automatically during freshness validation.

**R14 — Validation ordering.** Validation order:
1. PRD-118 coherent-generation validation
2. PRD-119 freshness validation
3. `output_path.write_text(...)`

FAIL: `output_path.write_text(...)` executes before freshness validation
succeeds.

**R15 — Unit-test proof.** `tests/test_dashboard_renderer.py` adds
deterministic tests covering:
- coherent fresh live publish succeeds
- coherent stale live publish raises `StalePublishError`
- coherent fresh inactive-session publish succeeds
- coherent stale inactive-session publish raises `StalePublishError`
- malformed `payload.meta.timestamp` raises `StalePublishError`
- missing `payload.meta.timestamp` raises `StalePublishError`
- non-ui stale render succeeds (output path under `tmp_path` with no
  `"ui"` segment, e.g. `tmp_path / "out" / "dashboard.html"`)
- PRD-118 coherent-generation mismatch still raises
  `CoherentPublishError` before freshness evaluation runs
- stderr captured on freshness failure contains all R7 fields

Tests manufacture staleness in-process (frozen timestamps via
monkeypatching `datetime.now` or by injecting payload dicts with old
`meta.timestamp`); no committed fixture files are introduced.
FAIL: Any listed branch lacks deterministic unit-test coverage, or any test
depends on a non-existent committed fixture file.

## DATA FLOW
```
payload / run / market_map
  -> PRD-118 coherent-generation validation
  -> PRD-119 freshness validation (payload.meta.timestamp only)
  -> PASS -> renderer writes output_path
  -> FAIL -> StalePublishError + non-zero CLI exit + R7 stderr diagnostics
```

No new artifact types. No payload schema changes. No `generation_id`
changes. No scheduler modifications.

## FAIL CONDITIONS (summary)
- ui publish with coherent-but-stale artifacts
- freshness validation bypass
- malformed/missing timestamp silently tolerated
- stale artifact auto-replacement
- freshness validation omitted for ui publish
- PRD-118 validation weakened or reordered
- missing stderr diagnostics
- `output_path.write_text(...)` before freshness validation passes
- wrong exception type on stale rejection
- committed test fixtures introduced
