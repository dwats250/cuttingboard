PRD-174 — Populate trend-structure OHLCV on STAY_FLAT hourly runs

Status: COMPLETE

LANE
HIGH-RISK

CLASS
EXECUTION

WHY NOW
The hourly dashboard renders DATA_UNAVAILABLE for vs-SMA50/SMA200/VWAP and
relative volume on every STAY_FLAT run, which is most days. Stand-down days are
exactly when the dashboard's only job is environmental context. This is the data
layer PRD-175/176/177 sit on; it lands first.

MAX EXPECTED DELTA
30 production LOC, 1 production file. Exceeding stops implementation and
re-triggers EXECUTION review.

GOAL
Trend structure renders real values for all six config.TREND_STRUCTURE_SYMBOLS on
every hourly run regardless of regime.posture, while preserving graceful
degradation when market data is genuinely unavailable.

SCOPE
- In cuttingboard/runtime/__init__._execute_notify_run, ensure the
  history_by_symbol passed to _write_trend_structure_snapshot (call site ~line
  539) covers config.TREND_STRUCTURE_SYMBOLS on every hourly run, independent of
  regime.posture and independent of the trade-candidate set.
- Add a flat-posture acceptance test driving the hourly notify path.

OUT OF SCOPE
- Renderer changes. The DATA_UNAVAILABLE display path is preserved; it stops
  firing under normal data conditions.
- Changes to which symbols are tracked, or to the trade-candidate OHLCV fetch.
- Any change to cuttingboard/trend_structure.py (builder/sentinel contract).
- Non-hourly notify modes (daily/sunday/fixture/intraday).

FILES
M cuttingboard/runtime/__init__.py
M tests/test_runtime_trend_structure_refresh.py

CHANGE SURFACE
- cuttingboard/runtime/__init__.py: edits confined to the OHLCV population that
  feeds the trend-structure writer inside _execute_notify_run. No public-function
  signature changes. No change to regime, qualification, sizing, posture, contract
  assembly, or artifact write ordering. The trade-candidate OHLCV population
  (lines ~424-428) is not removed; a TREND_STRUCTURE_SYMBOLS fetch is guaranteed
  for the snapshot writer.

REQUIREMENTS

R1 — Posture-independent trend OHLCV
On every run where notify_mode in _HOURLY_MODES, the history_by_symbol mapping
passed to _write_trend_structure_snapshot contains an entry for each symbol in
config.TREND_STRUCTURE_SYMBOLS for which fetch_ohlcv returns a non-None frame,
regardless of regime.posture (including "STAY_FLAT").
FAIL: a STAY_FLAT hourly fixture run produces logs/trend_structure_snapshot.json
in which any of the six symbols has null sma_50, sma_200, or vwap while its
fetch_ohlcv path returns a usable frame.

R2 — Degradation preserved
A symbol whose fetch_ohlcv returns None still resolves to its existing
unavailable sentinel and the writer does not raise.
FAIL: a forced-None fetch for a trend symbol raises, or writes a record missing
the data_status field.

R3 — No collateral behavior change
Non-hourly notify modes and the trade-candidate OHLCV fetch are unchanged; no
trading-decision, regime, qualification, or sizing output differs.
FAIL: any non-hourly mode's snapshot output changes, or trend_structure.py is
modified.

DATA FLOW
_execute_notify_run -> fetch_ohlcv(each TREND_STRUCTURE_SYMBOL) ->
history_by_symbol -> _write_trend_structure_snapshot ->
logs/trend_structure_snapshot.json -> dashboard_renderer (unchanged).

FAIL CONDITIONS
- STAY_FLAT hourly run leaves any trend symbol with null computed metrics when
  data is available.
- Forced-None fetch raises or yields a malformed record.
- Any non-hourly mode output changes, or trend_structure.py is edited.

VALIDATION
Manual:
- Run the hourly notify path against a fixture with regime.posture="STAY_FLAT";
  confirm logs/trend_structure_snapshot.json has non-null sma_50/sma_200/vwap and
  data_status "OK" for symbols with data, sentinel only where fetch returned None.
- python3 -m pytest tests/test_runtime_trend_structure_refresh.py -q passes,
  including the new flat-posture test.
- Full suite green; ruff clean.
