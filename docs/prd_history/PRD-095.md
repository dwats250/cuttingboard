# PRD-095 — Scheduled Pipeline Morning Readiness Guard

**STATUS:** DRAFT

---

## GOAL

Add a scheduled-readiness guard to the hourly pipeline so the morning dashboard publication path fails visibly when required runtime artifacts or public dashboard HTML artifacts are missing, malformed, structurally incomplete, or contaminated with local test paths.

This PRD is DRAFT only. Implementation has not started.

---

## SCOPE

1. Add a future readiness validation script at `scripts/check_readiness.py`.
2. Add a future GitHub Actions workflow step in `.github/workflows/hourly_alert.yml` after "Publish hourly artifacts" and before "Push hourly artifacts".
3. Validate required JSON artifacts exist, parse, and contain required fields.
4. Validate `ui/dashboard.html` and `ui/index.html` exist.
5. Validate public HTML artifacts do not contain `pytest-of-`, `/tmp/pytest`, or `/tmp/`.
6. Validate public HTML artifacts contain required structural markers.

---

1. Do not modify runtime logic.
2. Do not modify notifications.
3. Do not modify market_map generation.
4. Do not modify schemas.
5. Do not modify trade logic, grading, lifecycle, or data fetching.

## REQUIREMENTS

**R1 — Readiness script is stdlib-only and offline.**

`scripts/check_readiness.py` must use only the Python standard library. It must not call network services, shell out to subprocesses, fetch market data, regenerate artifacts, or depend on market hours.

FAIL: The script imports third-party dependencies, uses subprocess, performs network I/O, or mutates pipeline artifacts.

---

**R2 — Required JSON artifacts are validated.**

The readiness script must verify that required JSON artifacts exist, parse as JSON, and contain required top-level fields needed by the scheduled dashboard publication path.

Required artifacts include:

- `logs/latest_hourly_payload.json`
- `logs/latest_hourly_run.json`

FAIL: Missing files, malformed JSON, or missing required fields are allowed to pass.

---

**R3 — Public dashboard HTML artifacts are present.**

The readiness script must verify both public dashboard artifacts exist:

- `ui/dashboard.html`
- `ui/index.html`

FAIL: Either missing HTML artifact is allowed to pass.

---

**R4 — Public dashboard HTML artifacts are free of local/test path contamination.**

The readiness script must fail if either public HTML artifact contains any forbidden pattern:

```python
("pytest-of-", "/tmp/pytest", "/tmp/")
```

FAIL: Either HTML artifact containing `pytest-of-`, `/tmp/pytest`, or `/tmp/` passes readiness.

---

**R5 — Public dashboard HTML artifacts contain required structural markers.**

The readiness script must fail if either public HTML artifact is missing any required marker:

- `id="system-state"`
- `id="macro-tape"`
- `id="candidate-board"`
- `RUN SNAPSHOT`

FAIL: Either HTML artifact missing one or more markers passes readiness.

---

**R6 — Workflow gate runs after publication and before push.**

`.github/workflows/hourly_alert.yml` must run the readiness script after the "Publish hourly artifacts" step and before the "Push hourly artifacts" step.

FAIL: The workflow can push generated dashboard artifacts before readiness validation runs.

---

**R7 — Failures are visible and actionable.**

Readiness failures must identify the file and failed condition, including the missing artifact, missing field, missing marker, parse error, or forbidden contamination pattern.

FAIL: The script exits unsuccessfully without enough detail to identify the failed artifact or condition.

## DATA FLOW

Future readiness validation flow:

```
scheduled hourly pipeline
  -> publish hourly artifacts
  -> scripts/check_readiness.py
       -> validate JSON artifacts
       -> validate public HTML artifact presence
       -> validate public HTML contamination rules
       -> validate public HTML structural markers
  -> push hourly artifacts only if readiness passes
```

No runtime data flow changes are planned.

- F7: Either HTML artifact is missing `id="system-state"` and readiness passes.
