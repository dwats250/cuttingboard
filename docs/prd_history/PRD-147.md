# PRD-147 — prd_close.sh must not parse user input as re.sub template

STATUS
IN PROGRESS

LANE
MICRO

GOAL
`scripts/prd_close.sh` crashed during PRD-145 closeout with `re.PatternError: bad escape \d at position 270`, raised from the `re.subn` call that rewrites the `**Last work completed:**` line of `docs/PROJECT_STATE.md`. Root cause: the script interpolates the user-supplied `--summary` value into the *replacement* argument of `re.subn`, where Python interprets backslash escapes (`\g<name>`, `\1`, `\d`, etc.) as regex templates. My PRD-145 summary contained `re.findall(r'PRD-(\d+)', line)` verbatim, and the `\d` was parsed as an invalid template escape. The same defect exists for `--title` at the `**Last completed PRD:**` rewrite. Fix by using the callable replacement form of `re.subn` — callables bypass template parsing entirely — for the two interpolations that consume user input.

SCOPE
- Replace the string-form replacement in the `Last completed PRD` `re.subn` call (scripts/prd_close.sh:146-150) with a callable: `lambda _m: f"**Last completed PRD:** {prd_id} - {title} (commit {commit_hash})"`.
- Replace the string-form replacement in the `Last work completed` `re.subn` call (scripts/prd_close.sh:155-159) with a callable: `lambda _m: f"**Last work completed:** {today} — {summary}"`.
- Leave the other `re.subn` calls (lines 99, 137, 164) alone — their replacements interpolate only system-controlled values (`commit_hash`, `today`, numeric `tests`/`added`) that cannot legitimately contain backslash sequences.

NOTES
- An alternative fix is `summary.replace("\\", "\\\\")` and `title.replace("\\", "\\\\")` before interpolation, but that conflates "escape for replacement template" with the visible string the user typed. The callable form is cleaner and has zero collateral.
