# PRD-019 — Notification Decision Audit / Delivery Safety Layer

## SUMMARY

Build a deterministic notification decision layer that records why a notification
was sent, suppressed, retried, or skipped.

This layer must:
- Be purely observational (no strategy logic)
- Use the canonical contract as input
- Produce exactly one decision per pipeline run
- Be written to audit logs

This extends PRD-017/018 by making notification behavior explainable and safe for
future delivery systems (Telegram, HTML, React, Android).

---

## OBJECTIVE

Every pipeline run must produce exactly one notification decision:

- SENT
- SUPPRESSED
- RATE_LIMITED
- ERROR
- DISABLED

Each decision must include a deterministic reason from a fixed enum.

## REQUIREMENTS

### 1. Canonical Decision Builder

Implement as a pure function in `cuttingboard/output.py`:

```python
def build_notification_decision(
    contract: dict,
    *,
    notifications_enabled: bool = True,
    rate_limited: bool = False,
    send_error: str | None = None,
    sent: bool = False,
    retry_count: int = 0,
) -> dict:
```

`contract` must always be a plain `dict`. Callers are responsible for serializing
any dataclass before passing it. The builder never imports or references pipeline
dataclasses directly.

Return JSON-safe dict:

```
{
    "schema_version": 1,
    "decision": "SENT" | "SUPPRESSED" | "RATE_LIMITED" | "ERROR" | "DISABLED",
    "reason": str,
    "contract_status": str | None,
    "market_regime": str | None,
    "execution_posture": str | None,
    "tradable": bool | None,
    "top_setup_count": int,
    "sent": bool,
    "retry_count": int,
    "send_error": str | None,
}
```

---

### 2. Reason Enum (STRICT)

Allowed values:

```
TOP_SETUP_FOUND
STAY_FLAT
SYSTEM_HALTED
NO_TRADABLE_SETUP
NOTIFICATIONS_DISABLED
RATE_LIMITED
SEND_ERROR
UNKNOWN_CONTRACT_STATE
```

---

### 3. Decision Logic (ORDERED — FIRST MATCH WINS)

**A — Notifications disabled**

```
decision = "DISABLED"
reason = "NOTIFICATIONS_DISABLED"
```

---

**B — Rate limited**

```
decision = "RATE_LIMITED"
reason = "RATE_LIMITED"
```

---

**C — Send error**

```
decision = "ERROR"
reason = "SEND_ERROR"
```

---

**D — Sent successfully**

```
decision = "SENT"
```

Then:

- if `tradable == True` AND `top_setup_count > 0`
  → `reason = "TOP_SETUP_FOUND"`
- elif `contract_status == "STAY_FLAT"`
  → `reason = "STAY_FLAT"`
- elif `contract_status == "ERROR"`
  → `reason = "SYSTEM_HALTED"`
- else
  → `reason = "UNKNOWN_CONTRACT_STATE"`

---

**E — Not sent (suppressed)**

```
decision = "SUPPRESSED"
```

Then:

- if `contract_status == "STAY_FLAT"`
  → `reason = "STAY_FLAT"`
- elif `contract_status == "ERROR"`
  → `reason = "SYSTEM_HALTED"`
- elif `tradable != True` OR `top_setup_count == 0`
  → `reason = "NO_TRADABLE_SETUP"`

  Note: `tradable != True` covers both `False` and `None`. A missing `tradable`
  field (None) is treated the same as False — no confirmed tradable setup.

- else
  → `reason = "UNKNOWN_CONTRACT_STATE"`

---

### 4. Contract Field Extraction (SAFE)

Only read:

```
contract["run_status"]      → contract_status
contract["market_regime"]   → market_regime
contract["execution_posture"] → execution_posture
contract["tradable"]        → tradable
contract["top_setups"]      → top_setup_count = len(...)
```

Fallbacks (applied when key is absent or value is None):

```
contract_status   = None
market_regime     = None
execution_posture = None
tradable          = None
top_setup_count   = 0
```

Must never raise on missing or malformed fields.

---

### 5. Audit Integration

`write_notification_audit` in `audit.py` must accept an optional
`notification_decision: dict | None = None` parameter and include it in the
audit record.

Every notification path must write exactly one audit record containing:

- `retry_count`
- `notification_decision`

---

### 6. Runtime Wiring

**Wiring model:** `build_notification_decision` is called inside `send_telegram`
(in `output.py`), after the final send outcome is known. `send_telegram` accepts
one new optional parameter:

```python
contract: dict | None = None
```

`runtime.py` is responsible for serializing the pipeline contract to `dict` before
passing it to `send_telegram`. Use `dataclasses.asdict()` or equivalent. If no
contract is available, pass `None` — the builder will treat it as `{}`.

Execution flow:

1. Build contract (existing pipeline)
2. Serialize contract to dict (`runtime.py`)
3. Build notification message (existing PRD-017)
4. Attempt send — `send_telegram(..., contract=contract_dict)`
5. Inside `send_telegram`: after final outcome, call `build_notification_decision`
6. Pass result to `write_notification_audit` as `notification_decision`
7. Exactly one audit record written

Constraints:

- No multiple sends
- No duplicate audit writes
- Decision reflects final state after retries
- `send_telegram` return type remains `bool` (no interface break)

---

This PRD must NOT modify:

- candidate ranking
- flow logic
- regime logic
- options logic
- ORB logic
- macro scoring
- entry/exit/position sizing

Notification layer = audit + explanation only.

## ACCEPTANCE CRITERIA

**AC1 — Builder exists**

```python
from cuttingboard.output import build_notification_decision
callable(build_notification_decision) == True
```

---

**AC2 — Sent top setup**

```python
contract = {
    "run_status": "OK",
    "market_regime": "CLEAN",
    "execution_posture": "TRADE_ALLOWED",
    "tradable": True,
    "top_setups": [{"symbol": "SPY"}],
}
result = build_notification_decision(contract, sent=True, retry_count=1)
```

Expected:
```
decision == "SENT"
reason == "TOP_SETUP_FOUND"
top_setup_count == 1
sent == True
retry_count == 1
```

---

**AC3 — Stay flat suppressed**

```python
contract = {
    "run_status": "STAY_FLAT",
    "tradable": False,
    "top_setups": [],
}
result = build_notification_decision(contract, sent=False)
```

Expected:
```
decision == "SUPPRESSED"
reason == "STAY_FLAT"
```

---

**AC4 — System halted**

```python
contract["run_status"] = "ERROR"
```

Expected:
```
decision == "SUPPRESSED"
reason == "SYSTEM_HALTED"
```

---

**AC5 — Notifications disabled**

```python
result = build_notification_decision({}, notifications_enabled=False)
```

Expected:
```
decision == "DISABLED"
reason == "NOTIFICATIONS_DISABLED"
```

---

**AC6 — Rate limited**

```python
result = build_notification_decision({}, rate_limited=True)
```

Expected:
```
decision == "RATE_LIMITED"
reason == "RATE_LIMITED"
```

---

**AC7 — Send error**

```python
result = build_notification_decision({}, send_error="timeout")
```

Expected:
```
decision == "ERROR"
reason == "SEND_ERROR"
```

---

**AC8 — Missing fields safe**

```python
result = build_notification_decision({}, sent=False)
```

Must not crash.

Expected:
```
top_setup_count == 0
decision == "SUPPRESSED"
reason == "NO_TRADABLE_SETUP"
```

`tradable` is `None` (absent), which satisfies `tradable != True`. `top_setup_count`
is 0. Both conditions hold → `NO_TRADABLE_SETUP`.
