# PRD-140 — Document `pre_push_check.sh` in CLAUDE.md git hygiene

STATUS
PROPOSED

LANE
MICRO

GOAL
`scripts/pre_push_check.sh` exists in the repo and runs `ruff` plus a
targeted dashboard-renderer pytest before push, but
`CLAUDE.md § git hygiene and artifact discipline` only references
`scripts/clean_generated_artifacts.sh` and `scripts/pre_commit_sanity.sh`.
New sessions reading CLAUDE.md cannot discover the pre-push helper. Add
a one-line reference so the script is discoverable through the same
path the other two helpers are.

SCOPE
- Add a single bullet under `CLAUDE.md § git hygiene and artifact
  discipline § Before committing` (or an adjacent "Before pushing"
  line) that names `scripts/pre_push_check.sh` and states what it runs.
- Track this PRD in registry/state per existing convention.

OUT OF SCOPE
- Modifying the contents of `scripts/pre_push_check.sh`.
- Adding the script to any hook or automation.
- Documenting other scripts (`check_readiness.py`, `gitnexus-analyze.sh`).
