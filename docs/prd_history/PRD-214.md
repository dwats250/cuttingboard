CLASS
CONSUMER

WHY NOW
The macro tape renders each driver twice — once in the driver grid, then again as
a per-driver "risk-ON/OFF vote (…)" line with an italic gloss. That is ~10
restated lines that dominate the panel while adding no information the grid and
MACRO BIAS headline don't already carry. It is the largest vertical-space sink on
the dashboard for the least new signal.

MAX EXPECTED DELTA
40 production LOC (dashboard_renderer.py macro-tape section).

GOAL
Replace the always-expanded per-driver evidence block with (a) an always-visible
one-line risk-vote tally under MACRO BIAS and (b) the existing per-driver evidence
rows relocated into a default-collapsed disclosure, preserving the PRD-177/PRD-191
rationale audit while removing the clutter.

SCOPE
- Add a one-line tally under the MACRO BIAS headline: risk-off count / risk-on
  count and the resulting bias (e.g. "Risk votes: 3 off / 1 on → SHORT").
- Wrap the existing per-driver evidence rows (vote + interpretation gloss) in a
  default-collapsed `<details>` ("MACRO EVIDENCE ▶"), mirroring the existing
  MACRO PRESSURE disclosure pattern.

OUT OF SCOPE
- Changing vote-tallying logic, the pro-/contra-cyclical classification, or the
  interpretation strings (owned by macro_tape_layout.py — untouched).
- Removing the per-driver rows entirely (they remain, just collapsed).

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dashboard_renderer.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — macro-tape render (~L2064–2188) and
one _CSS rule for the tally line: add the tally string and a `<details>` wrapper.
macro_tape_layout.py is NOT modified; the tally reuses the existing long/short
vote counts already computed at ~L1788–1812.

REQUIREMENTS

R1 — Always-visible vote tally
An always-visible one-line tally under MACRO BIAS shows the risk-off and risk-on
counts and the derived bias.
FAIL: no tally line renders, or its bias term disagrees with the MACRO BIAS
headline (LONG/SHORT/MIXED).

R2 — Per-driver evidence collapsed by default
The per-driver "risk-ON/OFF vote (…)" evidence rows render inside a
default-collapsed `<details>` element, not in the always-expanded body.
FAIL: any per-driver evidence row renders outside a `<details>` element.

R3 — Rationale preserved
The relocated evidence rows retain the exact PRD-177/PRD-191 vote + interpretation
wording (no string changes).
FAIL: an evidence row's vote or interpretation text differs from the pre-change
string.

DATA FLOW
long_votes/short_votes (existing) → tally line → HTML
per-driver evidence rows (existing strings) → wrapped in `<details>` → HTML

FAIL CONDITIONS
- No tally line, or tally bias disagrees with MACRO BIAS headline.
- Any per-driver evidence row rendered outside `<details>`.
- Any evidence vote/interpretation string changed.

VALIDATION
Manual:
- Render the dashboard; MACRO BIAS is followed by one tally line; the per-driver
  detail is hidden until the disclosure is expanded.
- Expanding the disclosure shows the same vote/gloss text as before.
Automated:
- Update test_prd177_r3_* / test_prd191_* to locate the evidence rows inside the
  disclosure; new test asserts the tally line and its agreement with the headline.
