RATIONALE

- Required edits 1-3 protect an existing repo invariant: the test suite
  monkeypatches runtime collaborators and constants through the
  `cuttingboard.runtime` namespace. A scoping PRD whose central promise
  is "near-zero test churn via a re-export facade" must be correct about
  what a re-export actually preserves (import-time bindings) versus what
  it does not (attribute patches resolved inside relocated modules).
  This is the difference between a behavior-preserving split and a
  silently test-breaking one â€” exactly the realizability discipline in
  `CLAUDE.md Â§ PRD-author disciplines` (downstream-consumer audit).
- Required edits 4-6 enforce that a binding "acyclic layering" doctrine
  match the real call graph; otherwise the first extraction PRD that
  follows the roadmap (hourly before artifacts) immediately violates the
  no-clusterâ†’cluster rule. A cut-line map that is binding must be
  buildable.
- Required edit 7 enforces `docs/PRD_TEMPLATE.md` FAIL-line discipline
  (binary, observable): an acceptance gate that cannot be executed as
