## STEP 1b hardening — confirmed present
The resolver fails closed on *unrecognized structured output*, not just on the known
string: no recognized codex events → exit 2; no `turn.completed` → exit 3; an
item.error with an unrecognized message → exit 5. "Couldn't find the error string"
never silently equals "no error." A negative test
(`test_prd207_reworded_fallback_still_fails_closed`) pins this.
