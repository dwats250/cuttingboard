PRD-126 — Fixture Mode No-Live-OHLCV Boundary

STATUS
COMPLETE

LANE
STANDARD

CLASS
EXECUTION + PATCH

WHY NOW
Codex read-only review found fixture-backed runtime execution can still
call the live OHLCV fetch path after fixture quote loading.

MAX EXPECTED DELTA
<= 20 production LOC in `cuttingboard/runtime.py`. Test changes are
limited to fixture-mode regression coverage.

GOAL
Ensure fixture-backed runs never perform live OHLCV fetches while
preserving normal live-mode OHLCV behavior.

SCOPE
- Block the direct `runtime.fetch_ohlcv` path during fixture-backed
  runtime OHLCV lookup.
- Preserve existing cache-only OHLCV behavior for fixture-backed derived
  metric computation.
- Preserve existing missing-data degradation when fixture/cache-only OHLCV
  is unavailable.
- Add a regression test that fails if fixture mode calls direct
  `runtime.fetch_ohlcv`.
- Preserve normal non-fixture/live OHLCV fetch behavior.

OUT OF SCOPE
- No OHLCV cache TTL changes.
- No quote freshness changes.
- No notification semantic changes.
- No dashboard changes.
- No payload or contract schema changes.
- No market_map changes.
- No artifact regeneration.
- No logs, reports, or UI changes.
- No broad fixture-mode redesign.
- No live fetch policy changes outside fixture-backed runtime execution.
- No hourly alert alignment.
- No readiness ordering changes.
- No banned-text readiness guard.

ROOT CAUSE
incomplete mock boundary — fixture mode patched
`cuttingboard.derived.fetch_ohlcv` but `_run_pipeline()` also uses the
direct `runtime.fetch_ohlcv` import for candidate qualification OHLCV.

CHANGE SURFACE
- `cuttingboard/runtime.py`: fixture-mode OHLCV boundary only. No
  `_run_pipeline()` signature changes, live fetch policy changes outside
  fixture-backed runs, notification changes, dashboard changes, contract
  schema changes, market_map changes, or PRD-125 cache TTL changes.
- `tests/test_fixture_mode.py`: deterministic regression test for the
  direct `runtime.fetch_ohlcv` fixture leak only.
- `docs/PRD_REGISTRY.md`: add exactly one PRD-126 row with status
  `IN PROGRESS` and link to `docs/prd_history/PRD-126.md`.
- `docs/PROJECT_STATE.md`: set active PRD to PRD-126 only.

REQUIREMENTS

R1 — Fixture mode blocks direct runtime OHLCV live fetch
When fixture-backed mode is active, `_run_pipeline()` MUST NOT call the
live direct `runtime.fetch_ohlcv` path.

FAIL: a fixture-backed `_run_pipeline()` execution invokes
`runtime.fetch_ohlcv` for candidate OHLCV.

R2 — Fixture mode remains deterministic
When fixture-backed mode is active and OHLCV data is unavailable from
fixture/cache-only sources, the pipeline MUST degrade through existing
missing-data behavior and MUST NOT fall back to network OHLCV fetch.

FAIL: missing fixture/cache-only OHLCV in fixture-backed mode triggers a
network OHLCV fetch instead of continuing with unavailable OHLCV.

R3 — Existing non-fixture behavior remains unchanged
Normal non-fixture/live mode MUST still be able to use the regular
`runtime.fetch_ohlcv` path for candidate OHLCV.

FAIL: live mode is routed through a fixture-only cache guard or prevented
from calling `runtime.fetch_ohlcv`.

R4 — PRD-125 behavior remains unchanged
This PRD MUST NOT modify `_is_fresh_ohlcv_cache()` or OHLCV TTL behavior.

FAIL: `cuttingboard/ingestion.py` or PRD-125 cache freshness behavior is
modified.

R5 — Regression test fails on live fixture OHLCV
Tests MUST include a deterministic regression that spies or patches the
direct `runtime.fetch_ohlcv` path and fails if fixture-backed runtime
execution calls it.

FAIL: fixture-mode tests pass while direct `runtime.fetch_ohlcv` is still
called during fixture-backed candidate OHLCV lookup.

DATA FLOW
Fixture file -> `_load_inputs()` -> fixture-backed validation ->
`_fixture_cache_only_ohlcv()` -> derived + candidate OHLCV cache-only
lookup -> existing missing-data qualification behavior.

Live inputs -> `_load_inputs()` -> validation -> normal
`runtime.fetch_ohlcv` candidate OHLCV lookup.

FAIL CONDITIONS
- FAIL if fixture-backed `_run_pipeline()` calls direct `runtime.fetch_ohlcv`.
- FAIL if fixture-backed missing OHLCV falls back to network fetch.
- FAIL if non-fixture/live candidate OHLCV cannot call `runtime.fetch_ohlcv`.
- FAIL if `_is_fresh_ohlcv_cache()` or PRD-125 TTL behavior changes.
- FAIL if notification, dashboard, payload/contract, market_map, logs,
  reports, UI, workflow, or config files are modified.
- FAIL if files outside the FILES list are modified.
