CLASS
CONSUMER

WHY NOW
The Macro Tape currently uses GLD and SLV ETF proxies as the only
metals reference. Spot gold and spot silver futures (`GC=F`, `SI=F`)
are a complementary macro signal alongside the ETF row — they add
observational context without replacing the tradable proxies. A
dedicated spot-metals row improves macro readability while leaving
regime logic, qualification, contract, sidecars, and the existing
GLD/SLV tradables row untouched. This follows the PRD-122 precedent
that introduced `CL=F` (oil futures) into the universe as an
observational macro driver.

MAX EXPECTED DELTA
~20–40 production LOC across `cuttingboard/config.py` (universe and
bounds) and `cuttingboard/delivery/dashboard_renderer.py` (new render
row). No edits to `cuttingboard/runtime.py`, contract, payload,
regime, qualification, sizing, market_map, notifier, CI, or schema.

`cuttingboard/runtime.py` is intentionally excluded from FILES. The
implementation must first attempt to flow `GC=F`/`SI=F` through the
existing `ALL_SYMBOLS → fetch → normalize → renderer` path. If
implementation later proves a runtime.py propagation hook is
required, stop and report the exact missing propagation point before
editing it; do not preemptively edit runtime.py.

GOAL
Add a dedicated "spot metals" row to the Macro Tape rendering spot
gold (display label `XAU`, sourced from `GC=F`) and spot silver
(display label `XAG`, sourced from `SI=F`). The row must render
immediately above the existing macro-drivers row, beneath MACRO BIAS,
and must remain observational-only.

SCOPE
- Add `GC=F` and `SI=F` to `cuttingboard/config.py` `MACRO_DRIVERS`
  list (NOT `COMMODITIES`). This automatically extends `ALL_SYMBOLS`
  (composed at `config.py:156`) and `NON_TRADABLE_SYMBOLS` (composed
  at `config.py:151`); the latter fences the new symbols out of
  qualification iteration without any per-symbol gate.
- Add `PRICE_BOUNDS` entries for `GC=F` and `SI=F`. Add
  `SYMBOL_SOURCE_PRIORITY` entries only if required (yfinance only,
  matching the CL=F precedent).
- Add `"gold": "GC=F"` and `"silver": "SI=F"` to
  `cuttingboard/contract.py` `_MACRO_DRIVER_SYMBOLS`. Extend
  `_OPTIONAL_MACRO_DRIVERS` from `{"oil"}` to
  `{"oil", "gold", "silver"}` so missing/invalid quotes degrade
  gracefully without breaking `assert_valid_contract`.
- Add a new module-level constant in `dashboard_renderer.py`,
  `_TAPE_SPOT_METAL_DEFS = [("XAU", "gold"), ("XAG", "silver")]`,
  mapping display label → `macro_drivers` payload key (mirrors
  `_TAPE_DRIVER_DEFS`'s `("OIL", "oil")` shape).
- Emit a new macro-tape sub-row above `<div class="macro-drivers-row">`
  and below MACRO BIAS, using the same `tape_value_map` lookup pattern
  as the drivers row, and the same `macro-tape-slot` cell pattern.
- Preserve the existing tradables grid (`_TAPE_MM_SYMBOLS` = SPY,
  QQQ, GLD, SLV, XLE, GDX) byte-for-byte.

OUT OF SCOPE
- No regime, qualification, sizing, policy, or trade-decision logic
  changes.
- No contract, payload, market_map, notifier, sidecar, evaluation,
  performance, or CI changes.
- No schema redesign.
- No removal or repurposing of GLD/SLV/XLE/GDX from
  `_TAPE_MM_SYMBOLS`.
- No CSS class renames or removals; no new top-level dashboard blocks;
  no changes to the trend-structure / macro-pressure / candidate-board
  panels.
- No additions to `HALT_SYMBOLS` or `REQUIRED_SYMBOLS`.
- No additions to `SYMBOL_UNITS` (the new symbols default to
  `"usd_price"`).
- No edits to `cuttingboard/runtime.py` (see MAX EXPECTED DELTA note).
- No watchlist sidecar changes.

REQUIREMENTS

R1 — Spot-metals row insertion point
In `render_dashboard_html()`, a new
`<div class="macro-spot-metals-row">` wrapper is emitted
**immediately above** the `<div class="macro-drivers-row">`
element. No content other than insignificant whitespace sits between
the MACRO BIAS line and the spot-metals row; no content other than
insignificant whitespace sits between the spot-metals row and the
macro-drivers row.

FAIL: In the rendered HTML for a representative fixture, the byte
index of `class="macro-drivers-row"` precedes the byte index of
`class="macro-spot-metals-row"`, OR any non-whitespace element
appears between MACRO BIAS and `class="macro-spot-metals-row"`, OR
any non-whitespace element appears between
`class="macro-spot-metals-row"` and `class="macro-drivers-row"`.

R2 — Futures-style spot sourcing via macro_drivers plumbing
XAU and XAG values are sourced from `GC=F` (CME gold front-month)
and `SI=F` (CME silver front-month) via the **existing macro_drivers
payload plumbing**: `config.MACRO_DRIVERS` → ingestion → normalization
→ `contract._build_macro_drivers` (emitting optional `"gold"` and
`"silver"` driver entries) → payload → renderer. No ETF proxy,
mining equity, hardcoded transform, or alternate ingestion path is
used.

FAIL: The renderer references `GLD`, `SLV`, `GDX`, `PAAS`, or any
mining-equity ticker when computing the XAU or XAG cell values; OR
`cuttingboard/ingestion.py` or `cuttingboard/normalization.py` is
modified; OR `_TAPE_SPOT_METAL_DEFS` maps `"XAU"`/`"XAG"` to a
`macro_drivers` payload key other than `"gold"`/`"silver"`; OR
`contract._MACRO_DRIVER_SYMBOLS["gold"]`/`["silver"]` resolves to a
symbol other than `"GC=F"`/`"SI=F"`.

R3 — Existing tradables row preservation
The `_TAPE_MM_SYMBOLS` literal at `dashboard_renderer.py:663` is
byte-identical to HEAD: `["SPY", "QQQ", "GLD", "SLV", "XLE", "GDX"]`.
The tradables-grid render block (lines 1762–1772 at HEAD) is
byte-identical to HEAD.

FAIL: `_TAPE_MM_SYMBOLS` differs from HEAD; OR rendered HTML for
`<div class="macro-tradables-grid">` differs from HEAD's render for
the same input fixture.

(a) `GC=F` and `SI=F` are members of `config.MACRO_DRIVERS` and
    therefore members of `config.NON_TRADABLE_SYMBOLS` (composed at

(b) The literal strings `"XAU"`, `"XAG"`, `"GC=F"`, and `"SI=F"`

    `cuttingboard/qualification.py`, `cuttingboard/execution_policy.py`,
    `cuttingboard/market_map.py`, `cuttingboard/market_map_lifecycle.py`,
    `cuttingboard/trend_structure.py`, any file under
    `cuttingboard/notifications/`, `cuttingboard/macro_pressure.py`,
    `cuttingboard/correlation.py`, `cuttingboard/overnight_policy.py`,
    `cuttingboard/entry_quality.py`, `cuttingboard/invalidation.py`,
    `cuttingboard/intraday_state_engine.py`, or any sidecar/sizing
    module.

    `cuttingboard/contract.py` is the one exception — it may contain
    `"GC=F"` and `"SI=F"` inside `_MACRO_DRIVER_SYMBOLS` only, and
    `"gold"`/`"silver"` inside `_OPTIONAL_MACRO_DRIVERS` only. The
    display labels `"XAU"`/`"XAG"` must NOT appear in `contract.py`.

FAIL: (a) `"GC=F" in config.NON_TRADABLE_SYMBOLS` is False, OR

R8 — Graceful degradation on fetch failure

corresponding spot-metals cell renders the same fallback string the
tradables-grid uses (`"N/A"`). The pipeline does not HALT; the run
status does not change; the notification path is not suppressed.

exception from `render_dashboard_html()`, (b) causes a non-zero
pipeline exit, (c) flips run status (`TRADES | NO TRADE | HALT`),
(d) suppresses the hourly notification, or (e) propagates to any

→ `ingestion.fetch_all_quotes()` reads `ALL_SYMBOLS`
  (composed from `MACRO_DRIVERS + INDICES + COMMODITIES + HIGH_BETA`)

strict additive consumer of the existing macro_drivers payload
plumbing pioneered by PRD-122 for oil. `NON_TRADABLE_SYMBOLS`
(composed from `MACRO_DRIVERS`) automatically fences GC=F/SI=F out
