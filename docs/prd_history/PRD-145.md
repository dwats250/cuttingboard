# PRD-145 — Sequencing-gate parser keys on row-owner cell only

STATUS
COMPLETE

LANE
MICRO

GOAL
The UserPromptSubmit sequencing-gate hook at `.claude/hooks/prd_eval.sh` emits false-positive "PRD-NNN still IN PROGRESS with no commit" warnings for PRDs that are already COMPLETE. Observed twice this session against PRD-141 and PRD-143. Root cause: the Python block that builds the `in_progress` set runs `re.findall(r'PRD-(\d+)', line)` against the entire registry row, so a row like `| PRD-142 | — | PATCH PRD-141 ... | IN PROGRESS | ...` yields `{141, 142}` — the title cell's "PATCH PRD-141" reference is treated as another IN PROGRESS row owner. Fix the parser to read the row-owner cell (first `|`-delimited column) only, so the IN PROGRESS set contains exactly the PRDs whose own rows are IN PROGRESS. No sequencing semantics change; no other hook behavior changes.

SCOPE
- Replace the `in_progress` builder inside `.claude/hooks/prd_eval.sh` SEQUENCE_WARNING block with cell-scoped extraction:
  - Skip lines that do not start with `|`.
  - Skip lines without `IN PROGRESS`.
  - Split on `|`, take cells[1] as the row-owner cell.
  - Apply `re.findall(r'PRD-(\d+)', owner_cell)` to that cell only.
- Leave the prompt-side `submitted` extractor (line 117) alone — it parses the user prompt, not the registry, so it is not subject to this bug.
- No change to skip-phrase logic, the REGISTRY GAP block, the PRD-body detection regex, or any other branch.
