## WHY NOW

`logs/market_map.json` is a single shared artifact written by BOTH the
default/fixture pipeline (`runtime.py:361-363`) and the hourly pipeline
(`runtime.py:624-625`) to the same path (`MARKET_MAP_PATH`, `runtime.py:176`).
Every other hourly artifact is isolated behind a `latest_hourly_*` prefix
(`LATEST_HOURLY_RUN_PATH` `:172`, `LATEST_HOURLY_CONTRACT_PATH` `:173`,
`LATEST_HOURLY_PAYLOAD_PATH` `:174`) — market_map is the lone exception. Because
the renderer reads market_map from `logs/market_map.json`
(`dashboard_renderer.py:2297`) while the hourly workflow points payload/run at
the hourly-prefixed files (`hourly_alert.yml:94-97`), any local process that
rewrites the shared market_map between hourly emit and hourly render breaks the
PRD-118 R3 lineage-coherence gate (`validate_coherent_publish`,
`dashboard_renderer.py:362-427`, exact generation_id equality). CI masks this
by running atomically; local dev does not. Surfaced by the 2026-05-27
verification recon (PRD-158 §11).

## GOAL

Give the hourly pipeline its own market_map artifact so the shared
`logs/market_map.json` can never be the source of an hourly render's lineage
mismatch — closing the local-dev PRD-118 R3 coherence hazard without touching
the coherence gate itself.

## SCOPE

  (e.g. `LATEST_HOURLY_MARKET_MAP_PATH = logs/latest_hourly_market_map.json`),

## REQUIREMENTS

R1 — Hourly pipeline writes an isolated market_map artifact and leaves the
shared file untouched.
The hourly notify run writes its market_map to `logs/latest_hourly_market_map.json`
and does NOT modify the content or mtime of `logs/market_map.json` during that
run.

FAIL: after an hourly notify run, `logs/latest_hourly_market_map.json` does not
exist, OR the shared `logs/market_map.json` content/mtime changed during the
hourly run.

R2 — Renderer accepts an explicit market_map path.
The renderer CLI accepts `--market-map-path`; when supplied it loads market_map
from that path, and when omitted it defaults to `<logs-dir>/market_map.json`
(current behavior preserved).

FAIL: the renderer has no `--market-map-path` argument, OR supplying it does not
change which file market_map is loaded from, OR omitting it changes the default
path away from `<logs-dir>/market_map.json`.

R3 — Hourly workflow points the renderer at the hourly market_map.
`.github/workflows/hourly_alert.yml` invokes the renderer with the hourly
market_map path and force-adds the new artifact.

FAIL: the hourly workflow's renderer invocation does not pass the hourly
market_map path, OR the new artifact is not in the workflow's force-add set.

R4 — The fix closes the actual hazard end-to-end.
With the shared `logs/market_map.json` carrying a *mismatched* generation_id and
`logs/latest_hourly_market_map.json` matching the hourly payload/run
generation_id, rendering `ui/dashboard.html` with
`--market-map-path logs/latest_hourly_market_map.json` succeeds (no
`generation_id` mismatch) — proving the explicit path flows through both the
CLI pre-validation read and the `write_dashboard()` validation read, and that
the poisoned shared file is no longer consulted.

FAIL: a render with a mismatched shared `logs/market_map.json` but a matching
hourly market_map still raises a `generation_id` mismatch (i.e. some load site
still reads the shared file), OR a same-run coherent triple is rejected solely
because of the path change.

## DESIGN DECISIONS (resolved)

- **D1 — RESOLVED to Approach A.** Dedicated hourly artifact +
  `--market-map-path`; mirrors the existing `latest_hourly_*` convention. B
  (atomic four-write) declined. Dustin's call, 2026-06-05.
- **D2 — RESOLVED to hourly-only.** The default/fixture path stays
  `logs/market_map.json`; only the hourly read/write is isolated.
- **D3 — RESOLVED to (b).** The hourly previous-map read targets
  `logs/latest_hourly_market_map.json` for symmetric isolation — no residual
  shared-file disk dependency. `_load_previous_market_map` and
  `_write_market_map_file` take an explicit `path` (default `MARKET_MAP_PATH`)
  so default/fixture behavior is byte-identical.

## DATA FLOW

hourly run → `_write_market_map_file(hourly path)` → `logs/latest_hourly_market_map.json`
hourly workflow → renderer `--market-map-path logs/latest_hourly_market_map.json`
renderer → `_resolve_market_map(explicit path)` → `validate_coherent_publish` (unchanged)

## FAIL CONDITIONS

- Hourly run writes the shared `logs/market_map.json` (R1).
- Renderer lacks a working `--market-map-path` / default regresses (R2).
- Hourly workflow doesn't point at / force-add the hourly market_map (R3).
- Same-run hourly render fails coherence due to the path change (R4).
