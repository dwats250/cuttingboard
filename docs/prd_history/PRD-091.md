# PRD-091 - Candidate Validation Context

**Status:** COMPLETE
**Owner:** Codex

---

## GOAL

Render setup context for A+/A/B candidates using existing fields that the renderer already receives but currently ignores. The `preferred_trade_structure` and `what_to_look_for` fields are computed by the market_map engine and present in every symbol entry. Surfacing them on high-grade candidate cards shows the trader what to watch for without changing any data, grading, or trade logic.

---

## SCOPE

1. Modify `cuttingboard/delivery/dashboard_renderer.py`.
2. Modify `tests/test_dashboard_renderer.py`.
3. Add a module-level constant `_UNAVAILABLE_WATCH` to `dashboard_renderer.py` with the value `"Market data unavailable for this run; review during live market session."`.
4. In `_render_candidate_card`, inside the `if is_high:` block, after the REASON field and before the `level_anchor` assignment:
   - If `entry.get("preferred_trade_structure")` is not `None`, render it under label `PLAY`.
   - For each item in `entry.get("what_to_look_for") or []`: if `item` is truthy and `item != _UNAVAILABLE_WATCH`, render it under label `WATCH`.
5. C/D/F card rendering (`if not is_high:` branch) is not modified.
6. Do not modify `_mm_symbol` or any other test helper.
7. Do not edit `ui/dashboard.html` or `ui/index.html`.

## REQUIREMENTS

- **R1:** `_UNAVAILABLE_WATCH` is defined as a module-level constant in `dashboard_renderer.py` with the exact value `"Market data unavailable for this run; review during live market session."`.
  FAIL: The string is absent from `dashboard_renderer.py` as a named constant.

- **R2:** For an A+/A/B candidate card where `preferred_trade_structure` is a non-None string, the value renders under label `PLAY` in the HTML output.
  FAIL: The `preferred_trade_structure` value is absent from rendered HTML when the field is non-None.

- **R3:** For an A+/A/B candidate card where `what_to_look_for` contains valid items (non-empty, not equal to `_UNAVAILABLE_WATCH`), each valid item renders under label `WATCH` in the HTML output.
  FAIL: Any valid `what_to_look_for` item is absent from rendered HTML.

- **R4:** For an A+/A/B candidate card where `preferred_trade_structure` is `None` and `what_to_look_for` is `[]`, neither `PLAY` nor `WATCH` labels appear in rendered HTML.
  FAIL: `PLAY` or `WATCH` label appears when fields are empty/None.

- **R5:** For an A+/A/B candidate card where `what_to_look_for` contains only `_UNAVAILABLE_WATCH`, no `WATCH` label appears in rendered HTML.
  FAIL: `WATCH` label appears when the only item is the unavailable sentinel.

- **R6:** For a C, D, or F candidate card, neither `PLAY` nor `WATCH` labels appear in rendered HTML regardless of the field values.
  FAIL: `PLAY` or `WATCH` label appears in any non-high-grade render.

- **R7:** All existing dashboard renderer tests pass without modification.
  FAIL: Any existing test fails after this change.

## IMPLEMENTATION NOTES

Two changes to `dashboard_renderer.py`.

**1. New module-level constant** (after `_HIGH_GRADES` at line 45):

```python
_UNAVAILABLE_WATCH = "Market data unavailable for this run; review during live market session."
```

**2. New rendering block in `_render_candidate_card`**, inserted inside `if is_high:` after the REASON block (after line 734) and before `level_anchor = ...` (line 736):

```python
        pts = entry.get("preferred_trade_structure")
        if pts is not None:
            w(f'  <div class="label">PLAY</div><div class="value">{_esc(pts)}</div>')
        for item in (entry.get("what_to_look_for") or []):
            if item and item != _UNAVAILABLE_WATCH:
                w(f'  <div class="label">WATCH</div><div class="value">{_esc(item)}</div>')
```

No new CSS required — `label` and `value` classes already apply. No new parameters. No new imports. No changes to `_mm_symbol` or any test helper.

Before: `preferred_trade_structure` and `what_to_look_for` were computed by the engine and written to every market_map symbol entry, but silently ignored by the renderer.

After: A+/A/B cards show what to watch for. C/D/F cards are unchanged.
