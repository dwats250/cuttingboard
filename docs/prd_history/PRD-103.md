PRD-103 — Dashboard Data Contract Gap Patch

STATUS
COMPLETE

GOAL
Patch the system_state contract, hourly run summary, payload adapter, and dashboard renderer so that outcome, permission, confidence, and reason are deterministic, non-null, and truthfully displayed in the live dashboard.

ROOT CAUSE
missing fail condition — `_build_hourly_run_summary()` hardcodes `permission=None`, `_build_system_state()` omits `confidence`, and no test asserted these fields produce real values in the hourly path, allowing the permission display to silently degrade to a dash and "Changes Since Last Run" to emit SOURCE_MISSING instead of a deterministic label.

SCOPE
- Add `confidence` to `_build_system_state()` in contract.py
- Inject `outcome`, `permission`, `reason` into `contract["system_state"]` in runtime.py after contract build
- Fix `"permission": None` hardcode in `_build_hourly_run_summary()` to derive from posture
- Extend payload adapter `build_report_payload()` to carry `outcome`, `confidence`, `permission` in `summary`
- Update dashboard renderer: fallback Permission display from `payload["summary"]` when `run.get("permission")` is None; replace SOURCE_MISSING with NO_PREVIOUS_RUN when `previous_run is None`
- Deterministic tests for all five changes
- Regenerate `ui/dashboard.html` + `ui/index.html`

OUT OF SCOPE
- CSS or layout changes
- New trade logic or qualification changes
- Market map grading changes
- Symbol scoring changes
- Notification logic changes
- Broad refactoring of any module

REQUIREMENTS

R1 — system_state.confidence emitted by contract builder
`_build_system_state()` in contract.py returns a dict that includes `"confidence"`: `float(regime.confidence)` when regime is not None, `None` when regime is None.

FAIL: `contract["system_state"].get("confidence")` is absent or non-float when regime is provided.

R2 — system_state carries outcome, permission, reason after runtime build
After `contract["outcome"] = outcome` is set in runtime.py, `contract["system_state"]` contains: `"outcome"` (matching `contract["outcome"]`), `"permission"` (derived from posture via `_PERMISSION_LINES`, or `"No trades permitted. System halted."` if system_halted), `"reason"` (equals `stay_flat_reason` if set, else None).

FAIL: Any of outcome/permission/reason absent from `contract["system_state"]` in the production contract path.

R3 — hourly run summary has non-null permission string
`_build_hourly_run_summary()` in runtime.py derives `permission` from the posture label using `_PERMISSION_LINES` instead of hardcoding `None`. The produced dict has a non-None string permission when regime is available.

FAIL: `latest_hourly_run.json` permission field is null after a successful live run.

R4 — payload summary carries outcome, confidence, permission
`build_report_payload()` in payload.py includes `"outcome"`, `"confidence"`, `"permission"` in `payload["summary"]`, sourced from `contract["system_state"]`. `assert_valid_payload()` passes with these fields present.

FAIL: `latest_hourly_payload.json` summary missing any of outcome, confidence, permission.

R5 — dashboard Permission field shows real text when payload has permission
The dashboard renderer reads `run.get("permission")` first; if None, falls back to `payload.get("summary", {}).get("permission")`. The Permission label renders the resolved string, not `&#8212;`, when either source has a value.

FAIL: Dashboard HTML Permission field shows `&#8212;` when payload["summary"]["permission"] is non-null.

R6 — Changes Since Last Run renders NO_PREVIOUS_RUN instead of SOURCE_MISSING
When `previous_run is None`, the "Changes Since Last Run" block renders `NO_PREVIOUS_RUN` not `SOURCE_MISSING`. SOURCE_MISSING is reserved for actual missing/unparseable source artifacts.

FAIL: Dashboard HTML contains `SOURCE_MISSING` in the run-delta section when `previous_run is None`.

R7 — History and diagnostics panels never render empty boxes
History panel: shows either populated rows or explicit `NO_HISTORY`. Artifact diagnostics: always renders at least the payload and run source lines. Neither panel renders an empty `<div class="value"></div>`.

FAIL: Any of the two panels emits an empty visible div when rendered with default inputs.

DATA FLOW
contract["system_state"] populated by _build_system_state() [confidence added]
  → runtime.py injects outcome/permission/reason into system_state after build
  → _build_hourly_run_summary() derives permission from posture [not None]
  → build_report_payload() reads system_state → payload["summary"]["outcome/confidence/permission"]
  → dashboard_renderer reads run["permission"] || payload["summary"]["permission"] → renders Permission label
  → dashboard_renderer renders NO_PREVIOUS_RUN when previous_run is None

- Dashboard renders SOURCE_MISSING in run-delta block
- Dashboard renders &#8212; for Permission when payload has a permission string
