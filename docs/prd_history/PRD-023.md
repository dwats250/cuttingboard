PRD-023 — GLD–DXY Correlation Policy Layer

STATUS
COMPLETE
Commit: 314ca46

GOAL
Add a GLD–DXY correlation advisory layer that computes a risk_modifier and correlation state without mutating qualification results.

SCOPE
- Add cuttingboard/correlation.py: GLD–DXY correlation computation, ALIGNED/CONFLICT/NEUTRAL state
- Add cuttingboard/trade_policy.py: risk_modifier derivation from correlation state
- Update cuttingboard/contract.py: add correlation block with state and risk_modifier fields
- Update cuttingboard/options.py: read risk_modifier (advisory only, no strike changes)
- Update cuttingboard/runtime.py: wire correlation computation into pipeline
- Add tests/test_correlation.py and tests/test_trade_policy.py

OUT OF SCOPE
- No mutation of qualification results based on correlation
- No changes to regime vote model
- No changes to strike selection logic

FILES
A cuttingboard/correlation.py
A cuttingboard/trade_policy.py
M cuttingboard/contract.py
M cuttingboard/options.py
M cuttingboard/config.py
M cuttingboard/runtime.py

A tests/test_correlation.py
A tests/test_trade_policy.py

REQUIREMENTS

R1 — Correlation Computation
correlation.py MUST compute GLD–DXY correlation and return one of: ALIGNED / CONFLICT / NEUTRAL.

FAIL: correlation.py returns a state value outside those three.

R2 — Risk Modifier Advisory Only
risk_modifier MUST appear in the contract as an advisory field.
risk_modifier MUST NOT alter qualification gate outcomes or strike prices.

FAIL: A candidate's qualification result changes based on risk_modifier.

R3 — Contract Integration
contract MUST include a correlation block with: state, risk_modifier.

FAIL: Contract is missing correlation block or either field.

DATA FLOW
GLD price + DXY price → correlation.py → state → trade_policy.py → risk_modifier → contract.correlation block

FAIL CONDITIONS
- Correlation state outside ALIGNED/CONFLICT/NEUTRAL
- risk_modifier mutates qualification result
- Contract missing correlation block

VALIDATION
Run: pytest tests/test_correlation.py tests/test_trade_policy.py -q
Manual: live run — confirm correlation block in contract, qualification results unchanged from prior run.
