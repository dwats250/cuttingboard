# PRD-077 — Sunday Futures Pre-Report

**STATUS:** COMPLETE

---

## GOAL

Generate a deterministic Sunday pre-market context report using live
futures/metals/FX/crypto data without producing trade decisions or
mutating the core pipeline.

---

## SCOPE

- Add Sunday post-15:30 ET detection path
- Produce a valid pipeline run with regime classification, macro drivers
  snapshot, and market_map generation
- Force output contract:
  - outcome: `"NO_TRADE"`
  - `system_state.stay_flat_reason`: `"PREMARKET_CONTEXT"`
  - `system_state.session_type`: `"SUNDAY_PREMARKET"`
- Write standard artifacts:
  - `logs/latest_hourly_contract.json`
  - `logs/latest_hourly_payload.json`
  - `logs/latest_hourly_run.json`
  - `logs/market_map.json`
- Propagate to payload:
  - `payload.meta.session_type = "SUNDAY_PREMARKET"`
- Dashboard banner: `"SUNDAY PRE-MARKET CONTEXT — NO CASH SESSION"`
- Notifications (if enabled): prefix title with `[PREMARKET]`

---

## OUT OF SCOPE

- No trade approvals
- No qualification gating changes
- No execution policy changes
- No candidate forcing or fabrication
- No new data sources
- No UI redesign beyond banner/label
- No schedule system changes beyond time gate

## REQUIREMENTS

### R1 — Time Gate

Implementation in `cli_main` + `_resolve_effective_mode` (`runtime.py`):

1. Before calling `_resolve_effective_mode`, compute:
   ```python
   now_et = time_utils.convert_utc_to_et(datetime.now(timezone.utc))
   ```
2. Add `now_et: datetime` as a third parameter to `_resolve_effective_mode`.
3. Logic:
   ```python
   if (requested_mode == MODE_LIVE
           and run_date.weekday() == 6
           and now_et.hour * 60 + now_et.minute >= 15 * 60 + 30):
       return MODE_SUNDAY
   # else: existing fallthrough
   ```

- **FAIL:** Sunday run before 15:30 ET resolves to `MODE_SUNDAY`
- **FAIL:** Sunday run at or after 15:30 ET does not resolve to `MODE_SUNDAY`

---

### R2 — Contract Override

Injection point: `runtime.py`, immediately after `build_pipeline_output_contract`
returns and outcome is set (mirrors existing pattern at line 901):

```python
contract["outcome"] = OUTCOME_NO_TRADE
if mode == MODE_SUNDAY:
    contract["system_state"]["stay_flat_reason"] = "PREMARKET_CONTEXT"
```

Must not depend on qualification output.

- **FAIL:** `outcome != "NO_TRADE"` on any Sunday run
- **FAIL:** `system_state.stay_flat_reason != "PREMARKET_CONTEXT"` on Sunday run

---

### R3 — Pipeline Execution (Constraint, no new implementation)

Existing `MODE_SUNDAY` behavior: regime, macro pressure, market_map already
run. No code changes required. Must not regress.

- **FAIL:** regime or market_map absent from Sunday run artifacts

---

### R4 — Contract → Payload Propagation

Injection point: `runtime.py`, same block as R2:
```python
contract["system_state"]["session_type"] = "SUNDAY_PREMARKET"
```

In `build_report_payload` (`payload.py`): after meta is assembled, add:
```python
session_type = ss.get("session_type")
if session_type is not None:
    meta["session_type"] = session_type
```

`assert_valid_payload` uses `_require_keys` (checks missing, not extra) —
no validator changes needed.

- **FAIL:** `payload.meta.session_type` missing when mode is `SUNDAY_PREMARKET`
- **FAIL:** `payload.meta.session_type` present on any non-Sunday run

---

### R5 — Dashboard Banner

In `dashboard_renderer.py`: after loading payload, read
```python
session_type = meta.get("session_type")
```
Render banner at top of output if `session_type == "SUNDAY_PREMARKET"`:
```
SUNDAY PRE-MARKET CONTEXT — NO CASH SESSION
```
No other layout changes.

- **FAIL:** banner absent on Sunday run
- **FAIL:** banner present on weekday run

---

### R6 — Notification Formatting

In `build_notification_message` (`output.py`): read
`contract["system_state"].get("session_type")`.
If `== "SUNDAY_PREMARKET"`, prefix title: `f"[PREMARKET] {title}"`.

Body must not contain `"TRADE ACTIVE"` or `"ALLOW_TRADE"` (already satisfied
since Sunday runs produce no trade candidates).

- **FAIL:** notification title missing `[PREMARKET]` on Sunday run

---

### R7 — Determinism

Identical inputs → identical artifacts. No new randomness introduced.

- **FAIL:** artifacts differ between two runs with same inputs

## DATA FLOW

```
cli_main (runtime.py)
  → compute now_et
  → _resolve_effective_mode(mode, run_date, now_et)
     Sunday + >=15:30 ET → MODE_SUNDAY
     Sunday + <15:30 ET  → MODE_LIVE

_run_pipeline(MODE_SUNDAY)
  → regime, macro, market_map run (unchanged)
  → qualification skipped (unchanged, line 704)
  → build_pipeline_output_contract(...)
  → contract["outcome"] = OUTCOME_NO_TRADE
  → contract["system_state"]["stay_flat_reason"] = "PREMARKET_CONTEXT"
  → contract["system_state"]["session_type"] = "SUNDAY_PREMARKET"

_write_payload_artifacts(contract) / _write_hourly_artifacts(...)
  → build_report_payload(contract)
     ss.get("session_type") → payload["meta"]["session_type"]

dashboard_renderer.py
  → payload.meta.session_type == "SUNDAY_PREMARKET" → render banner

build_notification_message(contract) [if notification enabled]
  → session_type detected → prepend [PREMARKET]
```

---

## ACCEPTANCE CRITERIA

- Sunday >=15:30 ET → `session_type` in contract, payload, dashboard banner
- Sunday <15:30 ET → no `session_type`, no banner, normal pipeline
- `outcome == "NO_TRADE"` on every Sunday run
- No regression in weekday run behavior
