PRD-083 — Dashboard Data Freshness and Source Visibility

STATUS: COMPLETE

GOAL:
Make dashboard unavailable/stale data states explicit without changing trading logic, data fetching, candidate qualification, macro regime calculation, notification logic, market_map writer/schema, or dashboard styling.

SCOPE:
Dashboard renderer/data-display layer only.

OUT OF SCOPE:
- No new APIs.
- No quote-fetching changes.
- No trading logic changes.
- No candidate qualification changes.
- No macro regime changes.
- No notification changes.
- No dashboard redesign.
- No new sections.
- No market_map writer/schema changes.
- No contract schema changes.

FILES:
- cuttingboard/delivery/dashboard_renderer.py
- tests/test_dashboard_renderer.py
- tests/test_dash_timestamps.py
- ui/dashboard.html
- ui/index.html

VERIFIED DATA FLOW:
- Main timestamp:
  Existing payload/contract timestamp used by format_dashboard_timestamp(value) -> Timestamp block.
  Freshness display should reuse the same timestamp value already rendered in the dashboard.

- Macro Tape tradable values:
  market_map["symbols"][sym]["current_price"] -> Macro Tape tradable ticker rows.
  Existing _format_tape_value returns "--" when value is not a finite number.
  Replace silent "--" for tradable tickers with DATA_UNAVAILABLE.

- Macro Pressure:
  Existing macro pressure render block in dashboard_renderer.py -> volatility/dollar/rates/bitcoin/overall labels.
  Missing pressure object or missing pressure key -> render FIELD_MISSING.
  Existing valid pressure value -> render unchanged.

- Candidate Board:
  logs/market_map.json is loaded via _load_json_optional and passed as dict | None into render_dashboard_html.
  None means source missing.
  market_map["symbols"] feeds candidate-board/market-map visibility.
  Existing generic "MARKET MAP UNAVAILABLE" must be replaced with the most specific knowable reason:
  - SOURCE_MISSING
  - PARSE_ERROR
  - STALE
  - NO_CANDIDATES
  - DATA_UNAVAILABLE

- Candidate Board parse error:
  Do not modify _load_json_optional globally.
  Wrap only the _load_json_optional call for logs/market_map.json in render_dashboard_html.
  RuntimeError from malformed market_map JSON -> Candidate Board PARSE_ERROR state.

- Changes Since Last Run:
  Existing run-delta block.
  If previous_run is None or unavailable, render SOURCE_MISSING.
  If previous_run exists but required fields are absent, render FIELD_MISSING.
  Existing valid display remains unchanged.

- History:
  Existing history block.
  If history_runs is empty or None, render NO_HISTORY rather than a blank/dead section.
  Existing valid history display remains unchanged.

FRESHNESS RULE:
Use the existing repo freshness constant if already available in dashboard renderer scope.

If no existing constant is safely available without dependency risk, define a renderer-local constant:

DASHBOARD_STALE_AFTER_SECONDS = 300

Main timestamp freshness:
- Use the same timestamp value already displayed in the dashboard.
- If timestamp is parseable and age <= threshold -> FRESH.
- If timestamp is parseable and age > threshold -> STALE.
- If timestamp parsing fails -> PARSE_ERROR.

market_map freshness:
- market_map currently has no embedded timestamp.
- Use os.path.getmtime("logs/market_map.json") for market_map freshness only.
- If file mtime age <= threshold -> FRESH.
- If file mtime age > threshold -> STALE.
- If file missing -> SOURCE_MISSING.
- If JSON parse fails -> PARSE_ERROR.
- Do not use file mtime for other sources unless already used by the renderer.

Implementation note:
Add market_map_path: Path | None = None parameter to render_dashboard_html (following the existing
macro_snapshot_path pattern). Compute mtime and load inside the renderer from that path.
In write_dashboard_from_files, pass market_map_path=logs_dir / "market_map.json" and remove the
manual _load_json_optional call for market_map.

REQUIREMENTS:
R1 - Macro Tape unavailable values
Replace silent "--" tradable ticker placeholders with DATA_UNAVAILABLE when market_map["symbols"][sym]["current_price"] is missing, null, non-numeric, or non-finite.
FAIL if SPY/QQQ/GLD/SLV/XLE render "--" without an explicit unavailable reason.

R2 - Candidate Board unavailable states
Replace generic "MARKET MAP UNAVAILABLE" with the most specific available reason.
FAIL if the renderer can know SOURCE_MISSING, STALE, PARSE_ERROR, NO_CANDIDATES, or DATA_UNAVAILABLE but still renders only "MARKET MAP UNAVAILABLE".

R3 - Main dashboard freshness label
Add compact freshness/source visibility near the existing timestamp block.
Acceptable labels:
- FRESH
- STALE
- SOURCE_MISSING
- PARSE_ERROR
- DATA_UNAVAILABLE
FAIL if the timestamp card gives no data freshness/source status.

R4 - Fail closed
Missing files, missing fields, invalid timestamps, malformed market_map JSON, or malformed optional sidecar data must not crash dashboard rendering.
FAIL if dashboard rendering raises on missing/invalid optional sidecar data.

R5 - Preserve valid data display
When values are available, existing visual display should remain materially unchanged.
FAIL if valid macro pressure labels, valid ticker values, candidate rows, run-delta rows, or history rows are hidden, renamed, or degraded unnecessarily.

R6 - No logic mutation
Do not alter contract schema, runtime scheduling, trade decisions, candidate qualification, macro regime logic, notification output, or market_map writer/schema.
FAIL if non-rendering decision logic changes.

R7 - Null-safe rendering for Macro Pressure, Changes Since Last Run, and History
Macro Pressure: if pressure is None or missing required keys, render FIELD_MISSING instead of raising.
Changes Since Last Run: if previous_run is None or unavailable, render SOURCE_MISSING instead of a blank/dead section.
History: if history_runs is empty or None, render NO_HISTORY instead of a blank/dead section.
FAIL if any of these three blocks crashes or renders silently empty when its source is absent.

TESTS:
Add exactly 6 focused tests.

T1 - missing market map
Input/state: logs/market_map.json is unavailable or market_map argument is None.
Assert: Candidate Board renders SOURCE_MISSING or DATA_UNAVAILABLE, not generic-only MARKET MAP UNAVAILABLE.

T2 - stale market map
Input/state: logs/market_map.json exists and mtime is older than DASHBOARD_STALE_AFTER_SECONDS.
Assert: Candidate Board renders STALE.

T3 - parse-error market map
Input/state: logs/market_map.json is malformed JSON and _load_json_optional raises RuntimeError at the market_map load site.
Assert: Candidate Board renders PARSE_ERROR and dashboard does not crash.

T4 - missing tradable quote field
Input/state: market_map["symbols"] lacks current_price for SPY/QQQ/GLD/SLV/XLE or contains non-finite values.
Assert: DATA_UNAVAILABLE appears and silent "--" is not used for those tradable values without reason.

T5 - available tradable quote value
Input/state: market_map["symbols"][sym]["current_price"] contains a valid finite number.
Assert: value renders normally and DATA_UNAVAILABLE is not shown for that ticker.

T6 - null-safe secondary dashboard sections
Input/state: pressure is None or missing expected keys, previous_run is None, and history_runs is empty or None.
Assert: Macro Pressure renders FIELD_MISSING, Changes Since Last Run renders SOURCE_MISSING, History renders NO_HISTORY, and dashboard rendering does not crash.
