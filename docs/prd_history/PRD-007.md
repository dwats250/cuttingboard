PRD-007 — Imbalance Pullback Entry (FVG)

STATUS
COMPLETE
Commit: ab3d20b8

GOAL
Add Fair Value Gap (FVG) detection and a PULLBACK_IMBALANCE entry mode so the qualification and options layers can route entries through imbalance zones instead of market price.

SCOPE
- Add detect_fvg() pure function and FVGZone dataclass
- Extend QualificationResult with entry_mode and imbalance_zone fields
- Implement deterministic entry_mode resolution: DIRECT vs PULLBACK_IMBALANCE
- Integrate daily OHLCV into qualification pipeline (fallback-safe)
- Update options.py to override entry/stop via imbalance midpoint and zone bounds

OUT OF SCOPE
- No changes to regime layer
- No changes to sector router
- No changes to confirmation layers

FILES
M cuttingboard/config.py
M cuttingboard/intraday_state_engine.py
M cuttingboard/options.py
M cuttingboard/output.py
M cuttingboard/qualification.py

REQUIREMENTS

R1 — FVG Detection
detect_fvg() MUST identify gap zones from daily OHLCV bars.
FVGZone MUST contain: high, low, midpoint, direction.

FAIL: detect_fvg() returns a zone with missing midpoint or direction.

R2 — Entry Mode Resolution
QualificationResult.entry_mode MUST be DIRECT when no FVG zone is present.
QualificationResult.entry_mode MUST be PULLBACK_IMBALANCE when a valid FVG zone exists within range.

FAIL: entry_mode is PULLBACK_IMBALANCE when imbalance_zone is None.

R3 — Options Override
When entry_mode == PULLBACK_IMBALANCE, options.py MUST use imbalance midpoint as entry and zone bounds as stop reference.

FAIL: Options layer uses market price entry when entry_mode is PULLBACK_IMBALANCE.

R4 — OHLCV Fallback Safety
If daily OHLCV fetch fails, qualification MUST proceed with entry_mode=DIRECT.

FAIL: OHLCV fetch failure halts qualification.

DATA FLOW
daily OHLCV → detect_fvg() → FVGZone → QualificationResult.entry_mode → options.py strike/stop override

FAIL CONDITIONS
- PULLBACK_IMBALANCE entry with no zone
- OHLCV failure halts pipeline
- Options layer ignores entry_mode

VALIDATION
Run: pytest -q
Manual: fixture with known FVG zone — confirm entry_mode=PULLBACK_IMBALANCE, options entry matches zone midpoint.
