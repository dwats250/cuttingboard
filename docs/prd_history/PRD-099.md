# PRD-099 - Dashboard Artifact Generation Contract

**Status:** PROPOSED

## Goal

Enforce deterministic, coherent dashboard artifact generations so the published dashboard never renders mixed artifact states caused by out-of-order render/publish timing.

## Scope

- Introduce a deterministic shared `generation_id` propagated across latest run, payload, market map, contract, and dashboard diagnostics.
- Enforce coherent dashboard rendering using generation identity rather than timestamp-only comparisons.
- Formalize publish ordering as run, payload, market map, dashboard render, `ui/index.html` copy, then publish.
- Keep local and GitHub Actions publish behavior aligned.
- Preserve stale market-map protection, deterministic ASCII-safe diagnostics, and artifact contamination protections.

## Out Of Scope

- Trade logic, grading logic, and PRD-098 candidate visibility semantics.
- Dashboard layout redesign.
- Asynchronous publishing, external state stores, or databases.
- Weakened contamination tests or removed timestamp diagnostics.

## Requirements

- R1: Each artifact generation cycle emits one deterministic `generation_id`.
- R2: Normal dashboard rendering requires matching generation IDs across run, payload, and market map artifacts when those IDs are present.
- R3: Mixed generation IDs render explicit `MIXED_ARTIFACTS` degraded state.
- R4: Publish flow runs in deterministic order: run, payload, market map, dashboard render, `ui/index.html` copy, publish.
- R5: Local and CI publish flows use the same render/copy sequence.
- R6: Publishable artifacts are generated with `python3 -m cuttingboard.delivery.dashboard_renderer --output ui/dashboard.html`, then `cp ui/dashboard.html ui/index.html`.
- R7: Published dashboard artifacts continue rejecting local filesystem contamination patterns.
- R8: Dashboard artifact diagnostics include deterministic generation ID metadata.

## Validation

- `python3 -m pytest tests/ -q`
- Verify generation ID equality across coherent artifacts.
- Verify mixed generations trigger `MIXED_ARTIFACTS`.
- Verify publish sequence updates `ui/dashboard.html` then `ui/index.html`.
- Verify contamination checks still pass with `! grep -r "pytest-of-\|/tmp/pytest\|/tmp/" ui/`.
