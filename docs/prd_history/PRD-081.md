PRD-081 — Dashboard Timestamp Display Hardening

STATUS
COMPLETE

GOAL
Add a display-layer formatter that renders timestamps in Pacific time (PT) with the original UTC value shown below it, without mutating any artifact, contract, or runtime value.

SCOPE
- New helper `format_dashboard_timestamp(value: str) -> tuple[str, str]` in `dashboard_renderer.py`
  - Returns `(pacific_line, original_line)` e.g. `("2026-05-04 11:11:28 PT", "2026-05-04 18:11:28 UTC")`
  - Uses `America/Los_Angeles` via `zoneinfo.ZoneInfo` (stdlib, no fixed offset)
  - Labels the display value "PT" (not PST/PDT)
  - On parse failure: returns `("", cleaned_original)` — no crash
  - Never mutates the input string
- Update dashboard header Timestamp block (`dashboard_renderer.py:725-726`) to display both lines
- Update html_renderer.py footer to use the same readable original-UTC string (no PT line needed there)
- Add `tests/test_dash_timestamps.py` with the four required test cases

OUT OF SCOPE
- runtime.py, regime.py, qualification.py, output.py — untouched
- Contract JSON schema — untouched
- Artifact write logic — untouched
- Notification/Telegram timing — untouched
- GitHub workflow schedule — untouched
- history run timestamps in dashboard (line 880 slices HH:MM only — already readable)
- Any JS-side timestamp formatting

FILES
M cuttingboard/delivery/dashboard_renderer.py
M cuttingboard/delivery/html_renderer.py
A tests/test_dash_timestamps.py

REQUIREMENTS

R1 — format_dashboard_timestamp helper
A module-level function `format_dashboard_timestamp(value: str) -> tuple[str, str]` exists in
`cuttingboard/delivery/dashboard_renderer.py`.

Given a Zulu string `"2026-05-04T18:11:28Z"`:
- Returns `("2026-05-04 11:11:28 PT", "2026-05-04 18:11:28 UTC")`

Given an offset string `"2026-05-04T18:11:28+00:00"`:
- Returns a pacific_line with "PT" and an original_line showing the original cleaned for readability

Given an invalid/empty string:
- Returns `("", cleaned_original)` where cleaned_original is the input stripped of T and Z noise
- Does not raise

Input string is never mutated.

FAIL: `format_dashboard_timestamp` is not importable from `cuttingboard.delivery.dashboard_renderer`.
FAIL: Zulu input does not produce a "PT" pacific line.
FAIL: Zulu input does not produce an "UTC" original line.
FAIL: Invalid input raises an exception.
FAIL: Input string value changes after the call.

R2 — Dashboard header renders PT + original
The Timestamp block in the dashboard header (`id="dashboard-header"`) renders:
```
TIMESTAMP
Pacific: 2026-05-04 11:11:28 PT
Original: 2026-05-04 18:11:28 UTC
```
Two `<div class="value">` lines (or equivalent), one for Pacific and one for Original.
When the pacific conversion is unavailable (parse failure), only the Original line renders.

FAIL: Dashboard header renders a raw `2026-05-04T18:11:28Z` mashed ISO string with no spacing.
FAIL: Dashboard header shows no Pacific time when input is a valid Zulu timestamp.
FAIL: Dashboard header crashes or omits the Timestamp block on an invalid timestamp string.

R3 — html_renderer footer readability
`html_renderer.py` footer reads "Generated at YYYY-MM-DD HH:MM:SS UTC" (or the cleaned original)
instead of the raw ISO string. Uses the `original_line` from `format_dashboard_timestamp`.

FAIL: Footer still renders a raw mashed ISO string (e.g., `2026-05-04T18:11:28Z` with no spacing).

R4 — No artifact mutation
The `payload["meta"]["timestamp"]` value in the dict passed to `render_signal_forge_dashboard`
and `render_html` is identical before and after the call.

FAIL: `payload["meta"]["timestamp"]` differs after rendering.

R5 — Focused tests
`tests/test_dash_timestamps.py` contains exactly these four test cases (names may vary):
1. Zulu → pacific_line contains "PT", original_line contains "UTC"
2. Offset → pacific_line contains "PT", original_line is readable
3. Invalid → no exception raised, pacific_line is empty string
4. No mutation — input dict value unchanged after render call

FAIL: Any of the four test cases is absent.
FAIL: Any of the four test cases fails.

DATA FLOW
payload["meta"]["timestamp"]  →  format_dashboard_timestamp(value)  →  (pacific_line, original_line)
pacific_line   →  dashboard header "Pacific:" div (display only)
original_line  →  dashboard header "Original:" div (display only)
original_line  →  html_renderer footer (display only)
payload["meta"]["timestamp"]  →  unchanged in source dict

FAIL CONDITIONS
- format_dashboard_timestamp not importable from dashboard_renderer
- Zulu input produces no "PT" line
- Zulu input produces no "UTC" label on original line
- Invalid input raises instead of returning ("", cleaned)
- Input value mutated after call
- Dashboard header renders raw mashed ISO string
- Dashboard header crashes on invalid timestamp
- Footer still renders raw mashed ISO string
- payload["meta"]["timestamp"] changed after render
