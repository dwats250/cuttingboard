# PRD-121 - PRD Workflow Lane Classification and Review Discipline

STATUS: COMPLETE @ bd14b71
LANE: MICRO

## GOAL

Address three specific weaknesses surfaced by PRD-118 through PRD-120
retrospective evidence, without redesigning the existing governance
baseline:

(a) Small PRDs carry disproportionate ceremony. Add a coarse `LANE`
    axis (MICRO / STANDARD / HIGH-RISK) on top of the existing CLASS
    + Tier matrix so authors can declare the ceremony tier up front.

(b) Same-context review is currently described as "Claude's
    independent review" but is not adversarial when the author and
    reviewer are the same instance in the same context window. Add a
    Review Independence attestation that names the actual independence
    method, and require genuine independence for the HIGH-RISK lane.

(c) Mapping tables in recent PRDs (PRD-120 Trend Structure block) have
    produced unreachable branches because precedence rows referenced
    derived variables that shadowed later rows. Add a per-row
    reachability checklist to the PRD review template.

Also slim `docs/PROJECT_STATE.md` to verifiable fields only.

This is a narrow amendment grounded in PRD-118-120 observations. Not
a governance rewrite. The existing CLASS Matrix, HIGH-RISK FILES,
CHANGE SURFACE trigger, MAX EXPECTED DELTA, Adjudication Trigger, and
Forbidden Patterns Catalog are unchanged.

## TERMINOLOGY

- **LANE**: a ceremony-tier axis declared in every PRD header.
  Values: `MICRO | STANDARD | HIGH-RISK`. Lane is orthogonal to
  CLASS (GOVERNANCE / SIDECAR / CONSUMER / EXECUTION / CONTRACT /
  INFRA) and Tier (T0-T3).

- **MICRO lane**: a PRD whose change qualifies under the existing
  `docs/PRD_MICRO_TEMPLATE.md` eligibility criteria (docs-only,
  hook-only, test-helper-only, or process-only; ≤ 20 production-code
  lines; no HIGH-RISK FILES intersect; one deterministic FAIL
  condition).

- **STANDARD lane**: a PRD that does not qualify for MICRO and whose
  `FILES` list does not intersect any HIGH-RISK FILES entry in its
  CLASS row of the CLASS Matrix.

- **HIGH-RISK lane**: a PRD whose `FILES` list intersects any
  HIGH-RISK FILES entry in its CLASS row of the CLASS Matrix, OR
  whose CLASS is EXECUTION or CONTRACT, OR whose default Tier is T0.

- **Review Independence**: the property that the review artifact was
  produced under one of the following conditions:
  - `fresh-context`: a separate session from the PRD draft, in which
    the PRD-draft text was not present in the context window at the
    time of review.
  - `different-model`: a model architecture or vendor different from
    the PRD draft author (e.g., Claude draft + Codex review, or vice
    versa).
  - `same-context`: same instance, same session as the PRD draft.
    Acceptable for MICRO and STANDARD; INSUFFICIENT for HIGH-RISK.

- **Mapping table**: any tabular precedence structure inside a PRD
  where rows are evaluated in order and the first match wins
  (e.g., PRD-120 SOURCE-HEALTH MAPPING per block, regime vote
  matrix, qualification gate ordering, source classification by
  enum).

      review INSUFFICIENT; Review Independence must be
      `fresh-context` OR `different-model`.

    `MICRO | STANDARD | HIGH-RISK`.

    FAIL: Either template lacks the `LANE:` line in the header

R4. **Review Independence attestation block in

R5. **Mapping-Table Reachability Checklist in
    `docs/PRD_REVIEW_TEMPLATE.md`.** A new
    `## Mapping-Table Reachability Checklist` section requires the
    reviewer to confirm, for each mapping-table row in the PRD:
    1. The row condition is expressed in terms of a raw input

       rows.
    2. If a derived variable is referenced, the derivation function
       is named.
    3. The precedence order is explicit (e.g., "first match wins"
       or numbered ordering).
    4. At least one test fixture in the PRD's R14 (or equivalent
       test list) is named in which this row is the matching row.

    FAIL: The template lacks the section, or any one of the four
    sub-checks is omitted.

R10. **ASCII-only.** All FILES entries remain ASCII per CLAUDE.md
     governance discipline.

R11. **Lane Downgrade Prohibition.** A PRD whose `FILES` list
     intersects any HIGH-RISK FILES entry for its CLASS in the
     `## CLASS Matrix`, OR whose CLASS is `EXECUTION` or `CONTRACT`,
     OR whose default Tier is T0, MUST declare `LANE: HIGH-RISK`.
     Authors and reviewers cannot select MICRO or STANDARD for such
     changes regardless of diff size. Lane is a ceremony axis; it
     cannot be used to bypass the review intensity required by the
     existing CLASS Matrix.

     FAIL: Any PRD merged after PRD-121 with HIGH-RISK FILES
     intersection, CLASS EXECUTION, CLASS CONTRACT, or default
     Tier T0 declares `LANE: MICRO` or `LANE: STANDARD` in its
     header.

R12. **MICRO Eligibility Safety Net.** `LANE: MICRO` is invalid if
     the change alters any of the following, even when the diff is
     ≤ 20 production-code lines:

     - Executable trading-decision behavior (regime classification,
       qualification gates, sizing, posture, contract assembly).
     - Artifact contracts or payload / market_map / run schemas.
     - Publication gates (`validate_coherent_publish`, freshness
       windows, coherent-generation checks, mixed-artifact gating).
     - Runtime artifact write ordering.
     - Dashboard truth semantics (renderer-derived decision-bearing
       values, threshold-to-label synthesis, source-health
       classification, lineage classification).
     - Notification truth semantics (block reason, decision label,
       artifact correlation).

     This rule is additive to the existing
     `docs/PRD_MICRO_TEMPLATE.md` eligibility criteria — it
     enumerates the specific behavior surfaces that disqualify
     a change from MICRO even if the diff appears small.

     FAIL: A PRD merged after PRD-121 declared `LANE: MICRO` whose
     FILES list contains any of: `cuttingboard/runtime.py`,
     `cuttingboard/output.py`, `cuttingboard/payload.py`,
     `cuttingboard/contract.py`, `cuttingboard/regime.py`,
     `cuttingboard/qualification.py`, `cuttingboard/market_map.py`,
     `cuttingboard/execution_policy.py`,
     `cuttingboard/trade_decision.py`, `cuttingboard/trade_policy.py`,
     any new sidecar writer, any schema definition file, or any
     function flagged by the Forbidden Patterns Catalog under
     `renderer-derived semantics` / `threshold→label synthesis`.

## DATA FLOW

PRD author drafts PRD-NNN.md
  -> declares STATUS: PROPOSED + LANE: <value> in header
  -> author OR cross-review hook authors PRD-NNN.review.codex.md
     -> reviewer ticks Review Independence checkbox (1 of 3)
     -> reviewer fills Mapping-Table Reachability Checklist if any
        mapping tables present in PRD
  -> implementation proceeds, scope-locked to FILES list (existing rule)
  -> closeout: PRD-NNN.md STATUS: COMPLETE @ <hash>, registry row,
     PROJECT_STATE update (slimmed format)

The LANE axis adds no new artifacts. It adds a declaration in two
existing artifacts (PRD draft header, review file) and a
verification checkpoint at the cross-review gate.
