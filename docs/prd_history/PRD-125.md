PRD-125 — OHLCV Cache Freshness Contract

STATUS
IN PROGRESS

LANE
STANDARD

CLASS
EXECUTION + PATCH

WHY NOW
Codex read-only review found daily OHLCV cache freshness is wired to
the 5-minute quote freshness threshold, causing avoidable live refreshes
and derived-history loss despite the documented 12-hour cache contract.

MAX EXPECTED DELTA
<= 12 production LOC in `cuttingboard/ingestion.py`. Test changes are
limited to `tests/test_derived.py`.

GOAL
Make daily OHLCV cache freshness use `config.OHLCV_STALE_HOURS` while
preserving quote freshness under `config.FRESHNESS_SECONDS`.

SCOPE
- Correct `_is_fresh_ohlcv_cache()` to use `config.OHLCV_STALE_HOURS`.
- Test that OHLCV cache younger than the TTL is accepted.
- Test that OHLCV cache at or older than the TTL is rejected.
- Preserve a regression test proving quote freshness still uses
  `config.FRESHNESS_SECONDS`.
- Update any test encoding the incorrect 5-minute OHLCV TTL.

OUT OF SCOPE
- No quote freshness behavior changes.
- No live fetch policy or fixture mode changes.
- No `runtime.py`, notification, dashboard, payload, contract, market_map,
  logs, reports, UI, or artifact-publication changes.
- No changes to trading logic, qualification, or cache paths.
- No changes to `config.OHLCV_STALE_HOURS` or `config.FRESHNESS_SECONDS`.

ROOT CAUSE
ambiguous requirement — the code reused the quote freshness constant for
daily OHLCV cache freshness despite a separate documented OHLCV TTL.

CHANGE SURFACE
- `cuttingboard/ingestion.py`: `_is_fresh_ohlcv_cache()` only. No
  signature, live fetch policy, cache path, or quote ingestion changes.
- `tests/test_derived.py`: OHLCV cache TTL and quote freshness tests only.
- `docs/PRD_REGISTRY.md`: when implementation begins, add exactly one
  PRD-125 row with status `IN PROGRESS` and a link to
  `docs/prd_history/PRD-125.md`; after merge, update that same row to
  `COMPLETE` and record the implementation commit hash.
- `docs/PROJECT_STATE.md`: update active/completed PRD state only.

REQUIREMENTS

R1 — OHLCV cache younger than TTL is fresh
`_is_fresh_ohlcv_cache()` MUST return true iff the cached daily OHLCV
latest-bar age is >= 0 and < `config.OHLCV_STALE_HOURS` hours.

FAIL: a cached OHLCV DataFrame with latest index timestamp younger than
`config.OHLCV_STALE_HOURS` is rejected as stale.

R2 — OHLCV cache at or beyond TTL is stale
`_is_fresh_ohlcv_cache()` MUST return false when the cached daily OHLCV
latest-bar age is >= `config.OHLCV_STALE_HOURS` hours.

FAIL: a cached OHLCV DataFrame with latest index timestamp at or older
than `config.OHLCV_STALE_HOURS` is returned without attempting live refresh.

R3 — Quote freshness remains unchanged
Quote validation and normalized quote freshness MUST remain governed by
`config.FRESHNESS_SECONDS`; this PRD MUST NOT widen quote freshness.

FAIL: any quote freshness test accepts a quote older than
`config.FRESHNESS_SECONDS` because of `OHLCV_STALE_HOURS`.

R4 — Incorrect 5-minute OHLCV expectation is removed
Tests MUST NOT treat a daily OHLCV cache younger than 12 hours as stale
solely because it is older than `config.FRESHNESS_SECONDS`.

FAIL: `tests/test_derived.py` asserts that a daily OHLCV cache younger
than 12 hours is stale solely because it is older than
`config.FRESHNESS_SECONDS`.

DATA FLOW
`data/cache/*_ohlcv.parquet` -> `pd.read_parquet()` ->
`_is_fresh_ohlcv_cache()` -> cached OHLCV returned when latest-bar age
is < `config.OHLCV_STALE_HOURS`.

FAIL CONDITIONS
- FAIL if `_is_fresh_ohlcv_cache()` uses `config.FRESHNESS_SECONDS`.
- FAIL if OHLCV cache younger than 12 hours is rejected.
- FAIL if OHLCV cache at or older than 12 hours is accepted.
- FAIL if quote freshness is widened beyond `config.FRESHNESS_SECONDS`.
- FAIL if files outside the FILES list are modified.
