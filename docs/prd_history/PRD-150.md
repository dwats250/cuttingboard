PRD-150 — Five-Tier Symbol Classification System

STATUS
PROPOSED

LANE
HIGH-RISK

CLASS
EXECUTION

WHY NOW
The binary STAY_FLAT short-circuit in `qualify_all` collapses every
symbol's evaluation into a single posture-level decision, destroying
per-symbol visibility on days when the regime is mildly defensive. A
five-tier classification restores graduated visibility through a new
sidecar artifact and a narrow audit-trail extension to the existing
`rejections[]` channel, without expanding the `trade_candidates[]`
shape or manufacturing fake `decision_trace` records. The PRIME path
remains the sole gateway to actual trade emission.

The authoritative tier-visibility surface is
`reports/daily_classification.md`. The `rejections[]` CLASSIFICATION
channel functions as an R5 demotion audit trail in the contract
artifact for symbols that would have been PRIME but were demoted to
QUALIFIED by concentration cap or flow-gate opposition. All other
non-PRIME tiers (DEVELOPING, WATCHLIST, REJECT, and QUALIFIED-via-
soft-miss) route through the pre-existing QUALIFICATION /
WATCHLIST emission paths in `_build_rejections` and are visible via
the sidecar.

GOAL
Replace the regime-level STAY_FLAT short-circuit in `qualify_all`
(`cuttingboard/qualification.py:151–166`) with a per-symbol
five-tier classification (PRIME | QUALIFIED | WATCHLIST | DEVELOPING
| REJECT), surface non-PRIME tiers through the existing
`rejections[]` channel with new tier-specific reason literals, emit
a new sidecar artifact summarizing the classification, add a new
top-level run status `STATUS_NO_TRADE = "NO_TRADE"` for the
"posture-allowed, zero PRIME after gating" case, and gate the
existing aggregate Telegram notification body to PRIME-tier symbols
only (with a second QUALIFIED summary notification appended).

R3 — Tier assignment logic
A pure function `classify_symbol(qualification_result, regime)` in
`cuttingboard/classification.py` returns a `ClassificationTier` per
the following decision tree, evaluated in order; first match wins.
Concentration caps and flow-gate demotion are handled in R5
post-steps, not by `classify_symbol`.

  PRIME      iff all hard gates 1–4 pass
                AND ≤ 1 soft-gate miss (of 7 soft gates: gates 5–11)
                AND regime.posture ∈ {AGGRESSIVE_LONG,
                                      CONTROLLED_LONG,
                                      DEFENSIVE_SHORT}
  QUALIFIED  iff all hard gates 1–4 pass
                AND exactly 2 soft-gate misses
                AND regime.posture ∈ {AGGRESSIVE_LONG,
                                      CONTROLLED_LONG,
                                      DEFENSIVE_SHORT,
                                      NEUTRAL_PREMIUM}
  WATCHLIST  iff all hard gates 1–4 pass
                AND 3 or 4 soft-gate misses
                AND regime.posture != CHAOTIC
  DEVELOPING iff exactly 1 hard-gate failure
                AND ≤ 2 soft-gate misses
                AND regime.posture != CHAOTIC
  REJECT     otherwise (≥ 2 hard failures, ≥ 5 soft misses,
                CHAOTIC posture, or any state not matching above).

Posture asymmetry note: NEUTRAL_PREMIUM is excluded from PRIME but
included in QUALIFIED by design. PRIME is reserved for directional
postures with non-neutral confidence; NEUTRAL_PREMIUM remains
tradeable per CLAUDE.md but at one tier below.

Flow-gate-and-continuation note: `apply_flow_gate` downgrades and
EXPANSION-regime continuation results are NOT inputs to
`classify_symbol`. Continuation-path QualificationResults flow
through the same decision tree as primary-path results; flow-gate
downgrades are applied as a separate post-step (R5).

R5 — Concentration cap and flow-gate demotion (orthogonal post-steps)
After `classify_symbol` assigns initial tiers, two pure post-steps
run in this fixed order. Both may only demote PRIME → QUALIFIED.

Step A — `apply_concentration_caps(classifications)`:
  - Combined per-symbol cap: > 3 PRIME entries for the same symbol
    across primary + continuation paths in one session → demote
    excess to QUALIFIED, ordering by soft-miss count (highest
    first), then path tag (primary before continuation), then
    stable input-order index. The per-symbol cap is RETAINED AS
    DEFENSIVE GUARDRAIL even though current `qualify_all` produces
    at most one primary + at most one continuation per symbol
    (max 2 PRIME entries reachable today). Future evaluation-path
    additions remain bounded by the cap.
  - Total cap: > 8 PRIME entries total → demote excess PRIME to
    QUALIFIED using the same ordering.

Step B — `apply_flow_gate_demotion(classifications, flow_snapshot)`:
  - For every classification record whose current tier == PRIME
    and whose flow-alignment from `apply_flow_gate` is `OPPOSES`,
    demote to QUALIFIED. Orthogonal to caps; runs after Step A.

Each demoted record carries `demotion_reason` ∈ {`NONE`,
`CONCENTRATION_CAP`, `FLOW_OPPOSED`,
`CONCENTRATION_CAP+FLOW_OPPOSED`}. Undemoted records carry
`demotion_reason == "NONE"`.

FAIL: A session output contains > 8 PRIME entries.

FAIL: A session output contains > 3 PRIME entries for the same
symbol across primary + continuation paths combined.

FAIL: `apply_concentration_caps` or `apply_flow_gate_demotion`
performs any tier transition other than PRIME → QUALIFIED.

FAIL: A symbol demoted by both cap and flow-gate without
`demotion_reason == "CONCENTRATION_CAP+FLOW_OPPOSED"`.

FAIL: A record with no demotion event carries
`demotion_reason != "NONE"`.

FAIL: `apply_concentration_caps` or `apply_flow_gate_demotion`
called 100 times with byte-identical inputs returns ≥ 2 distinct
output structures.

R7 — Notification gating via aggregate calls
The existing aggregate `send_notification(alert_title, alert_body)`
call at `cuttingboard/runtime.py:575` has its body built to list
only PRIME-tier symbols (gating happens in the body-construction
step, NOT inside `send_notification`). When zero PRIME symbols
exist, the entire PRIME `send_notification` call is suppressed (no
call invoked, "skip the call" semantics).

A second `send_notification` call is added AFTER the PRIME call
returns (or after suppression is recorded), sending a
QUALIFIED-tier summary (symbol, direction, soft-miss count,
demotion_reason). Suppressed when zero QUALIFIED symbols exist.
No new per-symbol Telegram loop is introduced.
