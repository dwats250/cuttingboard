PRD-031 — Claude Code Hooks: Commit Gate, File Guard, Test Gate, and State Snapshot

STATUS
IN PROGRESS

GOAL
Add Claude Code hooks to mechanically enforce three hard constraints: no
automatic git commit/push, protected file writes are blocked unless explicitly
scoped, and pytest runs after any Python file change.  A Stop hook captures
active-task state before compaction or session end.

SCOPE
- .claude/settings.json hook wiring (PreToolUse, PostToolUse, Stop events)
- .claude/hooks/git_gate.sh — blocks git commit / git push in Bash calls; enforces dirty repo guard
- .claude/hooks/protect_files.sh — blocks Write/Edit to protected paths
- .claude/hooks/test_gate.sh — runs pytest -q after scoped Python file changes
- .claude/hooks/stop_snapshot.sh — writes state snapshot on Stop
- .claude/state/active_prd.txt — convention: Claude writes this when beginning a PRD
- docs/CLAUDE_HOOKS.md — hook workflow documentation

OUT OF SCOPE
- Hooks that attempt to generate Claude reasoning output (PRD review, critique text)
- PreCompact/PostCompact hooks (event type does not exist in Claude Code)
- Fully automatic commit or push under any condition
- Changing trading engine logic
- Changing PRD numbering history
- External notification transports
- CI workflow changes

REQUIREMENTS

R1 — Git Commit/Push Gate
A PreToolUse/Bash hook must intercept every Bash tool call and exit non-zero
if the command contains `git commit` or `git push`.  The hook must print the
reason for blocking.

Approval override: the hook reads the session transcript at transcript_path and
searches user-role messages for the exact phrase APPROVE COMMIT.  If found,
the hook exits 0 and allows the command.  Without APPROVE COMMIT appearing in
a user message in the current session, no git commit or git push may proceed.

FAIL: git commit or git push executes in a Bash tool call without the literal
phrase APPROVE COMMIT appearing in a user-role message in the same session.

R2 — Protected File Guard
A PreToolUse hook on Write and Edit must exit non-zero when the target path
matches any protected pattern:
  - .env
  - .env.*
  - .git/*
  - *.lock
  - .github/workflows/*
  - secrets*

The hook must read .claude/state/active_prd.txt to resolve the active PRD.
If the target path appears in the FILES section of the active PRD document,
the hook exits 0 (allowed).  If active_prd.txt is missing, empty, or the PRD
document cannot be read, the hook must fail closed (exit non-zero).

FAIL: Claude writes to a protected path without it being listed in the active
PRD FILES section.  FAIL: hook allows a protected-path write when
active_prd.txt is missing or empty.

active_prd.txt lifecycle:
  WRITE — Claude writes this file immediately after Dustin approves a PRD for
  implementation.  Format: single line containing exactly PRD-XXX.
  READ — protect_files.sh reads this file before allowing protected-path
  exceptions.  stop_snapshot.sh reads this file when writing snapshot.json.
  FAIL CLOSED — if the file is missing or empty when required by a hook, the
  hook must block the action.

R3 — Test Gate
A PostToolUse hook on Write and Edit must detect when the modified path matches:
  - cuttingboard/*.py
  - tests/*.py

If the path matches, the hook must run .venv/bin/pytest -q tests/ and emit the
result.  The hook must debounce repeated runs: if pytest ran within the last
10 seconds (tracked via .claude/state/pytest_last_run), the hook skips the
run and prints the elapsed time.

FAIL: A scoped .py file is written and pytest does not run (outside debounce
window).  FAIL: pytest runs more than once within a 10-second window on
sequential edits.

R4 — Stop Snapshot
A Stop hook must write .claude/state/snapshot.json containing exactly:
  - active_prd  (string — from active_prd.txt, or "none" if missing/empty)
  - branch      (string — from git rev-parse --abbrev-ref HEAD)
  - commit      (string — from git rev-parse --short HEAD)
  - timestamp   (ISO 8601 UTC)
  - next_action (string — from .claude/state/next_action.txt, or "unknown")

At session start or after compaction, if snapshot.json exists Claude must read
it and summarize: active PRD, branch, last commit, and next required action.

FAIL: snapshot.json is not written on Stop.  FAIL: snapshot.json is missing
any of the five required fields.

R5 — Dirty Repo Guard
When the git_gate hook fires on a git commit or git push call, it must run
git status --short and identify any dirty or untracked files that are not
listed in the FILES section of the active PRD.

If unrelated dirty or untracked files exist, the hook must print them and
state that commit/push approval must not be requested until they are resolved.
Claude must not ask for commit/push approval while unrelated dirty or untracked
files remain.

FAIL: Claude asks Dustin for commit/push approval while unrelated dirty or
untracked files exist outside the active PRD FILES list.

DATA FLOW
Bash tool call → PreToolUse/Bash → git_gate.sh → transcript check + dirty guard → block or pass
Write/Edit tool call → PreToolUse/Write+Edit → protect_files.sh → active_prd.txt → PRD FILES check → block or pass
Write/Edit tool call → PostToolUse/Write+Edit → test_gate.sh → debounce check → pytest → result output
Session Stop → Stop → stop_snapshot.sh → active_prd.txt + git state → .claude/state/snapshot.json

FAIL CONDITIONS
- git commit or git push runs without APPROVE COMMIT in a user session message
- Write or Edit succeeds on a protected path not in the active PRD FILES section
- protect_files.sh allows a protected write when active_prd.txt is missing or empty
- A scoped .py file is written and pytest does not run (outside debounce window)
- Pytest runs more than once within a 10-second debounce window
- Stop fires and snapshot.json is missing or lacks any required field
- Any hook script does not exist or is not executable
- Claude asks for commit/push approval while unrelated dirty files exist

VALIDATION
Manual:
- chmod +x .claude/hooks/*.sh && bash -n .claude/hooks/*.sh
- Verify .claude/settings.json references all four scripts with correct event types
- Attempt git commit -m test in a Bash call without APPROVE COMMIT; confirm hook blocks
- Type APPROVE COMMIT then attempt git commit; confirm hook allows
- Attempt Write to .env; confirm hook blocks
- Attempt Write to .env with active PRD listing .env in FILES; confirm hook allows
- Remove .claude/state/active_prd.txt; attempt Write to .env; confirm hook blocks (fail closed)
- Write cuttingboard/config.py; confirm pytest runs in hook output
- Write same file again within 10 seconds; confirm debounce message, no duplicate run
- Write a non-Python file; confirm pytest does not run
- Trigger Stop; confirm .claude/state/snapshot.json exists with all five fields
- cat docs/CLAUDE_HOOKS.md; confirm all four hooks are documented

COMMIT PLAN

- git status --short
- commit only after Dustin explicitly types APPROVE COMMIT
- push only after Dustin explicitly types APPROVE COMMIT
