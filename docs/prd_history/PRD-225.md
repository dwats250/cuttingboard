CLASS

WHY NOW
Live-site report (Dustin, 2026-07-02 screenshot): on phone widths the XLE trend
row renders all cells on one line while SPY/QQQ/GDX/GLD/SLV wrap `VWAP N/A` to
a second line — the #87 inline-flow change works only for the MIX row. Root
cause: the Alignment token is the mobile row's ONLY variable-width cell
(`MIX` = 3ch vs `BULL`/`BEAR` = 4ch), and the 4ch rows overflow the viewport by
about that one character.

MAX EXPECTED DELTA
6 production LOC (CSS strings + one cell class), all in
cuttingboard/delivery/dashboard_renderer.py.

GOAL
All six trend rows wrap identically — and, on typical phone widths, identically
on ONE line: alignment cells share a fixed min-width so row width no longer
depends on the token, and the mobile inter-cell gap is tightened to absorb the
reclaimed character.

SCOPE
- Tag the Alignment cell (index 3) with a `ts-align` class at the emission site
  (mirroring the existing `ts-intraday` index-based tagging — class-based, so
  the PRD-165 column-collapse re-indexing cannot mistarget it).
- Mobile CSS (inside the existing `@media(max-width:640px)` block):
  `.ts-table td.ts-align{min-width:4ch}` — BULL/BEAR/MIX all occupy 4ch, making
  every row's total width identical.
- **Fix the latent mobile padding override (found during browser verification;
  amended into scope before implementation).** The PRD-213 mobile rule
  `.ts-table td{white-space:nowrap!important;padding:0}` has NEVER applied its
  `padding:0` half: each `<td>` carries inline `style="padding:2px 8px"`, and
  inline style beats a class rule without `!important` (only `white-space` had
  it). Every mobile cell silently keeps 16px of horizontal padding — 96px/row,
  ~28% of a 390px viewport — which is why #87's inline flow barely fits
  anywhere. Change to `padding:0!important`, realizing the original PRD-213
  intent; the flex `gap` (not padding) is the intended cell separator.
- Tighten the mobile row gap `gap:2px 10px` → `gap:2px 8px`, and right-size
  the two legacy mobile min-widths in text-width units (`td:first-child`
  3.2em → 4ch; `td:nth-child(2)` 5em → 7ch — the em values carried ~34px/row
  of dead width; 4ch/7ch still cover every symbol/price in the traded
  universe, preserving cross-card column alignment).
- Verified at real phone viewports with headless Chromium (before/after
  measurement over the actual published dashboard HTML), not guessed.

OUT OF SCOPE
- Desktop (>640px) table layout — untouched.
- Any change to cell content, column collapse logic, or the `ts-intraday`
  inline-flow decision (#87) — this makes #87's intent hold uniformly.
- Font-size or padding changes.

REQUIREMENTS

R1 — Alignment cell carries the uniform-width hook
Every rendered trend row's Alignment cell carries `class="ts-align"`, and the
mobile CSS block contains `.ts-table td.ts-align{min-width:4ch}`.
FAIL: a rendered trend section lacks `class="ts-align"` on the Alignment cell,
or the CSS token `.ts-table td.ts-align{min-width:4ch}` is absent.

R2 — Mobile spacing rules actually apply
The mobile row rule reads `gap:2px 8px`, and the mobile cell rule reads
`padding:0!important` so it defeats the inline `padding:2px 8px` (the PRD-213
override that silently never applied).
FAIL: `gap:2px 10px` still present, or the mobile td rule lacks
`padding:0!important` (grep of the rendered CSS).

R3 — Uniform one-line flow at phone width (verified where truth is determined
for layout: a real browser viewport)
At a 390px-wide viewport, all six trend rows render their Intraday (`VWAP N/A`)
cell on the same line as the Symbol cell (equal bounding-box top within the
row), for BULL, BEAR, and MIX alignment tokens alike.
FAIL: any row's Intraday cell top differs from its Symbol cell top at 390px in
the headless-Chromium check.

DATA FLOW
trend_alignment token → abbrev (BULL/BEAR/MIX) → td.ts-align (min-width:4ch,
uniform) → identical row widths → uniform inline flow of the Intraday cell

FAIL CONDITIONS
- Missing ts-align class or min-width CSS token (R1).
- gap:2px 10px survives (R2).
- Any trend row wraps VWAP while another does not at 390px (R3).

VALIDATION
Manual/browser (R3): headless Chromium at 390×844 over a rendered dashboard
with BULL/BEAR/MIX rows — compare per-row Symbol vs Intraday cell y-offsets.
Binary: all equal.
Automated: literal assertions in tests/test_dashboard_renderer.py for the
ts-align class and both CSS tokens (R1/R2) — each fails if the change is
reverted. (CI has no browser; R3 is validated at implementation time and the
CSS tokens pin the mechanism.)
