WHY NOW
Post PRD-158/165 the dashboard is cleaner but still organized around system
internals, not the decision sequence. It carries two debugging surfaces of no trader
value and presents macro bias as a bare conclusion. This is the single renderer pass
that also lands the SCOREBOARD (PRD-175) and RED FOLDER (PRD-176) sections, so the
HIGH-RISK consumer lane is reviewed once instead of three times.

MAX EXPECTED DELTA
300 production LOC across the renderer + layout module + regenerated UI. Exceeding
stops implementation and re-triggers CONSUMER review.

GOAL
Every rendered section maps to one of VISION's four questions in order
(environment -> what matters today -> tradable -> what invalidates / calibration);
the two debugging sections are gone; macro driver rows show their cyclicality-aware
vote and interpretation; and the SCOREBOARD and RED FOLDER sections render from the
PRD-175/176 sidecars.

SCOPE
- Cut the Run History (id="run-history", dashboard_renderer.py ~2257-2261) and
  Artifact Diagnostics (id="artifact-diagnostics", ~2263-2265) sections.
- Reorder remaining sections to the four-questions sequence: System State ->
  Macro Tape (Q1) -> Red Folder (Q2) -> Trend Structure -> Candidate Board (Q3) ->
  Run Delta -> Scoreboard (Q4 calibration). Conditional sections (coherence warning,
  premarket/Sunday banners, alert watchlist) keep their trigger logic.
- Add per-driver evidence to the Macro Tape: each MACRO_BIAS_DRIVERS row carries its
  cyclicality-aware vote and a fixed interpretation string (data already in the vote
  tally; this surfaces it). Headline bias line unchanged.
- Add the RED FOLDER section (reads cuttingboard.red_folder) and the SCOREBOARD
  section (reads logs/regime_history.jsonl), each with an empty-state line.

R2 — Four-questions order

R3 — Macro evidence rows

a fixed interpretation string; the headline bias still agrees with the vote tally
(PRD-160 invariant).

the tally.

With logs/regime_history.jsonl present, a scoreboard section renders up to 10 dated
rows; with it absent/empty, a single empty-state line renders (not a dead table).
FAIL: history present but no rows render, or history absent but no empty-state line.

R5 — Red folder section
With an event inside the loader window, a red-folder section lists it; with none, a
single empty-state line renders; with a missing/malformed schedule, the loader's
warning line renders.
FAIL: event in window not shown, or missing-file case renders silent absence.

R6 — Publish integrity
PRD-118 coherence and PRD-119 freshness gates pass unchanged; ui/dashboard.html and
ui/index.html are byte-identical.
FAIL: either gate regresses, or the two UI files differ.
