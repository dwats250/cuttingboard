PRD-024 — Contract UI Consumer — Static HTML Read-Only Decision Surface

STATUS
COMPLETE
Commit: 6f97d12

GOAL
Add a static HTML/JS UI that reads the pipeline contract JSON and renders a read-only decision surface in the browser. No server required.

SCOPE
- Add ui/index.html: single-page app structure
- Add ui/styles.css: visual styling
- Add ui/app.js: contract reader and renderer
- Add ui/mock_contract.json: fixture for development and validation

OUT OF SCOPE
- No Python module changes
- No server component
- No contract schema changes
- UI is read-only — no writes to contract or pipeline state

FILES
A ui/index.html
A ui/styles.css
A ui/app.js
A ui/mock_contract.json

REQUIREMENTS

R1 — Static Only
UI MUST function by opening ui/index.html directly in a browser with no server.

FAIL: UI requires a running server to render.

R2 — Contract Fidelity
UI MUST read and render all top-level contract fields: status, regime, trade_candidates, system_state, correlation.

FAIL: Any top-level contract field is absent from the rendered output.

R3 — Read Only
UI MUST NOT write to any contract file or trigger any pipeline execution.

FAIL: UI writes to any file or makes any network request beyond loading the contract JSON.

R4 — Mock Fixture
ui/mock_contract.json MUST be a valid contract snapshot usable for offline UI validation.

FAIL: Loading mock_contract.json in the UI produces a render error.

DATA FLOW
ui/index.html → ui/app.js → loads contract JSON → renders decision surface

FAIL CONDITIONS
- UI requires a server
- Contract field absent from render
- UI writes to any file
- Mock fixture produces render error

VALIDATION
Manual: open ui/index.html in browser, load mock_contract.json — confirm all contract fields render, no console errors.
