# PRD-053 — Graded Market Map Sidecar

STATUS
READY

GOAL
Add a read-only graded market map sidecar that writes `logs/market_map.json` and provides deterministic, trader-focused market visibility for the primary symbol universe without changing execution behavior or existing user-facing artifacts.

SCOPE
- Add `cuttingboard/market_map.py`
- Add runtime sidecar artifact write to `logs/market_map.json`
- Grade exactly:
  - SPY
  - QQQ
  - GDX
  - GLD
  - SLV
  - XLE
- Allow `USO` only as ENERGY_CONTEXT if already present in runtime data
- Use existing runtime data only:
  - normalized quotes
  - derived metrics (EMA, ATR, momentum)
  - structure results
  - intraday metrics (VWAP, ORB, PDH/PDL)
  - regime state
  - watch summary
  - existing bar windows if available
- Produce deterministic per-symbol market visibility fields
- Add unit tests for builder and runtime artifact wiring

OUT OF SCOPE
- Contract schema changes
- Payload schema changes
- Dashboard changes
- Alert or notification changes
- Trade decision changes
- Execution policy changes
- Qualification changes
- New gates or trade blocking logic
- New data providers or fetches
- Paper trading or execution simulation
- Making USO a primary graded symbol

FILES
A `cuttingboard/market_map.py`
M `cuttingboard/runtime.py`
A `tests/test_market_map.py`
M `tests/test_operationalization.py`
M `docs/PRD_REGISTRY.md`

---

REQUIREMENTS

R1 — Sidecar-only artifact
The implementation MUST write the graded market map only to `logs/market_map.json`.

FAIL:
- Any mutation of `logs/latest_contract.json`
- Any mutation of `logs/latest_payload.json`
- Any mutation of dashboard output
- Any mutation of notification output
- Any mutation of trade_candidates, trade_decisions, execution policy, or qualification outputs

---

R2 — Primary universe
The market map MUST contain exactly one record for each:
- SPY
- QQQ
- GDX
- GLD
- SLV
- XLE

USO MAY appear only as ENERGY_CONTEXT and MUST NOT be graded.

FAIL:
- Missing primary symbol
- Extra graded symbol
- USO graded as a primary symbol

---

R3 — Required per-symbol fields
Each symbol MUST include exactly:

- symbol
- asset_group
- grade
- bias
- structure
- setup_state
- confidence
- watch_zones
- fib_levels
- what_to_look_for
- invalidation
- preferred_trade_structure
- reason_for_grade

If data is unavailable:
- Use null OR
- Use a deterministic explanation string (e.g. "data unavailable")

FAIL:
- Missing field
- Renamed field
- Fabricated or non-deterministic values

---

R4 — Grade enum
grade MUST be exactly one of:

- A+ = premium actionable
- A  = strong watch
- B  = developing
- C  = unclear / low quality
- D  = poor opportunity
- F  = data unavailable or structurally invalid

All symbols MUST receive a grade.

A+ MUST require ALL:
- regime alignment
- strong structure
- proximity to key level or trigger zone
- non-extended price

FAIL:
- Invalid grade
- Missing grade
- A+ assigned without satisfying all conditions

---

R5 — Bias enum
bias MUST be exactly one of:

- BULLISH
- BEARISH
- NEUTRAL
- MIXED

FAIL:
- Free-form bias values

---

R6 — Structure enum
structure MUST be exactly one of:

- TRENDING_UP
- TRENDING_DOWN
- PULLBACK
- BREAKOUT
- BREAKDOWN
- RANGE
- CHOPPY
- UNKNOWN

FAIL:
- Non-enum structure values

---

R7 — Setup state enum
setup_state MUST be exactly one of:

- ACTIONABLE
- STRONG_WATCH
- DEVELOPING
- EXTENDED
- CHOPPY
- LOW_QUALITY
- DATA_UNAVAILABLE

FAIL:
- Missing or free-form setup_state

---

R8 — Confidence enum
confidence MUST be exactly one of:

- HIGH
- MEDIUM
- LOW

Derived deterministically from:
- grade
- regime confidence
- structure quality

FAIL:
- Missing confidence
- Non-enum values

---

R9 — Watch zones structure
watch_zones MUST be a list of objects:

{
  "type": string,
  "level": float,
  "context": string
}

Examples:
- VWAP
- EMA21
- ORB_HIGH
- PRIOR_HIGH
- PRIOR_LOW

FAIL:
- Non-object entries
- Missing level values
- Non-numeric levels

---

R10 — Fibonacci levels
fib_levels MUST be:

- null if deterministic swing cannot be computed

OR

{
  "source": string,
  "swing_high": float,
  "swing_low": float,
  "retracements": {
    "0.382": float,
    "0.5": float,
    "0.618": float
  }
}

Rules:
- Must be derived from deterministic bar window (e.g. last N daily bars)
- No subjective swing detection

FAIL:
- Fabricated fib values
- Non-deterministic swing selection

---

R11 — Language constraint
The market map MUST NOT use:

- NO_TRADE
- STAY_FLAT
- BLOCKED
- REJECTED

Guidance MUST use:

- watch
- look for
- developing
- extended
- invalidation
- preferred structure

FAIL:
- Binary trade/no-trade phrasing

---

R12 — Determinism
The same input data MUST always produce identical market_map.json output.

FAIL:
- Non-deterministic ordering
- Random or time-dependent logic (excluding timestamps)

---

VALIDATION

- Unit test: schema exact match

- Unit test: structure enum enforcement
- Unit test: setup_state enum enforcement
- Unit test: confidence enum enforcement
