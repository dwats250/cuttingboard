PRD-110 — Narrow Trend Structure Snapshot Universe

STATUS
IN PROGRESS

GOAL
Narrow the `logs/trend_structure_snapshot.json` symbol universe from the full
tradable set (16 symbols) to a curated swing/structure watchlist of six
symbols: `SPY, QQQ, GDX, GLD, SLV, XLE`. The sidecar remains observe-only and
its schema, writer, ordering, and FAIL contract from PRD-107 are preserved.

This PRD is sidecar-only. It must not mutate trade decisions, qualification,
runtime outcome, contract, payload, alerts, dashboard HTML, market_map
grading, or any other artifact.

SCOPE
- Introduce a single config constant naming the trend-structure universe.
- Replace the call-site argument in `_write_trend_structure_snapshot()` so it
  passes the curated list instead of `_tradable_symbols()`.
- Add a deterministic test that the runtime helper passes exactly the
  curated set to `build_trend_structure_snapshot()`.
- Add an artifact-shape test asserting the snapshot's `symbols` keys equal
  the curated set.
- Preserve every existing PRD-107 FAIL condition (R1–R9).

OUT OF SCOPE
- Schema changes (`schema_version`, field names, enum values).
- New fields, new indicators, new timeframes.
- Changes to `trend_structure.py` builder logic.
- Changes to the writer (`_write_trend_structure_snapshot`) atomic pattern,
  ordering, or error handling.
- Changes to `_tradable_symbols()` or `config.NON_TRADABLE_SYMBOLS`.
- Adding the snapshot to qualification, contract, payload, dashboard, or
  notifications.
- Any new symbol fetched from network. The curated set is a strict subset of
  symbols already fetched by the existing pipeline.

REQUIREMENTS

R1 — Curated universe constant
`cuttingboard/config.py` defines:
`TREND_STRUCTURE_SYMBOLS: tuple[str, ...] = ("SPY", "QQQ", "GDX", "GLD", "SLV", "XLE")`
Order is fixed. Tuple, not list. Every entry is a member of `ALL_SYMBOLS`
and not a member of `NON_TRADABLE_SYMBOLS`.

FAIL: constant missing, mutable type, wrong order, contains a
`NON_TRADABLE_SYMBOLS` member, or contains a symbol absent from
`ALL_SYMBOLS`.

R2 — Runtime call site uses curated universe
`_write_trend_structure_snapshot()` in `cuttingboard/runtime.py` passes
`list(config.TREND_STRUCTURE_SYMBOLS)` as the `symbols=` argument to
`build_trend_structure_snapshot()`. It must NOT call `_tradable_symbols()`
for this purpose.

FAIL: call site references `_tradable_symbols()` for trend_structure, or
passes any other sequence.

R3 — Builder unchanged
`cuttingboard/trend_structure.py` is not modified. The builder remains a
pure function whose `symbols` argument controls output keys.

FAIL: diff in `cuttingboard/trend_structure.py`.

R4 — Writer behavior unchanged
The atomic `.tmp → replace` pattern, ordering after `_write_market_map_file()`,
exception swallowing, and `TREND_STRUCTURE_PATH` are unchanged.

FAIL: diff in writer body other than the `symbols=` argument source.

R5 — Snapshot symbol set equals curated universe
Under a deterministic isolated runtime fixture (monkeypatched `LOGS_DIR` /
`TREND_STRUCTURE_PATH`, mocked builder spy or stub OHLCV), the artifact
`trend_structure_snapshot.json["symbols"]` keys equal
`{"SPY","QQQ","GDX","GLD","SLV","XLE"}` exactly (set equality). Test MUST
NOT depend on a live hourly run, network, or wall-clock data freshness.

FAIL: any extra or missing key, or test relies on non-isolated artifact
path or live data.

R6 — Banned symbols still excluded
None of `^VIX, ^TNX, DX-Y.NYB, BTC-USD, IWM, PAAS, USO, NVDA, TSLA, AAPL,
META, AMZN, COIN, MSTR` appear in `data["symbols"]`.

FAIL: any banned symbol present.

R7 — No leakage
The symbol `trend_structure` and the new constant `TREND_STRUCTURE_SYMBOLS`
are not referenced in `cuttingboard/contract.py`, `cuttingboard/delivery/`,
`cuttingboard/notifications/`, `ui/`, `cuttingboard/qualification.py`,
`cuttingboard/regime.py`, or `cuttingboard/output.py`.

FAIL: leakage grep returns matches outside `cuttingboard/runtime.py`,
`cuttingboard/config.py`, `cuttingboard/trend_structure.py`, `tests/`,
`docs/`, and `logs/`.

R8 — PRD-107 FAIL contract preserved
All FAIL conditions R1–R9 from PRD-107 still hold under the narrowed
universe. In particular:
- Builder still emits `null` / `UNKNOWN` for missing data.
- Builder still raises on naive `generated_at`.
- Builder still rejects insufficient bars for SMA fields.
- No `"AT"` value appears.

FAIL: any PRD-107 FAIL fixture begins to fail.
