## GOAL

Reduce always-visible noise in the System State block by conditionally rendering
the `Halted` and `Kill Switch` fields only when their values are `True`. All other
System State signals remain unconditional. No runtime, contract, or notification
changes.

---

## SCOPE

Single behavior change in the dashboard renderer:
- `Halted` field renders only when `system_halted is True`.
- `Kill Switch` field renders only when `kill_switch is True`.
- All other fields (Regime, Confidence, Outcome, Permission, Reason, Run Snapshot)
  are unconditional and unchanged.

One test file is updated to match the new conditional rendering contract.

---

## OUT OF SCOPE

- Runtime / execution pipeline changes
- Notification, alert, or workflow changes
- Market map, schema, grading, lifecycle, or trade decision changes
- Generated UI files (`ui/index.html`)
- `PRD_REGISTRY.md`, `PROJECT_STATE.md`
- Any new fields, classes, or CSS not already present
- Changing when `system_halted` or `kill_switch` are set (set upstream, read here)

## REQUIREMENTS

### R1 â€” Halted field is conditional
Render `<div class="field"><div class="label">Halted</div>â€¦</div>` only when
`bool(system_halted) is True`. When `system_halted is False`, the `Halted` label
and its value do not appear anywhere in the System State block.

**FAIL:** `"Halted"` appears in the system-state block when `system_halted=False`.

### R2 â€” Kill Switch field is conditional
Render `<div class="field"><div class="label">Kill Switch</div>â€¦</div>` only when
`bool(kill_switch) is True`. When `kill_switch is False`, the `Kill Switch` label
and its value do not appear.

**FAIL:** `"Kill Switch"` appears in the system-state block when `kill_switch=False`.

### R3 â€” Normal run shows no Halted or Kill Switch label
When `system_halted=False` and `kill_switch=False` (default `_run()` fixture),
neither label appears in the system-state block.

**FAIL:** Either label appears in the system-state block for a default normal run.

### R4 â€” Halted label still appears after Permission when halted
When `system_halted=True`, the `Halted` label must appear in the HTML after the
`Permission` label. This preserves the string-split anchor used in
`test_halted_state_raw_reason_shown_as_reason_not_permission` (line 406).

**FAIL:** When `system_halted=True`, `"Halted"` does not appear after `"Permission"`
in the system-state block, causing the existing test's `split("Halted", 1)` to raise.

### R5 â€” Regime, Confidence, Outcome always visible
These three fields render unconditionally on every run, regardless of
`system_halted` or `kill_switch` values.

**FAIL:** Any of `"Regime"`, `"Confidence"`, `"Outcome"` is absent from the
system-state block on any run.

### R6 â€” Permission always visible
Permission renders unconditionally in all four branches already implemented
(halted, stay-flat, permission set, fallback dash). No change to Permission logic.

**FAIL:** `"Permission"` is absent from the system-state block on any run.

### R7 â€” PRD-089-PATCH halt reason behavior unchanged
When `system_halted=True` and a halt reason is available:
- Permission displays `HALTED`.
- The raw reason appears as a separate `Reason` field.
- The raw reason does not appear inside the Permission value.

**FAIL:** Any of the three assertions in `test_halted_state_raw_reason_shown_as_reason_not_permission` fails.

### R8 â€” Run Snapshot always visible
`RUN SNAPSHOT - STALE` or `RUN SNAPSHOT - CURRENT` appears in the system-state
block on every run. Pacific timestamp renders when available.

**FAIL:** `"RUN SNAPSHOT"` is absent from the system-state block.

### R9 â€” Test update: replace `test_halted_and_kill_switch_visible_in_system_state`
The existing test at line 419 asserts `"Kill Switch" in state` when `kill_switch=False`.
This assertion is incorrect under the new contract. Replace it with three tests:

- `test_normal_run_no_halted_or_kill_switch_in_system_state`:
  `_run(system_halted=False, kill_switch=False)` â†’ neither label in state block.

- `test_halted_run_shows_halted_not_kill_switch`:
  `_run(system_halted=True, kill_switch=False)` â†’ `"Halted" in state`, `"Kill Switch" not in state`.

- `test_kill_switch_run_shows_kill_switch`:
  `_run(system_halted=False, kill_switch=True)` â†’ `"Kill Switch" in state`.

**FAIL:** Any of the three replacement tests fails.

from tests.dash_helpers import _payload, _run
from cuttingboard.delivery.dashboard_renderer import render_dashboard_html

def _block(html):
    return html.split('id="system-state"', 1)[1].split('<div class="block"', 1)[0]

# Normal run: neither label
state = _block(render_dashboard_html(_payload(), _run(system_halted=False, kill_switch=False)))
assert "Halted" not in state,      "FAIL: Halted visible in normal run"
assert "Kill Switch" not in state, "FAIL: Kill Switch visible in normal run"
assert "Regime" in state,          "FAIL: Regime missing"
assert "Confidence" in state,      "FAIL: Confidence missing"
assert "Outcome" in state,         "FAIL: Outcome missing"
assert "Permission" in state,      "FAIL: Permission missing"
assert "RUN SNAPSHOT" in state,    "FAIL: RUN SNAPSHOT missing"
print("PASS: normal run")

# Halted run: Halted appears after Permission, Kill Switch absent, Permission=HALTED
state = _block(render_dashboard_html(_payload(), _run(system_halted=True, kill_switch=False)))
assert "Halted" in state,          "FAIL: Halted missing when halted"
assert "Kill Switch" not in state, "FAIL: Kill Switch visible when kill_switch=False"
assert "HALTED" in state,          "FAIL: Permission not HALTED when halted"
perm_pos = state.find("Permission")
halted_pos = state.find("Halted")
assert perm_pos < halted_pos,      "FAIL: Halted must appear after Permission (split anchor)"
print("PASS: halted run")

# Kill-switch run: Kill Switch appears, Halted absent
state = _block(render_dashboard_html(_payload(), _run(system_halted=False, kill_switch=True)))
assert "Kill Switch" in state,     "FAIL: Kill Switch missing when active"
assert "Halted" not in state,      "FAIL: Halted visible when system_halted=False"
print("PASS: kill-switch run")

# Both active
state = _block(render_dashboard_html(_payload(), _run(system_halted=True, kill_switch=True)))
assert "Halted" in state,          "FAIL: Halted missing"
assert "Kill Switch" in state,     "FAIL: Kill Switch missing"
print("PASS: both active")

## IMPLEMENTATION NOTES

**Renderer change (`dashboard_renderer.py` ~line 948):**

Before:
```python
w(f'    <div class="field"><div class="label">Halted</div>'
  f'<div class="value{halted_cls}">{_bool_str(system_halted)}</div></div>')
w(f'    <div class="field"><div class="label">Kill Switch</div>'
  f'<div class="value{ks_cls}">{_bool_str(kill_switch)}</div></div>')
```

After:
```python
if bool(system_halted):
    w(f'    <div class="field"><div class="label">Halted</div>'
      f'<div class="value{halted_cls}">{_bool_str(system_halted)}</div></div>')
if bool(kill_switch):
    w(f'    <div class="field"><div class="label">Kill Switch</div>'
      f'<div class="value{ks_cls}">{_bool_str(kill_switch)}</div></div>')
```

`halted_cls` and `ks_cls` assignments remain correct when their conditions are true.
No other code changes required.

**String-split anchor invariant:** `test_halted_state_raw_reason_shown_as_reason_not_permission`
splits the state block on `"Halted"` to isolate the Permission section
(`state.split("Permission", 1)[1].split("Halted", 1)[0]`). This works because
when `system_halted=True`, the `Halted` label is present and appears after
`Permission` (R4). Do not reorder Permission and Halted, and do not rename the label.
