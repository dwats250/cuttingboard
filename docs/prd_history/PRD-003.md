PRD-003 — Enforce Deterministic Failure Visibility in CI Pipeline

STATUS
COMPLETE
Commit: 809214bc

GOAL
Harden the GitHub Actions workflow so CI failures are always visible, deterministic, and not masked by silent exit codes or missing artifact uploads.

SCOPE
- Rewrite .github/workflows/cuttingboard.yml failure-path guards
- Ensure test failures produce non-zero exit and fail the workflow step
- Ensure artifact upload always runs on failure

OUT OF SCOPE
- No changes to any Python module
- No changes to test logic

FILES
M .github/workflows/cuttingboard.yml

REQUIREMENTS

R1 — Failure Visibility
CI workflow MUST exit non-zero when tests fail.
Artifact upload step MUST run on failure (if: failure()).

FAIL: Workflow reports success when pytest exits non-zero.

R2 — Deterministic Step Order
Failure-path steps MUST be ordered: test → artifact upload → notify.
No step may suppress exit codes silently.

FAIL: A failed test run produces a green CI status.

DATA FLOW
pytest exit code → workflow step outcome → artifact upload (on failure) → CI status

FAIL CONDITIONS
- CI passes when pytest reports failures
- Artifact upload step skipped on test failure

VALIDATION
Manual: trigger workflow with a known failing test; confirm CI fails and artifact is uploaded.
