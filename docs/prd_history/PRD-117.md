# PRD-117 — Session-Aware Inactive-State Labeling

STATUS: IN PROGRESS

GOAL
Differentiate expected no-session states from unhealthy stale/missing
artifact states so the dashboard communicates operational health
accurately during closed-market periods. Presentation-layer only.

SCOPE
1. Add session-aware inactive labeling for Trend Structure and Market
   Map sections when the payload indicates an enumerated inactive
   session_type and artifact lineage is coherent.
2. Preserve existing artifact_lineage_state semantics and classification.
3. Preserve PRD-116 unhealthy-lineage handling and precedence.
4. Preserve existing sunday_coherent context-block rendering.
5. Confine all changes to the renderer presentation layer.

OUT OF SCOPE
1. No trading logic changes.
2. No artifact_lineage_state contract changes.
3. No runtime.py changes.
4. No payload.py changes.
5. No new session_type enum values (current set: {"SUNDAY_PREMARKET"}).
6. No market_map or scheduler changes.
7. No dashboard redesign beyond the two targeted section labels.

FILES
- cuttingboard/delivery/dashboard_renderer.py
- tests/test_dashboard_renderer.py
- docs/prd_history/PRD-117.md
- docs/PRD_REGISTRY.md
- docs/PROJECT_STATE.md

TERMINOLOGY
- artifact_lineage_state ∈ {COHERENT, MIXED, STALE, MISSING}
- unhealthy lineage := {MIXED, STALE, MISSING}
- INACTIVE_SESSION_TYPES := {"SUNDAY_PREMARKET"} (renderer-local constant)
- inactive session := payload.meta.session_type ∈ INACTIVE_SESSION_TYPES
- live session := payload.meta.session_type is None (no inactive enum
  emitted by runtime)
- INACTIVE_SESSION_LABEL := "SESSION INACTIVE" (renderer-local constant)
- presentation label := renderer-only human-readable status text that
  does not modify underlying artifact classification

REQUIREMENTS

R1 — Preserve artifact lineage contract
The renderer must continue using artifact_lineage_state values exactly
as currently defined.
FAIL: Any new artifact_lineage_state value introduced or existing
values renamed.

R2 — Inactive-session presentation labeling
When artifact_lineage_state == "COHERENT" AND
payload.meta.session_type ∈ INACTIVE_SESSION_TYPES, the Trend Structure
section (id="trend-structure") and Market Map section
(id="candidate-board") must render INACTIVE_SESSION_LABEL in place of
live-session content.
FAIL: Inactive-session rendering omits INACTIVE_SESSION_LABEL at either
targeted section, OR emits a label other than INACTIVE_SESSION_LABEL.

R3 — Unhealthy lineage precedence
When artifact_lineage_state ∈ {MIXED, STALE, MISSING}, PRD-116
unhealthy-lineage labels take precedence over INACTIVE_SESSION_LABEL
regardless of session_type.
FAIL: Any payload with unhealthy lineage renders INACTIVE_SESSION_LABEL
at the targeted sections.

R4 — Trend Structure inactive-session rendering
Under R2 conditions, id="trend-structure" renders
INACTIVE_SESSION_LABEL without disabled unhealthy-lineage wording.
FAIL: Trend Structure renders STALE, MISSING, or unhealthy-lineage
disabled text under R2 conditions.

R5 — Market Map inactive-session rendering
Under R2 conditions, id="candidate-board" renders
INACTIVE_SESSION_LABEL without unhealthy-lineage wording.
FAIL: Candidate Board renders STALE, MISSING, or unhealthy-lineage
disabled text under R2 conditions.

R6 — Coherent live-session regression
When artifact_lineage_state == "COHERENT" AND
payload.meta.session_type is None, Trend Structure and Market Map
render normal live-session content with no INACTIVE_SESSION_LABEL.
FAIL: Live-session rendering displays INACTIVE_SESSION_LABEL.

R7 — Sunday context-block preservation
The existing sunday_coherent context-block path remains active and
unchanged in behavior. INACTIVE_SESSION_LABEL is additive at the two
targeted section IDs only.
FAIL: sunday_coherent context block is removed, relocated, or its
rendered output diverges from current behavior under matching inputs.

R8 — PRD-116 regression protection
Mixed/stale/missing artifact suppression and diagnostics behavior from
PRD-116 remains unchanged.
FAIL: Any PRD-116 unhealthy-lineage suppression or diagnostics
behavior regresses.

R9 — Renderer-only implementation
Implementation confined to files listed in FILES.
FAIL: Files outside FILES modified.

R10 — Unit-test proof of R2/R4/R5
tests/test_dashboard_renderer.py adds at least one test that renders a
fixture payload with meta.session_type="SUNDAY_PREMARKET" and coherent
lineage, then asserts INACTIVE_SESSION_LABEL appears inside the
elements identified by id="trend-structure" and id="candidate-board".
FAIL: No test asserts the rendered HTML contains
INACTIVE_SESSION_LABEL within both targeted section IDs.

DATA FLOW
payload.meta.session_type
  + artifact_lineage_state
  -> renderer presentation-layer status selection
  -> INACTIVE_SESSION_LABEL OR unhealthy-lineage labels OR
     live-session content
  -> dashboard HTML

No new artifacts. No new payload fields. No lineage-state mutations.
No scheduler/session-engine changes.
