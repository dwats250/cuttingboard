PROBLEM
PRD-189 dropped the 12:50 `prefetch` cron from the run-mode resolver (it now
resolves to `noop`) because, once un-frozen, the prefetch slot was both broken
and pointless:

- The prefetch step sets `PUBLISH_READY=true`, so the Commit-artifacts step
  renders `ui/dashboard.html`. But `execute_prefetch()` only fetches / normalizes
  / validates / derives — it writes no fresh `latest_payload` / `latest_run`. At
  the 12:50 UTC weekday slot the checked-in payload is the previous day's artifact
  (~16-24h old), and a weekday `session_type` is not in `INACTIVE_SESSION_TYPES`
  ({"SUNDAY_PREMARKET"}), so the PRD-119 freshness gate uses the 3h LIVE window
  and raises `StalePublishError` — the prefetch job fails red, daily.
- The OHLCV warm-up cache lives at `data/cache` (config.OHLCV_CACHE_DIR), which
  the Commit-artifacts step does NOT stage (it commits only `reports/`, `logs/`,
  `ui/`), and there is no `actions/cache`. So the warmed cache is discarded at the
  end of the prefetch run; the separate 13:00 live run re-fetches from scratch.
  Prefetch therefore gives no cross-run benefit today.

SCOPE (to be detailed at Stage 0)
- Make prefetch's warm-up actually persist so the live run benefits: either
  commit `data/cache` artifacts (with a sane retention/size policy) or wire
  `actions/cache` keyed on the trading day, so the 13:00 live run reads a warm
  cache instead of re-fetching.
- Make the prefetch slot publish-safe: either suppress the dashboard publish for
  `prefetch` (cache-commit without render) or have prefetch produce fresh,
  gate-passing publishable artifacts before `PUBLISH_READY=true`.
- Re-add the `50 12 * * 1-5` prefetch cron to the resolver (`_DEDICATED`) once the
  above hold, with a CI guard proving the prefetch slot does not trip the PRD-119
  freshness gate.
- Reconcile `docs/prd_history/INTERFACE_LOCK.md` — its prefetch row currently
  lists `Prefetch (12:50 UTC M-F)` as a live scheduled invocation, stale since
  PRD-189 dropped prefetch to noop-on-schedule (true now only for the manual
  `workflow_dispatch` path). Update the row to match whatever prefetch behavior
  this PRD lands.

NON-GOALS
- No change to the live / sunday cron-string resolution landed in PRD-189.
- No change to the intraday/orb routing owned by PRD-192.

GATE
Own PRD because it changes prefetch behavior and the publish path, and adds a
cache-persistence mechanism. Lane + review depth confirmed at Stage 0.
