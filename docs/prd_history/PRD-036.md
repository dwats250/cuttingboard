# PRD-036 — SLIM DASHBOARD RENDERER (READ-ONLY, PAYLOAD-ALIGNED)

STATUS: IN PROGRESS

## GOAL

Render a slim, mobile-first HTML dashboard from existing engine artifacts.

This is a strict presentation layer. No computation, inference, or logic is permitted.

## SCOPE

- Read logs/latest_payload.json
- Read logs/latest_run.json
- Render reports/output/dashboard.html

## OUT OF SCOPE

- Any trading logic (regime, qualification, scoring)
- Any modification of contract or payload
- Any new derived signals or computed fields
- Any use of logs/latest_contract.json
- Any live data fetching
- GitHub Pages deployment

## FILES

ADD:
- cuttingboard/delivery/dashboard_renderer.py
- tests/test_dashboard_renderer.py

MODIFY:
- cuttingboard/delivery/html_renderer.py (ONLY if shared helpers are reused)

DO NOT MODIFY:
- cuttingboard/delivery/payload.py
- cuttingboard/contract.py
- cuttingboard/runtime.py

## REQUIREMENTS

### R1 — Input Sources
- MUST read logs/latest_payload.json
- MUST read logs/latest_run.json
- Both required

### R2 — Read-Only Behavior
- MUST NOT mutate input dictionaries
- MUST NOT call engine, regime, qualification, or data modules
- MUST only write reports/output/dashboard.html

### R3 — Field Mapping (EXACT PATHS ONLY)

Verified against live logs/latest_payload.json and logs/latest_run.json.

```
HEADER
  timestamp         ← payload["meta"]["timestamp"]
  status            ← run["status"]
  market_regime     ← payload["summary"]["market_regime"]
  execution_posture ← run["posture"]

SYSTEM STATE
  tradable          ← payload["summary"]["tradable"]
  stay_flat_reason  ← payload["sections"]["validation_halt_detail"]["reason"]
                       IF payload["sections"]["validation_halt_detail"] is not null
                       ELSE null

PRIMARY SETUP       ← payload["sections"]["top_trades"][0]
  fields: symbol, direction, strategy_tag, entry_mode
  Render ONLY if len(payload["sections"]["top_trades"]) >= 1

SECONDARY SETUPS    ← payload["sections"]["top_trades"][1:5]
  fields per item: symbol, direction, strategy_tag, entry_mode
  Render ONLY if len(payload["sections"]["top_trades"]) >= 2
  MAX 4 items

RUN HEALTH
  system_halted ← run["system_halted"]
  kill_switch   ← run["kill_switch"]
  first_error   ← run["errors"][0] IF len(run["errors"]) > 0 ELSE null
  stale_data    ← run["data_status"] != "ok"
```

NO OTHER FIELDS PERMITTED.

### R4 — Allowed Transformations

Allowed:
- HTML escaping
- bool → "YES" / "NO"
- null → hidden section
- slicing [1:5]
- `run["data_status"] != "ok"` string equality check → bool (explicit comparison, not derivation)

Forbidden:
- deriving values beyond the above
- fallback logic that implies meaning
- ranking or sorting
- interpreting regime
- computing risk or levels

### R5 — HTML Layout (REQUIRED IDS)

Must include block ids:
- dashboard-header
- system-state
- primary-setup
- secondary-setups
- run-health

Visibility:
- primary-setup hidden if top_trades is empty
- secondary-setups hidden if fewer than 2 top_trades items
- stay_flat_reason hidden if null
- first_error hidden if null

### R6 — Style Constraints
- Single HTML file
- Inline CSS only
- Dark theme
- Max width: 640px
- No tables
- No JavaScript
- ASCII-only output

### R7 — Output Constraints
- Write reports/output/dashboard.html
- Must include <!doctype html>, <html>, <head>, <body>
- File size < 200KB
- Deterministic output (byte-for-byte)

## DATA FLOW (RENDERER SCOPE ONLY)

```
logs/latest_payload.json ─┐
                          ├── dashboard_renderer.py → reports/output/dashboard.html
logs/latest_run.json    ──┘
```

## FAIL CONDITIONS

- F1: latest_payload.json missing → FAIL
- F2: latest_run.json missing → FAIL
- F3: either file contains invalid JSON → FAIL
- F4: any required path in R3 is missing from input → FAIL
- F5: rendered value does not match value at specified JSON path → FAIL
- F6: input dicts are modified after render → FAIL
- F7: renderer imports or calls engine/market/regime/qualification modules → FAIL
- F8: required block IDs missing OR empty sections rendered → FAIL
- F9: JavaScript OR table markup OR non-ASCII present → FAIL
- F10: file missing OR >200KB OR invalid HTML structure → FAIL
- F11: identical inputs produce different output → FAIL
