WHY NOW
On a rising-yield reading the 10Y row votes risk-OFF correctly but its rationale
subtitle reads "easing yields favor risk" — vote and prose disagree. The vote is
direction-aware (dashboard_renderer.py:2017–2021: arrow ± a contra-cyclical flip
for rates/dollar/volatility), but the rationale is a FIXED string keyed only by
payload_key (dashboard_renderer.py:2024 → MACRO_BIAS_INTERPRETATION;
macro_tape_layout.py:86–94, "rates": "easing yields favor risk"). The string has
no direction term, so it renders identically whether yields rise or fall. The
current test (test_prd177_r3_macro_evidence_rows_present) only asserts the interp
string is NON-EMPTY — so an inverted rationale passes CI. That test gap is the
actual bug: the wrong text shipped because nothing checks agreement.

MAX EXPECTED DELTA
The interpretation strings become direction-keyed (per driver: a rising form, a
falling form, and a neutral/no-vote form); the renderer selects by the SAME
arrow/vote sign it already computes; the test asserts vote↔prose agreement.
~20–40 lines across macro_tape_layout.py + dashboard_renderer.py + the two tests.
No change to the vote computation or the macro-bias arithmetic.

GOAL
The rationale subtitle agrees in direction with the vote for every driver and
every direction (including the flat/neutral no-vote case), and a test enforces
that agreement.

SCOPE
- Make MACRO_BIAS_INTERPRETATION direction-keyed: each driver carries distinct
  rising / falling rationale text PLUS an explicit neutral / no-vote string.
- The renderer selects the rationale using the SAME arrow + contra-cyclical sign
  it already uses for the vote (single source of direction; no second sign calc).
- NEUTRAL / FLAT edge case (confirmed present): _pct_arrow (dashboard_renderer.py:
  959) and _direction_arrow (:975) return _FLAT ("→") on a near-zero reading, and
  the vote block (:2017) emits "no vote" when the arrow is neither _UP nor _DOWN.
  Direction-keying by up/down ALONE would leave the flat case falling back to an
  empty/stale string and half-recreate the original vote↔prose mismatch. The dict
  MUST therefore carry a neutral/no-vote form per driver (or a defined shared
  one), and the renderer must select it when the arrow is _FLAT.
- Strengthen the test: assert vote sign and rationale direction AGREE for the
  rising-yield and falling-yield cases (across all four drivers) AND that the
  flat/no-vote case renders the neutral string, not risk-on/off prose. Not done
  until this assertion exists and FAILS against today's static string.

OUT OF SCOPE
- No change to the vote computation or the macro-bias arithmetic; cyclicality
  classification (MACRO_BIAS_CONTRA_CYCLICAL / PRO_CYCLICAL) is unchanged.
- No new drivers; no change to the macro-tape data feed, thresholds, or the
  arrow/no-vote logic itself.
- No restyling of the evidence-row layout beyond the rationale text source.

REQUIREMENTS

R1 — Rationale is direction-keyed
Each driver's rationale is selected by direction from the same arrow/vote sign
the row already computes.
FAIL: the rendered rationale for a driver is identical for its rising and
falling readings.

R2 — Vote and rationale agree
For every driver and direction, a risk-OFF vote never renders risk-ON rationale
prose (and vice-versa).
FAIL: any row renders a vote sign that contradicts its rationale direction
(e.g. risk-OFF vote + "easing yields favor risk").

R3 — Flat/no-vote renders a neutral rationale
When the arrow is _FLAT and the vote is "no vote", the row renders the driver's
neutral rationale string, not a directional one and not empty.
FAIL: a flat reading renders empty rationale or a risk-on/off directional string.

R4 — Test enforces agreement, not mere presence
The macro-evidence test asserts vote-sign↔rationale-direction agreement (rising,
falling, and flat) and fails against today's static "easing yields favor risk"
on a rising 10Y.
FAIL: the test passes with the inverted (pre-fix) rationale.

DATA FLOW
macro driver arrow (rising / falling / flat) → vote sign (± contra-cyclical
flip; "no vote" when flat) → the SAME sign selects the direction-keyed (or
neutral) rationale → rendered subtitle agrees with the vote.

FAIL CONDITIONS
- A driver's rationale text is direction-invariant.
- A rendered vote sign contradicts its rationale prose.
- A flat reading renders empty or directional rationale.
- The agreement test passes against the pre-fix inverted string.

VALIDATION

  first.
- Manual: render against the current macro_drivers_snapshot.json (^TNX +1.20%,
