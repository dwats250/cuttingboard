WHY NOW
On a rising-yield reading the 10Y row votes risk-OFF correctly but its rationale
subtitle reads "easing yields favor risk" — vote and prose disagree. The vote is
direction-aware (dashboard_renderer.py:2087–2093: arrow ± a contra-cyclical flip
for contra-cyclical drivers, "no vote" when the arrow is neither ↑ nor ↓), but
the rationale is a FIXED string keyed only by payload_key (dashboard_renderer.py:
2094 → MACRO_BIAS_INTERPRETATION; macro_tape_layout.py:87–94, "rates": "easing
yields favor risk"). The string has no direction term, so it renders identically
whether yields rise or fall. The guard test
(test_prd177_r3_macro_evidence_rows_present, tests/test_dashboard_renderer.py:
3553–3554) only asserts the interp string is NON-EMPTY — so an inverted rationale
passes CI. That test gap is why the wrong text shipped: nothing checks agreement.

~15–30 lines across macro_tape_layout.py + dashboard_renderer.py + one test file.
No change to the vote computation or the macro-bias arithmetic.

CHANGE SURFACE

  payload_key → string map to payload_key → {rising, falling} strings, plus one
  private shared neutral constant.

  same sign, and the shared neutral string on the no-vote branch.

REQUIREMENTS

R1 — Rationale is direction-keyed and agrees with the vote
Each driver's rationale is selected by direction from the same arrow/vote sign the
row already computes; a risk-OFF vote never renders risk-ON rationale prose, and
vice-versa.
FAIL: a driver's rendered rationale is identical for its rising and falling
readings, OR a rendered vote sign contradicts its rationale direction (e.g.
risk-OFF vote + "easing yields favor risk").

R2 — Flat/no-vote renders the shared neutral string
When the arrow is flat (neither ↑ nor ↓) and the vote is "no vote", the row
renders the shared neutral rationale string — not a directional one and not empty.
FAIL: a flat reading renders empty rationale or a directional (risk-on/off)
string.

R3 — Test enforces agreement, not mere presence
The macro-evidence test asserts vote-sign ↔ rationale-direction agreement for the
rising and falling cases and the shared neutral string for the flat case, and
fails against today's static "easing yields favor risk" on a rising 10Y.
FAIL: the test passes with the inverted (pre-fix) static rationale.

DATA FLOW
macro driver arrow (rising ↑ / falling ↓ / flat →) → vote sign (± contra-cyclical
flip; "no vote" when flat) → the SAME sign selects the rising/falling rationale,
or the shared neutral string when flat → rendered subtitle agrees with the vote.

FAIL CONDITIONS
- A driver's rationale text is identical for its rising and falling readings.
- A rendered vote sign contradicts its rationale prose.
- A flat reading renders empty or directional rationale.
- The agreement test passes against the pre-fix static string.

VALIDATION

  phrasing; falling-10Y asserts the inverse; flat-10Y asserts the shared neutral

  current code first (RED), then passes after the fix.
- Manual: render against the current macro_drivers_snapshot.json (rising ^TNX) and
  confirm the 10Y row reads risk-OFF with a consistent subtitle.
