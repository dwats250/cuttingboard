# PRD-114 Review — Codex (Cross-Review)

Scope: cross-review of PRD-114 and PRD-114.review.claude.md per
PRD-113 governance. Findings tagged per the Review Failure Taxonomy.

## Top-line position

ACCEPT WITH AMENDMENTS

## Findings

F1 — CLASS challenge: ACCEPT. SIDECAR is the correct class because
PRD-114 introduces an observe-only producer artifact with no v1
consumer, no contract mutation, and no decision-path readback. CONSUMER
would be wrong unless this PRD also modified dashboard, notifications,
or another reader of finalized artifacts; PRD-114 explicitly forbids
that. The artifact's low immediate information value is a product
question, not a CLASS reclassification trigger. Taxonomy label:
factual drift.

F2 — `watch_reason` and `event_risk` semantics: MODIFY. Dropping
`event_risk` is necessary and correct. Keeping `watch_reason` is
acceptable only if it remains literal static metadata from the frozen
tuple and is renamed or described as non-dynamic human curation text in
R1/R3. Removing it entirely would be the strictest v1, but it is not
required if the PRD adds a fail condition that `watch_reason` cannot be
computed from price, volume, trend, calendar, news, or thresholds.
Taxonomy label: derived semantics.

F3 — Duplication with `market_map`: ACCEPT. The proposed four-field
schema does not duplicate `market_map` grading or
`trend_structure_snapshot` semantics because it avoids grade, lifecycle,
data_status, vwap, SMA, relative_volume, trend_alignment, entry_context,
and reason synthesis. The only overlap is `current_price`, which is a
pass-through scalar already available from `normalized_quotes`, not a
semantic grade. Taxonomy label: schema ambiguity.

F4 — Manual-state persistence: ACCEPT. The curated list living in-repo
is defensible because PRD-114 wants deterministic reviewable state,
stable tests, and no external/manual artifact dependency. A generated or
external list would either hide ownership outside the PRD FILES boundary
or add stale/malformed-input handling that v1 does not need. Taxonomy
label: ownership violation.

F5 — Minimum viable schema: MODIFY. The true absolute minimum for
symbol presence is `symbol` plus `current_price`, with the symbol key
itself already duplicating `symbol`. However, for the stated purpose of
a curated discretionary index, `sector_theme` is reasonable static
context. `watch_reason` is the only non-minimum field and should either
be removed or explicitly frozen as static curation metadata with no
dynamic semantics. Taxonomy label: scope creep.

F6 — Ranking-engine creep: MODIFY. The PRD mostly blocks ranking by
forbidding ranking, scoring, AI commentary, and dynamic fields, but
there is latent ordering semantics in "symbol order follows insertion
order of the frozen constant." That order can become a visual or
consumer priority ranking later. Add wording that constant order is
serialization-only and MUST NOT be interpreted as rank, priority, or
trade preference. Taxonomy label: consumer contamination.

F7 — Hidden coupling: MODIFY. The stated runtime placement immediately
after `_write_trend_structure_snapshot()` creates temporal adjacency but
not read coupling as long as the new helper only receives
`normalized_quotes` and `generated_at`. PRD-114 should say explicitly
that `_write_watchlist_snapshot()` MUST NOT read
`logs/trend_structure_snapshot.json`, `TREND_STRUCTURE_PATH`, or the
trend builder output in memory. Taxonomy label: hidden coupling.

F8 — Determinism / staleness: ACCEPT. Accepting `generated_at` from
runtime matches the trend_structure builder pattern: the builder formats
a provided aware datetime and does not call wall-clock APIs. With stable
JSON serialization, output bytes should be identical for identical
`normalized_quotes`, frozen constant order, and `generated_at`. The main
byte-drift risks are non-normalized float representations from quote
objects or accidental use of an unordered input for the universe; R1/R5
already constrain the universe order. Taxonomy label: non-determinism.

F9 — FILES recommendation: ACCEPT. The four-file FILES list is correct
for PROPOSED implementation scope: new builder, new tests, append-only
runtime hook, and artifact flow map row. `docs/PROJECT_STATE.md` and
`docs/PRD_REGISTRY.md` should remain excluded from FILES because PRD-113
allows registry/state changes on close as exceptions, not as normal
implementation surfaces. Taxonomy label: scope creep.

F10 — FAIL condition recommendation: MODIFY. Claude's F9 must be
tightened. In the current hourly runtime path, `_write_market_map_file()`
and `_write_trend_structure_snapshot()` run inside the hourly block after
contract/summary construction, outside the `if not
validation_summary.system_halted` branch. Therefore a
`_write_watchlist_snapshot()` call placed immediately after
`_write_trend_structure_snapshot()` would be reached on a validation HALT
unless explicitly guarded. Add R11: on `validation_summary.system_halted`,
the watchlist snapshot write MUST be skipped and the previous artifact
left untouched. Taxonomy label: stale-data ambiguity.

F11 — Deferred fields: ACCEPT. Deferring vwap relation, SMA relations,
relative_volume, trend_alignment, entry_context, and event_risk to
PRD-115+ is correct because they either already belong to
`trend_structure_snapshot` or require a future read-only join/consumer.
`event_risk` should be treated as rejected for v1, not casually deferred,
because calendar/news integration creates a new data source and freshness
contract. No deferred field belongs in v1. Taxonomy label: scope creep.

F12 — PRD-113 failure taxonomy application: MODIFY. Against the nine
forbidden patterns: hidden coupling is partially addressed but needs the
no-read-from-trend wording in F7; payload mutation is addressed by R7/R9;
renderer-derived semantics is addressed by no dashboard changes;
threshold→label synthesis is addressed only if F2 freezes or removes
`watch_reason`; wall-clock dependence is addressed by generated_at
injection; partial malformed recovery is not directly relevant because
there is no external input beyond quotes and the frozen constant; silent
fallback is addressed for missing quotes by explicit null; sidecar
mutation is addressed by the isolated artifact path; while-we're-here
mutations is addressed by R10. Claude did not explicitly call out the
serialization-order-as-rank risk or the trend snapshot no-read
requirement. Taxonomy label: schema ambiguity.

## Disagreements with Claude review

- Claude F9 says HALT already short-circuits before the trend-structure
  write and would therefore short-circuit the watchlist write. Current
  `runtime.py` does not support that: the hourly artifact write block is
  reached after the `not validation_summary.system_halted` branch.
- Claude accepts `watch_reason` as-is. I would require either removal or
  stronger wording that it is static curation metadata and cannot become
  a computed reason.
- Claude did not identify constant insertion order as a latent ranking
  surface; I recommend making the order serialization-only.

## Adjudication recommendation

Adjudication is conditional, not mandatory, because PRD-114 is
SIDECAR-class rather than CONTRACT-class. If the author accepts the R11
HALT skip, static `watch_reason` fence, no-read-from-trend fence, and
serialization-only ordering amendment, no adjudication artifact is
needed. If Claude maintains that HALT already short-circuits this write,
that unresolved runtime disagreement should be adjudicated before moving
PRD-114 to IN PROGRESS.
