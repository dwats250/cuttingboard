# PRD-090 - Candidate Board Display Tiers

**Status:** COMPLETE
**Owner:** Codex

---

## GOAL

Improve Candidate Board scanability without changing candidate logic.

When there are no actionable setups, the dashboard currently shows large lower-grade candidate cards in the main flow. This wastes screen space and makes the board feel noisy. The main dashboard should prioritize A+, A, and B setups while keeping C, D, and F candidates available for inspection.

---

## SCOPE

1. Modify `cuttingboard/delivery/dashboard_renderer.py`.
2. Modify `tests/test_dashboard_renderer.py`.
3. A+, A, and B tier groups render as `<div class="tier-group">` — no change from current.
4. C tier group renders as `<details class="tier-group" id="tier-c">` with `<summary class="tier-header">C — EARLY (N)</summary>`. Collapsed by default.
5. D/F tier group renders as `<details class="tier-group" id="tier-df">` with `<summary class="tier-header">D/F — FAILING (N)</summary>`. Collapsed by default.

## IMPLEMENTATION NOTES

Presentation-only Candidate Board restructuring.

In the tier render loop (lines 1078–1089 of `dashboard_renderer.py`), the `<div class="tier-group">` / `<div class="tier-header">` / `</div>` triple becomes conditional:

```python
is_low_tier = tier_grades.isdisjoint(_HIGH_GRADES)
if is_low_tier:
    w(f'  <details class="tier-group" id="tier-{tier_id}">')
    w(f'    <summary class="tier-header">{_esc(tier_label)} ({len(tier_syms)})</summary>')
else:
    w(f'  <div class="tier-group" id="tier-{tier_id}">')
    w(f'    <div class="tier-header">{_esc(tier_label)} ({len(tier_syms)})</div>')
for sym in tier_syms:
    _render_candidate_card(w, sym, symbols[sym], contract_entry=(contract_entry_map or {}).get(sym))
if is_low_tier:
    w("  </details>")
else:
    w("  </div>")
```

The CSS selector at line 265 changes from:
```python
"#artifact-diagnostics summary,#run-history summary{cursor:pointer;list-style:none}"
```
to:
```python
"#artifact-diagnostics summary,#run-history summary,details.tier-group summary{cursor:pointer;list-style:none}"
```

Before: C/D/F candidates rendered as full visible cards in the main flow.
After: C and D/F tiers are collapsed by default. A+/A/B rendering is unchanged. `_render_candidate_card` is not touched.
