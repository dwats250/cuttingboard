## Cohesiveness

GOAL -> SCOPE -> REQUIREMENTS -> DATA FLOW form a coherent chain. The
revised PRD resolves all internal gaps that were identified in the prior
draft:

1. Trend Structure mapping now includes explicit unhealthy-lineage
   (`MIXED` / `STALE` / `MISSING`) rows at the top of the precedence list,
   matching the PRD-116 disabled-state rendering at
   `dashboard_renderer.py:1208,1508-1513`.
2. Market Map mapping now includes explicit `STALE` and `MISSING` lineage
   rows, matching the renderer suppression branches at lines 1573-1588.
3. The mapping tables now state an explicit ordered evaluation rule
   ("first matching row wins"), removing ambiguity between simultaneous
   signals (resolves prior conflict between tests #15 and #20).
4. The FALLBACK branch for Trend Structure is now backed by a new pure
   helper `_trend_symbols_usable(snapshot)` that computes per-symbol
   completeness independently of the all-or-nothing
   `_trend_structure_records` helper, making test #7 reachable.
5. The Macro Tape MIXED branch was dropped because no per-slot freshness
   signal exists in `_build_tape_value_slots`; the mapping is now
   strictly OK / FALLBACK / MISSING.
6. The Permission display mapping now explicitly handles unhealthy
   lineage (UNKNOWN) before MONITOR_ONLY, so a MIXED_ARTIFACTS title
   does not coincide with a misleading MONITOR_ONLY Permission cell.
