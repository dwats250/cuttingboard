CLASS
CONSUMER

WHY NOW
The candidate level diagram plots each level at its true price-mapped y but labels
it only by name (PRIOR_LOW, 0.618, VWAP, ENTRY), so the ladder shows structure
without the dollar levels a trader acts on. The renderer already holds every price
float; it just never prints them.

MAX EXPECTED DELTA
25 production LOC (dashboard_renderer.py _render_level_diagram).

GOAL
Append the dollar value to every level label so each line reads as an actionable
price (PRIOR_LOW 75.00, 0.618 75.10, VWAP 75.05, ENTRY 75.00). Descriptive facts,
not forecasts — consistent with VISION "description, not prediction".

SCOPE
- In _render_level_diagram, append the formatted price (thousands-separated, 2dp)
  to each fib, zone, VWAP, and ENTRY label.

OUT OF SCOPE
- A separate numeric axis rail, a real OHLC/candlestick chart (would need intraday
  bar data — future PRD), or any change to which levels render.

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dash_level_diagram.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — the four labels.append(...) calls in
_render_level_diagram only. No change to line geometry, declutter, or inputs.

REQUIREMENTS

R1 — Priced labels
Every rendered level label includes its dollar value formatted to 2 decimals.
FAIL: a fib / zone / VWAP / ENTRY label renders without its numeric price.

DATA FLOW
fib_levels.retracements / watch_zones[].level / vwap_level / contract_entry
(existing floats) → label text "<name> <price>" → SVG

FAIL CONDITIONS
- Any level label renders without its price value.

VALIDATION
Manual:
- Render a card with levels; each diagram line shows "<name> <price>".
Automated:
- Test asserts the ENTRY and a zone label carry their price value.
