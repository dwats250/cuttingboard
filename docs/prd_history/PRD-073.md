# PRD-073 — Human-Readable Dashboard Trader View

**Status:** IN PROGRESS
**Depends on:** PRD-055, PRD-062, PRD-066

---

## GOAL

Convert the Signal Forge dashboard from engine-log readability to trader-facing readability. All rendering changes are cosmetic and organizational — deterministic, read-only, no logic mutations.

---

## SCOPE

- Decision title: trader-readable label derived from `outcome`, `system_halted`, `status`
- Posture labels: human-readable display strings
- Macro pressure driver labels: human-readable keys
- Candidate card structure: grade, lifecycle, bias, structure, setup_state (server path); policy_allowed context (UI path only)
- Section order and labels in `dashboard_renderer.py`
- Outcome/status badge rendering in `ui/app.js`

---

## OUT OF SCOPE

- No changes to decision logic, qualification gates, or regime engine
- No changes to `contract.py`, `regime.py`, `output.py`, or any Python module outside `cuttingboard/delivery/dashboard_renderer.py` and its test file
- No new data sources
- No new fields in any artifact
- No notification changes

---

## FILES

- `cuttingboard/delivery/dashboard_renderer.py`
- `tests/test_dashboard_renderer.py`
- `ui/app.js`
- `ui/index.html`
- `ui/styles.css`

---

## REQUIREMENTS

### RULE — FIELD AVAILABILITY

- Fields from `contract.json` (`policy_allowed`, `block_reason`, `size_multiplier`) are **UI PATH ONLY**.
- Server renderer (`dashboard_renderer.py`) MUST NOT reference `contract.json` fields.
- Server renderer MUST operate strictly from:
  - `latest_payload.json`
  - `latest_run.json`
  - `market_map.json`
- Any requirement referencing `contract.json` fields MUST explicitly state **"UI path only"**.

---

### R1 — Decision-first header

The decision title MUST be derived ONLY from:
- `outcome`
- `system_halted`
- `status`

This rule applies identically to both rendering paths.

**Decision title mapping:**

| Condition | Title |
|-----------|-------|
| `status in {"FAIL", "ERROR"}` or `system_halted == true` | `"SYSTEM HALT"` |
| `outcome == "TRADE"` | `"TRADE SETUP ACTIVE"` |
| `outcome == "NO_TRADE"` | `"NO TRADE"` |
| otherwise | `"MONITOR"` |

**UI path only — additional informational line:**

If any `trade_candidate` in `contract.json` has `policy_allowed == true`:
- Display: `"Actionable candidates present"`
- This MUST NOT override or alter the decision title.

FAIL: Decision title does not match mapping for given inputs.
FAIL: UI path informational line appears when no `policy_allowed == true` candidate exists.
FAIL: Server renderer reads `contract.json`.

---

### R2 — Posture label mapping

Display human-readable posture labels in both rendering paths.

| Internal value | Display label |
|----------------|---------------|
| `AGGRESSIVE_LONG` | Aggressive Long |
| `CONTROLLED_LONG` | Controlled Long |
| `EXPANSION_LONG` | Expansion Long |
| `NEUTRAL_PREMIUM` | Neutral Premium |
| `DEFENSIVE_SHORT` | Defensive Short |
| `STAY_FLAT` | Stay Flat |
| `CHAOTIC` | Chaotic |
| Unknown | raw value |

FAIL: Any posture value with a mapping renders the raw internal string instead of its label.

---

### R3 — Macro pressure driver labels

`build_macro_pressure()` returns these keys:

- `volatility_pressure`
- `dollar_pressure`
- `rates_pressure`
- `bitcoin_pressure`
- `overall_pressure`

Display mapping:

| Key | Display label |
|-----|---------------|
| `volatility_pressure` | Volatility |
| `dollar_pressure` | Dollar |
| `rates_pressure` | Rates |
| `bitcoin_pressure` | Bitcoin |
| `overall_pressure` | Overall |

FAIL: Any driver key with a mapping renders the raw key string instead of its label.

---

### R4 — Candidate card display

**UI path** (`ui/app.js`, reads `contract.json` + `latest_payload.json`):

Each candidate card must display:
- Symbol
- Grade
- Lifecycle state (`setup_state`)
- Bias
- Structure
- `policy_allowed` status: "Actionable" if `true`, "Blocked" if `false` (UI path only)
- `block_reason` if `policy_allowed == false` (UI path only)

**Server path** (`dashboard_renderer.py`, reads `latest_payload.json` + `market_map.json`):

Each candidate card must display:
- Grade
- Lifecycle state (if `setup_state` present)
- Bias
- Structure

Server path MUST NOT display "Actionable" or "Blocked" status.
Server path MUST NOT reference `policy_allowed` or `block_reason`.

FAIL: Server path renders "Actionable" or "Blocked" labels.
FAIL: Server path references `contract.json` fields.
FAIL: UI path omits `policy_allowed` status when the field is present.

---

### R5 — Section order (`dashboard_renderer.py`)

Render sections in this order:
1. Decision header (outcome + posture)
2. Macro tape
3. Macro pressure
4. Candidates
5. System state
6. Run delta

FAIL: Section order deviates from above sequence.

---

### R6 — Section labels (`dashboard_renderer.py`)

| Current label | New display label |
|---------------|-------------------|
| `run-delta` | "Changes Since Last Run" |
| `system-state` | "System State" |
| `macro-tape` | "Macro Tape" |

FAIL: Any renamed label still renders under its old internal name.

---

### R7 — Outcome/status badge (`ui/app.js`)

Apply CSS class to outcome badge based on value:

| Value | CSS class |
|-------|-----------|
| `"TRADE"` | `badge-trade` |
| `"NO_TRADE"` | `badge-no-trade` |
| `"HALT"` | `badge-halt` |
| other | `badge-unknown` |

FAIL: Badge class does not match outcome value.

---

### R8 — Empty state

If no candidates:
- Display: "No candidates evaluated this run."
- No errors thrown.

FAIL: Empty candidate set throws an error or renders blank.

---

### R9 — No logic changes

No changes to:
- Decision output values
- Qualification gates
- Regime engine
- Scoring
- Policy logic
- Payload structure
- Contract structure

FAIL: Any non-display field value changes as a result of this PRD.
