FINDINGS:

1. IWM dropout can loosen the global posture. Suppose EXPANSION fails and SPY, QQQ, DXY, and TNX cast RISK_ON while IWM, VIX level, VIX change, and BTC are neutral. With IWM present at 0.0: net=4, total=8, confidence=0.50, RISK_ON/STAY_FLAT. With IWM absent: net=4, total=7, confidence=0.571, RISK_ON/CONTROLLED_LONG. Qualification no longer short-circuits; contract status changes from STAY_FLAT to OK, system_state.tradable becomes true, and notification priority becomes HIGH instead of LOW. Execution policy may still block an eventual decision below its separate 0.60 threshold, but the decision, qualification, contract, and notification paths have already loosened.

2. A normalization-only NaN dropout is missing from the failure ledger. Given a non-HALT RawQuote with fetch_succeeded=True and pct_change_raw=NaN, normalize_quote returns None, while extract_fetch_failures does not register it because the fetch technically succeeded. Validation therefore reports N-1 attempted, N-1 validated, zero failed; 0.0 would report N attempted and N validated. The report, audit record, and premarket formatter can present the reduced denominator as fully healthy rather than identifying a failed symbol.

3. Missing BTC can fail open inside macro-pressure sizing. Given a qualified decision with confidence >=0.60 and other macro components that would produce MIXED pressure when BTC is present at 0.0, the present-zero path applies a 0.75 size multiplier. With BTC absent, _build_macro_drivers raises because bitcoin is required; _compute_overall_pressure catches the exception and returns UNKNOWN; execution_policy.py:239 treats UNKNOWN like NEUTRAL and allows the full 1.0 size. Full-pipeline contract construction later raises again and replaces final artifacts with ERROR/HALT, so the normal alert is prevented, but the pressure guard itself is not equivalent or conservative.

4. The active hourly path has two surface/artifact divergences. First, optional CL=F/GC=F/SI=F or non-HALT tradable GLD/GDX/SLV/XLE absence silently removes its notification row; 0.0 renders a level and either "+0.0%" or the tradable price. Second, alert_runner.py:97 invokes _execute_notify_run, which sends the normal hourly alert before building the contract. If required ^VIX, DXY, TNX, or BTC is absent, contract construction then raises, the exception handler sends a second failure alert, writes traceback/error artifacts, and does not save the hourly slot. The 0.0 path sends one alert, writes normal artifacts, and can save the slot.
