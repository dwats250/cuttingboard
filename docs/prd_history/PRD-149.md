GOAL
GitHub Actions' scheduled-cron event delivery is degraded enough on this repo to drop entire days of firings (2026-05-19: zero scheduled firings of either workflow over ~18+ hours, with manual `workflow_dispatch` working normally). The PRD-141 slot canonicalization and PRD-144 backup-cron redundancy both presume some scheduled firing reaches the runner. Add a Cloudflare Workers cron sidecar that fires `workflow_dispatch` against `hourly_alert.yml` on the same canonical UTC times the GH schedule targets. The existing PRD-141 slot dedup gate guarantees one-alert-per-PT-hour regardless of whether GH cron, the Worker, or both fire ‚Äî provided the workflow no longer unconditionally passes `--force-slot` for `workflow_dispatch`. Include the minimum workflow YAML edit needed to differentiate operator manual dispatch (keep `--force-slot`) from external-scheduler dispatch (drop `--force-slot`, route through dedup).

SCOPE
- New Cloudflare Worker under `tools/external_trigger/` that POSTs to `POST /repos/dwats250/cuttingboard/actions/workflows/hourly_alert.yml/dispatches` with `{ref: "main", inputs: {scheduler: "external_cron"}}` on each scheduled tick.
- Worker cron schedule mirrors `.github/workflows/hourly_alert.yml` (premarket warm-ups at 12:50/12:55/13:00/13:05/13:50 UTC plus intraday `*/30 14-21` UTC).
- `.github/workflows/hourly_alert.yml`: add `workflow_dispatch.inputs.scheduler` (enum: `operator | external_cron`, default `operator`) and update the `Run hourly alert` step to pass `--force-slot` only when `github.event_name == 'workflow_dispatch' AND inputs.scheduler != 'external_cron'`.
- README under `tools/external_trigger/` documenting deploy (`wrangler deploy`) and secret setup (`wrangler secret put GITHUB_TOKEN`, fine-grained PAT scoped to this repo with Actions Read and write).

OUT OF SCOPE
- Auto-deploy of the Worker via CI.
- Removal of any in-repo cron entry from `.github/workflows/hourly_alert.yml`.
- Monitoring/alerting on Worker failures.
- Any new Python module under `cuttingboard/`.
- A second Worker for `Cuttingboard Pipeline` workflow.
