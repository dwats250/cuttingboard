## REQUIRED EDITS

1. **Pin `openai/codex-action` to a reviewed release commit SHA** (currently
   `@v1` with a `TODO(pin)` comment). For a HIGH-RISK workflow that handles a
   secret on a public repo, a moving tag is a supply-chain hole: a retag could
   swap the action body under the proxied key. Resolve `@v1` to a full commit SHA
   and pin it before this workflow is trusted in production. Tracked by the
   in-file TODO and gated by the human merge; promote it from TODO to done.

## RECOMMENDED EDITS

1. **Verify the pinned `codex-version: "0.2.0"` and default `model: gpt-5-codex`
   are real/current** before the first run — a wrong model id or nonexistent CLI
   version fails the `review` job. The first-run self-review (close condition)
   will surface this; fix forward if it does.
2. **Record the pinned `codex-version` input as the authoritative version** in
   the artifact, not only `codex --version`: the Codex binary is installed by the
   action and may not be on `PATH` in the later assemble step (the `|| echo
   'unknown'` fallback would then record "unknown"). Belt-and-suspenders: emit
   both the pinned input value and the queried version.
3. **Confirm `actions/download-artifact@v7` major matches `upload-artifact@v7`**
   (the repo bumped upload to v7 in PRD-185; verify the download major exists and
   pairs). A mismatch fails the `commit` job at first run.
