## What changed (the redesign)
- `land` **no longer opens or auto-merges a PR.** It checks out the **base**,
  commits the artifact onto `codex-review/PRD-<N>-<short-sha>`, pushes **only that
  branch**, writes a job-summary pointing to it, and stops.
- **Rooted at the base, not `inputs.sha`** — so the branch is artifact-only; an
  unmerged-tip root would have let a later PR carry that PRD's implementation.
- `land` permissions reduced to **`contents: write` only** (no `pull-requests`
  scope), still **no key**. `review` unchanged: proxied key, `read-only`,
  `contents: read`, no write.
- **Fail-closed guard** retained (aborts unless the work branch is under
  `codex-review/`); never pushes `HEAD:${base}` / `HEAD:main`.
