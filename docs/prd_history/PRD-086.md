PRD-086 — Carry Forward current_price Through Sunday Market Map

STATUS
COMPLETE

GOAL
When MODE_SUNDAY produces a market_map with current_price=None for all tradable symbols (because normalized_quotes={} by design), inject_lifecycle must carry the previous market_map's current_price forward so the dashboard does not display DATA_UNAVAILABLE for SPY/QQQ/GLD/SLV/XLE on Sundays.

SCOPE
- inject_lifecycle in market_map_lifecycle.py: carry previous current_price when current is None
- Tests in tests/test_market_map_lifecycle.py covering the carry-forward behavior

OUT OF SCOPE
- No changes to _run_pipeline, runtime.py, or the Sunday fetch-block logic
- No changes to dashboard_renderer.py
- No new snapshot files or fallback mechanisms
- No changes to macro driver tape values
- No changes to any other market_map field (grade, bias, structure, etc.)

FILES
M cuttingboard/market_map_lifecycle.py
M tests/test_market_map_lifecycle.py

REQUIREMENTS

R1 — Carry previous current_price when current is None
When inject_lifecycle processes a symbol whose current_map current_price is None, and the previous_map has a non-None finite numeric current_price for that symbol, inject_lifecycle must set that symbol's current_price to the previous value.

FAIL: After inject_lifecycle, any symbol with current_price=None in current_map AND a real numeric current_price in previous_map still shows current_price=None in the result.

R2 — Do not overwrite a real current price
When current_map has a non-None numeric current_price for a symbol, inject_lifecycle must not replace it with the previous value regardless of what previous_map contains.

FAIL: A symbol with a real current_price in current_map has its current_price changed by inject_lifecycle.

R3 — No previous_map means no carry
When previous_map is None or does not contain the symbol, current_price remains None (no fabrication).

FAIL: current_price is non-None after inject_lifecycle when previous_map is None or lacks the symbol.

R4 — Non-finite values are not carried
previous_map current_price values that are None, non-numeric, or non-finite (inf, nan) are not carried forward.

FAIL: A non-finite previous current_price is injected into the result.

DATA FLOW
MODE_SUNDAY _run_pipeline
  → build_market_map(normalized_quotes={})
    → _build_symbol_record(quote=None) → current_price=None
  → inject_lifecycle(current_map, previous_map)
    → for each symbol: if current_price is None and previous has finite price → carry forward
  → _write_market_map_file(enhanced_market_map)
  → dashboard_renderer reads market_map.json → current_price present → value shown

FAIL CONDITIONS
- inject_lifecycle returns current_price=None for a symbol that has a real price in previous_map
- inject_lifecycle changes current_price for a symbol that already has a real price in current_map
- inject_lifecycle carries None, nan, or inf from previous_map
- inject_lifecycle carries a price when previous_map is None
- Any existing test in test_market_map_lifecycle.py breaks

VALIDATION
Manual:
1. Run: python3 -m pytest tests/test_market_map_lifecycle.py -v
   Expected: all tests pass including new carry-forward cases
2. Run: python3 -m pytest --tb=short -q
   Expected: full suite green, no regressions
3. Construct a minimal script: build a current_map with current_price=None for SPY,
   a previous_map with current_price=512.0 for SPY, call inject_lifecycle, confirm
   result["symbols"]["SPY"]["current_price"] == 512.0
