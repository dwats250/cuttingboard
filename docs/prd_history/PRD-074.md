# PRD-074 — Chart Context Layer (Level Diagram)

**Status:** IN PROGRESS
**Depends on:** PRD-073, PRD-029

---

## GOAL

Add a deterministic, read-only level diagram for each candidate to visually represent price context using key levels and a VWAP reference.

This layer MUST NOT influence trade decisions, policy, or scoring.

This is a **level diagram**, not a chart. Purpose: show relative positioning of entry vs VWAP and entry vs levels. Does NOT visualize price movement over time.

---

## SCOPE

- Render one static level diagram per candidate symbol
- Display entry price as the primary reference anchor
- Display scalar VWAP value as a single horizontal reference line (when available)
- Display fib_levels as horizontal level markers (when non-null)
- Display watch_zones as horizontal level markers (when non-empty)
- Diagram visually aligns with existing candidate card

---

## OUT OF SCOPE

- No candlestick or line price charts
- No price series (OHLCV arrays) — not available in UI data sources
- No VWAP time series — only scalar VWAP float is available
- No sparklines or time-axis rendering
- No RSI, MACD, moving averages, oscillators
- No prediction logic
- No signal generation
- No scoring changes
- No execution logic
- No macro logic changes
- No lifecycle changes
- No new artifact fields
- No new backend data sources
- No external API calls
- No parquet file reads

Future charting (sparkline / price context) is explicitly deferred to a separate PRD.

---

## FILES

- `ui/index.html`

## REQUIREMENTS

### RULE — DATA SOURCES

UI path reads only:

- `contract.json` → `trade_candidates[].entry` (entry price anchor)
- `contract.json` → `trade_candidates[].intraday_state.vwap` (scalar float, if present)
- `market_map` → `symbols[symbol].fib_levels` (null-safe)
- `market_map` → `symbols[symbol].watch_zones` (empty-safe)

No other data sources are permitted.

---

### R1 — Level diagram per candidate

Each candidate card must include a level diagram section.

The diagram must render the entry price as a labeled reference point.

If entry price is unavailable:
- Display: `"Chart unavailable — no price data"`
- Do not throw an error.

FAIL: Diagram throws an error when entry price is absent.
FAIL: Diagram renders when no entry price is available.

---

### R2 — Key levels overlay

If the candidate's market_map entry includes non-null `fib_levels` or non-empty `watch_zones`:
- Render each as a horizontal marker with a label:
  - `fib_levels`: label from the level object (e.g., `"0.618"`, `"0.5"`)
  - `watch_zones`: label from the zone object's type field

If no levels present:
- Do not render level overlays.

FAIL: Level markers render when `fib_levels` is null and `watch_zones` is empty.
FAIL: A rendered level marker has no label.

---

### R3 — VWAP reference line

If `trade_candidates[symbol].intraday_state.vwap` is present and `> 0`:
- Render as a single labeled horizontal reference line.
- Label: `"VWAP"`
- Must be visually distinguishable from the entry price reference and level markers.

If VWAP is absent or `<= 0`:
- Do not render VWAP line.

FAIL: VWAP line renders when vwap is absent or `<= 0`.
FAIL: VWAP line is visually indistinguishable from a level marker.

---

### R4 — Data source constraint

All diagram elements MUST derive from:
- Entry price from `contract.json` `trade_candidates`
- `fib_levels` / `watch_zones` from `market_map`
- `vwap` scalar from `contract.json` `trade_candidates[].intraday_state`

FAIL: Diagram makes an external HTTP request.
FAIL: Diagram reads any data source not listed above.

---

### R5 — No logic coupling

Diagrams MUST NOT:
- Influence decision outcome
- Influence `policy_allowed`
- Influence grading
- Influence lifecycle
- Introduce new fields into payload or contract

FAIL: Any decision-related field value changes after this PRD.

---

### R6 — Deterministic rendering

All diagram elements must be:
- Derived from existing artifact data
- Rendered identically for identical inputs

No randomness. No inferred levels. No derived calculations beyond direct field mapping.

FAIL: Same input produces different visual output across renders.

---

### R7 — Empty state

If no candidates:
- No diagrams rendered.
- No errors thrown.

FAIL: Error thrown when candidate list is empty.
