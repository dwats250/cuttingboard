PRD-013 — Flow Alignment Soft Gate in Qualification Pipeline

STATUS
COMPLETE
Commit: b17be17f

GOAL
Add a soft gate that downgrades PASS candidates to WATCHLIST when dominant speculative options flow opposes the candidate direction.

SCOPE
- Add cuttingboard/flow.py: FlowSnapshot dataclass, load_flow_snapshot(), flow direction classification
- Add FLOW gate to qualification.py as soft gate
- Add tests/test_flow.py

OUT OF SCOPE
- No changes to hard gates (1–4)
- No changes to regime or structure layers
- Gate never creates or upgrades candidates

FILES
A cuttingboard/flow.py
M cuttingboard/qualification.py
M cuttingboard/config.py
A tests/test_flow.py

REQUIREMENTS

R1 — Downgrade Only
The FLOW gate MUST only downgrade PASS → WATCHLIST.
The FLOW gate MUST NOT create new candidates or upgrade WATCHLIST → PASS.

FAIL: A WATCHLIST candidate is upgraded to PASS by the FLOW gate.

R2 — Directional Opposition
Gate triggers ONLY when dominant flow direction is the opposite of candidate direction.
Hedge, mixed, and absent flow MUST NOT trigger the gate.

FAIL: Gate triggers on hedge, mixed, or absent flow snapshot.

R3 — Determinism
load_flow_snapshot() MUST raise on any missing or invalid field.
No partial or default flow state MUST be accepted silently.

FAIL: load_flow_snapshot() returns a FlowSnapshot with a missing field instead of raising.

DATA FLOW
flow data file → load_flow_snapshot() → FlowSnapshot → qualify_candidate() FLOW gate → PASS | WATCHLIST

FAIL CONDITIONS
- WATCHLIST upgraded to PASS
- Hedge/mixed/absent flow triggers downgrade
- Partial flow snapshot accepted silently

VALIDATION
Run: pytest tests/test_flow.py -q (368 tests)
Manual: fixture with opposing flow — confirm PASS candidate becomes WATCHLIST.
