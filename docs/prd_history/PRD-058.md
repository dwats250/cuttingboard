# PRD-058 — Overnight Exit Guidance Layer

STATUS
COMPLETE

COMMIT
8f942c7

GOAL
Deterministically annotate each existing contract trade_candidate during the end-of-day window with overnight exit guidance to reduce avoidable overnight gap risk, especially for short-dated and structurally fragile options trades.

The layer must be downstream-only. It must not create trades, block trades, modify entry decisions, alter execution policy fields, or perform broker automation.

SCOPE
- Evaluate existing contract["trade_candidates"] during the EOD window only.
- Attach an overnight_policy object to each trade candidate during the EOD window.
- Use only existing internal runtime/contract inputs:
  - trade_candidates
  - system_state.market_regime
  - market_context.continuation_enabled
  - market_map.symbols[*].watch_zones
  - runtime timestamp
- Apply deterministic rule-based gating.
- Preserve existing trade decision and execution policy semantics.

OUT OF SCOPE
- No modification to:
  - trade qualification
  - trade decision status: ALLOW_TRADE / BLOCK_TRADE
  - execution policy fields: policy_allowed, policy_reason, size_multiplier
  - market_map generation
  - broker execution
  - notification send behavior
- No active position tracking.
- No unrealized PnL logic.
- No gamma flip logic unless gamma flip is later added as an existing internal source.

REQUIREMENTS

R1 - EOD Window
The EOD window MUST be defined by config constants:
- OVERNIGHT_POLICY_EOD_WINDOW_MINUTES
- OVERNIGHT_POLICY_MARKET_CLOSE_ET

If called outside the EOD window:
- no overnight_policy field is added
- existing contract fields remain unchanged
- no error is raised

R2 - Output Contract
During the EOD window, each trade_candidate MUST receive:

overnight_policy:
  decision: ALLOW_HOLD | REDUCE_POSITION | FORCE_EXIT
  reason: DTE_TOO_LOW | NEAR_KEY_LEVEL | REGIME_UNSTABLE | NO_EXPANSION_SUPPORT | SPREAD_FRAGILITY | PASS_ALL

overnight_policy MUST be absent outside the EOD window.

R3 - DTE Rule
Use trade_candidate["timeframe"] as the existing DTE source.

If trade_candidate["timeframe"] is numeric or a numeric string, treat it as DTE.

If DTE is missing, non-numeric, or less than OVERNIGHT_POLICY_MIN_HOLD_DTE:

decision: FORCE_EXIT
reason: DTE_TOO_LOW

Default config:
- OVERNIGHT_POLICY_MIN_HOLD_DTE = 10
- OVERNIGHT_POLICY_HARD_EXIT_DTE = 7

R4 - Key Level Proximity Rule
Use market_map["symbols"][symbol]["watch_zones"] as the only key-level source.

Use trade_candidate["entry"] as the candidate entry price.

If trade_candidate["entry"] is missing or non-numeric, fail closed:

decision: FORCE_EXIT
reason: NEAR_KEY_LEVEL

Relevant key-level types:
- VWAP
- ORB_HIGH
- ORB_LOW
- PRIOR_HIGH
- PRIOR_LOW
- EMA9
- EMA21
- EMA50

If the candidate entry price is within OVERNIGHT_POLICY_KEY_LEVEL_PROXIMITY_PCT of any relevant watch-zone level:

decision: FORCE_EXIT
reason: NEAR_KEY_LEVEL

Default config:
- OVERNIGHT_POLICY_KEY_LEVEL_PROXIMITY_PCT = 0.01

R5 - Regime Stability Rule
If contract["system_state"]["market_regime"] == "CHAOTIC":

decision: FORCE_EXIT
reason: REGIME_UNSTABLE

No new LOW_COHERENCE regime value is introduced in PRD-058.

R6 - Expansion Support Rule
If contract["market_context"]["continuation_enabled"] is not True, then a candidate that otherwise passes hard-exit rules MUST be reduced:

decision: REDUCE_POSITION
reason: NO_EXPANSION_SUPPORT

This replaces unrealized-PnL profit protection for v1 because current repo inputs do not contain active position PnL.

R7 - Spread Fragility Rule
If trade_candidate["strategy_tag"] contains SPREAD, increase exit severity by one level unless already FORCE_EXIT:
- ALLOW_HOLD becomes REDUCE_POSITION
- REDUCE_POSITION becomes FORCE_EXIT
- FORCE_EXIT remains FORCE_EXIT

If severity is increased by this rule, set:

reason: SPREAD_FRAGILITY

R8 - Pass Condition
If no rule triggers:

decision: ALLOW_HOLD
reason: PASS_ALL

R9 - Hard Override Rules
The following conditions always produce FORCE_EXIT and cannot be softened by later rules:
- DTE less than OVERNIGHT_POLICY_HARD_EXIT_DTE
- market regime is CHAOTIC
- near key level

Hard overrides are evaluated before reduction rules.

R10 - Determinism
Same inputs MUST always produce identical outputs.

The implementation MUST NOT use:
- randomness
- external data
- wall-clock reads inside the rule engine
- unordered iteration that affects output

Runtime must pass the timestamp into the rule engine.

R11 - Non-Interference
This layer MUST NOT alter:
- decision_status
- block_reason
- decision_trace
- policy_allowed
- policy_reason
- size_multiplier
- status
- outcome

The layer only annotates EOD exit guidance.

R12 - Missing Required Inputs
During the EOD window, missing required top-level inputs MUST raise RuntimeError.

Required inputs:
- contract
- contract["trade_candidates"]
- contract["system_state"]
- contract["market_context"]
- market_map

Candidate-level missing DTE is handled by R3 as DTE_TOO_LOW.

Missing symbol-level market_map data for a candidate MUST fail closed:

decision: FORCE_EXIT
reason: NEAR_KEY_LEVEL

R13 - Rule Order
Rules MUST execute in this exact order:
1. Validate required top-level inputs during EOD.
2. Determine EOD window.
3. If outside EOD, return unchanged contract.
4. Evaluate hard overrides:
   - DTE less than OVERNIGHT_POLICY_HARD_EXIT_DTE
   - CHAOTIC regime
   - near key level
5. Evaluate DTE minimum rule.
6. Evaluate expansion support rule.
7. Apply pass condition.
8. Apply spread fragility adjustment last.

R14 - Runtime Integration
Runtime MUST build market_map before applying overnight policy.

The integration point MUST be after the base contract is built and before:
- notification state key calculation
- notification formatting
- latest contract artifact write
- payload artifact write

This ensures downstream artifacts and notifications observe the same canonical contract.

ACCEPTANCE CRITERIA
- Outside EOD window, no overnight_policy field is added.
- During EOD window, every trade candidate receives overnight_policy.
- DTE < 10 outputs FORCE_EXIT / DTE_TOO_LOW.
- DTE < 7 remains a hard override.
- CHAOTIC regime outputs FORCE_EXIT / REGIME_UNSTABLE.
- Near watch-zone key levels output FORCE_EXIT / NEAR_KEY_LEVEL.
- Missing or non-numeric entry price fails closed as FORCE_EXIT / NEAR_KEY_LEVEL.
- Non-expansion support reduces otherwise clean candidates.
- Spread candidates increase exit severity by one level.
- Clean non-spread candidates output ALLOW_HOLD / PASS_ALL.
- Existing decision and execution policy fields are unchanged.
- Repeated runs with identical inputs produce identical contracts.

SUMMARY
PRD-058 adds a deterministic EOD-only overnight exit guidance layer. It uses existing contract and market_map data, fails closed on fragile or incomplete EOD conditions, and preserves all existing trade entry, decision, execution policy, and broker boundaries.
