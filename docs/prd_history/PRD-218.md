CLASS
CONSUMER

WHY NOW
The PRD-213 mobile reflow renders each symbol as a 6-row stacked card — six
symbols is a long scroll, too high a real-estate cost. The section is also uniform
grey; a text-heavy table benefits from directional colour on the one number that
matters (price). And the SMA arrows (↑50 ↑200) lack the space the macro tape uses
after its glyphs.

MAX EXPECTED DELTA
55 production LOC (dashboard_renderer.py trend render + _CSS + composite strings).

GOAL
Cut the trend-structure real-estate cost on mobile by reflowing each symbol to a
compact single row, colour the price by trend alignment (bullish green / bearish
red), and add a space after the SMA arrows for glyph consistency.

SCOPE
- Colour the Price cell by trend_alignment: BULLISH → green, BEARISH → red, else
  neutral (one _CSS class per direction, applied to the price `<td>`).
- Add a space after each SMA arrow in _TREND_STRUCTURE_COMPOSITE_DISPLAY
  (↑50 ↑200 → ↑ 50 ↑ 200); unavailable strings unchanged.
- Rework the `@media(max-width:640px)` rule so each row renders as one compact
  inline line (labels hidden, cells flow horizontally) instead of a stacked card.

OUT OF SCOPE
- Changing the column set, collapse logic, or any cell value beyond the SMA arrow
  spacing.

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dashboard_renderer.py

CHANGE SURFACE
cuttingboard/delivery/dashboard_renderer.py — _TREND_STRUCTURE_COMPOSITE_DISPLAY
literals, the trend-structure `<td>` render (add price colour class), and the _CSS
(price colour classes + the mobile media query). No change to column identity/order
or the composite lookup logic.

REQUIREMENTS

R1 — Alignment-coloured price
The Price cell carries a colour class keyed to the row's trend_alignment: bullish
green (#4caf50), bearish red (#f44336), neither otherwise.
FAIL: a BULLISH row's price cell lacks the green class, or a BEARISH row's lacks
the red class.

R2 — SMA arrow spacing
Every SMA composite string renders a space after each arrow (↑ 50 ↑ 200).
FAIL: any rendered SMA composite contains an arrow immediately followed by a digit
(e.g. "↑50").

R3 — Compact mobile row
Below 640px each symbol renders as one inline row (per-cell header labels hidden),
not a six-line stacked card.
FAIL: the mobile media query still sets the trend `<td>` to `display:flex` with a
visible `::before` data-label.

DATA FLOW
trend_alignment → price colour class → price `<td>`
_TREND_STRUCTURE_COMPOSITE_DISPLAY (spaced literals) → SMA cell
_CSS media query → compact inline mobile row

FAIL CONDITIONS
- Bullish/bearish price cell missing its colour class.
- Any rendered SMA composite with an arrow directly followed by a digit.
- Mobile media query still renders stacked flex rows with visible data-labels.

VALIDATION
Manual:
- Render at ≤640px: each symbol is one compact line; bullish prices green, bearish
  red; SMA reads "↑ 50 ↑ 200".
Automated:
- Update PRD-131 composite-vocabulary tests to the spaced strings; new tests for
  the price colour class and the compact media query.
