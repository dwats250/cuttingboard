# PRD-122 — Add WTI Crude Macro Visibility

## STATUS
COMPLETE

## CLASS
VISIBILITY

## TIER
T1

## LANE
STANDARD

## GOAL
Add WTI crude (`CL=F`) as an additive macro driver for dashboard visibility only.
No qualification, regime, scoring, policy, execution, pressure, halt, or
required-symbol semantics change. Missing or stale oil data degrades to a
neutral tape slot and never affects pipeline outcome.

---

# SCOPE

## In Scope
- Add `CL=F` to `MACRO_DRIVERS` in `cuttingboard/config.py`.
- Add `SYMBOL_SOURCE_PRIORITY["CL=F"] = ["yfinance"]` and
  `PRICE_BOUNDS["CL=F"] = (5, 250)`.
- Add `"oil": "CL=F"` to `_MACRO_DRIVER_SYMBOLS` in `cuttingboard/contract.py`
  so `_build_macro_drivers` emits `{"oil": {"symbol": "CL=F", "level": ...,
  "change_pct": ...}}` when the quote is valid.
- Add `("OIL", "oil")` to `_TAPE_DRIVER_DEFS` and an `"OIL"` branch in
  `_format_tape_value` (one-decimal level) in
  `cuttingboard/delivery/dashboard_renderer.py`.
- Document in `cuttingboard/macro_pressure.py` that `oil` is intentionally
  not a pressure component.
- Deterministic renderer tests: oil rendered, oil missing, oil stale.
- Update `docs/PROJECT_STATE.md` test count and PRD registry row at close.

## Out of Scope
- No additions to `HALT_SYMBOLS` or `REQUIRED_SYMBOLS`.
- No qualification, regime, scoring, policy, or execution logic changes.
- No new pressure component, no `oil_pressure`, no posture impact.
- No XLE/XOP/USO additions, no oil volatility, no derived oil indicators.
- No dashboard layout redesign or new sections.
- No new sidecar artifacts.
- No `SYMBOL_UNITS["CL=F"]` entry — `CL=F` is USD-priced and inherits
  `DEFAULT_UNITS = "usd_price"` (matches existing `BTC-USD` precedent).

# REQUIREMENTS

- **R1.** `MACRO_DRIVERS` in `cuttingboard/config.py` includes `"CL=F"`,
  positioned after `"BTC-USD"`.
  - **FAIL:** `"CL=F" not in cuttingboard.config.MACRO_DRIVERS`.

- **R2.** `cuttingboard/config.py` adds exactly
  `SYMBOL_SOURCE_PRIORITY["CL=F"] = ["yfinance"]` and
  `PRICE_BOUNDS["CL=F"] = (5, 250)`.
  No `SYMBOL_UNITS` entry is added — `CL=F` is USD-priced and inherits
  `DEFAULT_UNITS = "usd_price"` (matches the existing `BTC-USD` precedent).
  - **FAIL:** Either entry missing or differing from these values, OR a
    `SYMBOL_UNITS["CL=F"]` entry is introduced.

- **R3.** `CL=F` is NOT added to `HALT_SYMBOLS` or `REQUIRED_SYMBOLS`.
  Missing oil data does not halt the pipeline and does not mark the run
  required-symbol-incomplete.
  - **FAIL:** Either constant contains `"CL=F"`, OR a pipeline run with
    `CL=F` missing produces a different outcome from a baseline run
    before this PRD (oil-key additive field excluded from the comparison).

- **R4.** `_MACRO_DRIVER_SYMBOLS` in `cuttingboard/contract.py` contains
  `"oil": "CL=F"`. A new module-level constant
  `_OPTIONAL_MACRO_DRIVERS: frozenset[str] = frozenset({"oil"})`
  designates `oil` as optional. `_build_macro_drivers` SKIPS (does not
  emit) an optional driver when its quote is missing or non-finite,
  instead of raising; non-optional drivers retain raise-on-missing.
  `assert_valid_contract` enforces:
  (a) every non-optional driver key is present in `macro_drivers`,
  (b) `macro_drivers` keys are a subset of `_MACRO_DRIVER_SYMBOLS`,
  (c) each block actually present has the exact required field set
  (unchanged from current behavior).
  - **FAIL:** `oil` block missing when `CL=F` is valid; OR `oil` block
    present when `CL=F` is absent/invalid; OR validation raises on a
    `macro_drivers` dict that omits only optional keys; OR a
    non-optional driver becomes missing-tolerant.

- **R5.** `cuttingboard/macro_pressure.py` does NOT add an `oil_pressure`
  key to `_COMPONENT_KEYS` or an `oil` entry to `_COMPONENT_FIELDS`, and
  does not read the `oil` key. A code comment near `_COMPONENT_KEYS`
  documents that `oil` is a visibility-only driver and intentionally
  excluded from macro-pressure synthesis. `build_macro_pressure` output
  is identical whether `oil` is present in `macro_drivers` or not.
  - **FAIL:** `build_macro_pressure` output differs based on `oil`
    presence, OR `oil_pressure` appears anywhere in the output dict.

- **R6.** `_TAPE_DRIVER_DEFS` in
  `cuttingboard/delivery/dashboard_renderer.py` gains `("OIL", "oil")`
  after `("BTC", "bitcoin")`. `_format_tape_value` formats `"OIL"` as
  `f"{numeric:.1f}"`.
  - **FAIL:** Tape lacks an OIL slot when `macro_drivers["oil"]` is
    present, OR formatting differs from one-decimal.

- **R7.** Missing or stale oil data renders as `—` (arrow) and `--`
  (value), matching existing graceful-degrade semantics for
  `VIX`/`DXY`/`10Y`/`BTC`.
  - **FAIL:** Missing oil renders as anything other than `—`/`--`,
    OR absence of the `oil` key raises in the renderer.

- **R8.** Deterministic renderer tests cover: (a) full oil render
  (level + arrow), (b) `oil` key missing from `macro_drivers`, (c) stale
  `logs/macro_drivers_snapshot.json` fallback path containing an oil
  entry.
  - **FAIL:** Any of the three cases is unasserted, OR tests pass
    without exercising the new OIL branch.

- **R9.** `docs/PROJECT_STATE.md` is updated post-merge with new test
  count and active-PRD pointer (`PRD-122 COMPLETE @ <hash>`). Registry
  row added with merge commit hash.
  - **FAIL:** Registry row missing or status not `COMPLETE`, OR
    `PROJECT_STATE.md` not updated.

# FAIL CONDITIONS

- `HALT_SYMBOLS` or `REQUIRED_SYMBOLS` contains `"CL=F"`.
- `build_macro_pressure` reads or emits any `oil` key.
- `contract.macro_drivers["oil"]` present when `CL=F` quote is invalid.
- Renderer raises when `oil` key absent.
- Decision pipeline outcome differs for identical non-oil inputs with vs.
  without the oil driver wired in, excluding the additive
  `contract.macro_drivers["oil"]` visibility field.
- Tests pass without asserting all three (render / missing / stale) cases.
