PROVENANCE
audits/diag2-2026-06-24/qqq-closes-none.md (CONFIRMED root cause of recon F08).
The premarket path (_run_pipeline → _refresh_trend_structure_sidecar,
runtime/__init__.py:1012-1016) passes `history_by_symbol=ohlcv` — the
candidate-scoped dict (line 741: `{sym: fetch_ohlcv(sym) for sym in candidates …}`)
— BYPASSING the PRD-174 `_collect_trend_structure_history` fallback the hourly path
applies at its call site (line 549). A TREND_STRUCTURE_SYMBOL that is not a current
trade candidate (QQQ, structure=CHOP that run) therefore has no OHLCV history →
trend_structure `_close_series(None)` → DATA_UNAVAILABLE. Proof: every premarket
"CB report" publish shows QQQ DATA_UNAVAILABLE; every "CB hourly" publish (which
uses :549) shows it resolved — a path-divergence, not a data/cache problem.

WHY NOW
PRD-174 introduced `_collect_trend_structure_history` so trend renders for EVERY
TREND_STRUCTURE_SYMBOL independent of the candidate set, but applied it only at the
hourly call site (549). The premarket path was never wired to it, so the
premarket-published dashboard silently drops non-candidate trend rows to
DATA_UNAVAILABLE — the F08 symptom. This restores the PRD-174 invariant on the
premarket path.

CALLER AUDIT (required by charge; performed before choosing the approach)
- `_collect_trend_structure_history` (def :1881): called ONLY at :549
  (`_run_lightweight`, hourly), applying the fallback at the call site. Unit-tested
  directly (test_runtime_trend_structure_refresh.py:181,204); the hourly wiring is
  asserted by a static-source check (same file :219).
- `_refresh_trend_structure_sidecar` (def :1921): called ONLY at :1012
  (`_run_pipeline`, premarket), `mode == MODE_LIVE`-gated; internally calls the
  writer at :1933 with the raw history passed in. Exercised by
  test_runtime_trend_structure_refresh.py (mocks the writer; static-source check
  that _run_pipeline calls it, :143).
- `_write_trend_structure_snapshot` (def :1901): a PURE serializer (build snapshot
  over `symbols=config.TREND_STRUCTURE_SYMBOLS`, write JSON). Called at :547
  (hourly, post-collector) and :1933 (inside the sidecar). Called DIRECTLY by tests
  with controlled `history_by_symbol` (test_trend_structure.py:471,515) and guarded
  by an R2 negative test that it must not widen the symbol set (:492).

APPROACH DECISION (stated per charge: which, and why)
- PREFERRED — apply `_collect_trend_structure_history` INSIDE
  `_refresh_trend_structure_sidecar` (after its MODE_LIVE gate), so the
  premarket entry point cannot bypass the PRD-174 invariant. The sidecar's sole
  caller is premarket, so this fully covers that path; the hourly path keeps its
  :549 application. A behavioral guard test asserts the invariant on BOTH paths so a
  future bypass is caught.
- REJECTED — full single-source centralization inside `_write_trend_structure_snapshot`:
  the writer is a pure serializer called directly by unit tests with controlled
  history (test_trend_structure.py:471,492,515); injecting the network-fetching
  collector there would force `fetch_ohlcv` calls inside those unit tests and
  violate the writer's serializer role (and the R2 negative contract). REJECTED —
  routing the hourly path through the sidecar to centralize: the sidecar is
  MODE_LIVE-gated and the hourly path applies the collector at its own asserted call
  site (:219); rerouting risks the gate skipping the hourly write and churns that
  test for no behavioral gain.
- FALLBACK (if review prefers call-site symmetry) — mirror :549 at :1012 by passing
  `history_by_symbol=_collect_trend_structure_history(ohlcv)`. Behaviorally
  identical; less encapsulated (a future caller could bypass again). The PREFERRED
  option is chosen for bypass-resistance at the sidecar boundary.

GOAL
On the premarket `_run_pipeline`, a TREND_STRUCTURE_SYMBOL that is not a current
trade candidate renders full sma_50/sma_200 history in the trend-structure
snapshot, identical to the hourly path — closing F08.

SCOPE (read runtime/__init__.py first; ground every edit in the actual functions)
- In `_refresh_trend_structure_sidecar` (:1921-1937), after the `mode != MODE_LIVE`
  early-return, resolve the history via `_collect_trend_structure_history(history_by_symbol)`
  before calling `_write_trend_structure_snapshot`, so a non-candidate
  TREND_STRUCTURE_SYMBOL gets its `fetch_ohlcv` fallback. The premarket caller at
  :1012 (`history_by_symbol=ohlcv`) is unchanged in signature.
- No change to the hourly path (:547-549) — it already applies the collector.
- No change to `_write_trend_structure_snapshot` (preserve its pure-serializer role
  and the R2 negative contract) or to `_collect_trend_structure_history` itself.

COST (confirmed acceptable, per charge)
The premarket sidecar will now call `fetch_ohlcv` for non-candidate trend symbols
(≤6 total: SPY/QQQ/GDX/GLD/SLV/XLE), each cache-served when warm — EXACTLY what the
hourly path already does via :549. No new data source, schema, gate, or regime
change; no new fetch *kind*, only the same daily OHLCV fetch the hourly path makes.

NON-GOALS
- No change to regime, qualification, candidate generation, payload, or renderer
  (PRD-208 owns SMA presentation; this PRD only ensures the data is present).
- No change to the cache/fetch gates (that latent count-blind hole is PRD-209,
  HELD — orthogonal to F08).
- No widening of the trend symbol set (R2 negative contract preserved).
- No reroute of the hourly path; no change to the writer.

RISKS
- The sidecar now performs (cache-served) fetches under MODE_LIVE — mirror of the
  hourly path; bounded to ≤6 symbols. Mitigated by R3 (no behavior change for
  symbols already present in `ohlcv`).
- Static-source tests assert the hourly wiring (:219) and that _run_pipeline calls
  the sidecar (:143) — unaffected (hourly path untouched; sidecar still called).
