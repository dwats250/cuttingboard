# PRD-134 — Daily Pipeline Market Map Coherence Repair

STATUS: IN PROGRESS
LANE: STANDARD
CLASS: INFRASTRUCTURE (workflow-publish-gate; no decision-logic change)

## GOAL

Stop the daily Cuttingboard Pipeline workflow from publishing
ui/dashboard.html in "noop" mode, where no Python pipeline ran and the
on-disk artifact set has been split between an older daily live write
(latest_payload/run = live-*) and a more recent hourly_alert write
(market_map = hourly-*). PRD-118's `validate_coherent_publish` correctly
rejects this mix, surfacing as the recurring CoherentPublishError seen
in failed runs 25759504467, 25753693370, 25747005282, 25746783255,
25745013143 (all today). The fix preserves PRD-118 strictness verbatim
and changes no trading logic, no market data fetching, no schemas, and
no dashboard rendering.

## SCOPE

This PRD fixes the observed noop publish failure. It does NOT claim to
fix the theoretical race between a true non-noop daily pipeline run and
a simultaneous hourly_alert market_map write. That case has not been
observed today and is deferred with the PRD-135 trigger condition in
the cleanup plan.

- Add a `PUBLISH_READY=true` signal at the end of each of the eight
  `Run <mode>` steps in the daily Cuttingboard Pipeline workflow so the
  dashboard publish step is evidence-based rather than mode-name based.
- Gate `Commit artifacts` and `Push` steps on
  `success() && env.PUBLISH_READY == 'true'`. The "noop" path never
  sets the signal, so the publish steps are skipped exactly when no
  fresh Python pipeline artifact set was written.
- Add a deterministic regression test that
  `validate_coherent_publish` rejects the exact live-payload +
  hourly-market_map combination observed in the five failed runs
  cited above.
- No code change in `cuttingboard/`, no change in `dashboard_renderer`,
  no change in `runtime.py`, no change in hourly_alert workflow.

## REQUIREMENTS

- **R1 — Publish-ready signal from eight Run <mode> steps.** Each of
  the eight `Run <mode>` steps MUST set `PUBLISH_READY=true` in
  `$GITHUB_ENV` only AFTER its python command exits successfully. The
  eight steps are:
  - `Run prefetch`
  - `Run live pipeline`
  - `Run Sunday regime report`
  - `Verify run`
  - `Run ORB trajectory scan`
  - `Run post-ORB scan`
  - `Run mid-morning scan`
  - `Run power-hour scan`
  FAIL: any of the eight steps does not emit the signal on success,
  OR emits it before the python command, OR emits it
  unconditionally.

- **R2 — Commit artifacts gate.** The `Commit artifacts` step's `if:`
  condition MUST be:
  `${{ success() && env.PUBLISH_READY == 'true' }}`
  FAIL: condition differs, OR the step runs without the signal.

- **R3 — Push gate.** The `Push` step's `if:` condition MUST be:
  `${{ success() && env.PUBLISH_READY == 'true' }}`
  FAIL: condition differs, OR the step runs without the signal.

- **R4 — Strict gate preserved.** `validate_coherent_publish` source
  bytes MUST be byte-identical pre/post this PRD. No relaxation, no
  "newer-than" comparator, no fixture-mode loosening.
  FAIL: `cuttingboard/delivery/dashboard_renderer.py:382-422` diff is
  non-empty.

- **R5 — Regression test for observed mix.** A new test in
  `tests/test_dashboard_renderer.py` MUST construct an artifact triple
  with `payload.meta.generation_id = "live-20260512T113206Z"`,
  `run.generation_id = "live-20260512T113206Z"`,
  `market_map.generation_id = "hourly-20260512T195802Z"`, and assert
  `validate_coherent_publish` raises `CoherentPublishError` matching
  `r"generation_id mismatch"`. This locks the observed regression so
  any future relaxation of the gate is caught.
  FAIL: test absent, or test passes silently when the gate is
  hypothetically relaxed.

- **R6 — No suppression.** This PRD MUST NOT use `|| true`,
  `continue-on-error: true`, or any other suppressive construct on
  the publish gate, its preconditions, or any added step.
  FAIL: any suppressive token introduced.

- **R7 — Hourly path untouched.** Diff MUST NOT include
  `.github/workflows/hourly_alert.yml`, `cuttingboard/runtime.py`,
  `cuttingboard/notifications/`, `cuttingboard/market_map.py`, or
  `cuttingboard/delivery/dashboard_renderer.py`.
  FAIL: diff touches any of those paths.

## DATA FLOW

```
schedule cron fires
  → Determine run mode → job_mode ∈ {prefetch, live, sunday, verify,
                                       orb_trajectory, post_orb,
                                       midmorning, power_hour, noop}
  → Run <mode> step (one of eight)
      ├─ noop: SKIPPED                                  [no PUBLISH_READY]
      └─ any other: invokes python -m cuttingboard,
                    which writes a coherent live-* triple,
                    then sets PUBLISH_READY=true       [signal present]
  → Commit artifacts step (gated on signal)
      ├─ if env.PUBLISH_READY != 'true': SKIPPED
      └─ else: render dashboard → validate_coherent_publish → commit
  → Push step (same gate)
```

PRD-118 gate behavior is unchanged. The on-disk market_map.json can
still be stomped by hourly_alert; the publish step simply no longer
fires when there is no fresh local Python pipeline write to pair it
with.

## FAIL CONDITIONS
