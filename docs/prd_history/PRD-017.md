PRD-017 — Notification Delivery Stabilization: Rate Limit, Retry, and Aggregation

STATUS
COMPLETE
Commit: fc7f5e9

GOAL
Add a global rate limiter, bounded retry logic, and a contract-aware notification aggregation function to prevent duplicate sends and handle transient Telegram API failures.

SCOPE
- output.py: add 1.1s global rate limiter, bounded retry (429→2s, 5xx/timeout→1s, max 2 attempts), build_notification_message(contract)
- audit.py: add retry_count field to write_notification_audit
- runtime.py: build contract before send; single send_notification call per run
- tests/conftest.py: reset rate-limiter state between tests
- Add tests/test_prd017_notification_stabilization.py (28 tests)

OUT OF SCOPE
- No changes to Telegram credentials or transport configuration
- No changes to contract schema

FILES
M cuttingboard/output.py
M cuttingboard/audit.py
M cuttingboard/runtime.py
M tests/conftest.py
A tests/test_prd017_notification_stabilization.py

REQUIREMENTS

R1 — Rate Limiter
Global rate limiter MUST enforce minimum 1.1s between send_telegram calls.
Rate limiter state MUST be reset between test runs via conftest.py.

FAIL: Two send_telegram calls occur less than 1.1s apart in production.

R2 — Bounded Retry
429 responses MUST retry after 2s, max 1 retry.
5xx and timeout responses MUST retry after 1s, max 1 retry.
No other error codes MUST trigger retry.

FAIL: A non-retryable error code triggers a retry.

R3 — Single Send Per Run
runtime.py MUST call send_notification exactly once per run.

FAIL: A single run triggers more than one send_notification call.

R4 — retry_count Audit
write_notification_audit MUST record retry_count for every notification attempt.

FAIL: audit record is missing retry_count field.

DATA FLOW
contract → build_notification_message() → send_notification (once per run) → rate limiter → retry if eligible → send_telegram

FAIL CONDITIONS
- Multiple sends per run
- Retry on non-retryable error
- retry_count absent from audit record
- Rate limiter not reset between tests

VALIDATION
Run: pytest tests/test_prd017_notification_stabilization.py -q (28 tests)
Manual: live run — confirm single Telegram message, audit log shows retry_count.
