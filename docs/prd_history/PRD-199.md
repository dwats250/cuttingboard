WHY NOW
PRD-190 stabilized the trend-structure data path. The MACRO TAPE tradables block
(SPY/QQQ/GLD/SLV/GDX/XLE) still shows price only ("no arrows",
dashboard_renderer.py:2109) while macro-driver rows already show a sign-of-%-change
arrow via _pct_arrow. Adding the same arrow to tradables closes the asymmetry using
data already in hand (NormalizedQuote.pct_change_decimal).

MAX EXPECTED DELTA
One additive field in trend_structure._build_record + the tradables arrow
source/render in dashboard_renderer.py (retiring the dead _direction_arrow tradables
branch) + call-site plumbing + tests. No new fetch.

GOAL
Each MACRO TAPE tradables cell renders an up/down/flat arrow driven by the SIGN of
that symbol's daily %-change, alongside the existing price.

SCOPE
- trend_structure._build_record (trend_structure.py:245-281): emit one additive field
  daily_change_pct from quote.pct_change_decimal (quote already passed in); None when
  quote is None. test_record_has_required_keys (test_trend_structure.py:388-398) uses a
  subset check, so the add is non-breaking.
- dashboard_renderer _build_tape_slots tradables branch (dashboard_renderer.py:1050-1061):
  source the tradable arrow from the trend-structure record's daily_change_pct via
  _pct_arrow (1007), retiring the dead _direction_arrow(trade_framing.direction) path
  (1057). Thread the trend-structure records (extracted by _trend_structure_records:950;
  in render scope via the trend_structure_snapshot param:1640) into _build_tape_slots
  (called :1746).
- Render the arrow in the macro-tradables-grid cells (dashboard_renderer.py:2109-2119):
  add the arrow glyph beside the label; price value (current_price) formatting unchanged.

DEAD BRANCH (author discipline): the tradables branch of _build_tape_slots computes
_direction_arrow(trade_framing.direction) today, but the grid renders price only (2109)
and the vote tally reads only MACRO_ROW_1/2 — so that arrow is computed-but-unused. This
PRD retires it; no other reader of the tradables tape arrow exists.

REQUIREMENTS

R1 — Trend record carries daily %-change
_build_record emits daily_change_pct derived from quote.pct_change_decimal (None when
quote is None); all existing keys unchanged.
FAIL: a record for a symbol with a non-None quote omits daily_change_pct.

R2 — Tradables arrow is driven by %-change sign, not trade_framing.direction
The tradables tape arrow is _pct_arrow(daily_change_pct) from the trend-structure record;
trade_framing.direction no longer sources it.

R3 — Arrow renders in the tradables grid beside the price
Each macro-tradables-grid cell renders the arrow glyph and the price; missing %-change
renders the flat/dash sentinel with the price unchanged.
FAIL: the rendered tradables grid contains no arrow glyph for a tradable whose daily_change_pct is finite and non-zero.

R4 — No regression to macro rows or tradable price
Macro-driver arrows/votes and tradable price values are unchanged by this PRD for identical input.
FAIL: a macro-driver arrow/vote or a tradable price value changes vs the pre-PRD render for identical input.
