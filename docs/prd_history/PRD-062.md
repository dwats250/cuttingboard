PRD-062 — Macro Pressure Block in Signal Forge Dashboard

STATUS
IN PROGRESS

GOAL
Surface the macro_pressure snapshot (built in PRD-060 but never integrated)
as a read-only block in the Signal Forge dashboard, using the macro_drivers
already present in the payload.

SCOPE
- Call `build_macro_pressure` inside `dashboard_renderer.py`
- Add a `macro-pressure` block between `macro-tape` and `run-delta` sections
- Show volatility, dollar, rates, bitcoin component pressures and overall_pressure
- Handle empty macro_drivers gracefully (no-data state)
- Add tests to `tests/test_dashboard_renderer.py`

OUT OF SCOPE
- Changes to payload schema
- Changes to contract
- Changes to runtime.py
- Changes to notifications
- Changes to qualification or regime logic
- Any new data sources

FILES
M cuttingboard/delivery/dashboard_renderer.py
M tests/test_dashboard_renderer.py
M docs/PRD_REGISTRY.md
M docs/prd_index.json
M docs/PROJECT_STATE.md
A docs/prd_history/PRD-062.md

REQUIREMENTS

R1 — Macro Pressure Block Present
The rendered HTML MUST contain a `<div>` with `id="macro-pressure"`.

FAIL: `id="macro-pressure"` is absent from rendered HTML.

R2 — Component Pressures Rendered
The block MUST render exactly four component labels:
`VOL`, `DXY`, `RATES`, `BTC` — each with its pressure value
(`RISK_ON`, `RISK_OFF`, `NEUTRAL`, or `UNKNOWN`).

FAIL: Any of the four component labels is missing from the macro-pressure block.

R3 — Overall Pressure Rendered
The block MUST render the `overall_pressure` value
(`RISK_ON`, `RISK_OFF`, `MIXED`, `NEUTRAL`, or `UNKNOWN`).

FAIL: overall_pressure value is absent from the macro-pressure block.

R4 — No-Data Graceful Handling
When `macro_drivers` is empty (`{}`), the block MUST render a
no-data marker and MUST NOT raise.

FAIL: Rendering raises when macro_drivers is `{}`. OR: No no-data marker present when macro_drivers is `{}`.

R5 — Block Position
The macro-pressure block MUST appear after `id="macro-tape"` and before
`id="run-delta"` in the rendered HTML.

FAIL: macro-pressure appears before macro-tape or after run-delta in the output.

DATA FLOW
payload["macro_drivers"] → build_macro_pressure() → pressure snapshot
market_map → build_macro_pressure() → pressure snapshot
pressure snapshot → render_dashboard_html() → HTML macro-pressure block

FAIL CONDITIONS
- `id="macro-pressure"` absent from rendered HTML.
- Any of VOL, DXY, RATES, BTC labels missing from block.
- overall_pressure value absent from block.
- Renderer raises when macro_drivers is `{}`.
- No no-data marker when macro_drivers is `{}`.
- macro-pressure block appears before macro-tape or after run-delta.
