2. DIFF VERDICT

The slice fix itself is correct:

- `df.iloc[-(n+1+h):-(1+h)]` has exactly `n` rows.
- At the minimum length `n+1+h`, it selects the first `n` rows, followed by `h` completed hold rows and the current row.
- The rebuilt fixtures are valid OHLC and their expected breakout levels match the corrected window.

There is one gate-level defect: `CONTINUATION_HOLD_CANDLES > 1` excludes all hold candles from the lookback but validates only the oldest hold candle. The implementation must check every completed hold close, for example the slice `df.iloc[-(h+1):-1]["Close"]`, against `breakout_level`, and tests should cover `h=2` plus the exact minimum length.

The focused pytest run could not execute in this read-only environment because Python could not create any temporary file; the fixture verdict above is from literal static verification.

Commissioned answer: there is no blanket downstream assumption that continuation candidates are never accepted; several consumers explicitly preserve `entry_mode`. There are, however, two decisive implicit assumptions: decision assembly assumes every accepted result has an original candidate, and every geometry consumer assumes the resulting `TradeDecision` contains the geometry that actually qualified. The first assumption is currently protected only by candidate-scoped OHLCV fetching; the second is false on the first real production continuation acceptance.

4. OVERALL

ACCEPT WITH CHANGES. The PRD-259 window correction is mathematically correct for the default `CONTINUATION_HOLD_CANDLES = 1`, makes valid-OHLC continuation qualification satisfiable, and the rebuilt fixtures correctly model the new window. Before merging, change the HOLD gate to validate every configured hold candle and add `h > 1` plus minimum-length tests. The stale geometry, contradictory qualified/rejected output, notification semantics, and latent candidate join are serious merger-disposition risks exposed by the first acceptance, but they are downstream pre-existing assumptions rather than defects in the narrow window slice itself.
